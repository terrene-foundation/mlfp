Why interpretability matters

  • Regulatory: Singapore's MAS requires explainability for credit decisions
  • Debugging: a model that uses postcode as a proxy for race needs to be caught
  • Trust: stakeholders will not act on a prediction they do not understand
🔵 Theory: Interpretability is not a nice-to-have. It is a deployment prerequisite.

SHAP: Shapley values for ML

\[ \phi_i = \sum_{S \subseteq F \setminus \{i\}} \frac{|S|!\,(|F|-|S|-1)!}{|F|!} [f(S \cup \{i\}) - f(S)] \]
AxiomMeaning
EfficiencyContributions sum to the prediction
SymmetryEqual contributions → equal values
DummyUseless feature → zero value
LinearityValues add across model sums

SHAP waterfall

Each bar shows one feature's contribution. Green pushes up; red pushes down.

SHAP in code


import shap

explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)

# Global importance
shap.summary_plot(shap_values, X_test)

# Single prediction
shap.plots.waterfall(shap_values[0])

LIME: local explanations


from lime.lime_tabular import LimeTabularExplainer

explainer = LimeTabularExplainer(
    X_train.to_numpy(),
    feature_names=X_train.columns,
)
exp = explainer.explain_instance(
    X_test[0].to_numpy(),

    num_features=5,

Fairness: disparate impact

\[ \text{DI} = \frac{P(\hat{Y}=1 \mid G=\text{minority})}{P(\hat{Y}=1 \mid G=\text{majority})} \;\geq\; 0.8 \] 

rate_min = pred[group == "minority"].mean()
rate_maj = pred[group == "majority"].mean()
di = rate_min / rate_maj
print(f"DI: {di:.3f} — {'PASS' if di >= 0.8 else 'FAIL'}")

Fairness: equalised odds

TPR and FPR must be equal across groups:

\[ P(\hat{Y}=1 \mid Y=1, G=a) = P(\hat{Y}=1 \mid Y=1, G=b) \] 

for g in ["minority", "majority"]:
    mask = group == g
    tpr = recall_score(y_test[mask], pred[mask])
    print(f"  {g}: TPR = {tpr:.3f}")

The impossibility theorem

Not a limitation of your model. A mathematical fact.

Three mistakes to avoid

1. SHAP ≠ causation
SHAP shows what the model uses, not what causes the outcome.
2. Removing protected features ≠ fairness
Proxy features (postcode, school) carry the same signal.
3. Optimising one fairness metric blindly
Impossibility theorem: fixing one may break another.
