The Complete Supervised Model Zoo

SVM · KNN · Naive Bayes · Decision Trees · Random Forests

⏱ 90 minutes   ·   Prereq: 3.2   ·   Outcome: train and compare five model families on churn data
MLFP — Module 3: Supervised ML

Five families cover most non-deep supervised problems

  1. SVM — hyperplane with maximum margin
  2. KNN — vote of nearest training points
  3. Naive Bayes — Bayes' theorem with conditional independence
  4. Decision Trees — recursive axis-aligned splits
  5. Random Forests — bagged ensemble of decorrelated trees

Every family has a distinct geometry. The data tells you which geometry fits.

SVM — maximum margin

SVM — the quadratic program

\[ \min_{\boldsymbol{w}, b} \tfrac{1}{2}\|\boldsymbol{w}\|^2 + C\sum_i \xi_i \quad \text{s.t.}\quad y_i(\boldsymbol{w}^\top x_i + b) \ge 1 - \xi_i \]
  • Small $C$ = tolerate violations, smooth boundary (high bias)
  • Large $C$ = separate everything (high variance)
  • Only support vectors have $\alpha_i > 0$ in the dual — sparse solution

The kernel trick

\[ K(x, x') = \langle \phi(x), \phi(x') \rangle \quad\text{(never compute }\phi\text{ explicitly)} \]
Kernel$K(x, x')$Use for
Linear$x^\top x'$text, high-dim sparse
Polynomial$(x^\top x' + c)^d$degree-$d$ interactions
RBF$\exp(-\gamma \|x-x'\|^2)$smooth non-linear

KNN — the curse of dimensionality

$\lim_{d\to\infty}\; (\text{dist}_{\max} - \text{dist}_{\min}) / \text{dist}_{\min} \to 0$

Naive Bayes — fast baseline

\[ \hat{y} = \arg\max_y P(y) \prod_{j=1}^{d} P(x_j \mid y) \]
  • GaussianNB — continuous features, $P(x_j|y) = \mathcal{N}(\mu_{yj}, \sigma^2_{yj})$
  • MultinomialNB — count features (text bag-of-words)
  • BernoulliNB — binary features

Independence assumption is wrong — but rank-correct is easier than probability-correct. Great text baseline.

Decision trees — splitting criteria

\[ G = 1 - \sum_k p_k^2 \qquad H = -\sum_k p_k \log_2 p_k \]
\[ \text{IG} = H(\text{parent}) - \sum_j \tfrac{n_j}{n} H(\text{child}_j) \]

Try every feature, every threshold, pick the split with the largest IG. Recurse until stopping rule.

Trees partition feature space with axis-aligned cuts

Decision tree by hand — Gini


Parent: 10 churn, 6 retain  (n=16)
  p_churn = 10/16 = 0.625
  Gini = 1 - 0.625² - 0.375² = 0.469

Split → {8 churn, 1 retain}, {2 churn, 5 retain}
  G(left)  = 1 - (8/9)² - (1/9)² = 0.198
  G(right) = 1 - (2/7)² - (5/7)² = 0.408
  Gweighted = (9/16)(0.198) + (7/16)(0.408) = 0.290
  IG = 0.469 - 0.290 = 0.179

Random forest — why the trees are decorrelated

OOB = 1/e — free validation

\[ \lim_{n\to\infty}\left(1 - \frac{1}{n}\right)^n = \frac{1}{e} \approx 0.368 \]

Each tree is built on a bootstrap sample. ~36.8% of rows never enter that particular tree. Those out-of-bag rows are an automatic validation set — unbiased generalisation estimate with zero extra compute.

Five models, one dataset, same CV


models = {
    "SVM (RBF)":     SVC(kernel="rbf", C=1.0),
    "KNN (k=25)":    KNeighborsClassifier(n_neighbors=25),
    "GaussianNB":    GaussianNB(),
    "Decision Tree": DecisionTreeClassifier(max_depth=6),
    "Random Forest": RandomForestClassifier(n_estimators=200,
                                            oob_score=True),
}

for name, m in models.items():
    cv_res = cross_validate(m, X, y, cv=StratifiedKFold(5),

Pick the right tool

ModelFirst choice when
SVMsmall, clean, high-dim; text with linear kernel
KNNsmall data, low dim, interpretable boundaries
Naive Bayestext baseline, spam, sentiment, fast prototype
Decision Treeneed a human-readable rule set
Random Forestmost tabular problems, default when unsure

Three traps

1. Unscaled features with SVM or KNN — dollars dominate years. Always standardise.
2. KNN with very large $k$ — collapses to base rate. Tune by CV.
3. Reporting accuracy on imbalanced data — meaningless. Use F1/AUC. More in 3.5.
