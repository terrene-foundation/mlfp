Five stages, one pipeline

ETL: the universal data pattern

  • Extract: get data from its source (file, API, database)
  • Transform: clean, encode, scale, engineer features
  • Load: write the result to a destination
🟢 Foundations: Every data project you will ever build follows this pattern. The tools change; the structure does not.

🟢 Foundations: GET = "give me data". POST = "here is data". Response is usually JSON — parse it into a DataFrame.

Step 3: Transform


df = (
    df_raw
    .with_columns(
        pl.col("pickup_datetime")
          .str.to_datetime("%Y-%m-%d %H:%M:%S"),
    )
    .filter(pl.col("fare_amount") > 0)
    .with_columns(
        pl.col("fare_amount")
          .fill_null(pl.col("fare_amount").median()),
        hour=pl.col("pickup_datetime").dt.hour(),
        day_of_week=pl.col("pickup_datetime").dt.weekday(),
    )
    .drop_nulls(subset=["pickup_zone"])
)
print(f"Clean: {df.shape}")

Step 4: PreprocessingPipeline


pipeline = PreprocessingPipeline(

Four mistakes to avoid

1. Dropping nulls without checking count
Check null_count() first. 25% nulls = 25% data loss.
2. Fitting pipeline on test data
fit_transform(train), transform(test). Never fit on both.
3. Hardcoding file paths
Use the data loader. A pipeline that only works on your laptop is not a pipeline.
4. No re-profiling after cleaning
Always compare before/after. Cleaning can introduce new issues.
