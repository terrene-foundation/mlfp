Lesson 3.5 Model Evaluation, Imbalance, and Calibration

A model that reports 0.8 should mean "80% chance". A model with 92% accuracy on 10%-positive data is not necessarily any good. In the next 75 minutes you will learn how to evaluate classifiers honestly on imbalanced data using precision, recall, F1, ROC, PR, and log-loss; handle imbalance with SMOTE, cost-sensitive learning, and focal loss; and calibrate model probabilities so they can be used as decision thresholds. The running dataset is Singapore credit scoring with a 100:1 business cost ratio: missing a default costs $10 000; a false alarm costs $100.

The Confusion Matrix and Its Offspring

Every classification metric ultimately comes from the 2×2 confusion matrix that counts predictions against labels.

The formulas you must know

$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$ $$\text{Precision} = \frac{TP}{TP + FP} \qquad \text{Recall} = \frac{TP}{TP + FN}$$ $$F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$

Precision asks "when I predicted positive, how often was I right?". Recall asks "of the actual positives, how many did I catch?". F1 is their harmonic mean — it punishes extreme imbalance between the two.

🟢 Foundations — Accuracy is lying on imbalanced data

On our 12%-positive credit data, predicting "no default" for every single applicant scores 88% accuracy. F1 for that same classifier is exactly zero (precision = 0, recall = 0). If you only report accuracy, you cannot distinguish a useful model from the one that gives up.

ROC and PR Curves: Threshold-Free Views

A classifier outputs a probability score. A threshold turns that score into a hard prediction. Different thresholds give different precision/recall trade-offs. Instead of picking one threshold, ROC and PR curves show the trade-off at every threshold.

Why ROC-AUC fails on imbalanced data

ROC is defined by the true positive rate (TPR = recall) against the false positive rate ($FPR = FP / (FP + TN)$). When the negative class is huge, $TN$ dwarfs $FP$ and $FPR$ barely moves even when $FP$ grows by thousands. So the ROC curve still hugs the upper-left corner — the AUC looks great — while precision is quietly atrocious.

Precision-recall has no such blind spot. It ignores $TN$ entirely. $\text{Precision} = TP/(TP + FP)$ punishes every false positive, regardless of how many true negatives there are. For the credit data:

  • Predict "no default" for everyone: ROC-AUC = 0.5, AP ≈ 0.12 (the base rate).
  • Predict top 5% by score as default: ROC-AUC might be 0.92, AP around 0.45.
  • Report AP as your honest number; ROC-AUC is a sanity check at best.
🔵 Theory — AUC-PR vs AUC-ROC

AUC-PR (average precision) is the integral of precision over the recall axis. On highly imbalanced data, two models with nearly identical ROC-AUC can have PR-AUCs that differ by 30%. The rule of thumb: if the positive class is below 20%, report PR-AUC; if above 20%, either works.

Handling Imbalance: Four Techniques

1. SMOTE — synthetic minority oversampling

SMOTE (Chawla et al., 2002) creates synthetic minority-class examples by interpolating between each minority example and its $k$ nearest minority neighbours. For a minority point $x_i$ with a neighbour $x_{nn}$, a synthetic point is $x_{\text{new}} = x_i + \alpha(x_{nn} - x_i)$ for $\alpha \in [0, 1]$.

Pros: the training set becomes balanced without duplicating existing points. Cons: SMOTE breaks badly when (a) minority points are clustered in dense regions and SMOTE interpolates across the gap creating implausible samples, (b) the feature space has high dimensionality and "nearest" is no longer informative (the curse again), (c) minority points are noise, in which case SMOTE amplifies the noise.

⚠ SMOTE is not a free lunch

SMOTE-trained models are not automatically better. Modern practitioners usually prefer cost-sensitive learning or focal loss — both of which do not fabricate data. SMOTE remains useful when you must use a model family that ignores sample weights; otherwise pick a technique that adjusts the loss directly.

2. Cost-sensitive learning — weights in the loss

If missing a default costs $10 000 and a false alarm costs $100, tell the model that. Pass sample_weight or class_weight into the classifier: 

from sklearn.ensemble import RandomForestClassifier
# class_weight='balanced' sets weights inversely proportional to class frequencies
model = RandomForestClassifier(class_weight="balanced")

# Or a custom cost ratio that matches the business
model = RandomForestClassifier(class_weight={0: 1, 1: 100})

The loss is now $\sum_i w_i L(y_i, \hat{y}_i)$ and the optimiser naturally cares more about the high-weight examples. No fake data, no distribution distortion.

3. Focal loss — down-weight easy examples

Focal loss (Lin et al., 2017) was invented for dense object detection where the imbalance is extreme (1000+ background boxes per foreground box). It modifies binary cross-entropy to down-weight already-correctly- classified examples:

$$\text{FL}(p_t) \;=\; -\alpha_t \,(1 - p_t)^\gamma\, \log(p_t)$$

where $p_t$ is the predicted probability for the true class, $\alpha_t$ balances classes, and $\gamma \ge 0$ is the focusing parameter. When $\gamma = 0$, focal loss is just weighted cross-entropy. When $\gamma > 0$, the factor $(1 - p_t)^\gamma$ is tiny for well-classified examples and close to 1 for hard ones. The loss automatically focuses on the hard cases.

🔵 Theory — Why focal loss works

Standard cross-entropy on an easy example with $p_t = 0.9$ still contributes $-\log(0.9) \approx 0.1$ — enough that over millions of easy examples it dominates the gradient. Focal loss with $\gamma = 2$ multiplies that by $(1 - 0.9)^2 = 0.01$, reducing the contribution 100×. Hard examples with $p_t = 0.3$ get multiplied by $(1 - 0.3)^2 = 0.49$ — barely changed. Net effect: the gradient is dominated by hard examples and the model keeps learning on them.

4. Threshold tuning from the business cost matrix

The default threshold 0.5 is rarely optimal. With a cost matrix $(c_{FP}, c_{FN})$, the expected cost of predicting positive is:

$$\mathbb{E}[\text{cost}] = c_{FP}(1 - p) + c_{FN} \cdot 0$$

and of predicting negative is:

$$\mathbb{E}[\text{cost}] = c_{FP} \cdot 0 + c_{FN} \cdot p$$

You should predict positive iff $c_{FN} \cdot p > c_{FP}(1 - p)$, i.e. when $p > c_{FP} / (c_{FP} + c_{FN})$. For our 100:1 cost matrix that is $p > 100 / 10100 \approx 0.0099$. The optimal threshold is about 1%, not 50%. Any doubt the model has about a loan is enough to flag it when the cost of missing a default is 100× the cost of a false alarm.

Calibration: Making Probabilities Mean What They Say

A model is well-calibrated if, among the samples it predicts with probability $p$, exactly a fraction $p$ are positive. Raw gradient-boosted classifiers are typically over-confident — they output probabilities closer to 0 and 1 than the truth warrants. That is fine if you only care about rankings but catastrophic if you feed the probabilities into downstream cost calculations.

Platt scaling — fit a sigmoid on the model scores

Platt scaling (1999) fits a one-parameter sigmoid $\sigma(Ax + B)$ on top of the raw model output. Find $A, B$ by maximum likelihood on a held-out calibration set. Cheap — one parameter plus a bias — and works when the uncalibrated scores have an approximately sigmoidal shape relative to the truth.

Isotonic regression — non-parametric calibration

When the miscalibration is not sigmoidal, isotonic regression fits a non-decreasing piecewise-constant function instead. More flexible than Platt, but needs more data (at least a few hundred samples per calibration fold) to avoid overfitting. 

from sklearn.calibration import CalibratedClassifierCV
from sklearn.ensemble import GradientBoostingClassifier

base = GradientBoostingClassifier()

# Sigmoid (Platt)
cal_platt = CalibratedClassifierCV(base, method="sigmoid", cv=5)
cal_platt.fit(X_tr, y_tr)

# Isotonic
cal_iso = CalibratedClassifierCV(base, method="isotonic", cv=5)
cal_iso.fit(X_tr, y_tr)

The Brier score — a proper scoring rule

How do you measure calibration quality? The Brier score is the mean squared error between predicted probability and actual label:

$$\text{BS} \;=\; \frac{1}{N}\sum_{i=1}^{N}(p_i - y_i)^2$$

Lower is better. Brier is a proper scoring rule: the expected Brier is minimised uniquely by reporting the true probability. A model that cheats by pushing everything to 0 or 1 scores worse than one that reports honest uncertainty. Report Brier after calibration to show the improvement.

Worked Example: End-to-End on Credit Scoring

📝 Goal

Train a LightGBM on 12%-positive credit data. Compare: (baseline, SMOTE, class_weight, threshold tuned, calibrated). Report accuracy, AUC-PR, Brier, and business cost at the optimal threshold. Show which combination wins.

Step 1 — Baseline. 
import lightgbm as lgb
import numpy as np
from sklearn.metrics import (
    average_precision_score, brier_score_loss,
    precision_recall_curve,
)

base = lgb.LGBMClassifier(n_estimators=400, learning_rate=0.05)
base.fit(X_tr, y_tr)
p_base = base.predict_proba(X_te)[:, 1]

print(f"Baseline  AUC-PR = {average_precision_score(y_te, p_base):.4f}")
print(f"Baseline  Brier  = {brier_score_loss(y_te, p_base):.4f}")
Step 2 — Cost-sensitive class weights. 
weighted = lgb.LGBMClassifier(n_estimators=400, learning_rate=0.05,
                              class_weight={0: 1, 1: 100})
weighted.fit(X_tr, y_tr)
p_w = weighted.predict_proba(X_te)[:, 1]
Step 3 — Threshold from the cost matrix. 
COST_FP, COST_FN = 100, 10_000
thresh_opt = COST_FP / (COST_FP + COST_FN)  # ≈ 0.0099

pred_opt = (p_w >= thresh_opt).astype(int)
fp = ((pred_opt == 1) & (y_te == 0)).sum()
fn = ((pred_opt == 0) & (y_te == 1)).sum()
cost = fp * COST_FP + fn * COST_FN
print(f"Total business cost at t={thresh_opt:.4f}: ${cost:,}")
Step 4 — Post-hoc calibration. 
from sklearn.calibration import CalibratedClassifierCV

cal = CalibratedClassifierCV(weighted, method="isotonic", cv=3)
cal.fit(X_tr, y_tr)
p_cal = cal.predict_proba(X_te)[:, 1]

print(f"Calibrated  Brier = {brier_score_loss(y_te, p_cal):.4f}")

The Brier score should drop from ~0.11 to ~0.08 after calibration. The ranking (AUC-PR) stays almost the same — calibration only moves probabilities, not orderings.

Common Mistakes

⚠ Mistake 1 — Reporting accuracy on imbalanced data

Already warned twice in the course. This one is the cheque your stakeholders will cash — so you must not write it in the first place. Report AUC-PR and Brier after calibration.

⚠ Mistake 2 — SMOTE-ing the test set

SMOTE is training-only. If you SMOTE the test set, you are measuring the model against synthetic points that do not exist in production. Split first, SMOTE the training split only.

⚠ Mistake 3 — Leaving the threshold at 0.5

The default 0.5 is the Bayes-optimal threshold only if the two error costs are equal and the classes are balanced. Neither holds in most real problems. Derive the threshold from the cost matrix.
