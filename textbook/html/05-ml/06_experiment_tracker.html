Chapter 6: ExperimentTracker

Overview

ExperimentTracker records every aspect of your ML experiments: parameters, step-based metrics, tags, and run status. It uses an async context manager pattern that automatically marks runs as COMPLETED on normal exit and FAILED on exception.

Concepts

The Context Manager Pattern

async with tracker.run(experiment_name, run_name) as ctx: handles the full lifecycle:

  1. Auto-creates the experiment if it does not exist.
  2. Sets run status to RUNNING on entry.
  3. Marks COMPLETED on normal exit.
  4. Marks FAILED if an exception propagates out.
NaN/Inf rejection: ExperimentTracker rejects NaN and Inf metric values. These indicate bugs in training code, and storing them would corrupt comparison logic.

Key API Reference

MethodPurpose
ExperimentTracker(conn, artifact_root)Create tracker
tracker.run(exp_name, run_name)Async context manager for a run
ctx.log_params(dict)Log parameters (string values)
ctx.log_metric(key, value, step)Log metric at a step
ctx.log_metrics(dict, step)Log multiple metrics
ctx.set_tag(key, value)Set a run tag
tracker.compare_runs(run_ids)Side-by-side comparison
tracker.get_metric_history(run_id, key)Step-based metric entries
tracker.search_runs(exp, filter_params)Find runs by parameters

Code Walkthrough

Step 1: Track a Run

from kailash_ml.engines.experiment_tracker import ExperimentTracker

async with tracker.run("churn_experiment", run_name="baseline") as ctx:
    await ctx.log_params({
        "model_class": "RandomForestClassifier",
        "n_estimators": "100",
    })
    await ctx.log_metric("train_loss", 0.65, step=0)
    await ctx.log_metric("train_loss", 0.42, step=1)
    await ctx.log_metrics({"accuracy": 0.92, "f1": 0.89})
    await ctx.set_tag("team", "ml-platform")

baseline_run_id = ctx.run_id

Step 2: Verify Completion

run = await tracker.get_run(baseline_run_id)
assert run.status == "COMPLETED"
assert run.metrics["accuracy"] == 0.92

Step 3: Failed Run Handling

try:
    async with tracker.run("churn_experiment", run_name="crashed") as ctx:
        raise RuntimeError("Training crashed")
except RuntimeError:
    pass

failed = await tracker.get_run(ctx.run_id)
assert failed.status == "FAILED"

Step 4: Compare Runs

comparison = await tracker.compare_runs([baseline_id, improved_id])
assert comparison.metrics["accuracy"][0] == 0.92  # baseline
assert comparison.metrics["accuracy"][1] == 0.95  # improved

Step 5: Metric History

history = await tracker.get_metric_history(baseline_run_id, "train_loss")
# Returns [MetricEntry(step=0, value=0.65), MetricEntry(step=1, value=0.42), ...]

Step 6: Search Runs

results = await tracker.search_runs(
    "churn_experiment",
    filter_params={"model_class": "GradientBoostingClassifier"},
)
assert len(results) == 1

Key Takeaways

  • The async with tracker.run() pattern automatically manages run lifecycle.
  • Step-based metrics enable training curve visualization.
  • compare_runs() provides instant side-by-side comparison.
  • search_runs() filters by parameter values.
  • NaN/Inf values are rejected to maintain data integrity.
