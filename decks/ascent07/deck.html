7.1 Linear Regression as a Neural Network

Foundations

You already know linear regression from M3:

\hat{y} = w_1 x_1 + w_2 x_2 + b
x = inputs  |  w = weights  |  b = bias  |  ŷ = prediction

Now let's draw this as a picture...

Time: ~2 min. Start from the familiar. Every student has seen linear regression. We're going to show that a neural network is just a visual way to represent this same equation.

Linear Regression IS a Neural Network

Foundations
Key insight: A neural network with 1 layer and no activation function IS linear regression. There is no magic — just a visual way to represent the same math.
Time: ~3 min. Draw the connection slowly. Point at each wire and say "this wire carries weight w1". The output node just sums up all the weighted inputs plus bias. This IS the linear regression equation. Neural networks are not magic -- they are just a way of composing simple operations.

Training: Gradient Descent

Theory

We want to find weights that minimise the loss:

L = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2
Mean Squared Error — same loss as OLS regression
Gradient of loss w.r.t. weight
\frac{\partial L}{\partial w_1} = -\frac{2}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i) \cdot x_{i1}
Update rule
w_1 \leftarrow w_1 - \eta \frac{\partial L}{\partial w_1}

When the learning rate \eta is set correctly and we use all data, this recovers the OLS solution.

Time: ~3 min. Walk through the gradient computation. The key point: gradient descent is just "take a small step in the direction that reduces the error". For linear regression with MSE, this converges to the exact same answer as the closed-form OLS solution. The neural network framing gives us a GENERAL optimisation method that works even when there is no closed-form solution.

From OLS to Gradient Descent: Why Bother?

Theory

OLS (closed form)

\mathbf{w} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}
  • Exact solution in one step
  • Requires matrix inversion: O(p³)
  • Only works for linear models
  • Only works for MSE loss

Gradient Descent (iterative)

\mathbf{w} \leftarrow \mathbf{w} - \eta \nabla L(\mathbf{w})
  • Approximate, converges over iterations
  • O(np) per iteration — scales to billions
  • Works for ANY differentiable model
  • Works for ANY differentiable loss
Gradient descent is the universal training algorithm. It is what makes deep learning possible.
Time: ~3 min. This is why we care about gradient descent even though OLS gives us the exact answer for linear regression. OLS is a special case. Gradient descent is the GENERAL method. Once students accept gradient descent for linear regression (where they can verify it gives the right answer), the leap to neural networks is just "more layers, same optimisation".

Code: Linear Regression as a Neural Network

Foundations 
import torch
import torch.nn as nn

# Define: single neuron, no activation = linear regression
model = nn.Linear(in_features=2, out_features=1)

# Loss and optimiser — same as OLS
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

# Training loop
for epoch in range(1000):
    predictions = model(X_train)           # Forward pass
    loss = criterion(predictions, y_train)  # Compute loss
    loss.backward()                         # Compute gradients
    optimizer.step()                        # Update weights
    optimizer.zero_grad()                   # Reset gradients

# Compare: model.weight ≈ sklearn's .coef_
# Compare: model.bias   ≈ sklearn's .intercept_
Kailash bridge: TrainingPipeline wraps this loop with early stopping, checkpointing, and logging. You write the architecture; Kailash handles the engineering.
Time: ~3 min. Run this code live if possible. Show that the weights converge to the same values as sklearn LinearRegression. The four steps (forward, loss, backward, step) are the heartbeat of ALL neural network training. Students will see this pattern hundreds of times.

7.2 The Interaction Problem

Foundations

Linear regression cannot capture interactions between features unless you manually create them:

Linear model

\hat{y} = w_1 x_1 + w_2 x_2 + b

Can only draw straight lines. Cannot learn "price goes up when BOTH area AND location are good".

With manual interaction

\hat{y} = w_1 x_1 + w_2 x_2 + w_3 x_1 x_2 + b

YOU had to create x_1 x_2. What if there are 100 features? 4,950 possible pairs.

Time: ~3 min. Connect to M2 feature engineering. Students manually created interaction terms. With 100 features, there are C(100,2) = 4,950 pairs, and that's just 2-way interactions. 3-way? 161,700. This does not scale. What if the network could discover interactions automatically?

Hidden Layers: Automatic Interaction Discovery

Foundations

Each hidden neuron computes: h_j = \sigma(w_{j1}x_1 + w_{j2}x_2 + b_j)

The activation function \sigma creates non-linear combinations of inputs — interactions are discovered automatically!

Time: ~3 min. This is the pivotal slide. The hidden layer IS learned feature engineering. Each hidden neuron creates a new feature that is a non-linear combination of the inputs. The activation function is what makes it non-linear -- without it, stacking linear layers is still just linear regression (we'll prove this soon).

The XOR Problem: Why Hidden Layers Matter

Foundations

XOR truth table

x₁x₂y
000
011
101
110

No single straight line can separate the 0s from the 1s. Linear models fail.

With a hidden layer

h₁ = learns "at least one input is 1"

h₂ = learns "both inputs are 1"

output = h₁ AND NOT h₂

The hidden layer re-represents the data in a new space where it IS linearly separable!

Hidden layers = learned feature engineering. They transform data into a space where the problem becomes easy.
Time: ~3 min. XOR is the classic demonstration. Minsky and Papert (1969) showed single-layer perceptrons cannot learn XOR, which contributed to the first "AI winter". The hidden layer solves this by creating new features (h1, h2) in which the problem IS linearly separable. This is exactly what feature engineering does -- but automatically.

What the Hidden Layer Actually Learns

Theory

A hidden layer with k neurons computes a feature transformation:

\mathbf{h} = \sigma(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1) \quad \in \mathbb{R}^k
x ∈ R² (original features) → h ∈ Rᵌ (learned features)

This is exactly a feature transformation: n original features → k learned features.

Compare with M6 USML:
PCA: n → k (linear, fixed transformation)
Autoencoder: n → k (non-linear, learned transformation)
Hidden layer: n → k (non-linear, learned, task-optimised)
Time: ~3 min. Make the equivalence explicit. PCA finds the k directions of maximum variance (unsupervised, linear). An autoencoder learns a non-linear compression (unsupervised, learned). A supervised hidden layer learns features that are optimised for the TASK (prediction). This is why supervised DL features are often better than unsupervised ones -- they are shaped by the loss function.

7.3 Where DL's Power Comes From

Foundations

If we stack layers WITHOUT activation functions:

Layer 1
\mathbf{h}_1 = \mathbf{W}_1 \mathbf{x} + \mathbf{b}_1
Layer 2
\hat{\mathbf{y}} = \mathbf{W}_2 \mathbf{h}_1 + \mathbf{b}_2 = \mathbf{W}_2(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1) + \mathbf{b}_2 = \underbrace{(\mathbf{W}_2\mathbf{W}_1)}_{\mathbf{W}'}\mathbf{x} + \underbrace{(\mathbf{W}_2\mathbf{b}_1 + \mathbf{b}_2)}_{\mathbf{b}'}
Still just linear regression! Without activation functions, stacking layers adds parameters but no representational power. The product of linear transformations is itself a linear transformation.
Time: ~3 min. This is a crucial proof. Students must understand WHY activation functions are essential. Without them, 100 layers collapse to a single linear transformation. It's the non-linearity that creates representational power. This is also why the choice of activation function matters so much.

The Power Hierarchy

Foundations
ArchitectureWhat It Can LearnDecision Boundary
Linear (no hidden layer)Linear relationships onlyStraight line / hyperplane
1 hidden layer + activationAny continuous functionSmooth curves
Deep network (many layers)Hierarchical featuresComplex, compositional boundaries
Universal Approximation Theorem (Cybenko, 1989): A single hidden layer with enough neurons can approximate ANY continuous function on a compact domain to arbitrary precision.

But "enough neurons" might mean exponentially many. Deep networks are exponentially more efficient than wide ones (Telgarsky, 2016).

Time: ~3 min. The Universal Approximation Theorem sounds like "1 layer is enough" but it is misleading. Yes, 1 layer CAN approximate anything, but it might need an exponential number of neurons. Deep networks achieve the same approximation with polynomially many parameters. This is why "deep" learning, not just "wide" learning, took off.

Depth = Hierarchical Feature Learning

Foundations
Each layer builds on the previous. Layer 1 finds edges. Layer 2 combines edges into textures. Layer 3 combines textures into parts. This compositional structure is why depth is so powerful.
Time: ~3 min. This is one of the most important slides. The hierarchy of features mirrors how the human visual cortex works (V1 detects edges, V2 detects shapes, IT detects objects). This is NOT a coincidence -- both systems are optimised for the same task (visual recognition). Zeiler and Fergus (2013) visualised these learned features in real CNNs.

Width vs Depth

Advanced

Wide network (1 layer, many neurons)

  • Can approximate any function (UAT)
  • May need exponentially many neurons
  • Learns flat, non-compositional features
  • Hard to generalise: memorises rather than learns structure

Deep network (many layers, fewer neurons)

  • Polynomially efficient for hierarchical patterns
  • Learns compositional features (edges → parts → objects)
  • Better generalisation (parameter sharing)
  • But: harder to train (vanishing gradients — we'll solve this)
Telgarsky (2016): There exist functions that a depth-k network represents with O(1) neurons per layer, but a depth-(k-1) network needs O(2^k) neurons. Depth is exponentially more parameter-efficient than width.
Time: ~3 min. Advanced audience insight. The mathematical result by Telgarsky gives a formal justification for why deep networks are better. The intuition: composing functions is more efficient than summing them. Think of it like describing a face: "two eyes above a nose above a mouth" (compositional) vs. listing every possible pixel pattern that could be a face (flat).

7.4 The Neuron

Foundations

A single neuron does two things:

Step 1: Weighted sum

z = \sum_{i=1}^{n} w_i x_i + b

"How important is each input?"

Step 2: Activation

a = \sigma(z)

"Should this neuron fire?"

Think of it as a tiny decision-maker: it weighs evidence (step 1), then decides yes/no/how much (step 2).

Time: ~2 min. The biological analogy: a neuron receives signals (inputs), integrates them (weighted sum), and fires if the total exceeds a threshold (activation). The artificial neuron does the same thing with math. The activation function is what makes it non-linear.

Layers as Matrix Operations

Theory

A layer with m neurons taking n inputs is a matrix multiply:

\mathbf{a} = \sigma(\mathbf{W}\mathbf{x} + \mathbf{b})
W ∈ Rm×n  |  x ∈ Rn  |  b ∈ Rm  |  a ∈ Rm

Forward pass through a 3-layer network:

Layer 1
\mathbf{h}_1 = \sigma(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1)
Layer 2
\mathbf{h}_2 = \sigma(\mathbf{W}_2 \mathbf{h}_1 + \mathbf{b}_2)
Output
\hat{\mathbf{y}} = \mathbf{W}_3 \mathbf{h}_2 + \mathbf{b}_3
Time: ~3 min. Emphasise the dimensions. If layer 1 has 64 neurons and the input has 10 features, W1 is 64x10 (64 neurons, each looking at 10 inputs). The output is a 64-dimensional vector -- 64 learned features. This is the feature engineering happening inside the network.

Parameter Count

Theory

For a network with layers [n, h₁, h₂, ..., hₖ, m]:

\text{Parameters} = \sum_{l=1}^{L} (n_{l-1} \times n_l + n_l) = \sum_{l=1}^{L} n_l(n_{l-1} + 1)
weights + biases for each layer

Example: [784, 256, 128, 10]

Activation Functions: The Complete Catalog

Foundations

The activation function decides how a neuron responds. Without it, everything collapses to linear regression.

Time: ~3 min. Give the practical decision guide first. Most students will use ReLU for everything and that is fine. The nuances (GELU for transformers, sigmoid for gates) come later when they need them. The key point: the activation function is what gives the network its non-linear power.

Sigmoid and Tanh: The Originals

Theory

Sigmoid

\sigma(z) = \frac{1}{1 + e^{-z}}

Gradient: \sigma'(z) = \sigma(z)(1 - \sigma(z))

Maximum gradient at z=0: \sigma'(0) = 0.25

Problem: gradient ≤ 0.25 always — multiplied across layers, it vanishes

Tanh

\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}

Gradient: \tanh'(z) = 1 - \tanh^2(z)

Maximum gradient at z=0: \tanh'(0) = 1.0

Better: zero-centred, stronger gradient

Still saturates at extremes

Time: ~3 min. Show why sigmoid causes vanishing gradients. Maximum gradient is 0.25. Through 10 layers: 0.25^10 = 9.5e-7. The gradient signal from the loss barely reaches early layers. Tanh is better (max gradient 1.0) but still saturates. Both were used historically; ReLU replaced them for hidden layers.

ReLU and Variants

Theory

ReLU

\text{ReLU}(z) = \max(0, z)

Gradient: 0 if z < 0, 1 if z > 0

No vanishing gradient (for z > 0)

Dying ReLU: if z < 0, gradient = 0 forever

Leaky ReLU

\text{LeakyReLU}(z) = \max(\alpha z, z), \quad \alpha \approx 0.01

Small gradient even for negative z — prevents dying neurons

ELU

\text{ELU}(z) = \begin{cases} z & z > 0 \\ \alpha(e^z - 1) & z \le 0 \end{cases}

Smooth at zero, negative values push mean toward zero

Swish / SiLU

\text{Swish}(z) = z \cdot \sigma(z)

Self-gated, smooth, non-monotonic. Often outperforms ReLU.

Time: ~3 min. ReLU is the workhorse. Its gradient is either 0 or 1 -- perfect for gradient flow. The "dying ReLU" problem is real: if a neuron gets a very negative input, it will output 0 and never recover because the gradient is also 0. Leaky ReLU fixes this cheaply. ELU is smoother. Swish/SiLU is used in many modern architectures.

GELU: The Transformer Activation

Advanced
\text{GELU}(z) = z \cdot \Phi(z) \approx z \cdot \sigma(1.702z)
Φ = Gaussian CDF = probability that a standard normal is ≤ z

Why GELU for transformers?

  • Stochastic regularisation interpretation: randomly zeroes out neurons proportional to their magnitude
  • Smooth approximation of ReLU — no sharp corner at 0
  • Empirically outperforms ReLU in attention-heavy architectures
  • Used in BERT, GPT-2, GPT-3, and most modern LLMs

Mish

\text{Mish}(z) = z \cdot \tanh(\text{softplus}(z))

Even smoother than GELU. Used in YOLOv4+ and some modern vision models.

\text{softplus}(z) = \ln(1 + e^z)

Time: ~3 min. Advanced audience. GELU has a beautiful probabilistic interpretation: it weights each input by the probability that it is "positive enough" under a Gaussian. This creates a soft gating effect -- inputs near zero are partially transmitted, very negative inputs are suppressed, very positive inputs pass through. This is similar to dropout but deterministic and differentiable.

Softmax: The Output Activation

Theory

For multi-class classification, the output layer uses softmax:

\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}
Converts K raw scores (logits) into a valid probability distribution that sums to 1

Properties

  • All outputs ∈ (0, 1)
  • All outputs sum to 1
  • Preserves ranking: largest logit → highest probability
  • Temperature scaling: \frac{e^{z_i/T}}{\sum e^{z_j/T}}

Temperature

  • T → 0: hard argmax (one-hot)
  • T = 1: standard softmax
  • T → ∞: uniform distribution
  • Used in knowledge distillation and LLM sampling
Time: ~3 min. Softmax converts raw logits into probabilities. The temperature parameter controls how "confident" the distribution is. Students will encounter temperature again in M5's LLM sampling (top-p, top-k). Connect: temperature scaling in softmax IS the same concept as temperature in LLM generation.

7.5 Loss Functions for Regression

Theory
LossFormulaWhen to use
MSE \frac{1}{n}\sum(y_i - \hat{y}_i)^2 Default. Penalises large errors heavily.
MAE \frac{1}{n}\sum|y_i - \hat{y}_i| Robust to outliers. Non-smooth at 0.
Huber \begin{cases} \frac{1}{2}(y-\hat{y})^2 & |y-\hat{y}| \le \delta \\ \delta|y-\hat{y}| - \frac{1}{2}\delta^2 & \text{otherwise} \end{cases} Best of both: MSE for small errors, MAE for large.
Log-cosh \sum \log(\cosh(y_i - \hat{y}_i)) Smooth approximation of MAE. Twice differentiable.
Time: ~3 min. MSE is the default for regression. It is equivalent to maximum likelihood under Gaussian noise assumption. MAE is robust to outliers. Huber combines both -- MSE when the error is small, MAE when it is large. The delta parameter controls the threshold. Log-cosh is like Huber but smoother, which helps optimisers.

Loss Functions for Classification

Theory

Binary Cross-Entropy

L = -[y\log(\hat{y}) + (1-y)\log(1-\hat{y})]

For 2-class problems. Output: sigmoid.

Categorical Cross-Entropy

L = -\sum_{k=1}^{K} y_k \log(\hat{y}_k)

For K-class problems. Output: softmax.

Focal Loss (Lin et al., 2017)

L = -\alpha(1-\hat{y})^\gamma \log(\hat{y})

Down-weights easy examples. Crucial for class imbalance.

\gamma = 0: standard CE. \gamma = 2: typical for detection.

Hinge Loss

L = \max(0, 1 - y \cdot \hat{y})

SVM-style. Margin-based. Less common in DL.

Time: ~3 min. Cross-entropy is the workhorse for classification. It has a beautiful property: when combined with softmax, the gradient simplifies to (prediction - target). Focal loss is critical for the kind of class imbalance problems they saw in M4 (the Credit Suisse case). It reduces the contribution of easy, well-classified examples so the model focuses on hard ones.

Embedding Losses (Preview)

Theory

Contrastive Loss

L = y \cdot d^2 + (1-y) \cdot \max(0, m-d)^2

Pull similar pairs close, push dissimilar pairs apart beyond margin m.

Triplet Loss

L = \max(0, d(a,p) - d(a,n) + m)

Anchor-positive-negative: make a closer to p than to n by at least margin m.

Why these matter: Embedding losses are how networks learn to create feature representations where similar items are close together. This is the heart of transfer learning and the foundation of text/image search.
Time: ~2 min. Preview for the embeddings section (Section G). Students don't need to memorise these yet. The key insight: these losses train the network to produce useful embeddings, not classifications. The embeddings ARE the learned features.

Weight Initialisation: Why It Matters

Theory

If we initialise weights too large or too small, activations explode or vanish after a few layers.

Xavier / Glorot (2010)

\text{Var}(w) = \frac{2}{n_{\text{in}} + n_{\text{out}}}

For sigmoid/tanh. Maintains variance of activations across layers.

Derivation: Set Var(output) = Var(input) assuming linear activations.

He / Kaiming (2015)

\text{Var}(w) = \frac{2}{n_{\text{in}}}

For ReLU. Accounts for ReLU killing half the activations (factor of 2).

Derivation: ReLU zeroes out ~50% of activations, doubling the required variance.

LSUV (Layer-Sequential Unit-Variance): Initialise weights, then rescale each layer so that activations have unit variance. Works for any activation function. More robust than analytical methods.
Time: ~3 min. Wrong initialisation is a silent killer. The network might train but converge to a bad solution, or not converge at all. Xavier is for sigmoid/tanh, He/Kaiming is for ReLU. PyTorch uses Kaiming uniform by default for linear layers. In practice, most students will never change the default, but they should understand WHY it works.

Fixup Initialisation for Deep ResNets

Advanced

For very deep residual networks (100+ layers), even He initialisation can cause training instability.

Fixup (Zhang et al., 2019): Scale residual branch weights by L^{-1/(2m-2)} where L is depth and m is blocks per residual branch.

This allows training very deep ResNets without batch normalisation, achieving comparable accuracy.

Why care? Batch normalisation adds complexity, introduces train/inference discrepancy, and breaks with small batch sizes. Fixup shows that careful initialisation can replace it entirely.
Time: ~2 min. Advanced only. Fixup is a beautiful result: it shows that batch normalisation is not fundamentally necessary -- it is compensating for bad initialisation. With the right initialisation, you can train 100+ layer networks without batch norm. This has implications for inference (no running statistics to track) and small-batch training.

7.6 The Chain Rule

Theory

If y = f(g(x)), then:

\frac{dy}{dx} = \frac{dy}{dg} \cdot \frac{dg}{dx}
Rate of change of y w.r.t. x = (rate of change of y w.r.t. g) × (rate of change of g w.r.t. x)

For neural networks, the loss is a composition of MANY functions:

L = \text{loss}(\text{output}(\text{activation}(\text{linear}(\text{activation}(\text{linear}(x))))))

The chain rule lets us compute how each weight affects the loss, no matter how deep the network.

Time: ~3 min. The chain rule is calculus 101. But for students who have not taken calculus, frame it as: "if changing w by a tiny amount changes z by a tiny amount, and changing z by a tiny amount changes the loss by a tiny amount, then we can multiply these tiny effects to get the total effect of w on the loss." This is the fundamental insight of backpropagation.

Worked Example: 2-Layer Network

Theory

Forward pass (storing all intermediate values):

Layer 1
z_1 = W_1 x + b_1 \quad \rightarrow \quad a_1 = \sigma(z_1)
Layer 2 (output)
z_2 = W_2 a_1 + b_2 \quad \rightarrow \quad \hat{y} = z_2
Loss
L = \frac{1}{2}(y - \hat{y})^2

We store z₁, a₁, z₂ during the forward pass. We need them for the backward pass.

Time: ~2 min. Set up the forward pass. Emphasise that we STORE all intermediate values. This is why neural networks use so much memory during training -- you need the activations from every layer for the backward pass. Inference only needs the current layer's output.

Backward Pass: Computing Every Gradient

Theory
Step 1: Loss → output
\frac{\partial L}{\partial \hat{y}} = -(y - \hat{y})
Step 2: Output → W₂, b₂
\frac{\partial L}{\partial W_2} = \frac{\partial L}{\partial \hat{y}} \cdot a_1^T \qquad \frac{\partial L}{\partial b_2} = \frac{\partial L}{\partial \hat{y}}
Step 3: Propagate to hidden layer
\frac{\partial L}{\partial a_1} = W_2^T \frac{\partial L}{\partial \hat{y}}
Step 4: Through activation
\frac{\partial L}{\partial z_1} = \frac{\partial L}{\partial a_1} \odot \sigma'(z_1)
Step 5: Hidden → W₁, b₁
\frac{\partial L}{\partial W_1} = \frac{\partial L}{\partial z_1} \cdot x^T \qquad \frac{\partial L}{\partial b_1} = \frac{\partial L}{\partial z_1}
Time: ~5 min. Walk through each step slowly. The pattern is: compute gradient of loss w.r.t. output of current layer, then use that to compute gradient w.r.t. weights (for the update) and w.r.t. input (to propagate backward). The key operation at each layer: multiply by the transpose of the weight matrix to propagate backward, and element-wise multiply by the activation derivative.

Backpropagation: The Algorithm

Foundations

Four steps, every iteration:

  1. Forward pass: Compute and store all intermediate values
  2. Compute loss: Compare prediction to target
  3. Backward pass: Compute gradients layer by layer, output to input
  4. Update weights: w \leftarrow w - \eta \cdot \nabla L
# PyTorch makes this automatic
for batch in dataloader:
    # 1. Forward pass
    predictions = model(batch.x)
    # 2. Compute loss
    loss = criterion(predictions, batch.y)
    # 3. Backward pass (automatic!)
    loss.backward()
    # 4. Update weights
    optimizer.step()
    optimizer.zero_grad()
PyTorch autograd handles step 3 automatically. You define the forward pass; PyTorch records the computational graph and computes all gradients for you.
Time: ~3 min. Reassure students: they will NOT compute gradients by hand in practice. PyTorch autograd does it. But understanding the algorithm is essential for debugging (why is my loss not decreasing? why are my gradients zero?) and for understanding advanced techniques (gradient clipping, gradient accumulation, mixed precision).

Jacobians and Vector-Jacobian Products

Advanced

For a function \mathbf{f}: \mathbb{R}^n \to \mathbb{R}^m, the Jacobian is:

J = \begin{bmatrix} \frac{\partial f_1}{\partial x_1} & \cdots & \frac{\partial f_1}{\partial x_n} \\ \vdots & \ddots & \vdots \\ \frac{\partial f_m}{\partial x_1} & \cdots & \frac{\partial f_m}{\partial x_n} \end{bmatrix}

Backpropagation never computes the full Jacobian. Instead, it computes Vector-Jacobian Products (VJPs):

\mathbf{v}^T J \quad \text{where } \mathbf{v} = \frac{\partial L}{\partial \mathbf{f}}

VJPs are computed in O(n) time, not O(nm). This is why backpropagation scales.

Time: ~3 min. For the mathematically inclined. The full Jacobian for a layer with 1000 inputs and 1000 outputs is a 1M-element matrix. But we only need the VJP: how does a scalar loss change when we change the input? This is a single vector, not a matrix. JAX and PyTorch both use VJPs internally.

Gradient Checking: Debugging Backpropagation

Theory

Numerical gradient approximation (for debugging only):

\frac{\partial L}{\partial w} \approx \frac{L(w + \epsilon) - L(w - \epsilon)}{2\epsilon}
Central difference, ε ≈ 10&sup{-7}

Compare with the analytical gradient from backpropagation:

\text{relative error} = \frac{|g_{\text{analytical}} - g_{\text{numerical}}|}{|g_{\text{analytical}}| + |g_{\text{numerical}}|} < 10^{-5}
Only for debugging! Numerical gradients are O(n) forward passes per parameter. For a network with 1M parameters, that's 1M forward passes just to check. Use only on small test networks.
Time: ~2 min. Gradient checking is how you verify that your custom layer's backward pass is correct. PyTorch has torch.autograd.gradcheck() for this. Students writing custom autograd functions should always run gradient checking.

7.7 Vanishing Gradients

Theory

The gradient of the loss w.r.t. an early layer's weights:

\frac{\partial L}{\partial w_1} = \frac{\partial L}{\partial a_L} \cdot \frac{\partial a_L}{\partial a_{L-1}} \cdot \ldots \cdot \frac{\partial a_2}{\partial a_1} \cdot \frac{\partial a_1}{\partial w_1}

If each factor |\frac{\partial a_l}{\partial a_{l-1}}| < 1, the product vanishes exponentially:

LayersSigmoid maxGradient magnitude
50.25&sup5;~10&sup{-3}
100.25&sup{10}~10&sup{-6}
200.25&sup{20}~10&sup{-12}

Solutions

  • ReLU: gradient is 0 or 1, no shrinking
  • Skip connections: gradient highway (ResNet)
  • Proper initialisation: He/Xavier
  • Batch normalisation: prevents extreme values
Time: ~3 min. This is why deep networks were considered impossible to train before 2010. The gradient signal from the loss reaches early layers as an astronomically small number, so those layers learn nothing. The solutions -- ReLU, skip connections, proper init -- are what unlocked the "deep" in deep learning.

Exploding Gradients and Gradient Clipping

Theory

If each factor |\frac{\partial a_l}{\partial a_{l-1}}| > 1, the product explodes:

\text{5 layers, factor 2:} \quad 2^5 = 32 \qquad \text{20 layers:} \quad 2^{20} = 1,048,576

Gradient Clipping

By value

g_i = \text{clip}(g_i, -\tau, \tau)

Simple but changes gradient direction.

By norm (preferred)

\mathbf{g} = \begin{cases} \mathbf{g} & \|\mathbf{g}\| \le \tau \\ \tau \frac{\mathbf{g}}{\|\mathbf{g}\|} & \|\mathbf{g}\| > \tau \end{cases}

Preserves gradient direction, scales magnitude.

# PyTorch gradient clipping
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
Time: ~3 min. Exploding gradients cause NaN losses and divergent training. Gradient clipping by norm is the standard solution. The max_norm parameter (typically 1.0 or 5.0) is a hyperparameter. In transformers, gradient clipping is almost always used. It is also essential for RNNs and LSTMs.

Dead Neurons (Dying ReLU)

Foundations

The problem

If a neuron's pre-activation z becomes very negative:

  • ReLU output = 0
  • ReLU gradient = 0
  • No gradient → no learning → permanently dead

Can happen with large learning rates that push weights too far negative.

Solutions

  • Leaky ReLU: small gradient (0.01) for negative inputs
  • ELU: exponential curve for negative inputs
  • Lower learning rate: prevent extreme weight updates
  • Batch normalisation: keeps pre-activations near zero
# Use Leaky ReLU to prevent dead neurons
nn.LeakyReLU(negative_slope=0.01)
Time: ~2 min. In practice, dying ReLU is rarely a catastrophic problem for well-initialised networks with reasonable learning rates. But students should know it exists and how to diagnose it (check for layers where a large fraction of activations are always zero).

7.8 Parallelised Backpropagation

Foundations

Modern models are too large for a single GPU. How do we train them across multiple GPUs?

Time: ~3 min. Overview of the four parallelism strategies. In practice, large models use ALL of these simultaneously (3D parallelism). GPT-3 used data + model + pipeline parallelism across thousands of GPUs.

Data Parallelism in Detail

Theory 
# PyTorch DistributedDataParallel (DDP)
model = torch.nn.parallel.DistributedDataParallel(model, device_ids=[local_rank])
Time: ~3 min. Data parallelism is the simplest and most common approach. Each GPU has a complete copy of the model, processes a different slice of the batch, computes local gradients, then ALL-REDUCE averages the gradients across all GPUs. The update is applied identically on all GPUs. Synchronous DDP is the standard in PyTorch. Effective batch size scales linearly with GPUs.

Model Parallelism and the Bubble Problem

Theory

When the model does not fit on one GPU, split layers across GPUs:

Time: ~3 min. Model parallelism is necessary for large models but has a fundamental problem: when GPU 0 is computing the forward pass for layers 1-4, GPUs 1 and 2 are idle. When GPU 2 computes the backward pass, GPUs 0 and 1 are idle. This creates "bubbles" of wasted compute. Pipeline parallelism solves this.

Pipeline Parallelism: Filling the Bubbles

Theory

GPipe (Huang et al., 2019)

  • Split mini-batch into M micro-batches
  • Pipeline micro-batches through stages
  • While GPU 0 processes micro-batch 2, GPU 1 processes micro-batch 1
  • Bubble fraction: \frac{S-1}{S+M-1} where S = stages, M = micro-batches
  • More micro-batches → smaller bubbles

PipeDream: 1F1B Schedule

  • Alternate: 1 Forward, 1 Backward
  • Each GPU stays busy: forward on one micro-batch, backward on another
  • Near-perfect GPU utilisation after warmup
  • Trades memory for utilisation
Interleaved 1F1B: Split each GPU's layers into non-contiguous chunks. Further reduces bubble.
Time: ~3 min. GPipe is the simpler approach: do all forwards, then all backwards, but on micro-batches. PipeDream interleaves forward and backward passes on different micro-batches, keeping GPUs busy. In practice, large model training (GPT-3, PaLM) uses interleaved 1F1B with hundreds of GPUs.

Tensor Parallelism

Advanced

Split individual operations (matrix multiplies) across GPUs:

\mathbf{Y} = \mathbf{X}\mathbf{W} = \mathbf{X}[\mathbf{W}_0 | \mathbf{W}_1] = [\mathbf{X}\mathbf{W}_0 | \mathbf{X}\mathbf{W}_1]
Column-parallel: split weight matrix columns across GPUs, each computes partial output

Megatron-LM style

  • First linear: column-parallel (split output features)
  • GeLU activation (no communication needed)
  • Second linear: row-parallel (split input features)
  • All-reduce after second linear

Sequence Parallelism

Split along the sequence dimension instead of (or in addition to) the feature dimension. Critical for long-context models. LayerNorm and dropout are computed on sequence-parallel chunks.

Time: ~3 min. Tensor parallelism is the most fine-grained. Instead of splitting layers, you split individual matrix operations. Megatron-LM showed how to do this efficiently for transformer blocks. Column-parallel splits the first projection, row-parallel splits the second, with only one all-reduce per block. Sequence parallelism further distributes the non-tensor-parallel operations.

ZeRO: Zero Redundancy Optimiser

Advanced

In data parallelism, each GPU stores: model parameters + gradients + optimiser states. ZeRO partitions these:

StagePartitionedMemory per GPUReduction
ZeRO-1Optimiser states~60% of baseline~1.7x
ZeRO-2+ Gradients~38% of baseline~2.6x
ZeRO-3+ Parameters~1/N of baseline~Nx
Why Adam is expensive: Adam stores 2 additional tensors per parameter (momentum + squared gradient). For a 7B model: 28GB parameters (FP32) + 28GB momentum + 28GB variance = 84GB just for optimiser state. ZeRO-1 partitions this across N GPUs.

Time: ~3 min. ZeRO is the key insight that makes training very large models feasible with data parallelism. Instead of duplicating everything on every GPU, partition it. The trade-off is communication: parameters must be gathered before each forward pass and gradients must be reduced after each backward pass. DeepSpeed (Microsoft) implements this.

Mixed Precision Training

Theory

The idea

  • Store weights in FP32 (master copy)
  • Compute forward/backward in FP16 or BF16
  • Accumulate gradients in FP32

Benefits

  • 2x speed (hardware FP16 units)
  • ~50% memory reduction
  • Same accuracy (with loss scaling)

Loss scaling

FP16 underflows for very small gradients. Multiply the loss by a large constant (e.g., 1024), compute gradients (now 1024x larger), then divide by 1024 before the update.

# PyTorch automatic mixed precision
scaler = torch.amp.GradScaler()
with torch.amp.autocast('cuda'):
    output = model(input)
    loss = criterion(output, target)
scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
BF16 vs FP16: BF16 has the same exponent range as FP32 (8 bits), so no loss scaling needed. FP16 has only 5 exponent bits, causing underflow. Modern GPUs (A100+, H100) support BF16 natively.
Time: ~3 min. Mixed precision is essentially free performance. Every modern training run uses it. BF16 is preferred when available because it eliminates the need for loss scaling. FlashAttention-3 on H100s can use FP8 for even more speed (1.3 PFLOPs/s), achieving 2.6x lower numerical error than naive FP8 through block quantization and incoherent processing.

D. Optimizers

How to navigate the loss landscape efficiently

Time: ~1 min. Section divider. The optimiser is what actually updates the weights using the gradients computed by backpropagation. The choice of optimiser affects convergence speed, final accuracy, and generalisation.

7.9 SGD: Stochastic Gradient Descent

Foundations
w \leftarrow w - \eta \nabla L(w; x_{\text{batch}})
η = learning rate (the most important hyperparameter in DL)

Three flavours

VariantBatch sizeTrade-off
Batch GDAll dataStable but slow
Mini-batch32-512Best trade-off
Stochastic1Noisy but fast

The learning rate controls step size:

  • Too large: oscillate, diverge
  • Too small: slow convergence, stuck in local minima
  • Just right: converges to good solution

Typical starting point: 0.01 for SGD, 0.001 for Adam

Time: ~3 min. SGD is the foundation. All other optimisers are modifications of SGD. The noise in mini-batch SGD is actually beneficial -- it helps escape shallow local minima and improves generalisation. The learning rate is the single most important hyperparameter in deep learning. Always tune it first.

SGD with Momentum

Theory

Physical analogy: a ball rolling downhill with momentum.

Velocity update
v \leftarrow \beta v + \nabla L(w)
Weight update
w \leftarrow w - \eta v

Without momentum: oscillates in steep directions, slow in flat directions

With momentum (\beta=0.9): accelerates through consistent gradients, dampens oscillations

Nesterov Accelerated Gradient

v \leftarrow \beta v + \nabla L(w - \eta \beta v)

"Look ahead" before computing gradient. Reduces overshooting. Converges faster for convex problems.

Time: ~3 min. Momentum is the first major improvement over vanilla SGD. The velocity term accumulates past gradients. If the gradient consistently points in the same direction, momentum builds up and takes larger steps. If the gradient oscillates, momentum dampens it. Nesterov's insight: compute the gradient at the "look-ahead" position, which gives a correction that reduces overshooting.

AdaGrad and RMSProp

Theory

AdaGrad (Duchi et al., 2011)

Accumulate
G \leftarrow G + (\nabla L)^2
Update
w \leftarrow w - \frac{\eta}{\sqrt{G + \epsilon}} \nabla L

Per-parameter learning rate. Frequent features get smaller updates.

Problem: G only grows → learning rate → 0

RMSProp (Hinton, unpublished)

Exponential moving average
G \leftarrow \gamma G + (1-\gamma)(\nabla L)^2
Update
w \leftarrow w - \frac{\eta}{\sqrt{G + \epsilon}} \nabla L

Fixes AdaGrad: uses exponential moving average instead of sum. Learning rate adapts but does not vanish.

Time: ~3 min. AdaGrad's insight: different parameters need different learning rates. A weight that receives large gradients (common features) should use a smaller learning rate. A weight that receives small gradients (rare features) should use a larger learning rate. RMSProp fixes AdaGrad's decaying learning rate by using a moving average instead of a sum.

Adam: The Default Optimiser

Theory

Adam combines momentum (first moment) with RMSProp (second moment):

First moment (mean)
m \leftarrow \beta_1 m + (1-\beta_1)\nabla L
Second moment (variance)
v \leftarrow \beta_2 v + (1-\beta_2)(\nabla L)^2
Bias correction
\hat{m} = \frac{m}{1-\beta_1^t} \qquad \hat{v} = \frac{v}{1-\beta_2^t}
Update
w \leftarrow w - \eta \frac{\hat{m}}{\sqrt{\hat{v}} + \epsilon}

Defaults: \beta_1 = 0.9, \beta_2 = 0.999, \epsilon = 10^{-8}, \eta = 0.001

Time: ~3 min. Adam is the go-to optimiser for most DL tasks. It adapts the learning rate per-parameter AND uses momentum. The bias correction terms account for the fact that m and v are initialised at zero and would be biased toward zero in early iterations. In practice, Adam with default hyperparameters works surprisingly well.

AdamW: Decoupled Weight Decay

Theory

L2 Regularisation in Adam (wrong)

w \leftarrow w - \eta \frac{\hat{m}}{\sqrt{\hat{v}} + \epsilon}

where \nabla L includes \lambda w term

Problem: the adaptive scaling in Adam also scales the regularisation, weakening it for parameters with large gradients.

AdamW (correct)

w \leftarrow w - \eta\left(\frac{\hat{m}}{\sqrt{\hat{v}} + \epsilon} + \lambda w\right)

Weight decay is applied AFTER the Adam update, not mixed into the gradient.

The regularisation is applied uniformly to all parameters.

Always use AdamW, not Adam + L2. The difference is subtle but significant. Loshchilov & Hutter (2019) showed that decoupled weight decay generalises better.
Time: ~3 min. This is a subtle but important point. In SGD, L2 regularisation and weight decay are equivalent. In Adam, they are NOT because Adam's adaptive learning rates scale the L2 gradient term differently for each parameter. AdamW applies weight decay directly to the weights, bypassing Adam's adaptive scaling. Always use AdamW.

Cutting-Edge Optimisers

Advanced

Lion (Chen et al., 2023)

Discovered by program search. Uses only the sign of the momentum, not the magnitude. Lower memory than Adam (no second moment). Competitive or better performance.

Sophia (Liu et al., 2023)

Second-order optimiser using diagonal Hessian approximation. Up to 2x faster convergence than AdamW on LLM pre-training. Automatically adapts to loss landscape curvature.

Muon (2025)

Orthogonalised gradient momentum via Newton-Schulz iteration. Expands the compute-loss Pareto frontier over AdamW. Strong hyperparameter transferability across scales. Promising for large-scale model training.

LAMB (You et al., 2020)

Layer-wise adaptive moments. Enables large-batch training (batch size 64K+) without accuracy loss. Used for BERT pre-training.

Time: ~3 min. Advanced audience only. Lion is interesting because it was found by SEARCHING over optimiser algorithms -- the AI designed the AI's optimiser. Sophia uses second-order information (curvature) cheaply. Muon is the latest contender, showing strong results in recent benchmarks. None of these have replaced AdamW as the universal default, but they are actively researched.

E. Regularisation

Preventing overfitting in powerful models

Time: ~1 min. Section divider. Deep networks have millions of parameters -- they can memorise the training data. Regularisation prevents this.

7.10 Dropout

Foundations

The idea

During training, randomly set each neuron's output to zero with probability p.

  • Forces the network to not rely on any single neuron
  • Like training an ensemble of sub-networks
  • Prevents co-adaptation of features

At inference

Use all neurons, but scale outputs by (1-p). Or use inverted dropout: scale by 1/(1-p) during training, no scaling at inference.

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 256)
        self.drop = nn.Dropout(p=0.5)
        self.fc2 = nn.Linear(256, 10)

    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = self.drop(x)  # 50% zeroed
        return self.fc2(x)

Typical: p=0.1-0.3 for small layers, p=0.5 for large layers.

Time: ~3 min. Dropout is the most widely used regularisation technique in DL. Srivastava et al. (2014). The ensemble interpretation: with p=0.5 and n neurons, dropout samples from 2^n possible sub-networks during training. At test time, the full network approximates the ensemble average. Inverted dropout (multiply by 1/(1-p) during training) is the standard implementation because it means no modification is needed at inference time.

Batch Normalisation

Theory
Normalise
\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}
Scale and shift (learnable)
y_i = \gamma \hat{x}_i + \beta

Training

\mu_B, \sigma_B^2 computed per mini-batch. Running averages tracked for inference.

Inference

Uses running mean/variance (fixed, not computed from current batch).

Does it really fix "internal covariate shift"? Santurkar et al. (2018) showed BatchNorm works mainly because it makes the loss landscape smoother (Lipschitz continuous gradients), not because it reduces covariate shift.
Time: ~3 min. Batch normalisation (Ioffe & Szegedy, 2015) was a breakthrough that enabled training of much deeper networks. The learnable gamma and beta allow the network to undo the normalisation if needed. The controversy: the original paper's explanation (reducing internal covariate shift) was challenged by Santurkar et al. (2018), who showed the real benefit is a smoother optimisation landscape.

Normalise across all features within a single sample. No batch dependency. Identical at train and inference time.

Why LayerNorm dominates transformers: Transformers process variable-length sequences with variable batch sizes. BatchNorm would need running statistics per position, which is impractical. LayerNorm works identically regardless of batch or sequence length.

Other Regularisation Techniques

Theory

Weight Decay

Penalise large weights: add \lambda\|w\|^2 to loss. Bayesian interpretation: Gaussian prior on weights. Keeps model simple.

Early Stopping

Monitor validation loss. Stop when it increases for N epochs (patience). Implicit regularisation -- limits the effective number of training iterations.

Data Augmentation

Images: flip, rotate, crop, colour jitter, Mixup, CutMix. Text: synonym replacement, back-translation. Increases effective dataset size.

Label Smoothing

Replace hard targets [0, 1] with soft targets [ε/K, 1-ε+ε/K]. Prevents overconfidence. Default ε=0.1.

Stochastic Depth

Randomly skip entire residual blocks during training. Survival probability decreases linearly with depth. Reduces training time and acts as regulariser.

SAM (Sharpness-Aware Minimisation)

Seek parameters in flat loss regions: \min_w \max_{\|\epsilon\| \le \rho} L(w + \epsilon). Flat minima generalise better. 2x compute cost.

Time: ~3 min. Survey of additional techniques. Weight decay and early stopping are used in almost every training run. Data augmentation is essential for vision (CutMix especially). Label smoothing and stochastic depth are used in most modern architectures. SAM finds flat minima (which generalise better) at the cost of 2x compute. Recent variants like GCSAM (2025) combine gradient centralisation with SAM for better stability.

F. Convolutional Neural Networks

Architecture-driven feature engineering for spatial data

Time: ~1 min. Section divider. CNNs are the first major example of ARCHITECTURE driving feature learning. The convolution operation constrains the network to learn local, translation-invariant features -- exactly the right inductive bias for images.

7.11 The Convolution Operation

Foundations

Think of it as sliding a small detector across the image:

Convolution = learned feature detection. The network learns WHAT to detect (the kernel values). Each kernel becomes a specialised feature detector.
Time: ~3 min. The convolution operation is elegant in its simplicity. Slide a small window (kernel) across the image, at each position compute a dot product. The kernel's values determine what pattern it detects. A vertical edge detector has negative values on the left, positive on the right. The network LEARNS these values through backpropagation -- it discovers the best detectors for the task.

Convolution Parameters

Theory

Output size formula

o = \frac{n + 2p - k}{s} + 1
n = input size  |  p = padding  |  k = kernel size  |  s = stride

Receptive field

The input region that affects a given output neuron. Grows with depth: after L layers of 3×3 convolutions, receptive field = 2L + 1.

Parameter count

\text{params} = k_h \times k_w \times C_{\text{in}} \times C_{\text{out}} + C_{\text{out}}

A 3×3 conv with 64 input and 128 output channels: 3×3×64×128 + 128 = 73,856 parameters

Compare to fully connected: 64×32×32 → 128×32×32 = 536M parameters!

Key insight: Convolutions share weights across spatial positions. This is why CNNs have far fewer parameters than fully connected networks.
Time: ~3 min. The parameter efficiency of convolutions is a key design insight. Weight sharing (same kernel applied at every position) means the number of parameters depends only on kernel size and channels, NOT on the spatial dimensions. This is why a CNN can process a 1000x1000 image with the same number of parameters as a 32x32 image. This is also the inductive bias: the same feature should be detected regardless of WHERE it appears in the image.

CNN Architecture Components

Foundations

Pooling

  • Max pooling: take the maximum value in each window
  • Average pooling: take the mean
  • Reduces spatial dimensions, increases receptive field
  • Provides translation invariance

Global Average Pooling

Replace fully connected layers with global average over each feature map. Reduces parameters dramatically. Used in most modern architectures.

1×1 Convolutions

Kernel size = 1. Does not look at spatial neighbours. Purpose: mix channels, reduce/increase channel dimensions.

Think of it as a per-pixel fully connected layer across channels.

Strided Convolutions

Use stride > 1 instead of pooling. Learned downsampling. Common in modern architectures (replaces max pooling).

# Common CNN block
nn.Conv2d(64, 128, kernel_size=3, padding=1)
nn.BatchNorm2d(128)
nn.ReLU()
nn.MaxPool2d(kernel_size=2, stride=2)
Time: ~3 min. Pooling reduces spatial resolution and increases the effective receptive field. Max pooling keeps the strongest activation (most prominent feature). Global average pooling (Lin et al., 2013) replaced the massive FC layers at the end of CNNs, reducing parameters and overfitting. 1x1 convolutions (Network in Network) are channel mixers -- they allow the network to combine features across channels without looking at spatial context.

ResNet: The Skip Connection Revolution

Theory

The residual block

y = F(x) + x
x = input (identity shortcut)  |  F(x) = learned residual

Instead of learning the full mapping H(x), learn the residual F(x) = H(x) - x.

If the optimal transformation is close to identity, F(x) just needs to be close to zero (easier to learn).

Why it works

  • Gradient highway: gradients flow directly through the skip connection, bypassing many layers
  • \frac{\partial y}{\partial x} = \frac{\partial F}{\partial x} + I — gradient is always at least I
  • No vanishing gradient problem
  • Enabled training of 152-layer (and 1000+ layer) networks
class ResBlock(nn.Module):
    def forward(self, x):
        return F.relu(self.conv2(
            F.relu(self.conv1(x))) + x)
Time: ~3 min. ResNet (He et al., 2015) is arguably the most important architecture innovation since backpropagation. The skip connection is simple but profound: it creates a "gradient highway" that lets gradients flow directly from the loss to early layers. The network only needs to learn the RESIDUAL -- the difference from identity. This is much easier than learning the full transformation from scratch.

Pre-Activation ResNet

Advanced

Original (post-activation)

x → Conv → BN → ReLU → Conv → BN → (+x) → ReLU

Problem: ReLU after addition can block gradient flow through the shortcut.

Pre-activation (He et al., 2016)

x → BN → ReLU → Conv → BN → ReLU → Conv → (+x)

Identity shortcut is completely clean. Better gradient flow. Better results on CIFAR and ImageNet.

ConvNeXt (2022): Modernised ResNet with transformer-inspired design choices: larger kernels (7×7), fewer activation functions, LayerNorm instead of BatchNorm, GELU instead of ReLU. Competitive with Vision Transformers while being purely convolutional. ConvNeXt V2 further improves accuracy with self-supervised pre-training.
Time: ~3 min. Pre-activation ResNet is a subtle but important improvement. The original ResNet has ReLU after the addition, which means the shortcut path also goes through a non-linearity. Pre-activation moves BN and ReLU before the convolutions, keeping the identity path completely linear. ConvNeXt showed that many of the transformer's advantages (large receptive field, fewer activations, better normalisation) can be imported into CNNs.

H. Advanced Topics

Generative Adversarial Networks (GANs)

Advanced

The game

  • Generator G: creates fake data from noise
  • Discriminator D: tries to distinguish real from fake
  • They improve each other through competition
\min_G \max_D \; \mathbb{E}[\log D(x)] + \mathbb{E}[\log(1-D(G(z)))]

Challenges

  • Mode collapse: generator produces only a few types of output
  • Training instability: oscillation, no convergence
  • Evaluation: no good loss metric for quality

WGAN (Arjovsky et al., 2017)

Uses Wasserstein (Earth Mover's) distance instead of JS divergence. More stable training, meaningful loss.

Time: ~3 min. GANs are conceptually beautiful: two networks playing a game until the generator produces data indistinguishable from real data. In practice, they are notoriously hard to train. WGAN improved stability significantly. Modern generative models (diffusion models) have largely replaced GANs for image generation, but GANs remain important for understanding adversarial training.

Autoencoders

Advanced

Architecture

Encoder compresses input → bottleneck → Decoder reconstructs input.

The bottleneck IS a learned compressed representation (embedding).

TypeSpecial property
VanillaReconstruct input
DenoisingReconstruct from corrupted input
SparsePenalise active neurons (L1 on activations)
ContractivePenalise sensitivity to input (Frobenius norm of Jacobian)

Variational Autoencoder (VAE)

z \sim \mathcal{N}(\mu, \sigma^2)

Reparameterisation trick: z = \mu + \sigma \odot \epsilon, \epsilon \sim \mathcal{N}(0,1)

L = L_{\text{reconstruct}} + D_{KL}(q(z|x) \| p(z))

KL term regularises the latent space to be smooth and continuous. Can generate new data by sampling z.

Time: ~3 min. Autoencoders are the purest form of "learned feature engineering": force the network to compress data through a bottleneck, and the bottleneck representation IS a set of learned features. VAEs add a probabilistic interpretation, making the latent space smooth and enabling generation of new data by sampling. The reparameterisation trick is how we backpropagate through a sampling operation.

Graph Neural Networks

Advanced

Message Passing Framework

For each node, at each layer:

  1. Aggregate: collect messages from neighbours
  2. Update: combine with own features
h_v^{(l+1)} = \text{UPDATE}\left(h_v^{(l)}, \text{AGG}(\{h_u^{(l)} : u \in \mathcal{N}(v)\})\right)

Key architectures

  • GCN: mean aggregation, shared weights
  • GAT: attention-weighted aggregation
  • GraphSAGE: sampled neighbourhood, generalises to unseen nodes

Applications

  • Social networks (friend recommendation)
  • Molecular graphs (drug discovery)
  • Knowledge graphs (entity relationships)
Time: ~3 min. GNNs extend the DL paradigm to graph-structured data. The message passing framework is analogous to convolution: instead of sliding a kernel over a grid, each node aggregates information from its neighbours. After L layers of message passing, each node's representation incorporates information from its L-hop neighbourhood. This is "architecture-driven feature engineering" for graphs.

State Space Models

Advanced

The Transformer limitation

Attention is O(n²) in sequence length. For a 100K-token document, that's 10 billion operations per layer.

S4 (Gu et al., 2022)

Structured State Spaces: maps continuous dynamical systems to discrete sequences. O(n log n) via FFT. Handles sequences of 16,000+ tokens.

Mamba (Gu & Dao, 2024)

Selective state spaces: input-dependent state transitions. Linear scaling in sequence length. Competitive with transformers on language tasks.

Mamba-3 (2026)

Complex-valued state updates, MIMO formulation. At 1.5B scale: +0.6pp average downstream accuracy vs. Mamba-2, with half the state size using MIMO variant.

Hybrid approaches (2025): TransMamba combines transformer and Mamba features. The consensus is forming: hybrids often outperform pure transformers or pure SSMs.
Time: ~3 min. State space models are the primary challenger to transformers for sequence modelling. The key insight: instead of attending to all positions (O(n^2)), maintain a compressed state that is updated recurrently (O(n)). Mamba made SSMs competitive by making the state transitions input-dependent (selective). The field is currently exploring hybrids that combine transformer and SSM components.
