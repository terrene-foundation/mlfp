Lesson 2.6 Logistic Regression and Classification Foundations

Linear regression predicts continuous outcomes. But many questions in business and medicine have binary answers — did the customer convert? did the patient survive? This lesson introduces logistic regression as linear regression in the log-odds space, derives the sigmoid link, interprets coefficients as odds ratios, and closes with one-way ANOVA for multi-group comparisons.

Why not linear regression for binary outcomes?

You could fit a linear regression with $y \in \{0, 1\}$. It would even work, sort of. But linear predictions can exceed 1 or drop below 0, which has no probability interpretation. And the constant-variance assumption is violated for binary data — the variance depends on the probability. We need a model whose output is bounded in $[0, 1]$ and whose optimisation respects the Bernoulli likelihood.

The sigmoid and the log-odds link

Define the odds of an event with probability $p$ as $p / (1 - p)$. The log-odds (or logit) is:

$$\text{logit}(p) \;=\; \log\!\left(\frac{p}{1 - p}\right)$$

As $p$ ranges from 0 to 1, the log-odds ranges from $-\infty$ to $+\infty$. Logistic regression assumes the log-odds is a linear function of the predictors:

$$\log\!\left(\frac{p}{1 - p}\right) \;=\; \beta_0 + \beta_1 x_1 + \cdots + \beta_k x_k \;=\; z$$
linear predictor $z$ lives in $(-\infty, \infty)$ so we transform it back to a probability

Inverting the link

Given $z$, we can recover $p$ by solving for it. Start from $\log(p/(1-p)) = z$:

$$\begin{aligned} \frac{p}{1 - p} &\;=\; e^z \\ p &\;=\; (1 - p)\,e^z \\ p(1 + e^z) &\;=\; e^z \\ p &\;=\; \frac{e^z}{1 + e^z} \;=\; \frac{1}{1 + e^{-z}} \;=\; \sigma(z) \end{aligned}$$

That last function $\sigma$ is the sigmoid. It squashes the linear predictor into a valid probability. Logistic regression is exactly "linear regression in the log-odds, then sigmoid to get back probabilities".

The Bernoulli likelihood

For a binary outcome $y_i \in \{0, 1\}$ with probability $p_i = \sigma(x_i^\top \beta)$, the likelihood is Bernoulli:

$$P(y_i \mid x_i, \beta) \;=\; p_i^{y_i}\,(1 - p_i)^{1 - y_i}$$

The log-likelihood over $n$ observations is:

$$\ell(\beta) \;=\; \sum_{i=1}^{n} \big[y_i \log p_i + (1 - y_i) \log(1 - p_i)\big]$$

Substitute $p_i = \sigma(x_i^\top \beta)$ and you get an equation with no closed-form solution. Unlike OLS, logistic regression has no $(X^\top X)^{-1} X^\top y$ — you optimise it numerically.

🔵 Theory: why no closed form?

For OLS, the gradient of the loss is linear in $\beta$, so setting it to zero gives a linear system. For logistic regression, the gradient is $X^\top (p - y)$ where $p$ itself depends non-linearly on $\beta$ through the sigmoid. Solving $X^\top (p(\beta) - y) = 0$ requires an iterative method like Newton-Raphson (also called Iteratively Reweighted Least Squares, IRLS). Scipy's minimize with BFGS does this under the hood.

Odds ratios: the multiplicative interpretation

From $\log(p / (1 - p)) = \beta_0 + \beta_1 x_1 + \cdots$, a one-unit increase in $x_j$ adds $\beta_j$ to the log-odds, which multiplies the odds themselves by $e^{\beta_j}$:

$$\frac{\text{odds}(x_j + 1)}{\text{odds}(x_j)} \;=\; e^{\beta_j}$$
$e^{\beta_j}$ is called the odds ratio
$\beta_j$$e^{\beta_j}$Interpretation
$+0.69$$\approx 2.0$Odds double per unit increase in $x_j$
$+0.10$$\approx 1.10$10% increase in odds per unit

⚠ Odds are not probabilities

An odds ratio of 2 does not mean "twice as likely". It means the odds — the ratio $p/(1-p)$ — are twice as big. For a baseline probability of 10%, doubling the odds gives about 18%, not 20%. Always convert back to probabilities when reporting to non-specialists.

Multiclass extensions

For outcomes with more than two classes, three common approaches:

  • One-vs-rest: fit $k$ binary classifiers, each separating one class from the rest. Pick the one with the highest predicted probability.
  • One-vs-one: fit $\binom{k}{2}$ binary classifiers and vote. More classifiers but each deals with less-imbalanced data.
  • Multinomial (softmax) regression: a single model with one coefficient vector per class, trained jointly: $P(y = j \mid x) = e^{z_j} / \sum_{m} e^{z_m}$. The natural extension and what sklearn uses by default.

One-way ANOVA: comparing three or more groups

ANOVA (Analysis of Variance) generalises the two-sample $t$-test to three or more groups. It asks: are the group means different? without saying which group differs from which (that is the job of post-hoc tests). The test decomposes total variance into between-group and within-group components:

$$\text{SS}_{\text{total}} \;=\; \text{SS}_{\text{between}} + \text{SS}_{\text{within}}$$
between: group means vs grand mean within: observations vs their own group mean

The test statistic is the ratio of mean squares:

$$F \;=\; \frac{\text{MS}_{\text{between}}}{\text{MS}_{\text{within}}} \;=\; \frac{\text{SS}_{\text{between}} / (k - 1)}{\text{SS}_{\text{within}} / (n - k)}$$

where $k$ is the number of groups and $n$ is the total sample size. Under $H_0$ (all groups have the same mean), $F \sim F_{k-1, n-k}$. Large $F$ → group means differ.

🔵 Theory: why $F$, not $t$?

With two groups, ANOVA's $F$ equals the square of the two-sample $t$-statistic. ANOVA is the generalisation of the $t$-test to more groups; it reduces to the same test in the two-group case.

Post-hoc tests: Tukey's HSD

ANOVA tells you that the group means are not all equal. It does not tell you which groups differ. Running pairwise $t$-tests inflates the false-positive rate (back to the multiple testing problem from 2.3). Tukey's HSD (Honest Significant Difference) corrects for this: it computes all pairwise differences and compares them to a critical value based on the studentised range distribution, giving family-wise error control:

$$\text{HSD} \;=\; q_{\alpha, k, n - k} \cdot \sqrt{\frac{\text{MS}_{\text{within}}}{n_{\text{group}}}}$$

Two group means are significantly different if their difference exceeds HSD.

Worked Example: ICU mortality prediction

📝 Goal

Predict ICU 30-day mortality from age, APACHE score, and admission type using logistic regression. Interpret the coefficients as odds ratios and evaluate with a confusion matrix and AUC.

Step 1 — Load and prepare. 
import polars as pl
import numpy as np
from scipy.optimize import minimize
from shared import MLFPDataLoader

df = MLFPDataLoader().load("mlfp02", "icu_patients.parquet")
df = df.with_columns(died=pl.col("outcome_30d").eq("died").cast(pl.Int8))

X_cols = ["age", "apache_score", "emergency_admission"]
X = df.select(X_cols).to_numpy()
X = np.column_stack([np.ones(len(X)), X])
y = df["died"].to_numpy()
Step 2 — Define the sigmoid and the loss. 
def sigmoid(z):
    # Numerically stable form
    return np.where(z >= 0, 1 / (1 + np.exp(-z)), np.exp(z) / (1 + np.exp(z)))

def neg_log_lik(beta, X, y):
    z = X @ beta
    p = sigmoid(z)
    eps = 1e-12
    return -np.sum(y * np.log(p + eps) + (1 - y) * np.log(1 - p + eps))
Step 3 — Optimise. 
beta0 = np.zeros(X.shape[1])
result = minimize(neg_log_lik, beta0, args=(X, y), method="BFGS")
beta_hat = result.x
names = ["intercept"] + X_cols
for name, b in zip(names, beta_hat):
    print(f"{name:>20s}: {b:+.4f}  (OR = {np.exp(b):.2f})")
Step 4 — Predict, confusion matrix, AUC. 
from sklearn.metrics import confusion_matrix, roc_auc_score

p_hat = sigmoid(X @ beta_hat)
y_hat = (p_hat >= 0.5).astype(int)
cm = confusion_matrix(y, y_hat)
auc = roc_auc_score(y, p_hat)
print("Confusion matrix:\n", cm)
print(f"Accuracy: {(y == y_hat).mean():.3f}")
print(f"AUC:      {auc:.3f}")
Step 5 — Interpret.

Suppose the coefficient on APACHE score is $\hat\beta = 0.08$, so $e^{0.08} \approx 1.08$. Each one-point increase in APACHE multiplies the mortality odds by about 1.08 — an 8% increase in odds per APACHE point, holding age and admission type constant. For a 20-point APACHE increase, odds multiply by $e^{1.6} \approx 4.95$.

Worked: ANOVA on HDB flat types

📝 Goal

Do 3-room, 4-room, 5-room, and executive flats have different average resale prices? Use one-way ANOVA and Tukey's HSD.

Step 1 — ANOVA. 
from scipy import stats

groups = {
    "3-room": df.filter(pl.col("flat_type") == "3 ROOM")["resale_price"].to_numpy(),
    "4-room": df.filter(pl.col("flat_type") == "4 ROOM")["resale_price"].to_numpy(),
    "5-room": df.filter(pl.col("flat_type") == "5 ROOM")["resale_price"].to_numpy(),
    "exec":   df.filter(pl.col("flat_type") == "EXECUTIVE")["resale_price"].to_numpy(),
}
F, p = stats.f_oneway(*groups.values())
print(f"ANOVA: F = {F:.1f}, p = {p:.3g}")
Step 2 — Tukey's HSD. 
from statsmodels.stats.multicomp import pairwise_tukeyhsd

prices = np.concatenate(list(groups.values()))
labels = np.concatenate([[k] * len(v) for k, v in groups.items()])
result = pairwise_tukeyhsd(prices, labels, alpha=0.05)
print(result)

Tukey tells you which pairs of flat types differ significantly. In practice all six pairs will be flagged because the sample is huge and the differences are real. With smaller samples or smaller effects, Tukey lets you say which groups carry the ANOVA result.

Common mistakes

⚠ Mistake 1: Thresholding at 0.5 blindly

The 0.5 cutoff is arbitrary. If one class is rare or one type of error is more costly, pick the threshold that optimises your business metric, not 0.5.

⚠ Mistake 2: Reporting accuracy on imbalanced data

95% accuracy sounds great — until you learn the base rate is 95%. Always report the confusion matrix and AUC alongside accuracy, or use balanced metrics like F1.

⚠ Mistake 3: Running many $t$-tests instead of ANOVA

With 4 groups you have 6 pairs. Running 6 $t$-tests at $\alpha = 0.05$ inflates family-wise error to about 26%. Use ANOVA first; if significant, use Tukey for post-hoc.
