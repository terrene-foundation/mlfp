RNNs and Sequence Models

Vanilla RNN · LSTM · GRU · temporal attention

⏱ 60 minutes · Prerequisite: 5.2 (PyTorch training loop)

Sequences need their own architecture

  • Variable length — 5 words or 500, no fixed input size
  • Temporal dependency — "bank" near "river" ≠ "bank" near "deposit"

The recurrent idea: carry a hidden state forward, update it step by step, reuse the same weights at every step.

The vanilla RNN update

\[ h_t = \phi(W_{xh}\, x_t + W_{hh}\, h_{t-1} + b) \]

Same weight matrices at every time step. Weight sharing is exactly like a conv kernel — but across time instead of space.

The RNN unrolled

The cruel flaw: vanishing gradients

Backprop through a long sequence = product of T Jacobians.

  • Each Jacobian has norm < 1 → gradient shrinks exponentially (vanish)
  • Each Jacobian has norm > 1 → gradient blows up (explode)

Vanilla RNNs are unusable past ~20 time steps. The 1997 fix: LSTM.

LSTM: cell state highway

Adds a second state — the cell state — that flows forward with only additive updates, protected by three gates.

Additive path → gradient flows through almost unchanged. That is the magic.

🔵 The cell state is the ancestor of ResNet's skip connections (2015).

LSTM gate equations — all six

\[ \begin{aligned} f_t &= \sigma(W_f[h_{t-1}, x_t] + b_f) && \text{forget}\\ i_t &= \sigma(W_i[h_{t-1}, x_t] + b_i) && \text{input}\\ \tilde C_t &= \tanh(W_C[h_{t-1}, x_t] + b_C) && \text{candidate}\\ C_t &= f_t \odot C_{t-1} + i_t \odot \tilde C_t && \text{cell update}\\ o_t &= \sigma(W_o[h_{t-1}, x_t] + b_o) && \text{output}\\ h_t &= o_t \odot \tanh(C_t) && \text{hidden} \end{aligned} \]

Walk the equations

  • Forget f_t ∈ [0,1]: "keep how much of past cell state?"
  • Input i_t ∈ [0,1]: "accept how much new content?"
  • Candidate \tilde C_t ∈ [-1,1]: "what is the new content?"
  • Cell update: forget some past + accept some new — additive
  • Output o_t: "how much of the cell to reveal?"
  • Hidden h_t: output-gated tanh of cell

Why it works: gradient highway

Vanilla RNN: \( \partial h_t / \partial h_{t-1} \) involves W_{hh} and a tanh derivative → decays exponentially.

LSTM: \( \partial C_t / \partial C_{t-1} \approx f_t \). If \( f_t \approx 1 \), gradient passes through unchanged.

🟢 Same insight as ResNet: give the gradient an additive path back.

GRU — a simpler cousin (2014)

\[ \begin{aligned} z_t &= \sigma(W_z[h_{t-1}, x_t]) && \text{update}\\ r_t &= \sigma(W_r[h_{t-1}, x_t]) && \text{reset}\\ \tilde h_t &= \tanh(W[r_t \odot h_{t-1}, x_t])\\ h_t &= (1 - z_t)\odot h_{t-1} + z_t \odot \tilde h_t \end{aligned} \]

One gate merged, no separate cell state. Fewer parameters, usually competitive with LSTM.

LSTM regressor in PyTorch


class LSTMRegressor(nn.Module):
    def __init__(self, input_size=1, hidden_size=32, num_layers=2):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers,
                            batch_first=True, dropout=0.1)
        self.head = nn.Linear(hidden_size, 1)

    def forward(self, x):
        out, _ = self.lstm(x)               # (B, L, H)
        last = out[:, -1, :]                # final time step
        return self.head(last).squeeze(-1)

Training — clip those gradients


opt = torch.optim.Adam(model.parameters(), lr=1e-3)
for epoch in range(30):
    for xb, yb in loader:
        pred = model(xb)
        loss = F.mse_loss(pred, yb)
        opt.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(
            model.parameters(), max_norm=1.0)   # ★ essential
        opt.step()

Without gradient clipping, RNNs explode on ~1% of batches and NaN your loss.

Temporal attention

Instead of taking only the last time step, compute a learned weighted sum:


out, _ = self.lstm(x)                         # (B, L, H)
scores = self.attn(out).squeeze(-1)           # (B, L)
weights = F.softmax(scores, dim=-1).unsqueeze(-1)
context = (weights * out).sum(dim=1)          # (B, H)

Four traps

  • Forgetting gradient clipping — always clip L2 norm to 1.0
  • Wrong batch_first — default is (L, B, F); modern is (B, L, F)
  • Stateful state leakage — don't reuse h across unrelated sequences
  • BPTT memory blow-up — truncate when seq_len × hidden × batch is too large
