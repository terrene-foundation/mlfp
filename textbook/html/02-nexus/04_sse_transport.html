Chapter 4: SSE Transport

Overview

Server-Sent Events (SSE) provide one-way real-time streaming from server to client over HTTP. Nexus uses SSE to stream EventBus events to browser dashboards, monitoring tools, and other clients. This chapter covers register_sse_endpoint(), the EventBus class, event publishing and subscription, and SSE filtering.

Prerequisites

  • Completed Chapters 1–3
  • Understanding of HTTP streaming concepts
  • Familiarity with async generators in Python

Concepts

What Are Server-Sent Events?

SSE is a W3C standard for one-way streaming from server to client over HTTP. Unlike WebSockets (bidirectional), SSE is simpler and sufficient for monitoring dashboards and event feeds.

data: {"event_type": "handler.called", "timestamp": "...", ...}\n\n

The EventBus

Nexus's EventBus bridges synchronous publishers and asynchronous consumers via a janus.Queue. It maintains a bounded history (default 256 events) and supports filtered subscriptions.

Code Walkthrough

Step 1: Register SSE Endpoint

from nexus import Nexus, register_sse_endpoint

app = Nexus(api_port=8000, enable_durability=False)
register_sse_endpoint(app)

Step 2: Publish Events

from nexus.events import EventBus, NexusEvent, NexusEventType

bus = app._event_bus

event = NexusEvent(
    event_type=NexusEventType.HANDLER_CALLED,
    handler_name="greet",
    data={"user": "Alice"},
)
bus.publish(event)

# Convenience method
bus.publish_handler_registered("my_handler")

Step 3: Event Serialization

event_dict = event.to_dict()
assert event_dict["event_type"] == "handler.called"

restored = NexusEvent.from_dict(event_dict)
assert restored.event_type == event.event_type

Step 4: Subscribe to Events

import asyncio

q_all = bus.subscribe()
q_filtered = bus.subscribe_filtered(
    lambda evt: evt.event_type == NexusEventType.HANDLER_CALLED
)

assert bus.subscriber_count == 2
SSE Streaming Format: Events are sent as data: {json}\n\n. If no events arrive within 15 seconds, a keepalive comment (: keepalive\n\n) is sent. Clients can filter by event type via ?event_type=handler.called.

Common Mistakes

MistakeProblemFix
Expecting history before dispatch loopEvents queued but not dispatchedRun bus.start() first
Forgetting register_sse_endpoint()No SSE endpoint createdSSE is opt-in
Very small EventBus capacityEvents evicted too quicklyUse capacity ≥ 256

Key Takeaways

  • register_sse_endpoint(app) adds GET /events/stream.
  • Events are published via bus.publish() (sync, thread-safe).
  • SSE format: data: {json}\n\n with 15s keepalive.
  • Optional ?event_type= query parameter for server-side filtering.
  • EventBus stores bounded history with LRU eviction.
