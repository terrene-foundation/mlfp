When prompting is not enough

  • Wrong tone for a regulatory environment
  • Hallucinates Singapore regulations that do not exist
  • Refuses to extract fields you need
  • Domain vocabulary the pre-trained model has never seen enough of

Full fine-tuning updates all ~7B weights. Too expensive, too destructive.

🟢 PEFT: update < 1% of weights. Base knowledge stays intact.

The LoRA insight

\[ W = W_0 + \Delta W = W_0 + BA, \quad r \ll \min(d, k) \]
  • $W_0 \in \mathbb{R}^{d \times k}$ — frozen base (e.g., 4096 × 4096 = 16.8M params)
  • $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$ — learned
  • At $r = 8$: $r(d+k) = 65\,536$ params — 250× reduction

Fine-tuning updates actually have low intrinsic rank. LoRA exploits this.

Picture of LoRA

Forward pass + init

\[ \mathbf{y} = W_0 \mathbf{x} + \frac{\alpha}{r}\, B A \mathbf{x} \]
  • $\alpha$: scaling that decouples "effect strength" from "rank"
  • Typical: $r \in \{4, 8, 16, 32\}$, $\alpha = r$ or $2r$
  • B initialised to 0 — step 0 is numerically identical to the base
  • A initialised with Kaiming uniform — breaks symmetry

LoRA from scratch — the minimal layer


class LoRALayer(nn.Module):
    def __init__(self, base_layer, rank=8, alpha=16):
        super().__init__()
        self.base_layer = base_layer
        for p in self.base_layer.parameters():
            p.requires_grad = False          # freeze W_0

        self.rank, self.alpha = rank, alpha
        self.scaling = alpha / rank
        self.lora_A = nn.Parameter(torch.empty(rank, base_layer.in_features))
        self.lora_B = nn.Parameter(torch.zeros(base_layer.out_features, rank))
        nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))

    def forward(self, x):
        return self.base_layer(x) + (x @ self.lora_A.T) @ self.lora_B.T * self.scaling

The other PEFT — adapter bottlenecks

Inserted after attention and FFN of each transformer block. Near-identity initialisation: zero the up projection.

Model merging — combining fine-tunes without retraining

  • TIES: Trim smallest updates → elect dominant sign → average sign-aligned updates
  • DARE: Randomly drop 90% of updates, rescale survivors by $1/(1-p)$
  • SLERP: spherical linear interpolation between two checkpoints
  • Task arithmetic: add/subtract delta vectors as if they were task gradients

SLERP in one formula

\[ \theta_{\text{slerp}} = \frac{\sin((1-t)\phi)}{\sin \phi}\,\theta_1 + \frac{\sin(t\phi)}{\sin \phi}\,\theta_2 \]
  • $\phi$ = angle between $\theta_1$ and $\theta_2$ on the parameter hypersphere
  • At $t = 0.5$ you get the geodesic midpoint
  • Smoother than linear averaging — fewer interference artefacts

Quantisation & QLoRA

  • GPTQ / AWQ — post-training 4-bit, calibrated
  • GGUF — llama.cpp; runs on CPU + Apple Silicon
  • bitsandbytes — on-the-fly during PyTorch training
  • QLoRA: 4-bit base + fp16 LoRA → fine-tune 65B on 1x 48GB GPU
Serious LoRA runs on constrained hardware should default to QLoRA.

AdapterRegistry is your single source of truth — every downstream lesson loads from it.

Reading the loss curves

  • Healthy: train ↓ smoothly, eval ↓ following with small gap
  • Overfitting: train ↓, eval ↑ — reduce epochs or raise dropout
  • Underfitting: both flat — raise LR or increase rank
  • Leakage: eval < 0.5 from step 1 — check train/eval split

Double the rank, double the params. No free lunch.

Four common mistakes

1. Wrong target modules. Start with q_proj and v_proj.
2. Rank too small for the task. Plateau early? Double r.
3. Forgetting to load the adapter at inference. Check the registry call.
4. Catastrophic forgetting on small datasets. Fewer epochs, lower LR, or Progressive Freezing.
