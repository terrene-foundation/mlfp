Structured schemas. LLM fills the parameters. You execute the function.

Agent design framework

  1. Goal: what is the agent supposed to achieve?
  2. Thought process: what reasoning steps?
  3. Specialist: be sharp — "data analysis researcher" not "researcher".
  4. Tools: versatile, fault-tolerant, with caching.

Kaizen agents

import os
from kaizen_agents.agents.specialized.react import ReActAgent

agent = ReActAgent(
    model=os.environ.get("LLM_MODEL", "gpt-4o-mini"),
    tools=[explore_tool, train_tool, viz_tool],
    max_llm_cost_usd=5.00,   # hard USD cap
)
result = await agent.run("Analyse HDB data and recommend a model")
⚙ Kailash: max_llm_cost_usd lives on the agent. The agent stops gracefully when the budget runs out — no separate tracker class in Kaizen 2.7.

Tool design principles

  • 3–5 tools max — more tools confuse the agent.
  • Clear descriptions — the LLM reads them to choose.
  • Error handling — tools should return errors, not crash.
  • Caching — avoid re-computing expensive operations.

Common mistakes

  1. Too many tools — agent performance degrades.
  2. No cost budget — reasoning loops burn money.
  3. Trusting agent output — agents hallucinate. Verify critical results.
  4. No observation logging — you need traces to debug agent behaviour.
