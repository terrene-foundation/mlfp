Chapter 1: Signatures

Overview

Signatures are the foundational abstraction in Kaizen's agent system. A Signature declares WHAT an agent does — its inputs and outputs — without specifying HOW. The docstring becomes the agent's system instructions, and InputField/OutputField descriptors define the structured interface.

Prerequisites

  • Kailash Kaizen installed (pip install kailash-kaizen)
  • Understanding of Python class syntax and type annotations

Concepts

What Is a Signature?

A declarative contract between programmer and AI agent. It specifies:

  • System prompt: The class docstring
  • Inputs: Fields the agent receives
  • Outputs: Fields the agent must produce

Why Signatures?

Signatures separate interface declaration from execution strategy, making agents composable, testable, and auditable.

Key API

APIDescription
InputField(description=, default=)Declare an input field
OutputField(description=)Declare an output field
cls._signature_inputsDict of all input fields
cls._signature_outputsDict of all output fields
field.requiredTrue if no default value
field.metadataExtra keyword arguments

Code Walkthrough

Step 1: Class-Based Signature

from kaizen import Signature, InputField, OutputField

class QASignature(Signature):
    """You are a helpful question-answering assistant."""

    question: str = InputField(description="The question to answer")
    context: str = InputField(description="Additional context", default="")
    answer: str = OutputField(description="Clear, accurate answer")
    confidence: float = OutputField(description="Confidence score 0.0 to 1.0")

The docstring becomes the system prompt. question is required (no default); context is optional.

Step 2: Verify Fields

assert "question" in QASignature._signature_inputs
assert "answer" in QASignature._signature_outputs

question_field = QASignature._signature_inputs["question"]
assert question_field.required is True

context_field = QASignature._signature_inputs["context"]
assert context_field.required is False
assert context_field.default == ""

Step 3: Intent and Guidelines

class CustomerSupportSignature(Signature):
    """You are a customer support agent for an e-commerce platform."""

    __intent__ = "Resolve customer issues efficiently and empathetically"
    __guidelines__ = [
        "Acknowledge concerns before proposing solutions",
        "Use empathetic language",
        "Escalate if unresolved in 3 turns",
    ]

    customer_message: str = InputField(description="Customer's message")
    response: str = OutputField(description="Support response")
    action: str = OutputField(description="Action: resolve, escalate, transfer")

Step 4: Inheritance

class DetailedQA(QASignature):
    """Extended QA with source citations."""

    sources: str = OutputField(description="List of sources used")

assert "question" in DetailedQA._signature_inputs   # Inherited
assert "sources" in DetailedQA._signature_outputs    # Added
Note: Child fields merge with parent fields. On name collision, the child field overrides the parent.

Step 5: Field Metadata

class RichSignature(Signature):
    """Signature with rich field metadata."""

    data: str = InputField(
        description="Input data",
        format="json",
        max_length=10000,
    )
    result: str = OutputField(description="Processed result")

data_field = RichSignature._signature_inputs["data"]
assert data_field.metadata.get("format") == "json"
assert data_field.metadata.get("max_length") == 10000

Common Mistakes

MistakeProblemFix
Forgetting the docstringAgent has no system promptAlways include a descriptive docstring
All fields requiredAgent fails on partial inputUse default="" for optional fields
Minimal output fieldsResponse lacks structureInclude all expected dimensions
Logic in signaturesSignatures are data, not codePut logic in execution strategy

Key Takeaways

  • Signatures declare WHAT an agent does (inputs/outputs), not HOW.
  • The class docstring becomes the agent's system prompt.
  • Fields without defaults are required; fields with defaults are optional.
  • __intent__ and __guidelines__ provide purpose and constraints.
  • Signatures support inheritance and field metadata.
