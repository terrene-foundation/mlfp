Chapter 2: Delegate

Overview

Delegate is the single entry point for autonomous AI execution in Kaizen. It composes an AgentLoop with optional governance and provides both streaming (run()) and synchronous (run_sync()) interfaces.

Concepts

Progressive Disclosure

Layer 2+ tools, system_prompt, max_turnsCustomized behavior

Code Walkthrough

Step 1: Minimal Delegate

import os
from dotenv import load_dotenv
load_dotenv()

from kaizen_agents import Delegate

model = os.environ.get("DEFAULT_LLM_MODEL", "claude-sonnet-4-20250514")
delegate = Delegate(model=model)

assert delegate._budget_usd is None  # No budget cap
Important: Model names must come from environment variables, never hardcoded. Call load_dotenv() before any os.environ access.

Step 2: Configured Delegate

configured = Delegate(
    model=model,
    tools=[],
    system_prompt="You are a helpful code reviewer.",
    max_turns=20,
)
assert configured._config.max_turns == 20

Step 3: Governed Delegate

governed = Delegate(model=model, budget_usd=5.0)

assert governed._budget_usd == 5.0
assert governed._consumed_usd == 0.0

Step 5: Budget Validation

try:
    Delegate(model=model, budget_usd=-1.0)
except ValueError as e:
    assert "non-negative" in str(e).lower()

try:
    Delegate(model=model, budget_usd=float("inf"))
except ValueError as e:
    assert "finite" in str(e).lower()

Common Mistakes

MistakeProblemFix
Hardcoding model namesBreaks on provider changesUse os.environ.get()
Forgetting load_dotenv()Env vars not loadedCall before os.environ
Ignoring BudgetExhaustedAgent exceeds budgetHandle in event loop
Using run() without asyncAsync generator errorUse async for or run_sync()

Key Takeaways

  • Delegate is the single entry point with three layers of progressive disclosure.
  • run() yields typed DelegateEvent instances for pattern matching.
  • run_sync() provides synchronous convenience for scripts.
  • budget_usd enables automatic cost tracking.
  • Model names always come from environment variables.
