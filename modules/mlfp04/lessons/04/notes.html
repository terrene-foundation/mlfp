Speaker Notes — Lesson 4.4 Anomaly Detection and Ensembles

Total time: ~75 minutes  ·  17 slides  ·  ~4 min per slide

This lesson is lighter on math than 4.2 and 4.3 but heavy on pragmatism. The most important idea to convey is that no single method is sufficient — blending beats individual methods. The second most important idea is that you cannot evaluate detectors without domain expertise.

Slide 1~2 minTitle

Anomaly Detection and Ensembles

Frame: "Same unsupervised problem, flipped. Instead of finding groups, find the points that don't belong to any group. Fraud, defective sensors, intrusions — the rare things that matter most."

→ "The opposite question."
Slide 2~3 minMotivation

The opposite question

Give a concrete scenario: "0.1% of financial transactions are fraud. You have a few labelled fraud cases from last year, but the patterns change every quarter. You cannot train a supervised classifier on 0.1% minorities. Anomaly detection lets you flag suspicious transactions using only the shape of the data itself."

→ "First: what does an outlier even look like?"
Slide 3~3 minThree types

Three types of outliers

Walk the three red points. (A) far from everything — obvious anomaly. (B) in a direction the data rarely goes — anomaly even though close in Euclidean distance. (C) in a low-density gap between clusters — anomaly even though "central". Different detectors catch different types.

→ "Simplest detector: Z-score."
Slide 4~3 minZ-score

Z-score: univariate, assumes normality

Walk the formula. "If data is normal, about 0.27% fall outside $\pm 3\sigma$ — those are your outliers. Works great for normally-distributed features. Fails badly on skewed data."

"Z-score is the 'how many standard deviations from the mean' measurement. 3 sigma is the classical threshold — anything beyond is considered rare."
→ "Skewed data needs IQR."
Slide 5~3 minIQR

IQR: robust to skew

Walk the rule. "Quartiles are unaffected by the extreme values — IQR is a robust statistic. This is exactly what box plot whiskers do. For revenue, session duration, transaction amount — use IQR instead of Z-score."

→ "But both are univariate. Real fraud is multivariate."
Slide 6~3 minUnivariate fails

Univariate methods miss combinations

Give the fraud example concretely: "Small transaction — normal. Midnight transaction — normal. Small + midnight + new device + unusual location — suspicious. No single feature is anomalous. The combination is. Univariate methods cannot catch this."

→ "Enter Isolation Forest."
Slide 7~5 minIsolation Forest diagram

Isolation Forest — few splits isolate anomalies

Walk the diagram. "The outlier on the upper right gets isolated by the first random split. The points in the dense cluster need many splits to be isolated from each other. So: few splits = outlier."

This is a beautifully simple idea. You do not need to compute densities or distances — just grow random trees and count path lengths. Scales linearly in the data size.

"Contrast with one-class SVM, which requires kernel matrix computation. Isolation Forest is $O(n \log n)$ per tree and embarrassingly parallel. It is the default for production anomaly detection."
→ "The formal score."
Slide 8~3 minScore formula

Isolation Forest score

Walk the formula. Explain $c(n)$ briefly — it normalises path length so scores are comparable across sample sizes. Emphasise the interpretation: 0.5 = normal, near 1 = strong anomaly.

→ "LOF for local density anomalies."
Slide 9~4 minLOF

LOF — local density matters

Walk the formula. "LOF compares the density around your point to the density around its neighbours. If your neighbours are in a dense pack and you're in a sparse area relative to them, you're anomalous."

"LOF is the right choice when your clusters have different densities — a point that is 'close' to one cluster in absolute distance can still be an outlier relative to that cluster's tight packing."

→ "Which method catches which?"
Slide 10~4 minMethod matrix

Method × outlier type matrix

Walk the matrix. Key insight: no single method catches all types. This is the motivation for blending.

→ "Blend them."
Slide 11~5 minBlending

Score blending — more robust than any single method

Walk the code. The three-step pattern: normalise to $[0, 1]$, average, threshold. Emphasise that you must normalise first — different methods use different ranges.

Business interpretation: "A point caught by all three is almost certainly anomalous. A point caught by one method is a candidate for human review. The number of methods that flagged a point is a confidence gradient — you act differently at each level."

→ "Kailash wraps this."
Slide 12~3 minEnsembleEngine

EnsembleEngine does this with one API

Quick. "Four blending modes: blend (weighted sum), stack (meta-learner), bag (bootstrapped), boost (sequential). For anomaly detection you almost always want blend with agg='mean' or agg='max'."

→ "The one tuning knob you must get right."
Slide 14~3 minContamination

Contamination matters

This is where most students mess up. "Contamination is your prior belief about how many points are anomalous. Set it to 10% when the real rate is 0.1% and your top-10 flagged points are all normal."

Practical advice: start with the known base rate, inspect the top $k$, adjust. There is no cross-validation here — you must have a domain expert look at the flagged points.
