Lesson 2.7 CUPED and Causal Inference

A/B tests from Lesson 2.4 are the gold standard for causal inference, but they are often noisy — the variance in your metric swamps the treatment effect. This lesson introduces the single most impactful variance-reduction technique in industry (CUPED), shows how to detect when your experiment is broken (SRM), and extends causal reasoning to settings where randomisation is impossible (Difference-in-Differences). By the end you will be able to cut experiment variance by up to 50% and measure policy effects from observational data.

CUPED: Control Using Pre-Experiment Data

CUPED is variance reduction using a covariate that is measured before the experiment starts. The idea is simple: if you know a user's spending last month, you can adjust their spending this month by how much it deviates from their historical average. The adjusted metric has lower variance, which means you can detect smaller effects or declare significance faster.

The derivation

Let $Y$ be the post-experiment metric and $X$ be the pre-experiment covariate. Define the adjusted metric:

$$Y_{\text{adj}} = Y - \theta \cdot (X - \mathbb{E}[X])$$

We want to choose $\theta$ to minimise the variance of $Y_{\text{adj}}$:

$$\text{Var}(Y_{\text{adj}}) = \text{Var}(Y) - 2\theta \cdot \text{Cov}(Y, X) + \theta^2 \cdot \text{Var}(X)$$
quadratic in $\theta$ minimum at $\theta^* = \text{Cov}(Y, X) / \text{Var}(X)$

Substituting $\theta^*$ back:

$$\boxed{\;\text{Var}(Y_{\text{adj}}) = \text{Var}(Y)(1 - \rho^2)\;}$$
$\rho$ = correlation between $Y$ and $X$ $\rho = 0.7 \Rightarrow 51\%$ variance reduction

The key insight: the higher the correlation $\rho$ between the pre and post metrics, the more variance CUPED removes. For e-commerce spending, $\rho$ between last month and this month is typically 0.6-0.8, giving 36-64% variance reduction.

Implementation

import polars as pl
import numpy as np

# Pre-experiment: last month's spending per user
# Post-experiment: this month's spending per user
df = loader.load("mlfp02", "ab_test_cuped.parquet")

X_pre = df["spend_pre"].to_numpy()
Y_post = df["spend_post"].to_numpy()

# Compute optimal theta
theta = np.cov(Y_post, X_pre)[0, 1] / np.var(X_pre)

# Adjusted metric
Y_adj = Y_post - theta * (X_pre - X_pre.mean())

print(f"Var(Y):     {np.var(Y_post):.2f}")
print(f"Var(Y_adj): {np.var(Y_adj):.2f}")
print(f"Reduction:  {1 - np.var(Y_adj)/np.var(Y_post):.1%}")
🔵 Theory

CUPED is mathematically identical to the residuals of a linear regression of $Y$ on $X$. The optimal $\theta$ is the regression coefficient $\hat\beta = \text{Cov}(Y, X) / \text{Var}(X)$. The adjusted metric is the residual $Y - \hat\beta X$. CUPED is regression adjustment applied to A/B test analysis.

🟣 Advanced

CUPED requires that the pre-experiment covariate is independent of the treatment assignment. Since $X$ is measured before the experiment, this is guaranteed by the randomisation. If you use a post-experiment covariate, you risk adjusting away the treatment effect itself.

SRM: Sample Ratio Mismatch

Before analysing any A/B test, check the split. If you randomised 50/50 but observe 52/48, something may be wrong — bot filtering, redirect bugs, or population filters could be biasing one arm.

Detection

from scipy import stats as scipy_stats

n_control = 4800
n_treatment = 5200
n_total = n_control + n_treatment

# Expected 50/50
chi2 = (n_control - n_total/2)**2 / (n_total/2) + \
       (n_treatment - n_total/2)**2 / (n_total/2)
p_srm = 1 - scipy_stats.chi2.cdf(chi2, df=1)

print(f"Chi-squared: {chi2:.2f}, p = {p_srm:.4f}")
if p_srm < 0.01:
    print("SRM detected — investigate before analysing results")
⚠ Warning

If SRM is detected, do not trust the A/B test results. Common causes: bots filtered unevenly, a redirect failing for one variant, or a population filter applied after randomisation. Fix the root cause and re-run the experiment.

Difference-in-Differences

When randomisation is impossible — you cannot randomly assign Singapore's cooling measures policy to some districts and not others — you need observational causal inference. Difference-in-Differences (DiD) compares the change in a treated group to the change in an untreated group:

$$\text{ATT} = (\bar{Y}_{\text{treat,post}} - \bar{Y}_{\text{treat,pre}}) - (\bar{Y}_{\text{control,post}} - \bar{Y}_{\text{control,pre}})$$
ATT = Average Treatment Effect on the Treated first difference removes level effects; second difference removes time effects

The parallel trends assumption

DiD requires that, absent treatment, the treated and control groups would have followed parallel trends. You cannot test this directly (it is about the counterfactual), but you can check it in the pre-period:

⚠ Warning

If the pre-period trends are not parallel, DiD is invalid. The treatment effect estimate will be biased by the divergent trends. Placebo tests — running DiD on a fake treatment date in the pre-period — help diagnose this.

Common Mistakes

⚠ Mistake 1: Using a post-experiment covariate in CUPED

The covariate must be measured before randomisation. A post-experiment covariate may be affected by the treatment, and adjusting for it removes part of the effect you are trying to measure.

⚠ Mistake 2: Ignoring SRM

Always check the split before analysing results. A 52/48 split on a 50/50 experiment is a red flag — the p-value on your treatment effect is meaningless if the groups are systematically different.

⚠ Mistake 3: Applying DiD without checking parallel trends

DiD with divergent pre-trends gives a biased estimate. Always plot the pre-period trends and run a placebo test.
