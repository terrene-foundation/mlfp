The RL loop

🟢 Foundations: all DL so far: static data. RL: learning from interaction. Agent acts, observes, learns.

Bellman equations

\[ V(s) = \mathbb{E}[R_{t+1} + \gamma\,V(S_{t+1}) \mid S_t=s] \] \[ Q^*(s,a) = \mathbb{E}[R_{t+1} + \gamma\,\max_{a'} Q^*(S_{t+1},a') \mid S_t=s, A_t=a] \]

$\gamma$ = discount factor. Future rewards worth less. DQN approximates $Q^*$ with a neural network.

DQN — Deep Q-Network


class DQN(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, 128), nn.ReLU(),
            nn.Linear(128, 128), nn.ReLU(),
            nn.Linear(128, action_dim))
    def forward(self, x): return self.net(x)

# Loss: (r + gamma * max Q_target(s',a') - Q(s,a))^2
  • Experience replay: store transitions, sample randomly.
  • Target network: stabilise training with delayed copy.
  • Discrete actions only.

PPO — Proximal Policy Optimization

\[ L^{\text{CLIP}} = \mathbb{E}\bigl[\min(r_t A_t,\;\text{clip}(r_t,1{-}\epsilon,1{+}\epsilon)A_t)\bigr] \]
  • $r_t = \pi_\theta / \pi_{\theta_\text{old}}$ — how much policy changed.
  • Clip prevents destructively large updates.
  • Works for continuous and discrete actions.

Bridge to M6: PPO → RLHF

🔵 Theory: RLHF uses PPO to maximise a reward model trained on human preferences. DPO skips the reward model entirely.

Common mistakes

  1. Complex reward shaping — start simple, add complexity only if agent fails.
  2. No advantage normalisation — PPO needs normalised advantages.
  3. Tiny replay buffer — DQN needs diverse experience.
