Chapter 7: TrainingPipeline

Overview

TrainingPipeline orchestrates the full model training workflow: validate data, split, train, evaluate, and optionally register the model. It integrates FeatureStore, ModelRegistry, and ExperimentTracker into a single train() call.

Concepts

ModelSpec & EvalSpec

ModelSpec declares what to train (class, hyperparameters, framework). EvalSpec declares how to evaluate (metrics, split strategy, thresholds). Together they fully specify a training run.

Split Strategies

StrategyBehaviorBest For
holdoutRandom train/test splitMost datasets
kfoldK cross-validation foldsSmall datasets
walk_forwardSequential split (no shuffle)Time-series

Threshold Gating

When min_threshold is set in EvalSpec, models that fail to meet the bar are not registered. This prevents deploying underperforming models.

Security: Model classes are restricted to an allowlist (sklearn, xgboost, lightgbm). Attempting to instantiate dangerous classes like os.system raises ValueError.

Code Walkthrough

Step 1: Define Specs

from kailash_ml.engines.training_pipeline import ModelSpec, EvalSpec

model_spec = ModelSpec(
    model_class="sklearn.ensemble.RandomForestClassifier",
    hyperparameters={"n_estimators": 20, "random_state": 42},
    framework="sklearn",
)

eval_spec = EvalSpec(
    metrics=["accuracy", "f1"],
    split_strategy="holdout",
    test_size=0.2,
)

Step 2: Train

result = await pipeline.train(
    data=df, schema=schema,
    model_spec=model_spec,
    eval_spec=eval_spec,
    experiment_name="tutorial_experiment",
)

assert result.threshold_met is True
assert result.registered is True
assert result.model_version.stage == "staging"

Step 3: Threshold Gating

strict_eval = EvalSpec(
    metrics=["accuracy"],
    split_strategy="holdout",
    test_size=0.2,
    min_threshold={"accuracy": 0.99},
)

strict_result = await pipeline.train(
    data=df, schema=schema,
    model_spec=model_spec, eval_spec=strict_eval,
    experiment_name="strict",
)

if not strict_result.threshold_met:
    assert strict_result.registered is False

Step 4: Evaluate and Retrain

# Evaluate on new data
eval_metrics = await pipeline.evaluate(
    model_name="tutorial_experiment",
    version=1, data=new_data,
    schema=schema,
    eval_spec=EvalSpec(metrics=["accuracy"]),
)

# Retrain with fresh data
retrain_result = await pipeline.retrain(
    model_name="tutorial_experiment",
    schema=schema, model_spec=model_spec,
    eval_spec=eval_spec, data=new_data,
)

Common Mistakes

MistakeWhat HappensFix
Missing columns in dataValueErrorInclude all schema features + target
Non-allowlisted model classValueErrorUse sklearn/xgboost/lightgbm
Invalid split strategyValueErrorUse holdout, kfold, or walk_forward
Unrealistic thresholdModel never registersSet thresholds from baseline results

Key Takeaways

  • TrainingPipeline integrates FeatureStore, ModelRegistry, and ExperimentTracker.
  • ModelSpec declares what to train; EvalSpec declares how to evaluate.
  • Three split strategies cover different data scenarios.
  • Threshold gating prevents weak models from being registered.
  • Model classes are allowlisted for security.
