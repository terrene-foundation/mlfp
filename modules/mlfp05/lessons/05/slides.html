The GAN setup — a minimax game

GAN minimax objective

\[ \min_G \max_D \;\bigl[\mathbb{E}_{x \sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]\bigr] \]
  • $D$ wants to maximise — correctly classify real and fake.
  • $G$ wants to minimise — fool $D$ into calling fakes real.
  • Training alternates: update $D$ for $k$ steps, then update $G$ for 1 step.

DCGAN architecture

DCGAN rules: no max pooling, no FC layers, batch norm everywhere except output, Tanh at generator output.

Mode collapse — the GAN failure mode

Generator finds one output that fools D and repeats it. WGAN fixes this.

WGAN — Wasserstein distance

\[ \min_G \max_{D \in \mathcal{D}} \;\bigl[\mathbb{E}_{x}[D(x)] - \mathbb{E}_{z}[D(G(z))]\bigr] \]
  • Replaces JS divergence with Wasserstein (Earth Mover's) distance.
  • Gradient penalty (WGAN-GP) enforces the Lipschitz constraint.
  • More stable training, meaningful loss curve, reduced mode collapse.
🔵 Theory: WGAN loss correlates with sample quality — you can actually use it to monitor training.

DDPM — Denoising Diffusion

  • More stable than GANs. Better diversity. No mode collapse.
  • Slower to sample (many denoising steps). Active research on speed.

FID — evaluating generated samples

\[ \text{FID} = \|\mu_r - \mu_g\|^2 + \text{Tr}\bigl(\Sigma_r + \Sigma_g - 2(\Sigma_r \Sigma_g)^{1/2}\bigr) \]
  • Compare statistics of real vs generated features (from Inception network).
  • Lower FID = closer to real data. FID = 0 means identical distributions.
  • Standard metric. Report it for every generative model.

DCGAN training loop


for epoch in range(epochs):
    for real_batch in dataloader:
        # Train D: maximise log D(x) + log(1-D(G(z)))
        z = torch.randn(batch_size, latent_dim)
        fake = generator(z)
        d_loss = -torch.mean(torch.log(D(real_batch))
                           + torch.log(1 - D(fake.detach())))
        d_loss.backward(); d_opt.step()

        # Train G: minimise log(1-D(G(z)))
        z = torch.randn(batch_size, latent_dim)
        fake = generator(z)
        g_loss = -torch.mean(torch.log(D(fake)))
        g_loss.backward(); g_opt.step()

WGAN-GP training differences


# Critic (not discriminator) — no sigmoid
# Gradient penalty replaces weight clipping
alpha = torch.rand(batch_size, 1, 1, 1)
interp = alpha * real + (1 - alpha) * fake.detach()
interp.requires_grad_(True)
d_interp = critic(interp)
gradients = torch.autograd.grad(d_interp, interp,
    grad_outputs=torch.ones_like(d_interp),
    create_graph=True)[0]
gp = ((gradients.norm(2, dim=1) - 1) ** 2).mean()

Singapore applications

  • Simulation: generate realistic traffic patterns for LTA planning models.

Common mistakes

  1. Training G too aggressively — D must stay ahead. Use k=5 D steps per G step.
  2. Ignoring mode collapse — monitor sample diversity, not just D loss.
  3. No FID evaluation — visual inspection is subjective. Report FID.
  4. Using GANs when diffusion is better — if quality matters more than speed, use diffusion.
