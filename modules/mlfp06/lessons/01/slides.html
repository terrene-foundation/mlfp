The transformer, one more time

Pre-training: one equation

\[ \mathcal{L}(\theta) = -\sum_{t=1}^{T} \log p_\theta(x_t \mid x_1, \ldots, x_{t-1}) \]
  • Predict the next token. That is the entire objective.
  • No labels — every token of internet text is both input and target.
  • Scaling laws: loss decays as a power law in params, data, compute.
🔵 Theory: grammar, world knowledge, reasoning, code, translation — all emerge as side effects of lowering one cross-entropy.

Tokenisation — you pay per token

Rule of thumb: 1 token ≈ 0.75 English words ≈ 4 characters. Context window = 128k tokens on modern frontier models.

Why prompting works

A prompt is not a command — it is a context. You are asking the model: "what tokens most naturally follow this exact string?"

Your prompt selects a neighbourhood of the training distribution. "Answer like a senior compliance officer" moves the spotlight across the model's knowledge — it does not teach the model anything new.

🟢 Mindset shift: stop thinking "instruct the model". Start thinking "make the desired output the natural continuation."

Zero-shot


prompt = f"""Classify this Singapore company report into one category.
Categories: Financial, Technology, Healthcare, Real Estate, Manufacturing.

Report excerpt:
{text[:800]}

Respond with ONLY the category name."""
  • Cheapest, fastest.
  • Most brittle — one stray word from the model and your parsing breaks.
  • Fix: inline the allowed labels, specify the output format.

Few-shot (in-context learning)


FEW_SHOT = [
    ("Revenue increased 15% driven by loan growth...", "Financial"),
    ("Cloud-native SaaS platform for enterprises...", "Technology"),
    ("Clinical trials for oncology drug...",         "Healthcare"),
    ("Integrated township in Jurong...",             "Real Estate"),
    ("Factory automation at Tuas fab...",            "Manufacturing"),
]

Diverse, clean, short. $k = 3$–$8$. Diminishing returns past 10 examples.

Chain-of-thought — "think step by step"


Classify this report. Think step by step:
1. Identify key terms and topics
2. Match those terms to the most relevant category
3. State your final classification on the last line

Report: "Revenue up 15% on loan growth..."

→ 1. Key terms: revenue, margin, loans
→ 2. These are banking concepts
→ 3. Category: Financial
🔵 Why it works: reasoning tokens become context for the answer. The model gets scratchpad space before committing.

Self-consistency: vote across samples

Kaizen Delegate — the LLM primitive


from kaizen_agents import Delegate

delegate = Delegate(
    model="gpt-4o-mini",
    max_llm_cost_usd=0.50,  # ← hard cap
)

response = ""
async for event in delegate.run(prompt):
    if hasattr(event, "text"):
        response += event.text
  • One interface across OpenAI / Anthropic / Ollama / local.
  • Streaming — abort early if the response is going bad.
  • Budget is not optional. A bug in a loop is the most common LLM bill disaster.

Stop parsing strings — use Signature


from kaizen import Signature, InputField, OutputField

class ReportExtraction(Signature):
    report_text: str = InputField(description="Report excerpt")

    category: str = OutputField(
        description="Financial | Technology | Healthcare | Real Estate | Manufacturing"
    )
    key_entities: list[str] = OutputField(description="Named entities")
    sentiment: str = OutputField(description="positive | negative | neutral")
    confidence: float = OutputField(description="Confidence in [0, 1]")

The agent writes the prompt, calls the model, parses to this schema, retries on mismatch. You get a typed object — not a string.

Using the signature


from kaizen_agents.agents.specialized.simple_qa import SimpleQAAgent

agent = SimpleQAAgent(
    signature=ReportExtraction,
    model="gpt-4o-mini",
    max_llm_cost_usd=1.0,
)

result = await agent.run(report_text=doc)

if result.confidence < 0.6:
    escalate_to_human(result)         # ← float, not "high"

for entity in result.key_entities:    # ← list[str], not a CSV string
    index_entity(entity)

Cost model

\[ \text{cost} = N_\text{in}\cdot r_\text{in} + N_\text{out}\cdot r_\text{out} \]
Model (Apr 2026 ballpark)Input / 1MOutput / 1M
gpt-4o-mini$0.15$0.60
gpt-4o$2.50$10.00
claude-3.5-sonnet$3.00$15.00

Output tokens cost 4× input. CoT is expensive because it is all output.

Temperature controls randomness

\[ p_i = \frac{\exp(\ell_i / T)}{\sum_j \exp(\ell_j / T)} \]
  • $T = 0$ — greedy. Same input → same output. Use for classification, extraction.
  • $T = 0.2$ — slight variety. Use for summarisation.
  • $T = 0.7$ — creative. Use for brainstorming and self-consistency sampling.
  • $T = 1.0$+ — loose. Use rarely.

Worked example — 10 Singapore reports

    _, c = await cot_classify(row["text"])
    results.append({"doc": row["text"][:40], "zero": z, "few": f, "cot": c})

comparison = pl.DataFrame(results)
print(comparison)

Rows where all three agree are high-confidence. Disagreements are exactly where you escalate to a human.

Four common mistakes

1. Parsing free-form text in production. Use Signature.
2. Batching 20 answers into one prompt. Error rate compounds. Use asyncio.gather.
3. No cost budget. A buggy loop can cost $1,000 overnight.
4. "T=0 is deterministic". Nearly, not fully — log every call for audit.
