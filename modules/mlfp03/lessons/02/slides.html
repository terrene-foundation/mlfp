Bias-Variance, Regularisation, and Cross-Validation

The most important equation in applied ML · L1 vs L2 geometry · Nested CV

⏱ 75 minutes   ·   Prereq: 3.1   ·   Outcome: derive the decomposition and tune a regularised model
MLFP — Module 3: Supervised ML

The canonical curve

Training error falls forever; test error is U-shaped. The gap is variance.

The setup

\[ y = f(x) + \varepsilon,\quad \mathbb{E}[\varepsilon]=0,\quad \mathrm{Var}(\varepsilon) = \sigma^2 \]

$f$ is the unknown true function. $\hat{f}$ is our fit on a random training sample. Error at a test point:

\[ \mathrm{Err}(x) = \mathbb{E}_{\mathcal{T}, \varepsilon}\bigl[(y - \hat{f}(x))^2\bigr] \]

The decomposition

\[ \mathrm{Err}(x) = \underbrace{(f - \mathbb{E}\hat{f})^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}[(\hat{f} - \mathbb{E}\hat{f})^2]}_{\text{Variance}} + \underbrace{\sigma^2}_{\text{Irreducible}} \]
  • Bias²: how wrong the average prediction is (too simple)
  • Variance: how much the prediction wobbles with new training sets (too flexible)
  • σ²: the part of $y$ that $x$ cannot explain; sets a hard floor on achievable error

The dart-board picture

Regularisation — trade variance for bias

\[ \text{Ridge: } \mathcal{L}(\boldsymbol\beta) + \lambda \sum_j \beta_j^2 \qquad \text{Lasso: } \mathcal{L}(\boldsymbol\beta) + \lambda \sum_j |\beta_j| \]

Both shrink coefficients. Only L1 drives them to exactly zero. Why? Geometry.

L2 (circle) vs L1 (diamond)

The diamond has corners on the axes; the ellipses preferentially hit those corners, producing sparsity.

Bayesian reading

  • Ridge ≡ MAP estimate under a Gaussian prior on $\boldsymbol\beta$
  • Lasso ≡ MAP estimate under a Laplace prior on $\boldsymbol\beta$
\[ \log p(\boldsymbol\beta \mid \text{data}) \propto -\frac{1}{2\tau^2}\sum_i(y_i - \boldsymbol\beta^\top x_i)^2 - \frac{\lambda}{2\tau^2}\sum_j \beta_j^2 \]

Maximising the log-posterior is algebraically identical to minimising ridge loss. The Laplace prior has a sharp spike at zero — the Bayesian image of the L1 diamond corner.

Elastic Net

\[ \mathcal{L}(\boldsymbol\beta) + \lambda\bigl(\alpha \|\boldsymbol\beta\|_1 + (1-\alpha)\|\boldsymbol\beta\|_2^2\bigr) \]

Mixed Laplace-Gaussian prior. Use when:

  • Lasso oversimplifies correlated features (arbitrary pick)
  • Ridge refuses to zero anything you know is irrelevant
  • You want a little of both

Nested CV — honest model selection


for train_idx, test_idx in outer_cv.split(X):
    X_tr, X_te = X[train_idx], X[test_idx]
    y_tr, y_te = y[train_idx], y[test_idx]

    search = GridSearchCV(Ridge(),
        param_grid={"alpha": np.logspace(-2, 3, 20)},
        cv=inner_cv, scoring="neg_mean_squared_error")
    search.fit(X_tr, y_tr)
    outer_scores.append(search.score(X_te, y_te))

print(f"Honest MSE: {-np.mean(outer_scores):,.0f}")

Inner loop chooses $\alpha$. Outer loop evaluates. Data used for tuning is never used for scoring.

Worked example — HDB overfitting


for degree in [1, 2, 3, 5, 9]:
    model = Pipeline([
        ("poly", PolynomialFeatures(degree)),
        ("lin",  LinearRegression()),
    ])
    mse = -cross_val_score(model, X, y, cv=5,
        scoring="neg_mean_squared_error").mean()
    print(f"d={degree}: CV MSE = {mse:,.0f}")

You will see the U-shape live: degree 1 underfits, degree 3 is optimal, degree 9 explodes.

Three expensive mistakes

1. Reporting GridSearchCV.best_score_ as the test score — optimistically biased.
2. Plain k-fold on grouped data (same patient in train and test).
3. Shuffling time-series — lets the model see the future.
