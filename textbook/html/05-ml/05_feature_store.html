Chapter 5: FeatureStore

Overview

FeatureStore bridges the gap between feature engineering and model training by providing a centralized, versioned repository for computed features. You register a FeatureSchema, compute features from raw data, store them in a database, and retrieve them by entity ID or time window with point-in-time correctness.

Concepts

Why a Feature Store?

Without a feature store, features are computed ad-hoc in notebooks, leading to training/serving skew (different code for training vs inference), feature duplication (multiple teams recompute the same features), and point-in-time leakage (training data includes future values). FeatureStore solves these by centralizing definitions and providing time-aware retrieval.

Point-in-Time Retrieval

get_training_set(schema, start, end) returns only rows where created_at falls within the window. This prevents lookahead bias when building training datasets from historical features.

Lazy retrieval: get_features_lazy() returns a polars LazyFrame instead of a DataFrame, enabling streaming operations on large feature sets without loading everything into memory.

Code Walkthrough

Step 1: Set Up Infrastructure

from kailash.db.connection import ConnectionManager
from kailash_ml import FeatureField, FeatureSchema
from kailash_ml.engines.feature_store import FeatureStore

conn = ConnectionManager("sqlite:///:memory:")
await conn.initialize()

fs = FeatureStore(conn, table_prefix="kml_feat_")
await fs.initialize()

Step 2: Register a Schema

schema = FeatureSchema(
    name="user_churn",
    features=[
        FeatureField(name="age", dtype="float64"),
        FeatureField(name="tenure_months", dtype="float64"),
        FeatureField(name="monthly_charges", dtype="float64"),
    ],
    entity_id_column="user_id",
)

await fs.register_features(schema)

Step 3: Compute and Store

computed = fs.compute(raw_data, schema)
stored_count = await fs.store(computed, schema)
assert stored_count == 20

Step 4: Retrieve by Entity ID

features = await fs.get_features(
    entity_ids=["u0", "u1", "u2"],
    feature_names=["age", "tenure_months"],
    schema=schema,
)
assert features.height == 3

Step 5: Build Training Set by Time Window

from datetime import datetime, timedelta, timezone

now = datetime.now(timezone.utc)
training_set = await fs.get_training_set(
    schema,
    start=now - timedelta(hours=1),
    end=now + timedelta(hours=1),
)
assert training_set.height == 20
Schema conflicts: Re-registering a schema with the same name but different features raises ValueError. Use unique names for different feature sets.

Key Takeaways

  • FeatureStore centralizes feature definitions and storage, preventing training/serving skew.
  • compute() validates data against the schema before storage.
  • get_training_set() provides point-in-time correct retrieval.
  • get_features_lazy() enables memory-efficient streaming.
  • Schema registration is idempotent but rejects conflicting schemas.
