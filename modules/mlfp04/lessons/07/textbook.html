Lesson 4.7 Recommender Systems — THE PIVOT

This is the single most important lesson in Module 4. We started the module with clustering and dimensionality reduction — unsupervised methods that discover features without any error signal. In 4.6 we saw NMF factorise a document-word matrix. Today we factorise a user-item matrix and watch the learned factors become embeddings — features discovered by optimisation. This is the Feature Engineering Spectrum pivot: the first time you see optimisation drive feature discovery. In Lesson 4.8, neural networks generalise this insight — hidden layer activations ARE embeddings, learned by minimising a loss function.

Why This Matters

Recommender systems generate a third of Amazon's revenue and 80% of Netflix's viewing hours. Singapore's retail landscape — from Shopee to FairPrice Online — runs on them. But the business value is not the only reason they matter for this curriculum. Recommender systems sit at the exact transition between classical unsupervised ML and deep learning:

Core Concepts

Content-based filtering

The simplest recommender: represent each item as a feature vector (genre, price, description TF-IDF), represent each user as the average of the items they liked, and recommend items whose feature vectors are closest to the user vector. 

import polars as pl
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# Item features: TF-IDF of product descriptions
item_features = tfidf_matrix  # shape (n_items, n_features)

# User profile: average of items they purchased
user_profile = item_features[purchased_indices].mean(axis=0)

# Recommend: top-k most similar items
sims = cosine_similarity(user_profile.reshape(1, -1), item_features)[0]
top_k = np.argsort(sims)[-10:][::-1]
🟢 Foundations

Content-based filtering does not need other users' data. It works even for brand-new users (no cold-start problem on the item side) as long as you have item features. The downside: it cannot discover surprising preferences — it only recommends more of what you already like.

Collaborative filtering

Collaborative filtering ignores item features entirely. It works with the user-item interaction matrix $R$ (ratings, purchases, clicks):

  • User-based CF: find users similar to you, recommend what they liked.
  • Item-based CF: find items similar to what you liked, recommend those.

Matrix factorisation — THE PIVOT

Factorise $R \approx U V^T$ where $U \in \mathbb{R}^{m \times k}$ (user embeddings) and $V \in \mathbb{R}^{n \times k}$ (item embeddings). The objective:

$$ \min_{U, V} \sum_{(u,i) \in \text{observed}} \bigl(r_{ui} - \mathbf{u}_u^T \mathbf{v}_i\bigr)^2 + \lambda\bigl(\|\mathbf{u}_u\|^2 + \|\mathbf{v}_i\|^2\bigr) $$
$r_{ui}$ — observed rating $\mathbf{u}_u^T \mathbf{v}_i$ — predicted rating $\lambda$ — regularisation

ALS (Alternating Least Squares) solves this by alternating: fix $U$, solve for $V$ in closed form; fix $V$, solve for $U$ in closed form. Each step is a standard least-squares regression. Repeat until convergence.

🔵 Theory — the connection

Compare to PCA (Lesson 4.3): PCA decomposes $X = U \Sigma V^T$. Matrix factorisation decomposes $R \approx U V^T$. Both find low-rank structure. The difference: PCA minimises reconstruction error on the full matrix; matrix factorisation minimises it only on observed entries. Both produce embeddings — PCA's are principal components; matrix factorisation's are user/item latent factors.

🟣 Advanced

SVD++ extends basic matrix factorisation with implicit feedback (views, clicks, time-on-page). The user embedding becomes $\mathbf{u}_u + |N(u)|^{-1/2} \sum_{j \in N(u)} \mathbf{y}_j$, where $N(u)$ is the set of items the user interacted with and $\mathbf{y}_j$ are implicit-factor vectors. This consistently outperforms explicit-only models because most user behaviour is implicit.

Implicit vs explicit feedback

Explicit feedback (star ratings) is what users deliberately tell you. Implicit feedback (clicks, views, purchases, dwell time) is what they show you through behaviour. Implicit data is 100x more abundant, always available, and often more predictive. Modern production recommenders use implicit feedback almost exclusively.

Hybrid systems

Combine content-based features (item descriptions, metadata) with collaborative signals (user-item interactions). Hybrids mitigate the cold-start problem (new items have features but no interactions) and improve coverage.

The Pivot Insight

Pause and absorb this. Matrix factorisation learns user and item embeddings by minimising reconstruction error. The embeddings are not hand-crafted features — they are features discovered by optimisation. This is the first time in the curriculum that a loss function drives feature creation.

In Lesson 4.8, we will see that a neural network does the exact same thing: hidden layer activations are embeddings, learned by minimising a loss function. The difference: neural networks can learn non-linear combinations through activation functions. Matrix factorisation produces linear embeddings; neural networks produce non-linear ones. But the core insight is identical: optimisation drives feature discovery.

Worked Example: Singapore Retail

Goal

Build three recommenders on Singapore retail data: content-based, user-based CF, and matrix factorisation. Compare recommendation quality and visualise learned embeddings.

Step 1.

Load and prepare the interaction matrix.

import polars as pl
import numpy as np
from shared import MLFPDataLoader

loader = MLFPDataLoader()
purchases = loader.load("mlfp04", "sg_retail_purchases.csv")

# Pivot to user-item matrix
R = purchases.pivot(
    on="product_id", index="user_id", values="rating"
).fill_null(0).drop("user_id").to_numpy()
Step 2.

Implement ALS matrix factorisation from scratch.

def als(R, k=20, lam=0.1, n_iter=20):
    m, n = R.shape
    U = np.random.randn(m, k) * 0.01
    V = np.random.randn(n, k) * 0.01
    mask = (R > 0).astype(float)

    for iteration in range(n_iter):
        # Fix V, solve for U
        for u in range(m):
            idx = mask[u] > 0
            V_u = V[idx]
            r_u = R[u, idx]
            U[u] = np.linalg.solve(
                V_u.T @ V_u + lam * np.eye(k), V_u.T @ r_u
            )
        # Fix U, solve for V
        for i in range(n):
            idx = mask[:, i] > 0
            U_i = U[idx]
            r_i = R[idx, i]
            V[i] = np.linalg.solve(
                U_i.T @ U_i + lam * np.eye(k), U_i.T @ r_i
            )
        # Track loss
        pred = U @ V.T
        err = np.sum(mask * (R - pred) ** 2)
        print(f"Iter {iteration+1}: MSE = {err / mask.sum():.4f}")

    return U, V

U, V = als(R, k=20)
Step 3.

Visualise the learned embeddings.

from sklearn.manifold import TSNE
import matplotlib.pyplot as plt

# Project item embeddings to 2D
V_2d = TSNE(n_components=2, random_state=42).fit_transform(V)

plt.figure(figsize=(10, 8))
plt.scatter(V_2d[:, 0], V_2d[:, 1], alpha=0.5, s=10)
plt.title("Learned Item Embeddings (2D t-SNE)")
plt.xlabel("Component 1")
plt.ylabel("Component 2")
plt.show()

Common Mistakes

  1. Training on the full matrix. You must hold out test interactions. Evaluate on unseen user-item pairs, not the ones used for training.
  2. Ignoring implicit data. Star ratings are rare; clicks are abundant. Weighting implicit feedback correctly is usually the biggest accuracy lever.
  3. Too many latent factors. $k = 20$ to $50$ is typical. Higher $k$ overfits sparse matrices and slows ALS convergence.
  4. Missing the pivot insight. The point is not just "build a recommender" — it is "optimisation discovers features". If you leave this lesson thinking it was about recommendations, go back and re-read the pivot section.
