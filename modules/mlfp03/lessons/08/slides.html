DataFlow: zero-config persistence


from kailash_dataflow import DataFlow, field

db = DataFlow()

@db.model
class Prediction:
    id: int = field(primary_key=True)
    customer_id: str
    probability: float
    model_version: int

await db.express.create(Prediction,
    customer_id="C001", probability=0.73,
    model_version=1)

Define a model, get CRUD for free. No SQL.

DriftMonitor: PSI

\[ \text{PSI} = \sum_{i=1}^{k} (p_i - q_i) \cdot \ln\!\left(\frac{p_i}{q_i}\right) \]
PSIMeaningAction
< 0.1No shiftContinue
0.1-0.25ModerateInvestigate
> 0.25SignificantRetrain

DriftMonitor: KS test

DriftMonitor code


from kailash_ml import DriftMonitor, DriftSpec

monitor = DriftMonitor(spec=DriftSpec(
    features=["income", "age", "credit_score"],
    psi_threshold=0.15,
    ks_alpha=0.05,
    check_interval="daily",
))

report = monitor.check(df_train, df_prod)
for alert in report.alerts:
    print(f"  {alert.feature}: PSI={alert.psi:.3f}")

Conformal prediction


# Distribution-free prediction intervals
residuals = np.abs(y_cal - model.predict(X_cal))
q90 = np.quantile(residuals, 0.90)

y_hat = model.predict(X_new)
interval = (y_hat - q90, y_hat + q90)
print(f"90% interval: [{interval[0]:.0f}, {interval[1]:.0f}]")
🔵 Theory: No distributional assumptions. Guaranteed coverage on exchangeable data.

Three mistakes to avoid

1. No drift monitoring
Silent degradation is the #1 production ML failure.
2. PSI on unused features
Monitor the features the model actually uses.
3. Point predictions without uncertainty
Conformal prediction adds intervals for free.
