Clustering in one picture

K-means: alternate two trivial steps

  1. Assign: each point → nearest centroid.
  2. Update: each centroid → mean of its points.

Both steps strictly decrease the inertia:

\[ J \;=\; \sum_{k=1}^{K}\ \sum_{\mathbf{x}\in C_k}\ \lVert \mathbf{x} - \boldsymbol{\mu}_k \rVert^2 \]

Squared Euclidean distance → K-means assumes spherical clusters.

K-means convergence in three snapshots

Picking K: elbow vs silhouette

  • Elbow: plot inertia vs K, look for the bend. Visual, subjective.
  • Silhouette: objective number in [−1, 1]. Pick K with highest average silhouette.

for k in range(2, 11):
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = km.fit_predict(X_scaled)
    sil_scores.append(silhouette_score(X_scaled, labels))
best_k = 2 + np.argmax(sil_scores)

Silhouette in one equation

\[ s(i) \;=\; \frac{b(i) - a(i)}{\max\{a(i),\,b(i)\}} \]
  • $a(i)$ = mean distance from $i$ to other points in its own cluster.
  • $b(i)$ = min mean distance from $i$ to another cluster.
🔵 Theory: $s \approx 1$ = clean. $s \approx 0$ = boundary. $s < 0$ = wrong cluster.

Hierarchical clustering: one tree, many cuts

Cut lower, more clusters. Cut higher, fewer.

Linkage: how do you measure cluster distance?

LinkageDistance between clustersTends to produce
singlemin pair distanceLong chains
completemax pair distanceCompact blobs
averagemean pair distanceBalanced, robust
wardΔ within-cluster varianceEqual-size blobs (K-means-like)
🔵 Theory: Ward's linkage = hierarchical K-means. Default choice.

DBSCAN: density, not centroids

Points are core (dense neighbourhood), border (reachable from core), or noise (label = −1).

Two knobs: $\varepsilon$ (radius) and min_samples. No $K$ needed.

HDBSCAN: DBSCAN at every density

  • Builds the full DBSCAN hierarchy across all $\varepsilon$.
  • Extracts the most stable clustering.
  • Only knob: min_cluster_size.
  • Handles clusters of varying density — DBSCAN cannot.

import hdbscan
hdb = hdbscan.HDBSCAN(min_cluster_size=50, min_samples=10)
labels = hdb.fit_predict(X_scaled)
# labels == -1  ⇒ noise

Spectral clustering in one line

Embed the similarity graph via the eigenvectors of its Laplacian, then run K-means in the embedding.

  • Good for non-convex clusters (moons, rings, trajectories).
  • Cost: $O(n^3)$ — only for datasets up to ~10K points.
  • Rarely needed in business data but essential when topology matters.

Three metrics, one decision

MetricRangeBetter whenWeakness
Silhouette[−1, 1]HigherFavours convex, similar-sized
Davies-Bouldin[0, ∞)LowerAssumes spherical
Calinski-Harabasz[0, ∞)HigherScales with n
⚠ Warning: All three degenerate if the data has no clusters. Use the gap statistic against a null reference to confirm structure exists.

Gap statistic — does the structure even exist?

\[ \text{Gap}(K) \;=\; \mathbb{E}\bigl[\log W_K^\star\bigr] - \log W_K \]
  • $W_K$ = within-cluster dispersion on your data.
  • $W_K^\star$ = same on a uniform null in the bounding box.
  • If best gap is at $K = 1$, your data has no clusters — an honest answer.

Standardise, always


from sklearn.preprocessing import StandardScaler
X_scaled = StandardScaler().fit_transform(X)
# Without this, a feature in "dollars" dominates
# over features in "counts" or "ratios".
Rookie mistake #1: clustering raw (unscaled) data. The feature with the biggest range wins, and the results are meaningless.

Worked example: e-commerce customer segmentation

  1. Load customers, select numeric features.
  2. Standardise. Sweep K-means $K=2..10$ by silhouette.
  3. Run GMM and HDBSCAN with the chosen $K$.
  4. Compare silhouette, DB, CH.
  5. Profile each cluster — assign business labels.

Four rookie mistakes

  1. Clustering unscaled data → worst feature dominates.
  2. Treating silhouette-optimal $K$ as "the true number of segments".
  3. Forgetting to filter HDBSCAN's -1 noise label before scoring.
  4. Letting clustering be the end state — the labels are features, not conclusions.
