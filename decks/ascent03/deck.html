Foundations

Zillow's Zestimate algorithm predicted home prices with impressive accuracy. They trusted it enough to buy 27,000 homes sight-unseen.

What Went Wrong?

Foundations

The Model Was Good

  • Median error ~2% on price estimates
  • Continuously retrained on new data
  • Best-in-class for real estate valuation

The System Was Bad

  • Uncalibrated confidence — model said "this house is worth $500K" without saying "but I could be off by $80K"

Foundations

What Is Prediction?

Foundations

Given inputs (features), guess the output (target).

Regression

Output is a continuous number

  • HDB resale price: $450,000
  • Tomorrow's temperature: 31.2°C

Classification

Output is a category

  • Loan default: Yes / No

Training vs Testing

Foundations

Why you cannot grade your own homework.

Training Set (80%)

The model learns from this data. It adjusts its parameters to fit these examples.

Test Set (20%)

Data the model has never seen. We check: does it generalise to new examples?

Analogy: A student who memorises past exam papers might score 100% on those papers, but fail the actual exam because it has new questions. The test set IS the actual exam.

What Is Overfitting?

Foundations

Memorising the training data instead of learning the pattern.

Underfitting

Model is too simple. Straight line through a curve. High error on both training and test data.

Just Right

Model captures the real pattern. Smooth curve that follows the trend. Low error on both sets.

Overfitting

Model memorises noise. Wiggly line through every point. Low training error, high test error.

Key insight: More complex is not always better. The goal is to learn the signal, not the noise.

Bias and Variance — The Darts Analogy

Foundations

Bias = how far the average prediction is from the truth. Variance = how spread out predictions are across different training sets.

Bias-Variance Decomposition

Theory

We want to predict target y = f(x) + \varepsilon, where \varepsilon \sim \mathcal{N}(0, \sigma^2).

Our model \hat{f}(x) is trained on a random sample D. What is the expected prediction error at a point x?

\text{EPE}(x) = \mathbb{E}_D\!\left[(y - \hat{f}(x))^2\right]
Expected Prediction Error: average squared error over all possible training sets D, and noise in y

Derivation — Step 1: Expand

Theory
Substitute y = f(x) + ε
\mathbb{E}\!\left[(f(x) + \varepsilon - \hat{f}(x))^2\right]
Add and subtract E[f̂(x)]
= \mathbb{E}\!\left[\bigl(\underbrace{f(x) - \mathbb{E}[\hat{f}(x)]}_{\text{bias term}} + \underbrace{\mathbb{E}[\hat{f}(x)] - \hat{f}(x)}_{\text{variance term}} + \underbrace{\varepsilon}_{\text{noise}}\bigr)^2\right]

Derivation — Step 2: Cross Terms Vanish

Theory
Expand the square — three squared terms + three cross terms

The cross terms vanish because:

  • \mathbb{E}[\varepsilon] = 0 — noise is zero-mean, independent of everything
  • \mathbb{E}[\mathbb{E}[\hat{f}] - \hat{f}] = 0 — by definition of expectation
  • Bias term f(x) - \mathbb{E}[\hat{f}(x)] is a constant w.r.t. D
\text{EPE}(x) = \underbrace{\bigl(f(x) - \mathbb{E}[\hat{f}(x)]\bigr)^2}_{\color{#DC2626}{\textbf{Bias}^2}} + \underbrace{\mathbb{E}\!\left[(\hat{f}(x) - \mathbb{E}[\hat{f}(x)])^2\right]}_{\color{#2563EB}{\textbf{Variance}}} + \underbrace{\sigma^2}_{\color{#6B7280}{\textbf{Irreducible Noise}}}

The Three Terms

Theory
TermWhat It MeasuresYou Control It By
Bias² How far your model's average prediction is from the truth More complex model (more features, deeper trees)
Variance How much your predictions change across different training sets Simpler model, more data, regularisation, ensembles
Noise σ² Randomness in the data itself. Cannot be reduced. Nothing — this is the floor
The trade-off: Reducing bias (more complex model) typically increases variance. Reducing variance (simpler model) typically increases bias. The art of ML is finding the sweet spot.

Double Descent — When the Trade-off Breaks

Advanced

Classical wisdom: more parameters = more overfitting. But modern deep learning uses billions of parameters and generalises well. Why?

Double descent (Belkin et al., 2019): Test error first decreases (underfitting zone), then increases at the interpolation threshold (classical overfitting), then decreases again in the over-parameterised regime.

In the over-parameterised regime, the model can fit training data exactly while still finding a smooth, low-norm solution — a phenomenon called benign overfitting.

This does NOT mean "just make everything bigger." It means the classical bias-variance picture is incomplete for modern architectures.

Regularisation — Why Constrain Your Model?

Foundations

Regularisation adds a penalty for complexity. It tells the model: "I will accept slightly worse training accuracy if your weights stay small and reasonable."

Analogy: When writing an essay, your teacher says "explain it in 500 words, not 5000." The constraint forces clarity — you cannot memorise every detail, so you focus on the important ideas.
\text{Loss}_{\text{regularised}} = \underbrace{\text{Loss}_{\text{data}}}_{\text{fit the data}} + \underbrace{\lambda \cdot R(\boldsymbol{w})}_{\text{penalty for large weights}}
λ controls how strongly we penalise — too high = underfitting, too low = overfitting

L1 vs L2 — Diamond vs Circle

Theory

L2 (Ridge) — Circle Constraint

R(\boldsymbol{w}) = \sum_j w_j^2 \leq t
  • Constraint region is a sphere
  • Loss contours touch the sphere at any point
  • Shrinks all weights toward zero, but rarely to zero
  • All features kept, but large weights penalised

L1 (Lasso) — Diamond Constraint

R(\boldsymbol{w}) = \sum_j |w_j| \leq t
  • Constraint region is a diamond (corners on axes)
  • Loss contours hit corners — where some w_j = 0
  • Produces sparse solutions (automatic feature selection)
  • Useful when you believe many features are irrelevant

Bayesian Interpretation

Theory

Regularisation is equivalent to placing a prior on the weights and doing MAP estimation:

\hat{\boldsymbol{w}}_{\text{MAP}} = \arg\max_{\boldsymbol{w}} \;\underbrace{\log p(D|\boldsymbol{w})}_{\text{log-likelihood (data loss)}} + \underbrace{\log p(\boldsymbol{w})}_{\text{log-prior (regulariser)}}
RegulariserPrior on WeightsEffect
L2 (Ridge)w_j \sim \mathcal{N}(0, \tau^2) — GaussianWeights shrink, none eliminated
L1 (Lasso)w_j \sim \text{Laplace}(0, b) — sharp peak at 0Weights shrink, many driven to exactly 0
The Laplace prior has heavier tails but a sharper peak at zero than the Gaussian — this is why L1 induces sparsity geometrically.

What Is a Decision Tree?

Foundations

A flowchart for making predictions. At each step, ask one question about the data and follow the answer.

Example — "Should I approve this loan?"
1. Is income > $50,000? → Yes → 2. Is credit score > 700? → Yes → Approve
                → No → Review
          → No → 3. Has collateral? → Yes → Review
                    → No → Deny

At each node, the tree finds the question (feature + threshold) that best separates the target outcomes.

Trees → Forests → Boosting

Foundations
MethodIdeaBias-Variance
Single Tree One decision flowchart Low bias, high variance (unstable)
Random Forest Train many trees on random subsets, vote Averaging reduces variance
Gradient Boosting Train trees sequentially — each corrects the previous tree's mistakes Reduces bias step by step
Bagging (Random Forest) reduces variance by averaging. Boosting reduces bias by iteratively fitting residuals. Boosting is the dominant paradigm for tabular data.

Gradient Boosting — Intuition

Foundations

Think of it as a team of specialists:

  1. Tree 1 makes a rough prediction
  2. Compute the residuals (mistakes)
  3. Tree 2 learns to predict only the mistakes of Tree 1
  4. Tree 3 learns to predict the remaining mistakes
  5. Final prediction = sum of all trees' contributions
\hat{y} = \sum_{m=1}^{M} \eta \cdot h_m(x)
M trees, each weighted by learning rate η (small steps to avoid overshooting)

XGBoost — The Objective Function

Theory

XGBoost improves gradient boosting by using second-order information (curvature) to find better splits.

\mathcal{L}^{(t)} = \sum_{i=1}^{n} \ell\!\left(y_i,\; \hat{y}_i^{(t-1)} + f_t(x_i)\right) + \Omega(f_t)
At step t: how well does the new tree ft improve the prediction? Plus a regularisation term Ω
\Omega(f) = \gamma T + \frac{1}{2}\lambda \sum_{j=1}^{T} w_j^2
γ penalises number of leaves T; λ penalises large leaf weights wj

XGBoost — 2nd-Order Taylor Expansion

Theory
Step 1: Taylor-expand the loss around the current prediction
\ell(y_i, \hat{y}_i^{(t-1)} + f_t(x_i)) \approx \ell(y_i, \hat{y}_i^{(t-1)}) + g_i f_t(x_i) + \frac{1}{2} h_i f_t(x_i)^2

Where we define the gradient and Hessian of the loss for each sample:

g_i = \frac{\partial\, \ell(y_i,\, \hat{y}^{(t-1)})}{\partial\, \hat{y}^{(t-1)}}, \qquad h_i = \frac{\partial^2 \ell(y_i,\, \hat{y}^{(t-1)})}{\partial\, (\hat{y}^{(t-1)})^2}
gi = 1st derivative (direction of steepest descent), hi = 2nd derivative (curvature)

XGBoost — Optimal Leaf Weights

Theory
Step 2: Drop constant term, group by leaf j, substitute ft(xi) = wj for sample i in leaf j
\tilde{\mathcal{L}}^{(t)} = \sum_{j=1}^{T}\!\left[G_j w_j + \frac{1}{2}(H_j + \lambda)\,w_j^2\right] + \gamma T

Where G_j = \sum_{i \in I_j} g_i and H_j = \sum_{i \in I_j} h_i are gradient and Hessian sums for leaf j.

Step 3: Differentiate w.r.t. wj, set to zero
w_j^* = -\frac{G_j}{H_j + \lambda}, \qquad \tilde{\mathcal{L}}^* = -\frac{1}{2}\sum_{j=1}^{T}\frac{G_j^2}{H_j + \lambda} + \gamma T

XGBoost — Split Gain Formula

Theory

To decide whether splitting a leaf is worthwhile, XGBoost computes the gain:

\text{Gain} = \frac{1}{2}\left[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L + G_R)^2}{H_L + H_R + \lambda}\right] - \gamma
Left child score + Right child score − Parent score − penalty for adding a leaf
Why second-order matters: Standard gradient boosting uses only gi (Newton's method with step size 1). XGBoost uses hi (Newton's method with curvature-adjusted steps), giving faster convergence and better handling of different loss functions.

LightGBM — Speed and Scale

Theory

Histogram-Based Splitting

  • Bin continuous features into ~256 discrete bins
  • Find best split by scanning bins instead of exact values
  • O(bins) instead of O(n log n) per feature
  • 10-20x faster than exact XGBoost on large data

GOSS — Gradient-based One-Side Sampling

  • Samples with large gradients contribute more to information gain
  • Keep all large-gradient samples, randomly sample small-gradient ones
  • Multiply sampled gradients by \frac{1-a}{b} to maintain distribution
  • Near-lossless accuracy with much less computation
Leaf-wise growth: LightGBM grows the leaf with largest loss reduction (leaf-wise), while XGBoost grows level-by-level. Leaf-wise can overfit more but converges faster.

NGBoost — Uncertainty from Boosting

Advanced

Standard gradient boosting predicts a point estimate. NGBoost predicts a full probability distribution.

\text{NGBoost outputs } P_\theta(y|x) \text{ instead of } \hat{y}(x)
  • Uses natural gradients (Fisher information matrix) instead of ordinary gradients
  • Natural gradient accounts for the geometry of the distribution space
  • Scoring rule: log-likelihood or CRPS (Continuous Ranked Probability Score)
Connection to Zillow: If Zillow had used distributional predictions instead of point estimates, they could have quantified their uncertainty per house and set risk-appropriate purchase prices.

Class Imbalance — The Problem

Foundations

In fraud detection, 99.9% of transactions are legitimate. A model that always predicts "not fraud" gets 99.9% accuracy.

99.9% accurate. 0% useful. The model never catches a single fraudster.

This is the class imbalance problem: when one class vastly outnumbers the other, accuracy becomes a misleading metric.

Better Metrics for Imbalanced Data

  • Precision — Of all "fraud" predictions, how many were actually fraud?
  • Recall — Of all actual fraud cases, how many did we catch?
  • F1 — Harmonic mean of precision and recall
  • AUPRC — Area under precision-recall curve (better than AUROC for imbalance)

SMOTE — And Why It Often Fails

Theory

What SMOTE Does

Synthetic Minority Oversampling Technique: creates new minority samples by interpolating between existing ones.

  1. Pick a minority sample
  2. Find its k nearest minority neighbours
  3. Create a synthetic point on the line between them

Failure Modes

  • Noisy boundaries: interpolation between classes creates ambiguous points
  • High dimensions: "nearest neighbour" loses meaning in 100+ features
  • Small clusters: SMOTE can bridge separate minority clusters, creating non-existent patterns
  • Information leak: SMOTE before train/test split leaks test information
SMOTE must only be applied to training data, never before the split. Many published papers have made this error.

Cost-Sensitive Learning

Theory

Instead of resampling the data, adjust the loss function to penalise misclassifying the minority class more heavily.

\mathcal{L} = -\sum_{i=1}^n \left[\alpha_i \cdot y_i \log(\hat{p}_i) + (1-y_i)\log(1-\hat{p}_i)\right]
αi is higher for minority class — a false negative costs more than a false positive

In XGBoost/LightGBM, this is the scale_pos_weight parameter:

# Ratio of negative to positive examples
scale_pos_weight = n_negative / n_positive  # e.g., 999 for 0.1% fraud
Advantage over SMOTE: No synthetic data, no information leak risk, works in any dimension, and integrates directly with the optimiser.

Focal Loss — Derivation

Theory

Standard cross-entropy loss, even with class weights, treats all examples equally in terms of difficulty. Focal Loss (Lin et al., 2017) down-weights easy examples so the model focuses on hard ones.

Start from binary cross-entropy
\text{CE}(p_t) = -\log(p_t)

where p_t = \hat{p} if y=1, else p_t = 1 - \hat{p}. (Probability assigned to the true class.)

Add the modulating factor (1 − pt)γ
\text{FL}(p_t) = -\alpha_t\,(1 - p_t)^\gamma \,\log(p_t)

Focal Loss — Why It Works

Theory
Scenariopt(1-pt)γ (γ=2)Effect
Easy correct0.950.0025Loss reduced 400x
Moderate0.700.09Loss reduced ~11x
Hard / misclassified0.100.81Loss barely changed
The modulating factor acts as a curriculum: easy examples contribute almost nothing to the gradient, so the model spends its optimisation budget on the hard cases — exactly the minority class examples that standard cross-entropy misses.

γ controls the focusing strength. γ=0 recovers standard CE. γ=2 is the default in the original paper. αt handles class balance; (1-pt)γ handles example difficulty. Together they are more effective than either alone.

Calibration — Can You Trust the Probabilities?

Foundations

When your model says "80% chance of loan default", does it really mean that 80% of similar cases actually default?

Calibrated model: Among all cases where the model predicts 80% probability, approximately 80% actually occur.
Uncalibrated model: The model says 80% but the true rate is 45%. You cannot use these probabilities for decisions.

Why calibration matters:

  • Setting thresholds (approve loans above 60% repayment probability)
  • Comparing risks across models or populations
  • Zillow: uncalibrated confidence led to over-aggressive purchases

Calibration Methods

Theory

Platt Scaling

Fit a logistic regression on the model's raw outputs:

p_{\text{cal}} = \frac{1}{1 + e^{-(Af(x) + B)}}
  • Parameters A, B fit on validation set
  • Works well when miscalibration is monotonic
  • Low data requirement

Isotonic Regression

Non-parametric: fit a monotone step function

  • No functional form assumed
  • More flexible than Platt
  • Needs more data (~1000+ validation samples)
  • Can overfit on small sets

ECE — Expected Calibration Error

\text{ECE} = \sum_{b=1}^{B}\frac{|B_b|}{n}\,|\text{acc}(B_b) - \text{conf}(B_b)|

Proper Scoring Rules

Advanced

A scoring rule S(p, y) is proper if it is minimised when the predicted distribution matches the true distribution. In other words: a proper score cannot be "gamed" by reporting anything other than your true belief.

Brier Score (Proper)

\text{BS} = \frac{1}{n}\sum_{i=1}^n (\hat{p}_i - y_i)^2

Strictly proper: unique minimum at \hat{p} = p_{\text{true}}. Decomposes into calibration + resolution + uncertainty.

Accuracy (NOT Proper)

\text{Acc} = \frac{1}{n}\sum_{i=1}^n \mathbb{1}[\hat{y}_i = y_i]

Depends only on the hard prediction, not the probability. A model predicting 0.51 and 0.99 get the same accuracy for a positive example. No incentive to calibrate.

Why Explain Your Model?

Foundations

A model predicts "deny this loan application." The applicant asks: why?

  • Regulatory requirement — EU AI Act Art. 13, Singapore PDPA
  • Debugging — is the model using sensible features or spurious correlations?
  • Trust — stakeholders will not deploy what they cannot understand
  • Fairness — does the model discriminate based on protected attributes?
Zillow again: If Zillow could have explained which features drove each price estimate, they might have caught that the model was over-relying on recent comps during a market bubble.

Shapley Values — Intuition

Foundations

Imagine a group project. Three students (A, B, C) submit a joint assignment and receive a grade. How much did each student contribute?

Shapley's answer (1953): Try every possible order in which students could have joined the team. Measure each student's marginal contribution in each order. Their fair share is the average of their marginal contributions across all orderings.

Replace "students" with features and "grade" with model prediction, and you have SHAP values for ML.

Shapley Axioms — The Only Fair Attribution

Theory

Shapley (1953) proved that only one attribution method satisfies all four fairness axioms simultaneously:

1. Efficiency: Contributions sum to the total. \sum_{j=1}^{p} \phi_j = f(x) - \mathbb{E}[f(x)]
2. Symmetry: If features i and j contribute equally in all coalitions, they receive equal attribution. \phi_i = \phi_j
3. Dummy (Null Player): A feature that never changes the prediction receives zero attribution. \phi_j = 0
4. Linearity (Additivity): For a model that is a sum of two models, SHAP values add: \phi_j(f + g) = \phi_j(f) + \phi_j(g)

The Shapley Value Formula

Theory
\phi_j = \sum_{S \subseteq \{1,\ldots,p\} \setminus \{j\}} \frac{|S|!\;(p - |S| - 1)!}{p!} \left[f(S \cup \{j\}) - f(S)\right]
Sum over all subsets S not containing feature j. Weight = probability of this ordering. Bracket = marginal contribution of adding feature j to coalition S.
Computational cost: 2^p subsets to evaluate. With 100 features, that is 2^{100} \approx 10^{30} evaluations. Exact Shapley values are intractable for real models.

TreeSHAP — Fast Exact SHAP for Trees

Theory

Lundberg et al. (2020) exploited the tree structure to compute exact Shapley values in polynomial time.

Key Insight

  • In a decision tree, a feature is "absent" if its split is ignored (sample follows both branches, weighted by training proportions)
  • Tree structure limits which coalitions matter
  • Dynamic programming over tree paths: O(TLD²) instead of O(2^p)

Complexity

MethodTime
Exact ShapleyO(2^p) — intractable
Kernel SHAPO(n \cdot p) per sample — approximate
TreeSHAPO(TLD^2) — exact

T = trees, L = leaves, D = max depth

Counterfactual Explanations

Advanced

SHAP answers: "Why was this prediction made?"
Counterfactuals answer: "What would need to change for a different outcome?"

Example: "Your loan was denied. If your annual income were $5,000 higher OR your credit utilisation were below 30%, the loan would have been approved."

Wachter et al. (2017) formalised this as an optimisation problem:

x' = \arg\min_{x'} \; \lambda\,(f(x') - y')^2 + d(x, x')
Find the closest point x' (minimum distance d) where the model predicts the desired outcome y'

Constraints: changes must be actionable (cannot change age) and plausible (income cannot jump from $30K to $3M).

Why Workflow Orchestration?

Foundations

A production ML system is not a single script. It is a pipeline of steps that must run in order, handle failures, and be reproducible.

Without Orchestration

  • Notebooks run manually, out of order
  • "It works on my machine"
  • No retry on failure
  • No audit trail

With Orchestration

  • Steps run in defined order
  • Parameters passed between steps
  • Failures logged and retried
  • Every run is reproducible

Hyperparameter Search — Finding the Sweet Spot

Foundations

Hyperparameters are settings you choose before training: learning rate, tree depth, regularisation strength. The model cannot learn these from data — you must search for the best combination.

Grid Search

Try every combination. Simple but exhaustive. 5 parameters × 5 values = 3,125 experiments.

Bayesian Optimisation

Use past results to predict which combination to try next. Focuses on promising regions. Typically finds the optimum in 30-50 trials.

Bayesian Optimisation — The Theory

Theory

Surrogate Model

Fit a Gaussian Process (GP) to map hyperparameters → validation metric.

The GP gives both a mean prediction (expected performance) and uncertainty (how unsure we are).

Acquisition Function

Balances exploitation (try where the GP predicts high performance) vs exploration (try where uncertainty is high).

\text{EI}(x) = \mathbb{E}\!\left[\max(f(x) - f^+, 0)\right]

Expected Improvement: how much better might this point be than the current best f^+?

Conformal Prediction — Guaranteed Uncertainty

Theory

Calibration gives better probabilities. Conformal prediction goes further: it provides prediction intervals with guaranteed coverage.

The guarantee: For a chosen confidence level α, the true value falls within the prediction interval at least (1-α) × 100% of the time, regardless of the model or data distribution.
\hat{C}(x) = \left\{y : s(x, y) \leq \hat{q}\right\}
Prediction set: all labels y whose nonconformity score s(x,y) is below quantile q̂ of calibration scores

For regression: Interval = [\hat{y} - \hat{q}, \hat{y} + \hat{q}] where \hat{q} is the (1-α) quantile of absolute residuals on a calibration set.

Elastic Net — The Best of Both

Theory

In practice, L1 alone can be unstable (selects one of several correlated features arbitrarily). Elastic Net combines both:

R(\boldsymbol{w}) = \alpha \sum_j |w_j| + (1-\alpha) \sum_j w_j^2
α=1 is pure Lasso; α=0 is pure Ridge. α=0.5 is a common default.
  • Groups of correlated features: L1 picks one, Elastic Net keeps all (with reduced weights)
  • Feature selection: still happens via the L1 component
  • Stability: the L2 component provides regularisation even when L1 saturates

XGBoost — Worked Example (Squared Loss)

Theory

For squared loss \ell(y, \hat{y}) = (y - \hat{y})^2:

Compute gi and hi
g_i = \frac{\partial (y_i - \hat{y}_i)^2}{\partial \hat{y}_i} = -2(y_i - \hat{y}_i) = -2r_i
h_i = \frac{\partial^2 (y_i - \hat{y}_i)^2}{\partial \hat{y}_i^2} = 2

For squared loss, the Hessian is constant — all samples have equal curvature. For logistic loss, h_i = \hat{p}_i(1-\hat{p}_i) varies per sample, which is where second-order information truly helps.

CatBoost — Ordered Boosting

Theory

The Target Leakage Problem

Standard gradient boosting computes residuals using all training data, including the sample being predicted. This creates a subtle form of overfitting called prediction shift.

Ordered Boosting

  • Randomly permute training samples
  • For sample i, compute residuals using only samples 1..i-1
  • Eliminates prediction shift
  • Native categorical feature handling (ordered target statistics)
LibraryKey InnovationBest For
XGBoost2nd-order Taylor, regularised objectiveGeneral purpose, GPU support
LightGBMHistogram splits, GOSS, leaf-wiseLarge datasets, speed
CatBoostOrdered boosting, native categoricalsCategorical-heavy data, less tuning

Fairness — When Models Discriminate

Foundations

Even if you remove race and gender from the features, a model can still discriminate. Why?

  • Proxy features: Postal code correlates with race. School name correlates with socioeconomic status.
  • Historical bias: Training data reflects past discrimination (fewer loans approved for certain groups).
  • Measurement bias: The target variable itself may be biased (arrests ≠ actual crime rates).
Disparate impact test (US 4/5ths rule): If the selection rate for a protected group is less than 80% of the highest selection rate, there is evidence of disparate impact.
\frac{\text{selection rate}_{minority}}{\text{selection rate}_{majority}} \geq 0.8

LIME — Local Interpretable Explanations

Theory

Where SHAP gives the theoretically optimal attribution, LIME provides a pragmatic alternative:

  1. Perturb the input around the instance to explain
  2. Get model predictions for each perturbation
  3. Fit a simple, interpretable model (linear regression) on the perturbations
  4. The coefficients of the simple model are the explanations
\xi(x) = \arg\min_{g \in G} \; \mathcal{L}(f, g, \pi_x) + \Omega(g)
Find simple model g that best approximates f locally (weighted by proximity πx) with low complexity Ω(g)
SHAP vs LIME: SHAP is mathematically grounded (Shapley axioms) but slower. LIME is fast and flexible but can produce inconsistent explanations (violates efficiency axiom).

Everyone Should Remember

  • Accuracy is misleading for imbalanced data — use F1, AUPRC, or calibrated probabilities.
  • Explainability is not optional — SHAP tells you why the model made each prediction.

Key Takeaways

Theory

If You Followed the Math

  • Bias-variance decomposition: EPE = Bias² + Variance + Noise — three independent, irreducible terms.
  • Regularisation is placing a prior on weights (L2 = Gaussian, L1 = Laplace).
  • XGBoost uses 2nd-order Taylor expansion for curvature-aware optimisation.
  • Focal Loss down-weights easy examples via (1-p_t)^\gamma, focusing on hard cases.
  • Shapley values are the unique attribution satisfying efficiency, symmetry, dummy, and linearity.
  • Calibration and proper scoring rules matter more than raw accuracy.

Key Takeaways

Advanced

For the Experts

  • Double descent: The classical bias-variance picture is incomplete for over-parameterised models.
  • NGBoost: Natural gradient boosting for distributional predictions with uncertainty.
  • Proper scoring rules: Brier score is proper (unique minimum at true probability); accuracy is not.
  • Counterfactual explanations: "What would need to change?" is more actionable than "What contributed?"
  • Conformal prediction: Distribution-free guaranteed coverage — the gold standard for uncertainty.

Cross-Validation — Reliable Performance Estimates

Theory

A single train/test split is noisy. K-fold cross-validation gives a more reliable estimate:

  1. Split data into K equal folds
  2. For each fold: train on K-1 folds, evaluate on the held-out fold
  3. Report the mean and standard deviation of the K scores
\widehat{\text{EPE}} = \frac{1}{K}\sum_{k=1}^{K} \ell(y^{(k)}, \hat{f}^{(-k)}(x^{(k)}))
Average loss across K held-out folds. The superscript (-k) means "trained without fold k."
For time series: Standard K-fold violates temporal ordering. Use time-series split (expanding window) instead. Kailash's EvalSpec handles this with cv_strategy="timeseries".

SHAP — Reading a Force Plot

Foundations

A SHAP force plot shows how each feature pushes a single prediction from the average prediction to its final value.

How to read it:
  • Red features push the prediction higher
  • Blue features push the prediction lower
  • Wider bars = larger contribution
  • Base value (left) = average prediction across all data
  • Output value (right) = this specific prediction

Global vs Local: A summary plot (beeswarm) shows all samples' SHAP values for global importance. A force plot explains one prediction. Both use the same Shapley values.

Proper Scoring Rules — Why Brier Works

Advanced

A scoring rule S(\hat{p}, y) is strictly proper if:

\mathbb{E}_{y \sim p}[S(\hat{p}, y)] > \mathbb{E}_{y \sim p}[S(p, y)] \quad \forall\, \hat{p} \neq p
The expected score is uniquely minimised when the reported probability equals the true probability

Brier Score Decomposition

\text{BS} = \underbrace{\text{REL}}_{\text{calibration}} - \underbrace{\text{RES}}_{\text{resolution}} + \underbrace{\text{UNC}}_{\text{uncertainty}}

REL measures calibration, RES measures how much predictions vary, UNC is entropy of the base rate.

Log Loss (also proper)

\text{LL} = -\frac{1}{n}\sum_i y_i\log\hat{p}_i + (1-y_i)\log(1-\hat{p}_i)

More sensitive to confident wrong predictions. Penalises \hat{p} \to 0 for y=1 very heavily.
