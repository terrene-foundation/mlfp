Lesson 4.8 DL Foundations — Neural Networks and the Training Toolkit

In Lesson 4.7 you watched matrix factorisation learn embeddings by minimising reconstruction error. A neural network does the exact same thing — hidden layer activations ARE embeddings, learned by minimising a loss function. The difference: neural networks can learn non-linear combinations through activation functions. Today you build a neural network from scratch — forward pass, backpropagation, gradient descent — then equip it with the full training toolkit: activations, dropout, batch normalisation, optimisers, learning rate schedules, and early stopping. This toolkit is your equipment for every architecture in Module 5.

Core Concepts

Neural network architecture

A feedforward neural network is a stack of layers. Each layer computes $\mathbf{z} = W\mathbf{x} + \mathbf{b}$ (a linear transformation) followed by an activation function $\mathbf{a} = f(\mathbf{z})$ (a non-linear transformation).

Forward pass

For each layer $l$:

$$ \mathbf{z}^{(l)} = W^{(l)} \mathbf{a}^{(l-1)} + \mathbf{b}^{(l)} \qquad \mathbf{a}^{(l)} = f\bigl(\mathbf{z}^{(l)}\bigr) $$

The final $\mathbf{a}^{(L)}$ is the prediction $\hat{y}$. The loss $\mathcal{L}(\hat{y}, y)$ measures how far the prediction is from the target.

Backpropagation

Backpropagation computes $\partial \mathcal{L} / \partial W^{(l)}$ for every layer by applying the chain rule backward through the network:

$$ \frac{\partial \mathcal{L}}{\partial W^{(l)}} = \frac{\partial \mathcal{L}}{\partial \mathbf{a}^{(L)}} \cdot \frac{\partial \mathbf{a}^{(L)}}{\partial \mathbf{z}^{(L)}} \cdots \frac{\partial \mathbf{z}^{(l)}}{\partial W^{(l)}} $$

Gradient descent

Update every weight: $W \leftarrow W - \eta \, \partial \mathcal{L} / \partial W$ where $\eta$ is the learning rate.

🔵 Theory

Linear regression with zero hidden layers IS a neural network: $\hat{y} = W \mathbf{x} + b$. Adding hidden layers lets the model "write its own parametric function". Two or more hidden layers can approximate any continuous function (universal approximation theorem).

Dropout

Randomly zero out neurons during training with probability $p$ (typically 0.1–0.5). Forces the network to learn redundant representations — no single neuron can be critical. Turn off during inference.

Batch normalisation

$$ \hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} \qquad y_i = \gamma \hat{x}_i + \beta $$

Normalise each layer's inputs to zero mean, unit variance, then rescale with learnable parameters $\gamma$ and $\beta$. Stabilises training and enables higher learning rates.

Learning rate schedules

Warmup + cosine: start with a small learning rate, ramp up over the first few epochs (warmup), then decay following a cosine curve. This is the standard schedule for transformer training and works well for most architectures.

Early stopping

Monitor validation loss each epoch. If it has not improved for $P$ consecutive epochs (patience), stop training and restore the best weights. Prevents overfitting without manual epoch tuning.

Hidden Layers as Automated Feature Engineering

🟣 Advanced — the deep insight

Hidden layer activations are learned representations of the input. The first hidden layer might learn simple features (edges in images, word associations in text). Deeper layers compose these into complex features (shapes, phrases). The output layer performs supervised prediction on these learned features. This is representation learning: the network simultaneously discovers features (unsupervised, in the hidden layers) and uses them for prediction (supervised, in the output layer). The hidden node values are embeddings — exactly like the matrix factorisation embeddings in 4.7, but non-linear.

Worked Example: HDB Price Prediction

Goal

Build a 3-layer neural network from scratch for HDB resale price prediction. Implement forward pass, backpropagation, gradient descent. Then add dropout, batch norm, Adam optimiser, and learning rate scheduling. Compare training curves with and without each technique.

Step 1.

Forward pass from scratch.

import numpy as np

def relu(z):
    return np.maximum(0, z)

def relu_grad(z):
    return (z > 0).astype(float)

def forward(X, W1, b1, W2, b2, W3, b3):
    z1 = X @ W1 + b1;    a1 = relu(z1)
    z2 = a1 @ W2 + b2;   a2 = relu(z2)
    z3 = a2 @ W3 + b3;   y_hat = z3  # linear output for regression
    return z1, a1, z2, a2, z3, y_hat
Step 2.

Backpropagation and weight update.

def backward(X, y, z1, a1, z2, a2, z3, y_hat,
             W1, b1, W2, b2, W3, b3, lr=0.001):
    m = X.shape[0]
    # Output layer gradient
    dz3 = (y_hat - y) / m
    dW3 = a2.T @ dz3;    db3 = dz3.sum(axis=0)
    # Hidden layer 2
    da2 = dz3 @ W3.T
    dz2 = da2 * relu_grad(z2)
    dW2 = a1.T @ dz2;    db2 = dz2.sum(axis=0)
    # Hidden layer 1
    da1 = dz2 @ W2.T
    dz1 = da1 * relu_grad(z1)
    dW1 = X.T @ dz1;     db1 = dz1.sum(axis=0)
    # Update
    W3 -= lr * dW3; b3 -= lr * db3
    W2 -= lr * dW2; b2 -= lr * db2
    W1 -= lr * dW1; b1 -= lr * db1
    return W1, b1, W2, b2, W3, b3
Step 3.

Training loop with loss tracking.

losses = []
for epoch in range(500):
    z1, a1, z2, a2, z3, y_hat = forward(X_train, W1, b1, W2, b2, W3, b3)
    loss = np.mean((y_hat - y_train) ** 2)
    losses.append(loss)
    W1, b1, W2, b2, W3, b3 = backward(
        X_train, y_train, z1, a1, z2, a2, z3, y_hat,
        W1, b1, W2, b2, W3, b3, lr=0.001)
    if epoch % 50 == 0:
        print(f"Epoch {epoch}: MSE = {loss:.4f}")

Common Mistakes

  1. Zero-initialising all weights. Every neuron computes the same gradient — symmetry is never broken. Use He initialisation for ReLU layers.
  2. Forgetting to turn off dropout at inference. Predictions become noisy and non-deterministic.
  3. Learning rate too high. Loss oscillates wildly or diverges. Start with $10^{-3}$ and use a schedule.
  4. No early stopping. The model overfits for 1000 epochs when it should have stopped at 200.
