Check the keys before joining


hdb_towns = set(hdb["town"].unique().to_list())
mrt_towns = set(mrt_stations["town"].unique().to_list())

matched = hdb_towns & mrt_towns      # intersection
unmatched = hdb_towns - mrt_towns    # difference

print(f"HDB towns: {len(hdb_towns)}")
print(f"Matched:   {len(matched)}")
print(f"Unmatched: {len(unmatched)}")

This check catches 90% of join errors before they happen.

Aggregate one side if keys aren't unique


# schools is one row per school; we want one row per town
school_counts = (
    schools
    .group_by("town")
    .agg(pl.col("school_name").count().alias("school_count"))
)

hdb_enriched = hdb_enriched.join(
    school_counts, on="town", how="left",
).with_columns(
    pl.col("school_count").fill_null(0),
)

If both sides have many rows per key, the join explodes into a Cartesian product. Aggregate first.

Handling nulls after a left join


# count the unmatched
n_null = hdb_enriched["distance_to_mrt_km"].null_count()

# fill with a sentinel
hdb_enriched = hdb_enriched.with_columns(
    pl.col("school_count").fill_null(0),
)

# or drop the unmatched
matched_only = hdb_enriched.filter(
    pl.col("distance_to_mrt_km").is_not_null()
)

Decide. Fill, drop, or model missingness — never let nulls silently propagate into calculations.

Set theory under the hood

\[ H \cap M = \text{towns in BOTH tables} \] \[ H \cup M = \text{towns in EITHER table} \] \[ H \setminus M = \text{towns in H, not in M} \]
  • INNER join → keys in $H \cap M$
  • LEFT join → all keys in $H$, with $H \cap M$ matched
  • OUTER join → all keys in $H \cup M$
  • Unmatched left rows → keys in $H \setminus M$

A desirability classifier


def desirability_tier(distance_km, schools):
    if distance_km is None:
        return "Unknown"
    if distance_km < 1.0 and schools >= 5:
        return "Prime"
    elif distance_km < 1.0:
        return "Good MRT access"
    elif schools >= 5:
        return "School-rich, farther MRT"
    else:
        return "Further out"

The function combines MRT distance and school count into a single label. None check handles unmatched left-join rows.

Applying the classifier per town


district_summary = (
    hdb_enriched.group_by("town")
    .agg(
        pl.col("resale_price").median().alias("median_price"),
        pl.col("distance_to_mrt_km").first().alias("dist_km"),
        pl.col("school_count").first().alias("schools"),
    )
)

# Apply the function row-by-row
for row in district_summary.iter_rows(named=True):
    tier = desirability_tier(row["dist_km"], row["schools"])
    print(f"  {row['town']:<20} {tier}")

group_by + agg collapses to one row per town, then the function classifies each town.

Four traps when joining

1. Inner where left was meant. 20,000 rows disappear silently.
2. Duplicate keys on both sides. Output explodes into Cartesian product.
3. Ignoring post-join nulls. null + 5 = null — cascading data loss.
4. Case or whitespace mismatch. "BISHAN" != "Bishan". Normalise first.
