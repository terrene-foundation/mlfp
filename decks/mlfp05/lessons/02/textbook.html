Lesson 5.2 CNNs and Computer Vision

Convolutional neural networks turned computer vision from a hand‑crafted feature‑engineering discipline into a learning problem. This lesson walks the architectural lineage — LeNet (1998), AlexNet (2012), VGG (2014), ResNet (2015), Squeeze‑and‑Excitation blocks (2017), and Vision Transformers (2020+) — and shows exactly how to build a modern residual CNN with SE blocks in PyTorch, then export it to ONNX with OnnxBridge.

Why Convolutions?

Suppose you want to classify a 224×224 colour image. Flattened it is a 150,528‑dimensional vector. A single fully‑connected layer mapping that to 1000 hidden units needs 150 million weights — and it would not generalise, because it treats a pixel in the top‑left corner as completely unrelated to the same pixel moved one column right.

Real images have two properties no MLP can exploit:

  1. Local structure. Adjacent pixels are strongly correlated; distant pixels are not. A good feature detector should only look at a small local window.
  2. Translation equivariance. A cat is a cat whether it is centred, shifted left, or shifted right. The same feature detector should work at every position.

A convolution enforces both by using the same small filter across every position in the image.

Convolution arithmetic

Given an input of width $W$, filter size $F$, padding $P$, and stride $S$, the output width is:

$$W_{\text{out}} \;=\; \left\lfloor \frac{W - F + 2P}{S} \right\rfloor + 1$$
$W$ — input width $F$ — filter size $P$ — padding $S$ — stride

Two common sanity checks you will run dozens of times in this module:

  • Same padding (stride 1, $P = (F-1)/2$) preserves the spatial size: 28×28 → 28×28.
  • Halving stride (stride 2, $P = 1$, $F = 3$) halves the spatial size: 28×28 → 14×14. You will see this over and over in ResNet and the convolutional autoencoder.
🔵 Theory — Parameter count

A Conv2d layer with $C_\text{in}$ input channels, $C_\text{out}$ output channels, and a $F \times F$ kernel has $C_\text{in} \cdot C_\text{out} \cdot F^2 + C_\text{out}$ parameters — independent of spatial size. An MLP on the same input would have parameters proportional to $H \cdot W$. This is why a CNN on a 224×224 image has ~10M parameters while the equivalent MLP would have ~150M.

ResNet: skip connections

Before ResNet, adding more layers to a CNN hurt accuracy after about 20 layers. The problem was not overfitting — it was that deep networks could not even fit the training data. Gradients vanished or exploded; the optimiser could not push updates through a long chain of non‑linearities.

ResNet's fix is absurdly simple. Instead of learning an output $H(x)$ directly, each block learns the residual $F(x) = H(x) - x$ and adds the input back:

$$H(x) \;=\; F(x) + x$$
$x$ — block input (shortcut) $F(x)$ — residual (learned) $H(x)$ — block output

The skip connection gives the gradient a clear path from the loss back to early layers — there is always an additive route that does not go through the non‑linearity. In the worst case, $F$ can learn to output zero and the block becomes the identity, so adding more layers never makes the loss worse.

Squeeze‑and‑Excitation blocks

SE blocks are a 2017 drop‑in upgrade that make a CNN pay attention to which channels are informative for the current input. For each feature map, the block:

  1. Squeezes each channel to a single number with global average pooling: $s_c = \text{GAP}(x_c)$.
  2. Excites each channel with a tiny MLP that outputs a weight in $(0, 1)$: $w = \sigma(W_2 \, \text{ReLU}(W_1 s))$.
  3. Scales each channel by its weight: $x'_c = w_c \cdot x_c$.

It adds only a handful of parameters per block and typically buys you 1–2% accuracy on ImageNet benchmarks. Modern ResNets use SE blocks by default.

Worked Example: A ResNet block with SE, trained and exported

📝 Goal

Build a small CNN that starts as plain conv, then upgrades to a ResNet basic block, then adds an SE block. Train on a synthetic shape‑classification dataset (three classes) and export the final model to ONNX.

Step 1 — imports and dataset. 
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.utils.data import DataLoader, TensorDataset
from kailash_ml import ModelVisualizer, OnnxBridge
import numpy as np

torch.manual_seed(42)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

# 28x28 grayscale synthetic shapes: 0=circle, 1=square, 2=triangle
def make_shapes(n=3000, size=28):
    X = np.zeros((n, 1, size, size), dtype=np.float32)
    y = np.random.randint(0, 3, n)
    # ... (see exercise 2 solution for the generator)
    return X, y

X_np, y_np = make_shapes()
X = torch.from_numpy(X_np).to(device)
y = torch.from_numpy(y_np).long().to(device)
loader = DataLoader(TensorDataset(X, y), batch_size=64, shuffle=True)
Step 2 — a basic residual block. 
class BasicBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn1   = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn2   = nn.BatchNorm2d(channels)

    def forward(self, x):
        identity = x
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = out + identity            # ← the skip
        return F.relu(out)

Two 3×3 convs with batch norm, an identity shortcut, and a final ReLU. Same spatial size in and out (because of padding=1). This is the exact building block of ResNet‑18 and ResNet‑34.

Step 3 — a squeeze‑and‑excitation block. 
class SEBlock(nn.Module):
    def __init__(self, channels, reduction=8):
        super().__init__()
        hidden = max(channels // reduction, 4)
        self.fc1 = nn.Linear(channels, hidden)
        self.fc2 = nn.Linear(hidden, channels)

    def forward(self, x):
        b, c, _, _ = x.shape
        s = x.mean(dim=[2, 3])                 # squeeze: GAP
        s = F.relu(self.fc1(s))
        s = torch.sigmoid(self.fc2(s))         # excite: (0, 1)
        return x * s.view(b, c, 1, 1)          # scale each channel
Step 4 — stack them into a tiny network. 
class TinyResNetSE(nn.Module):
    def __init__(self, n_classes=3):
        super().__init__()
        self.stem = nn.Sequential(
            nn.Conv2d(1, 16, 3, padding=1), nn.BatchNorm2d(16), nn.ReLU(),
        )
        self.block1 = BasicBlock(16)
        self.se1    = SEBlock(16)
        self.down   = nn.Conv2d(16, 32, 3, stride=2, padding=1)
        self.block2 = BasicBlock(32)
        self.se2    = SEBlock(32)
        self.head   = nn.Sequential(
            nn.AdaptiveAvgPool2d(1), nn.Flatten(),
            nn.Linear(32, n_classes),
        )

    def forward(self, x):
        x = self.stem(x)
        x = self.se1(self.block1(x))
        x = F.relu(self.down(x))
        x = self.se2(self.block2(x))
        return self.head(x)
Step 5 — train with a plain loop. 
model = TinyResNetSE().to(device)
opt = torch.optim.Adam(model.parameters(), lr=1e-3)
history = []

for epoch in range(15):
    running, correct = 0.0, 0
    for xb, yb in loader:
        logits = model(xb)
        loss = F.cross_entropy(logits, yb)
        opt.zero_grad(); loss.backward(); opt.step()
        running += loss.item() * xb.size(0)
        correct += (logits.argmax(1) == yb).sum().item()
    history.append(running / len(loader.dataset))
    print(f"epoch {epoch+1}: loss {history[-1]:.4f}  acc {correct/len(loader.dataset):.3f}")

Standard PyTorch training loop. Same structure as Lesson 5.1 — the only differences are cross‑entropy loss, the argmax to compute accuracy, and the fact that our model is much richer.

Step 6 — export to ONNX. 
bridge = OnnxBridge()
bridge.export(
    model=model.cpu().eval(),
    sample_input=torch.randn(1, 1, 28, 28),
    output_path="tiny_resnet_se.onnx",
)

OnnxBridge traces the model with the sample input, writes the ONNX file, and validates that a forward pass in ONNX Runtime gives the same output as PyTorch (within floating‑point tolerance). If the numbers drift, it raises — a common cause is using an unsupported op, which forces you to rewrite that piece of the network before deployment.

🟣 Advanced — Why Kaiming init matters

With zero‑mean Gaussian weights and ReLU activations, half the outputs die at every layer. To keep the forward‑pass variance constant you need to initialise weights with variance $2 / n_{\text{in}}$. Xavier init (variance $1/n_{\text{in}}$) was designed for tanh and is wrong for ReLU networks — use nn.init.kaiming_normal_ instead.

Vision Transformers in One Page

CNNs dominated vision from 2012 until about 2020. Then the Vision Transformer (ViT) arrived and matched ResNet‑50 on ImageNet with no convolutions at all. ViT splits an image into, say, 16×16 patches, flattens each patch to a vector, adds a learned positional embedding, and feeds the sequence into a standard Transformer encoder (which you will meet in Lesson 5.4).

For large‑scale pre‑training on billions of images, ViTs are now competitive or ahead. For small datasets and compute budgets, CNNs are still often the better choice. Lesson 5.7 shows how to fine‑tune a pre‑trained ResNet for a new task; the same techniques apply to ViTs.

Common Mistakes

⚠ Mistake 1: Forgetting to put BN in eval mode

BatchNorm behaves differently at training vs eval time. At train time it uses batch statistics; at eval time it uses running averages. If you forget model.eval() before validation, your metrics will be noisy or just plain wrong. Same thing when exporting to ONNX.

⚠ Mistake 2: Spatial size mismatch in the skip

The identity shortcut only works when the block's input and output have the same shape. When you downsample (stride 2) or change the channel count, you need a 1×1 convolution on the shortcut to match shapes: identity = self.shortcut(x).

⚠ Mistake 3: Learning rate too high for deep nets

Deep CNNs are fragile to learning rate. Start with 1e‑3 (Adam) or 1e‑1 (SGD + momentum + cosine schedule). If loss explodes to NaN in the first 100 steps, halve the LR and retry. Mixed precision makes this worse — add a gradient scaler.

⚠ Mistake 4: view vs reshape after a conv stack

After adaptive average pooling, use torch.flatten(x, 1) or nn.Flatten. Using .view(b, -1) works when the memory is contiguous but breaks silently if a preceding operation returned a non‑contiguous tensor. flatten is always safe.
