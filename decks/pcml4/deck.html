4.1 Clustering

Grouping similar things without being told what the groups are

What Is Clustering?

Foundations

Imagine you have 10,000 customers and no labels. You notice some buy luxury goods, others buy bulk groceries, others only buy on sale days.

Clustering finds these natural groups automatically.

K-Means: The Simplest Clustering Algorithm

Foundations

Think of it as "musical chairs with data points":

  1. Pick k chairs (centroids) — place them randomly
  2. Everyone sits at their nearest chair — assign each point to the closest centroid
  3. Move each chair to the middle of its group — recompute centroids
  4. Repeat until nobody changes chairs
You choose k. That's the big decision — and getting it wrong gives you meaningless groups. We'll see how to choose k shortly.

Lloyd's Algorithm: K-Means Formally

Theory

Objective: Minimise the within-cluster sum of squares (WCSS):

$$J = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2$$
x_i = data point  |  μ_k = centroid of cluster k  |  C_k = set of points in cluster k
E-step (Assignment)
$$c_i = \arg\min_k \|x_i - \mu_k\|^2$$
M-step (Update)
$$\mu_k = \frac{1}{|C_k|} \sum_{x_i \in C_k} x_i$$

Convergence and k-means++

Theory

Convergence guarantee

  • J is non-increasing at each step
  • Finite number of possible assignments
  • Therefore: always converges
  • But: converges to a local minimum, not necessarily the global one

k-means++ (Arthur & Vassilvitskii, 2007)

Smart initialisation that avoids bad starting positions:

  1. Pick first centroid uniformly at random
  2. For each remaining centroid, pick point with probability proportional to $D(x)^2$
  3. Points far from existing centroids are more likely chosen
$$P(x) = \frac{D(x)^2}{\sum_{x'} D(x')^2}$$
Guarantees O(log k)-competitive with optimal

Choosing k: The Elbow and Gap Statistic

Theory

Elbow method

Plot WCSS vs k. Look for the "elbow" where adding more clusters gives diminishing returns.

Problem: the elbow is often ambiguous.

Gap statistic (Tibshirani et al., 2001)

$$\text{Gap}(k) = E^*[\log W_k] - \log W_k$$
Compare observed WCSS to WCSS on uniformly distributed reference data

Choose smallest k such that:

$$\text{Gap}(k) \geq \text{Gap}(k+1) - s_{k+1}$$
Silhouette score — measures how similar a point is to its own cluster vs the nearest other cluster. Range [-1, 1], higher is better. $s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}$

When K-Means Fails: Non-Convex Clusters

Theory

K-means assumes spherical, equally-sized clusters. Real data often has:

  • Elongated or crescent-shaped clusters
  • Clusters within clusters
  • Unequal densities

Spectral Clustering

  1. Build similarity graph — connect nearby points (k-NN or ε-ball)
  2. Compute graph Laplacian: $L = D - W$ where D is degree matrix, W is adjacency
  3. Eigendecompose L — take the k smallest eigenvectors
  4. Run k-means in eigenspace — clusters now linearly separable
$$L_{\text{norm}} = D^{-1/2} L \, D^{-1/2} = I - D^{-1/2} W \, D^{-1/2}$$
Normalized Laplacian — invariant to cluster size differences

Eigengap Heuristic

Theory

The eigenvalues of L reveal the number of clusters:

  • k connected components → exactly k eigenvalues = 0
  • In practice: look for the largest gap between consecutive eigenvalues
  • Eigengap at position k means k is the right number of clusters
Normalized cuts (Shi & Malik, 2000)
Spectral clustering minimises the normalised cut: $$\text{Ncut}(A,B) = \frac{\text{cut}(A,B)}{\text{vol}(A)} + \frac{\text{cut}(A,B)}{\text{vol}(B)}$$ This balances cluster sizes — unlike the minimum cut which trivially isolates single points.

HDBSCAN: Density-Based Clustering

Theory

HDBSCAN finds clusters of varying density and labels outliers as noise.

  1. Core distance: distance to the k-th nearest neighbour
  2. Mutual reachability distance: $d_{\text{mreach}}(a,b) = \max\{\text{core}(a), \text{core}(b), d(a,b)\}$
  3. Build a minimum spanning tree on mutual reachability distances
  4. Convert to a condensed cluster tree — hierarchical density structure
  5. Extract clusters using excess of mass (stability-based selection)
Key advantage: You do NOT need to specify k. HDBSCAN discovers the number of clusters automatically. Points that don't belong to any cluster are labelled as noise (-1).

Information-Theoretic Clustering

Advanced

Instead of geometric distance, cluster by information content.

$$\min_{C} \; I(X; C) \quad \text{subject to} \quad I(Y; C) \geq I_0$$
Information Bottleneck (Tishby et al., 2000) — compress X into clusters C while preserving information about relevance variable Y

When to use

  • Text clustering (words carry information, not geometry)
  • Gene expression (correlation ≠ Euclidean distance)
  • Network community detection

Connection

Information Bottleneck was recently shown to explain deep learning generalisation (Shwartz-Ziv & Tishby, 2017) — layers progressively compress input while preserving label information.

4.2 EM Algorithm and Gaussian Mixture Models

Soft clustering — when points belong to multiple groups with different probabilities

Hard vs Soft Clustering

Foundations

K-means (hard)

Every point belongs to exactly one cluster.

"This customer IS a luxury buyer."

But what about the customer who buys luxury items on weekdays and bulk groceries on weekends?

GMM (soft)

Every point has a probability of belonging to each cluster.

"This customer is 70% luxury buyer, 25% bulk buyer, 5% sale hunter."

Captures the real-world ambiguity in group membership.

Gaussian Mixture Model assumes data comes from a mix of bell curves (Gaussians). Each cluster is one bell curve with its own centre, spread, and weight.

GMM: The Generative Model

Theory

A GMM assumes each data point is generated by:

  1. Pick a component k with probability $\pi_k$ (mixing weight)
  2. Draw from that component's Gaussian: $x \sim \mathcal{N}(\mu_k, \Sigma_k)$
$$p(x) = \sum_{k=1}^{K} \pi_k \; \mathcal{N}(x \mid \mu_k, \Sigma_k)$$
π_k = mixing weight (sums to 1)  |  μ_k = mean of component k  |  Σ_k = covariance of component k

Parameters to learn: $\theta = \{\pi_k, \mu_k, \Sigma_k\}_{k=1}^K$

Problem: We can't directly maximise the log-likelihood because we don't know which component generated each point.

The Log-Likelihood Problem

Theory

The log-likelihood is:

$$\ell(\theta) = \sum_{i=1}^{N} \log \left[ \sum_{k=1}^{K} \pi_k \; \mathcal{N}(x_i \mid \mu_k, \Sigma_k) \right]$$
The log of a sum has no closed-form MLE solution

Setting $\frac{\partial \ell}{\partial \mu_k} = 0$ gives:

$$\mu_k = \frac{\sum_{i=1}^{N} \gamma_{ik} \, x_i}{\sum_{i=1}^{N} \gamma_{ik}} \quad \text{where} \quad \gamma_{ik} = \frac{\pi_k \, \mathcal{N}(x_i \mid \mu_k, \Sigma_k)}{\sum_{j} \pi_j \, \mathcal{N}(x_i \mid \mu_j, \Sigma_j)}$$
Circular: μ_k depends on γ_ik, which depends on μ_k. EM breaks this circularity.

EM Algorithm: E-Step

Theory
E-step: Compute Responsibilities (Posterior)

For each point $x_i$ and component k, compute the posterior probability that component k generated $x_i$:

$$\gamma_{ik} = \frac{\pi_k \; \mathcal{N}(x_i \mid \mu_k, \Sigma_k)}{\sum_{j=1}^{K} \pi_j \; \mathcal{N}(x_i \mid \mu_j, \Sigma_j)}$$
Bayes' rule: prior × likelihood / evidence
Plain English: "Given our current best guess of the bell curves, how likely is it that this particular point came from bell curve k?" A point sitting between two bell curves gets partial credit from both.

EM Algorithm: M-Step

Theory
M-step: MLE Updates for Gaussian Mixture

Using the responsibilities from the E-step, update all parameters by maximising the expected complete-data log-likelihood:

Effective count:

$$N_k = \sum_{i=1}^{N} \gamma_{ik}$$

Updated mean:

$$\mu_k^{\text{new}} = \frac{1}{N_k} \sum_{i=1}^{N} \gamma_{ik} \, x_i$$

Updated mixing weight:

$$\pi_k^{\text{new}} = \frac{N_k}{N}$$

Updated covariance:

$$\Sigma_k^{\text{new}} = \frac{1}{N_k} \sum_{i=1}^{N} \gamma_{ik} \, (x_i - \mu_k^{\text{new}})(x_i - \mu_k^{\text{new}})^T$$

EM Convergence

Theory

Guarantee

EM monotonically increases the log-likelihood:

$$\ell(\theta^{(t+1)}) \geq \ell(\theta^{(t)})$$

Proof sketch: EM maximises a lower bound (ELBO) on $\ell(\theta)$ at each step.

  • Converges to a local maximum
  • Multiple random restarts help find global
  • BIC/AIC for model selection (choosing K)

4.3 Dimensionality Reduction

Projecting 100 columns to 2 so you can see patterns

The Curse of Dimensionality

Foundations

Your e-commerce dataset has 200 features: purchase history, browsing time, click patterns, demographics...

You cannot visualise 200 dimensions. You cannot compute distances reliably in 200 dimensions.

Dimensionality reduction = finding a smaller set of "super-features" that capture most of the information.

Analogy: A shadow is a 2D projection of a 3D object. You lose some information (depth), but a good shadow preserves the shape. PCA finds the best "angle" to cast the shadow.

PCA: Finding the Best Shadow

Theory

Goal: Find directions of maximum variance in the data.

Step 1: Centre the data
$$\tilde{X} = X - \bar{X} \quad \text{(subtract the mean of each feature)}$$
Step 2: Compute the covariance matrix
$$C = \frac{1}{N-1} \tilde{X}^T \tilde{X} \in \mathbb{R}^{d \times d}$$
Step 3: Eigendecompose
$$C \, v_j = \lambda_j \, v_j$$

v_j = j-th eigenvector (direction of variance)  |  λ_j = j-th eigenvalue (amount of variance in that direction)

Step 4: Project
$$Z = \tilde{X} \, V_k \quad \text{where } V_k = [v_1, \ldots, v_k] \text{ (top-k eigenvectors)}$$

PCA: Why Eigenvalues = Variance

Theory

The variance of data projected onto direction w is:

$$\text{Var}(Xw) = w^T C w$$

Maximising $w^T C w$ subject to $\|w\|=1$ (Lagrange multiplier):

$$\frac{\partial}{\partial w}\left[w^T C w - \lambda(w^T w - 1)\right] = 0 \implies Cw = \lambda w$$
The maximiser is an eigenvector, and the variance along that direction equals the eigenvalue.

Scree plot

Plot $\lambda_j$ vs j. Look for the "elbow" — dimensions after the elbow carry mostly noise.

Explained variance ratio: $\frac{\lambda_j}{\sum_i \lambda_i}$

SVD connection

PCA via SVD: $\tilde{X} = U \Sigma V^T$

Principal components = columns of $U\Sigma$

Eigenvalues: $\lambda_j = \frac{\sigma_j^2}{N-1}$

SVD is numerically more stable than eigendecomposing C directly.

PCA Reconstruction Error

Theory

If we keep only k of d components, the reconstruction error is:

$$\text{Error} = \sum_{j=k+1}^{d} \lambda_j = \text{total variance} - \text{retained variance}$$
Minimising reconstruction error is equivalent to maximising retained variance

The Eckart-Young theorem guarantees PCA gives the best rank-k approximation under the Frobenius norm:

$$\|\tilde{X} - \tilde{X}_k\|_F^2 = \sum_{j=k+1}^{d} \sigma_j^2$$
Rule of thumb: Keep enough components to explain 90-95% of variance. If 5 components out of 200 explain 93% of variance, those 5 "super-features" are almost as good as the original 200.

t-SNE: Preserving Local Neighbourhoods

Theory

PCA preserves global structure (overall variance). t-SNE preserves local structure (who is near whom).

Step 1: Define pairwise similarities in high-D
$$p_{j|i} = \frac{\exp(-\|x_i - x_j\|^2 / 2\sigma_i^2)}{\sum_{k \neq i} \exp(-\|x_i - x_k\|^2 / 2\sigma_i^2)}$$

Symmetrise: $p_{ij} = \frac{p_{j|i} + p_{i|j}}{2N}$

Step 2: Define pairwise similarities in low-D (Student-t with 1 DOF)
$$q_{ij} = \frac{(1 + \|y_i - y_j\|^2)^{-1}}{\sum_{k \neq l} (1 + \|y_k - y_l\|^2)^{-1}}$$

t-SNE: The KL Divergence Objective

Theory
Step 3: Minimise KL divergence between P and Q
$$\text{KL}(P \| Q) = \sum_{i \neq j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$
Penalises: nearby points in high-D mapped far apart in low-D (high p_ij, low q_ij)

Perplexity

Controls the effective number of neighbours:

$$\text{Perp}(P_i) = 2^{H(P_i)}$$

Typical range: 5-50. Lower = local structure, higher = global structure.

Why Student-t?

Heavy tails prevent the crowding problem: in 2D, there isn't enough room for moderately-distant points. The t-distribution gives them more space to spread.

Caution: t-SNE is for visualisation only. Do not cluster on t-SNE output. Do not interpret distances between clusters. Run it multiple times — different runs give different shapes.

UMAP: Topological Data Analysis

Theory

UMAP (McInnes et al., 2018) constructs a topological representation and optimises a cross-entropy objective.

Key ideas

  1. Build a fuzzy simplicial set (weighted k-NN graph) in high-D
  2. Construct a similar graph in low-D
  3. Minimise the cross-entropy between the two graphs
$$\mathcal{L} = \sum_{e} \left[ w_h(e) \log\frac{w_h(e)}{w_l(e)} + (1-w_h(e)) \log\frac{1-w_h(e)}{1-w_l(e)} \right]$$

UMAP vs t-SNE

t-SNEUMAP
SpeedO(N log N)O(N)
Global structurePoorBetter
DeterministicNoNearly
New point projectionNoYes (transform)
Theory basisKL divergenceRiemannian geometry

Kernel PCA and ICA

Advanced

Kernel PCA

PCA in a feature space induced by a kernel function:

$$K_{ij} = \phi(x_i)^T \phi(x_j) = k(x_i, x_j)$$
Eigendecompose K instead of C — captures nonlinear structure

Common kernels: RBF (Gaussian), polynomial, sigmoid.

ICA (Independent Component Analysis)

PCA finds uncorrelated components. ICA finds statistically independent components.

Key application: blind source separation ("cocktail party problem").

Maximises non-Gaussianity (kurtosis or negentropy) — by the Central Limit Theorem, mixtures are more Gaussian than sources.

4.4 Anomaly Detection and Ensembles

Finding the needles in the haystack, then combining detectors

What Is an Anomaly?

Foundations

An anomaly is a data point that does not conform to the expected pattern.

Types of anomalies

  • Point anomaly — single unusual observation (fraudulent transaction)
  • Contextual anomaly — normal value, wrong context (30 degrees C in Singapore: normal. In Iceland in January: anomaly)
  • Collective anomaly — group of points that together are unusual (DDoS attack pattern)

Back to Credit Suisse

The AML system was a supervised classifier on imbalanced data. Anomaly detection treats the problem differently: model the normal, then flag deviations.

Isolation Forest

Theory

Key insight: Anomalies are easier to isolate than normal points.

  1. Randomly select a feature
  2. Randomly select a split value between min and max
  3. Recursively partition until each point is isolated
  4. Anomaly score = average path length across trees
$$s(x, n) = 2^{-E[h(x)] / c(n)}$$
h(x) = path length, c(n) = average path in BST with n nodes

Scores close to 1 → anomaly (short path)

Scores close to 0.5 → normal (average path)

Time complexity: O(n log n) — scales to millions of records.

Local Outlier Factor (LOF)

Theory

LOF compares each point's local density to the density of its neighbours.

$$\text{LOF}_k(x) = \frac{\sum_{o \in N_k(x)} \frac{\text{lrd}_k(o)}{\text{lrd}_k(x)}}{|N_k(x)|}$$
lrd = local reachability density; LOF ≈ 1 = normal, LOF >> 1 = outlier

Strength

Detects anomalies in varying-density regions where global methods fail.

Weakness

O(n²) distance computations. Sensitive to k. Does not scale to very high dimensions.

Ensemble Methods: Combining Detectors

Theory

No single anomaly detector catches everything. Ensembles combine multiple detectors.

MethodHow It WorksWhen to Use
BlendingWeighted average of anomaly scoresDetectors have different strengths
StackingTrain a meta-model on detector outputsSome labelled anomalies available
BaggingSame algorithm on bootstrap samplesReduce variance of a single detector
BoostingFocus on missed anomalies iterativelyHigh recall requirement

4.5 NLP: Text to Topics

Teaching computers to read — from bag of words to BERTopic

What Is NLP?

Foundations

Natural Language Processing = teaching computers to work with human language.

In this lesson, we focus on one question: "What are people talking about?"

Topic modelling use cases

  • Customer support tickets → what issues are trending?
  • News articles → what events dominate coverage?
  • Product reviews → what features do people love/hate?
  • Research papers → what fields are emerging?

The evolution

  1. Bag of Words / TF-IDF
  2. Word2Vec (learned embeddings)
  3. BERTopic (modern pipeline)

TF-IDF: Words That Matter

Theory
$$\text{TF-IDF}(t, d) = \underbrace{\frac{f_{t,d}}{\sum_{t'} f_{t',d}}}_{\text{term frequency}} \times \underbrace{\log \frac{N}{|\{d : t \in d\}|}}_{\text{inverse document frequency}}$$
TF: how often this word appears in this document  |  IDF: how rare this word is across all documents
Plain English: A word is important to a document if it appears frequently in that document but rarely in others. "The" appears everywhere (low IDF). "Transformer" appears only in ML papers (high IDF).

Word2Vec: Words as Vectors

Theory

Skip-gram objective: predict context words from the centre word.

$$\max_\theta \sum_{t=1}^{T} \sum_{-c \leq j \leq c, j \neq 0} \log p(w_{t+j} \mid w_t)$$

With negative sampling, this simplifies to a binary classification:

$$\log \sigma(v_{w_O}^T v_{w_I}) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n} [\log \sigma(-v_{w_i}^T v_{w_I})]$$
Push context-word vectors close together, push random-word vectors apart
Result: Words with similar meanings end up near each other in vector space. "King - Man + Woman = Queen" — the geometry captures relationships.

BERTopic: The Modern Pipeline

Theory

BERTopic (Grootendorst, 2022) combines three techniques from this module:

c-TF-IDF (class-based TF-IDF) treats each cluster as a single "document" and computes TF-IDF to find the most representative words per topic: $\text{c-TF-IDF}(t, c) = \frac{f_{t,c}}{|c|} \times \log\frac{A}{f_t}$

Topic coherence — NPMI (Normalised Pointwise Mutual Information) measures how often top-k words co-occur. Higher NPMI = more coherent topics.

4.6 Drift Monitoring

Your model in production — detecting when the world changes

Why Models Degrade

Foundations

You deploy your model on Monday. By Friday, it's giving worse predictions. What happened?

Data drift

The input distribution changed.

Example: Your housing price model was trained on 2019-2022 data. In 2023, the Singapore property cooling measures shifted the entire market.

Concept drift

The relationship between inputs and outputs changed.

Example: During COVID, the relationship between restaurant location and revenue inverted — suburban locations outperformed CBD.

Models don't know they're wrong. Without monitoring, they silently degrade until someone notices the business impact — which could be months later.

PSI: Population Stability Index

Theory

PSI measures how much a feature's distribution has shifted between the reference (training) and current (production) data.

$$\text{PSI} = \sum_{i=1}^{B} (p_i^{\text{actual}} - p_i^{\text{expected}}) \times \ln\frac{p_i^{\text{actual}}}{p_i^{\text{expected}}}$$
Bin the feature into B buckets. Compare proportions in each bucket.
PSI ValueInterpretationAction
< 0.1No significant shiftContinue monitoring
0.1 - 0.25Moderate shiftInvestigate, consider retraining
> 0.25Significant shiftRetrain immediately

KS Test for Drift Detection

Theory

The Kolmogorov-Smirnov test compares two empirical distributions:

$$D = \sup_x |F_{\text{ref}}(x) - F_{\text{prod}}(x)|$$
Maximum vertical distance between two cumulative distribution functions

Advantages over PSI

  • No binning required
  • Distribution-free (non-parametric)
  • Exact p-value available

When to use which

  • PSI: production dashboards (easy to interpret, industry standard for credit risk)
  • KS: automated drift detection (statistical rigour, no binning bias)
  • Both: belt and suspenders for critical models

Performance Drift

Theory

Data drift is a leading indicator. Performance drift is the lagging confirmation.

Monitor these metrics

  • Accuracy / F1 on labelled production data
  • Prediction distribution shift (model output PSI)
  • Calibration drift (ECE over time)
  • Business metric correlation (revenue, conversion)

Governance obligation

Under the EU AI Act (Art. 9.4), high-risk AI systems must implement post-market monitoring. Singapore's AI Verify recommends continuous monitoring as part of responsible AI practices.

Drift monitoring is not optional — it is a regulatory expectation.

4.7 Deep Learning Foundations

From a single neuron to ResNets — the math behind neural networks

What Is a Neural Network?

Foundations

A neural network is layers of simple math that together learn complex patterns.

One neuron

Takes inputs, multiplies each by a weight, adds them up, and applies an activation function:

$$y = \sigma(w_1 x_1 + w_2 x_2 + \cdots + w_d x_d + b)$$

Like a dimmer switch: inputs come in, the neuron decides how much signal to pass through.

A network

Stack layers of neurons. Each layer transforms its input. The final layer produces the prediction.

  • Input layer: your raw features
  • Hidden layers: learned representations
  • Output layer: prediction

What Is Deep Learning?

Foundations

Deep learning = neural networks with many layers.

"Deep" refers to depth (number of layers), not philosophical depth.

Why depth matters

  • Shallow networks can approximate any function (universal approximation)
  • But deep networks need exponentially fewer neurons for the same complexity
  • Each layer builds on the previous — low-level features compose into high-level concepts

The hierarchy

In image recognition:

  • Layer 1: edges and textures
  • Layer 2: corners and shapes
  • Layer 3: object parts (eyes, wheels)
  • Layer 4+: whole objects (face, car)

Universal Approximation Theorem

Theory

Theorem (Cybenko, 1989; Hornik, 1991):

A feedforward network with a single hidden layer containing a finite number of neurons can approximate any continuous function on compact subsets of $\mathbb{R}^n$, to arbitrary accuracy.

Formally: for any continuous $f: [0,1]^d \to \mathbb{R}$ and $\epsilon > 0$, there exists:

$$g(x) = \sum_{j=1}^{N} \alpha_j \, \sigma(w_j^T x + b_j)$$

such that $\|f - g\|_\infty < \epsilon$

Existence, not construction. The theorem says the approximation exists but tells you nothing about how to find it. That's what training does.

Backpropagation: How Networks Learn

Theory

Goal: Compute $\frac{\partial L}{\partial w}$ for every weight w in the network, so we can update weights to reduce the loss.

Plain English: If the prediction is wrong, backprop figures out how much each weight contributed to the error, then adjusts each weight proportionally. Weights that contributed more to the error get adjusted more.

The mathematical tool: the chain rule of calculus, applied repeatedly through the computation graph.

Backprop: Computation Graph Walkthrough

Theory

Consider a 2-layer network: $x \to h \to \hat{y} \to L$

Forward Pass
$$h = \sigma(W_1 x + b_1) \qquad \hat{y} = W_2 h + b_2 \qquad L = \frac{1}{2}\|\hat{y} - y\|^2$$
Backward Pass (Chain Rule)
$$\frac{\partial L}{\partial W_2} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial W_2} = (\hat{y} - y) \cdot h^T$$
Propagate deeper
$$\frac{\partial L}{\partial W_1} = \underbrace{\frac{\partial L}{\partial \hat{y}}}_{(\hat{y}-y)} \cdot \underbrace{\frac{\partial \hat{y}}{\partial h}}_{W_2} \cdot \underbrace{\frac{\partial h}{\partial z_1}}_{\sigma'(z_1)} \cdot \underbrace{\frac{\partial z_1}{\partial W_1}}_{x^T}$$

Each factor corresponds to one edge in the computation graph. Backprop computes them right-to-left.

Backprop: The Annotated Chain

Theory
Key insight: The gradient at each layer is the product of all gradients downstream. This is why deep networks can have vanishing gradients (products of small numbers → 0) or exploding gradients (products of large numbers → infinity).

Gradient Descent: The Update Rule

Theory
$$W^{(t+1)} = W^{(t)} - \eta \frac{\partial L}{\partial W^{(t)}}$$
η = learning rate — too large: oscillate. Too small: slow convergence.

Stochastic Gradient Descent (SGD)

Compute gradient on a random mini-batch instead of the full dataset. Noisy but fast — and the noise helps escape local minima.

AdamW (the default)

Adaptive learning rates per parameter + momentum + weight decay:

$$\theta_{t+1} = \theta_t - \eta \left(\frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon} + \lambda \theta_t\right)$$
In practice: Use AdamW with learning rate ~1e-3, then reduce with a cosine schedule. This is the standard recipe for most deep learning tasks.

CNN: Convolution as Learned Features

Theory

A convolution slides a small filter (kernel) over the input, computing a dot product at each position.

$$(f * g)[n] = \sum_{m} f[m] \; g[n - m]$$
In 2D (images): the filter is a small matrix (e.g., 3×3) that learns to detect a specific pattern

Why convolution?

  • Parameter sharing: same filter applied everywhere (fewer parameters than fully connected)
  • Translation equivariance: a cat is a cat regardless of where it appears in the image
  • Local connectivity: each output depends on a small local region

Receptive field

The region of the input that influences a particular output neuron. Grows with depth:

Layer 1 (3×3 kernel): sees 3×3 pixels

Layer 2 (3×3 kernel): sees 5×5 pixels

Layer 3: sees 7×7 pixels

Deeper layers = larger receptive field = higher-level features.

CNN Architecture: Conv + Pool + FC

Theory

BatchNorm (Ioffe & Szegedy, 2015)

Normalises activations within each mini-batch. Stabilises training, allows higher learning rates.

$$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} \quad y_i = \gamma \hat{x}_i + \beta$$

Dropout (Srivastava et al., 2014)

Randomly zero out neurons during training (p = 0.5 for FC, 0.1-0.2 for conv). Acts as regularisation — prevents co-adaptation of neurons.

ResNet: The Skip Connection Revolution

Theory

Problem: Very deep networks (50+ layers) get worse accuracy, not better. This is the degradation problem.

Residual block (He et al., 2016):

$$y = F(x, \{W_i\}) + x$$
Instead of learning h(x) directly, learn the residual F(x) = h(x) - x. If the optimal mapping is close to identity, F(x) ≈ 0 is easier to learn than h(x) ≈ x.

Why it works

  • Gradient flows directly through the skip connection (no vanishing)
  • Each block only needs to learn the difference from identity
  • Effectively creates an ensemble of many shorter paths

Impact

ResNet-152 won ILSVRC 2015 with 3.57% top-5 error (surpassing human performance at ~5%). Skip connections are now standard in virtually all deep architectures.

Learning Rate Scheduling

Theory

The learning rate is the single most important hyperparameter in deep learning.

ScheduleFormulaWhen to Use
Step decay $\eta_t = \eta_0 \cdot \gamma^{\lfloor t/s \rfloor}$ Simple, predictable drops
Cosine annealing $\eta_t = \frac{\eta_0}{2}\left(1 + \cos\frac{\pi t}{T}\right)$ Smooth decay, good default
Warmup + cosine Linear increase then cosine decay Large models, transformers
One-cycle (Smith, 2019) Increase then decrease, with momentum mirroring Fast convergence
Practical recipe: Start with AdamW (lr=1e-3), warmup for 5-10% of training, then cosine decay to 1e-5. This works for most problems.

Attention: The Key Innovation

Theory

Core idea: Instead of processing a sequence left-to-right, let every position attend to every other position.

Scaled Dot-Product Attention (Vaswani et al., 2017)
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$
Q = queries ("what am I looking for?")  |  K = keys ("what do I contain?")  |  V = values ("what information do I carry?")

Why scale by √d_k?

For random Q, K vectors in $\mathbb{R}^{d_k}$:

$E[q^T k] = 0$,   $\text{Var}(q^T k) = d_k$

Large $d_k$ → large dot products → softmax saturates → near-zero gradients. Scaling by $\sqrt{d_k}$ keeps variance at 1.

Multi-head attention

Run h attention heads in parallel, each with different learned projections:

$$\text{MultiHead} = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O$$

Each head can attend to different relationships (syntax, semantics, position).

Flash Attention

Advanced

Problem: Standard attention is O(n²) in both compute and memory. For sequences of length 4096+, this is the bottleneck.

IO-aware algorithm (Dao et al., 2022)

  • Tiling: Process Q, K, V in blocks that fit in GPU SRAM
  • Kernel fusion: Compute softmax incrementally — never materialise the full n×n attention matrix in HBM
  • Recomputation: During backward pass, recompute attention from Q, K, V instead of storing it

Impact

  • 2-4x wall-clock speedup
  • 5-20x memory reduction
  • Exact (not approximate) — same output as standard attention
  • Enables training on much longer sequences (16K+ tokens)

Now the default in PyTorch 2.0+ via torch.nn.functional.scaled_dot_product_attention.

SAM and Vision Transformers

Advanced

SAM (Sharpness-Aware Minimisation)

Standard SGD finds a minimum. SAM finds a flat minimum (Foret et al., 2021):

$$\min_w \max_{\|\epsilon\| \leq \rho} L(w + \epsilon)$$
Optimise for the worst-case loss in a neighbourhood — flat minima generalise better

Two forward passes per step: one to find the adversarial perturbation, one to compute the gradient at the perturbed point.

ViT: Vision Transformer (Dosovitskiy et al., 2020)

Treats an image as a sequence of patches:

  1. Split image into 16×16 patches
  2. Flatten each patch to a vector
  3. Add positional embeddings
  4. Feed through a standard Transformer encoder

With enough data (JFT-300M), ViT matches or beats ResNet. Demonstrates that convolution is not a necessary inductive bias — attention alone suffices.

What Everyone Should Remember

  • Models degrade in production — drift monitoring (PSI, KS) is essential, not optional

If You Followed the Math

Theory
  • K-means minimises WCSS via alternating assignment and centroid update (Lloyd's algorithm)
  • EM breaks the circular dependency in GMM estimation via E-step (posterior) and M-step (MLE updates) — monotonically increases log-likelihood
  • PCA = eigendecomposition of the covariance matrix; eigenvalues are the variance in each direction
  • t-SNE minimises KL divergence between high-D Gaussian similarities and low-D Student-t similarities
  • Backpropagation = chain rule applied through the computation graph to compute gradients for all weights
  • Attention = softmax(QKᵀ/√d_k)V — scaling prevents softmax saturation

For the Experts

Advanced
  • Information Bottleneck provides an information-theoretic framework for clustering — and may explain deep learning generalisation
  • Kernel PCA captures nonlinear structure via the kernel trick; ICA finds statistically independent (not just uncorrelated) components
  • Flash Attention is IO-aware, not compute-aware — the key insight is never materialising the n×n matrix in HBM
  • SAM finds flat minima via minimax optimisation — connects optimisation geometry to generalisation
  • ViT demonstrates that the convolution inductive bias is not necessary given sufficient data — attention alone captures spatial relationships
