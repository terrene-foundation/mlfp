Lesson 3.7 Workflow Orchestration, Model Registry, and Hyperparameter Search

Until now you have built ML pipelines as sequential Python scripts. That works for prototyping — it does not work for production. This lesson introduces WorkflowBuilder to build ML pipelines as node graphs with automatic error handling, HyperparameterSearch to find optimal model configurations using Bayesian optimisation, and ModelRegistry to version and promote models through the deployment lifecycle.

WorkflowBuilder

⚙ Kailash Engine

WorkflowBuilder turns your ML pipeline into a directed graph of nodes connected by data edges. Each node is a self-contained unit — load data, preprocess, train, evaluate — and the runtime executes them in dependency order with error handling and retry logic. 

from kailash import WorkflowBuilder, PythonCodeNode, ConditionalNode

workflow = WorkflowBuilder("credit_scoring_pipeline")

# Define nodes
load_node = PythonCodeNode("load_data", code=load_data_fn)
preprocess_node = PythonCodeNode("preprocess", code=preprocess_fn)
train_node = PythonCodeNode("train", code=train_fn)
eval_node = PythonCodeNode("evaluate", code=evaluate_fn)
promote_node = ConditionalNode("promote_check",
    condition=lambda ctx: ctx["auc"] > 0.85)
register_node = PythonCodeNode("register", code=register_fn)

# Connect
workflow.connect(load_node, preprocess_node)
workflow.connect(preprocess_node, train_node)
workflow.connect(train_node, eval_node)
workflow.connect(eval_node, promote_node)
workflow.connect(promote_node, register_node, branch="yes")

# Execute
from kailash import Runtime
runtime = Runtime()
result = runtime.execute(workflow.build())

HyperparameterSearch

Grid search tries every combination and is wasteful. Random search is better but uninformed. Bayesian optimisation builds a surrogate model (Gaussian process) of the objective function and chooses the next point to evaluate based on an acquisition function (expected improvement). It finds good hyperparameters in far fewer evaluations. 

from kailash_ml import HyperparameterSearch, SearchSpace, SearchConfig

space = SearchSpace({
    "n_estimators": (100, 1000),
    "max_depth": (3, 10),
    "learning_rate": (0.01, 0.3, "log"),
    "min_child_weight": (1, 10),
})

config = SearchConfig(
    method="bayesian",
    n_trials=50,
    metric="auc",
    cv_folds=5,
)

search = HyperparameterSearch(model_class="xgboost")
best = search.run(X_train, y_train, space=space, config=config)
print(f"Best AUC: {best.score:.4f}")
print(f"Best params: {best.params}")

ModelRegistry

⚙ Kailash Engine

ModelRegistry versions every model, stores metadata (metrics, hyperparameters, data lineage), and manages the lifecycle: experiment → staging → production → retired. 

from kailash_ml import ModelRegistry, MetricSpec, ModelSignature

registry = ModelRegistry()

# Register the best model
registry.register(
    name="credit_scorer",
    model=best.model,
    metrics=MetricSpec(auc=best.score, f1=best.f1),
    signature=ModelSignature(
        input_schema=X_train.schema,
        output_schema={"probability": "Float64"},
    ),
    tags={"dataset": "credit_sg_v2", "author": "mlfp_capstone"},
)

# Promote to production
registry.promote("credit_scorer", version=1, stage="production")

Common Mistakes

⚠ Mistake 1: Grid search when Bayesian is available

Grid search with 5 hyperparameters at 5 values each = 3125 evaluations. Bayesian typically finds better results in 50.

⚠ Mistake 2: Promoting without a signature

A model without a signature cannot validate its inputs in production. Always define input and output schemas.

⚠ Mistake 3: No conditional gate on promotion

Auto-promoting every model is dangerous. Always gate promotion on evaluation metrics — the ConditionalNode pattern.
