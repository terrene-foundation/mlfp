One idea that powers four tools

  • Dimensionality reduction — compress 784 pixels into 16 numbers
  • Denoising — cleaner HDB listing photos without sliders
  • Anomaly detection — fraud scores itself as reconstruction error
  • Generation — synthetic patient records (VAE)

All from "compress then reconstruct". Unsupervised — no labels needed.

Compression forces the network to learn structure

Loss = \(\|x - \hat{x}\|^2\). Squeezing 784 dims through 8 forces the network to discover the structure that matters.

Vanilla AE in 15 lines of PyTorch


class VanillaAE(nn.Module):
    def __init__(self, d_in=256, d_latent=8):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(d_in, 64), nn.ReLU(),
            nn.Linear(64, d_latent),
        )
        self.decoder = nn.Sequential(
            nn.Linear(d_latent, 64), nn.ReLU(),
            nn.Linear(64, d_in), nn.Sigmoid(),
        )
    def forward(self, x):
        z = self.encoder(x)
        return self.decoder(z), z

Denoising AE — corrupt input, clean target


class DenoisingAE(VanillaAE):
    def add_noise(self, x, sigma=0.3):
        return torch.clamp(x + sigma * torch.randn_like(x), 0, 1)

# loss = F.mse_loss(ae(ae.add_noise(batch))[0], batch)

Forcing reconstruction of the clean original makes the encoder throw away noise and keep signal.

🟢 A powerful regulariser for free — costs one line of code.

But vanilla AE cannot generate new data

If you sample a random z and pass it to the decoder, you get nonsense.

Why? Because the latent space is not a probability distribution — it is just a set of isolated points, one per training example. Between the points is undefined territory.

We need the latent space to be continuous and smooth.

Variational AE: make the latent a distribution

Encoder emits a mean \(\mu(x)\) and a log‑variance \(\log\sigma^2(x)\).

We sample \(z \sim \mathcal{N}(\mu, \sigma^2)\) and decode.

A KL term in the loss pulls the distribution toward a standard normal prior, so between‑training‑example points are still "on the map".

🔵 This is the whole idea behind VAEs. Two changes: (1) output distribution, (2) KL regulariser.

The ELBO — two terms

\[ \mathcal{L}_{\text{ELBO}} = \underbrace{\mathbb{E}_{q_\phi(z|x)}[\log p_\theta(x|z)]}_{\text{reconstruction}} \;-\; \underbrace{\mathrm{KL}(q_\phi(z|x) \| p(z))}_{\text{regulariser}} \]

For Gaussian prior \(p(z) = \mathcal{N}(0, I)\) the KL has a closed form:

\[ \mathrm{KL} = -\tfrac{1}{2} \sum_j (1 + \log \sigma_j^2 - \mu_j^2 - \sigma_j^2) \]

The reparameterisation trick

\[ z = \mu + \sigma \odot \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, I) \]

Noise enters from the side — gradients still flow through \(\mu\) and \(\sigma\).


def reparameterise(mu, logvar):
    std = torch.exp(0.5 * logvar)
    eps = torch.randn_like(std)
    return mu + eps * std

Why the trick is necessary

Solid lines = differentiable path (gradients flow). Dashed line = stochastic sampling (no gradient). The trick: inject randomness from the side so backprop still reaches mu and sigma.

VAE loss in code


def vae_loss(recon, x, mu, logvar):
    rec = F.mse_loss(recon, x, reduction="sum")
    kl  = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return rec + kl, rec, kl

Exactly the ELBO — one line for reconstruction, one line for KL.

Watch for posterior collapse: if KL dominates, the encoder learns μ=0, σ=1 for every x. Fix with β-VAE or KL warm-up.

Generating new images


vae.eval()
with torch.no_grad():
    z = torch.randn(16, 8, device=device)
    generated = vae.decoder(z).cpu()

Sample z from the prior — it lands somewhere sensible because the KL term made it so.

Pass through the decoder — you get a new image that resembles the training distribution but is not a copy.

Four traps for autoencoder builders

1. log σ² is not σ. std = torch.exp(0.5 * logvar).
2. Loss must match output activation. Sigmoid → BCE/MSE, not BCEWithLogits.
3. Posterior collapse. KL term dominating → β‑VAE or KL warm‑up.
4. Linear AE ≈ PCA. Non‑linearity is what buys you representation power.
