XGBoost on the wrong features loses to logistic regression on the right ones — every time.

The skill that separates a senior ML engineer from a junior one is not knowing more algorithms — it is knowing how to turn a real problem into a feature matrix.

🟡 Applied: Today's dataset is a Singapore ICU cohort. Target = in-hospital mortality. Prediction time = 24 hours after admission.

The seven-stage ML pipeline

Statistics vs ML

Statistics: explain the past. Machine learning: predict the future.

\[ \min_{f}\; \mathbb{E}_{(x, y)\sim \mathcal{D}}\bigl[L(y, f(x))\bigr] \]
  • Stats cares about the coefficients; ML cares about loss on future draws.
  • Features can be ugly, redundant, non-interpretable — as long as they reduce loss.
  • That shift in objective justifies everything we do today.

Point-in-time correctness

A feature is legal only if its value would be known at the moment the prediction is made.

Three selection families

  1. Filter — model-free; rank each feature against the target (MI, chi², ANOVA). Fast. Ignores interactions.
  2. Wrapper — re-train the model with candidate subsets (forward, backward, RFE). Accurate but slow ($O(d^2)$ fits).
  3. Embedded — selection falls out of training (L1, tree importance). One fit, family-specific.
🔵 Theory: Use the intersection as your robust core, the union as your candidate pool for tuning.

Mutual information — the default filter

\[ I(X; Y) \;=\; H(Y) - H(Y \mid X) \]
  • How much the entropy of $Y$ drops once you know $X$.
  • $= 0$ iff independent; captures non-linear dependencies.
  • Invariant to monotonic reparameterisation: unit-free.

Chi-squared — for categorical targets

\[ \chi^2 \;=\; \sum_i \frac{(O_i - E_i)^2}{E_i} \]

$O_i$ observed counts, $E_i$ expected under independence. Large $\chi^2$ ⇒ the feature's distribution differs from what independence would predict ⇒ it carries information.

Kailash engines: three verbs

  • FeatureEngineer — generate derived columns from declarative recipes
  • PreprocessingPipeline — chain encoders, scalers, splitters (polars-native)
  • FeatureStore — register schema so downstream models validate inputs

Load and clip to the first 24h


vitals_24h = (
    vitals.join(admissions.select(["admission_id", "admit_time"]),
                on="admission_id")
          .with_columns(
              hours_since_admit=(
                  pl.col("timestamp") - pl.col("admit_time")
              ).dt.total_hours()
          )
          .filter(pl.col("hours_since_admit") <= 24)
)

This is where point-in-time correctness is enforced. A vital at hour 48 cannot be a feature for an hour-24 prediction.

Aggregate irregular time-series


vital_features = vitals_24h.group_by("admission_id").agg([
    pl.col("heart_rate").mean().alias("hr_mean"),
    pl.col("heart_rate").std().alias("hr_std"),
    pl.col("heart_rate").max().alias("hr_max"),
    pl.col("systolic_bp").mean().alias("sbp_mean"),
    pl.col("systolic_bp").min().alias("sbp_min"),
    pl.col("spo2").min().alias("spo2_min"),
    pl.col("heart_rate").count().alias("n_vital_readings"),
])

n_vital_readings is informative missingness: sicker patients get monitored more.

Domain feature: shock index

\[ \text{shock\_index} \;=\; \frac{\text{heart rate}}{\text{systolic BP}} \]

Clinically, a value > 1 is a red flag for early sepsis. This single derived feature often outperforms the two raw inputs — tree models love it as a single split.


vital_features = vital_features.with_columns(
    shock_index=pl.col("hr_mean") / pl.col("sbp_mean")
)

Run all three selection methods


from sklearn.feature_selection import mutual_info_classif, RFE
from sklearn.linear_model import LogisticRegression
from sklearn.ensemble import RandomForestClassifier

# Filter
mi = mutual_info_classif(X.to_numpy(), y, random_state=42)

# Wrapper
rfe = RFE(LogisticRegression(max_iter=2000), n_features_to_select=10)
rfe.fit(X.to_numpy(), y)

# Embedded
rf = RandomForestClassifier(n_estimators=200, random_state=42).fit(X, y)

The three rankings will disagree — that is the point. Use the intersection as your robust core.

Three expensive mistakes

1. Fitting the scaler on the full dataset before splitting — leaks test-set statistics into training.
2. Using future-looking features — length_of_stay, final diagnosis, total meds over admission.
3. Selecting features on the full data, then splitting — selection has already seen the test set.
