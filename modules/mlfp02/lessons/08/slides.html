FeatureStore: point-in-time correctness


from kailash_ml import FeatureStore

store = FeatureStore()

               timestamp_col="date", entity_col="town")

# Features as of Jan 2024 — no future data leakage

Step 2: EDA


# Correlation with target
for col in df.columns:
    if col != "quality":
        r = df.select(
            pl.corr(pl.col(col), pl.col("quality"))
        ).item()
        print(f"  {col}: r = {r:.3f}")

Identify the top 3-5 predictors by correlation strength.

Step 3: Engineer + store features

Step 4: Model


pipeline = TrainingPipeline(model="linear_regression")
result = pipeline.fit(
    df_eng.drop("quality"), df_eng["quality"]
)
print(f"R²: {result.r_squared:.4f}")
print(f"Adj R²: {result.adj_r_squared:.4f}")

Step 5: Hypothesis tests


# Which predictors are significant at alpha = 0.05?
for name, coef in result.coefficients.items():
    if coef.p_value < 0.05:
        print(f"  {name}: beta={coef.value:+.4f}, "
              f"t={coef.t_stat:.2f}, p={coef.p_value:.4f}")

Report effect sizes alongside p-values. "Alcohol increases quality by 0.3 points per percent ($p < 0.001$)."

Step 6: Report

  • Summary: what did you find? (2-3 sentences)
  • Key predictors: effect direction, magnitude, significance
  • Model fit: $R^2$, adjusted $R^2$, $F$-statistic
  • Limitations: missing data, selection bias, confounders
  • Recommendation: what should the stakeholder do?
🟢 Foundations: A non-technical reader should understand every sentence in the report.

Three mistakes to avoid

1. Feature leakage from the future
2. Features without domain rationale
3. p-values without effect sizes
