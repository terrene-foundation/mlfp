Module 2 · Lesson 2.5

Linear Regression

OLS · $(X^\top X)^{-1} X^\top y$ · $t$-stats · $R^2$ · $F$-stat · Dummy encoding

⏱ 60 minutes · Prereq: 2.2, 2.3

The model

\[ y_i = \beta_0 + \beta_1 x_{i1} + \beta_2 x_{i2} + \cdots + \beta_k x_{ik} + \varepsilon_i \]
  • $\beta_0$: expected $y$ when all predictors are zero
  • $\beta_j$: change in $y$ per unit change in $x_j$, holding other predictors constant
  • $\varepsilon_i$: unobserved error; captures what linear combination can't

The picture

OLS = minimise the sum of squared vertical distances (residuals).

Deriving OLS — the loss

Stack: $y = X\beta + \varepsilon$. Minimise sum of squared residuals:

\[ S(\beta) = (y - X\beta)^\top (y - X\beta) \]

Expand:

\[ S(\beta) = y^\top y - 2\beta^\top X^\top y + \beta^\top X^\top X \beta \]

Deriving OLS — setting the gradient to zero

\[ \frac{\partial S}{\partial \beta} = -2X^\top y + 2 X^\top X \beta = 0 \]

Normal equations:

\[ X^\top X \beta = X^\top y \]

If $X^\top X$ is invertible:

\[ \boxed{\; \hat\beta = (X^\top X)^{-1} X^\top y \;} \]

Standard errors and $t$-stats

Under Normal errors, the covariance of $\hat\beta$ is:

\[ \text{Var}(\hat\beta) = \sigma^2 (X^\top X)^{-1} \]

Estimate $\sigma^2$ with residual variance (note $n - k - 1$ denominator — general Bessel):

\[ \hat\sigma^2 = \frac{e^\top e}{n - k - 1} \] \[ t_j = \frac{\hat\beta_j}{\text{SE}(\hat\beta_j)} \sim t_{n-k-1} \]

$R^2$: proportion of variance explained

\[ R^2 = 1 - \frac{\text{SS}_{\text{res}}}{\text{SS}_{\text{tot}}} = \frac{\text{SS}_{\text{reg}}}{\text{SS}_{\text{tot}}} \]

Adjusted $R^2$ penalises complexity:

\[ R^2_{\text{adj}} = 1 - (1 - R^2) \cdot \frac{n - 1}{n - k - 1} \]

$R^2$ only goes up as you add predictors — even useless ones. Always report adjusted.

$F$-statistic: model vs intercept-only

\[ F = \frac{\text{SS}_{\text{reg}} / k}{\text{SS}_{\text{res}} / (n - k - 1)} = \frac{R^2 / k}{(1 - R^2) / (n - k - 1)} \]

Under $H_0$ (all slopes zero), $F \sim F_{k, n-k-1}$.

Large $F$ → small p-value → model is doing something beyond just guessing $\bar y$.

Categorical variables — dummy encoding

3-room is the base. Coefficient on is_4room = price difference between 4-room and 3-room.

Worked: HDB load and design matrix


df = (MLFPDataLoader().load("mlfp02", "hdb_resale.parquet")
      .filter(pl.col("flat_type") == "4 ROOM")
      .with_columns(
          age=pl.col("year") - pl.col("lease_commence_date"),
          log_price=pl.col("resale_price").log()))
df_enc = df.to_dummies(["town"], drop_first=True)

y = df_enc["log_price"].to_numpy()
X = df_enc.select(["floor_area_sqm", "age",
                   "town_BISHAN", "town_TAMPINES",
                   "town_YISHUN"]).to_numpy()
X = np.column_stack([np.ones(len(X)), X])  # intercept

Worked: solve the normal equations


XtX = X.T @ X
Xty = X.T @ y
beta_hat = np.linalg.solve(XtX, Xty)

# Standard errors
n, p = X.shape
e = y - X @ beta_hat
sigma2 = (e @ e) / (n - p)
se = np.sqrt(np.diag(sigma2 * np.linalg.inv(XtX)))
t_stats = beta_hat / se

That's the core of OLS — four lines. Everything else is interpretation.

Worked: R² and F


ss_tot = ((y - y.mean())**2).sum()
ss_res = (e**2).sum()
r2 = 1 - ss_res/ss_tot
r2_adj = 1 - (1 - r2) * (n - 1) / (n - p)
F = (r2 / (p-1)) / ((1 - r2) / (n - p))

Typical HDB fit: $R^2 \approx 0.82$, $F > 1000$, p < $10^{-50}$.

Interpreting coefficients (log-price)

CoefficientMeaning
$\hat\beta_{\text{area}} = 0.012$+1 m² → +1.2% in price
$\hat\beta_{\text{age}} = -0.004$+1 year → −0.4% in price
$\hat\beta_{\text{BISHAN}} = 0.18$Bishan premium ≈ 18% over the base town

Ceteris paribus. Drop any predictor from the model and every remaining coefficient can change.

Non-linearity without leaving OLS

  • Polynomial: add $x^2$ to capture curvature (diminishing returns)
  • Interaction: $x_1 \cdot x_2$ — effect of $x_1$ depends on $x_2$
  • Log transform: $\log y$ when $y$ is skewed; coefficients become multiplicative

"Linear" means linear in the coefficients — not in the predictors.

Three rookie mistakes

1. Interpreting coefficients in isolation. Coefficients depend on the whole model.
2. Chasing $R^2$. It only goes up with more predictors. Use adjusted $R^2$ and cross-validation.
3. Multicollinearity. Correlated predictors → singular $X^\top X$ → exploding standard errors. Check VIF.
