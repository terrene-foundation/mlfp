Chapter 2: PreprocessingPipeline

Overview

PreprocessingPipeline transforms raw data into ML-ready features with a single setup() call. It auto-detects whether your task is classification or regression, identifies column types, imputes missing values, encodes categoricals, scales numerics, removes outliers, and splits into train/test sets. At inference time, transform() applies the exact same transformations learned during training.

Concepts

Task Type Detection

PreprocessingPipeline examines the target column to determine task type. If the target has few unique values relative to the dataset size, it is classified as classification. If the target has many unique continuous values, it is regression.

Imputation Strategies

StrategyBehavior
meanReplace nulls with the column mean (numeric) or mode (categorical)
medianReplace nulls with the column median (numeric) or mode (categorical)
dropRemove rows containing any null value

Categorical Encoding

EncodingBehaviorOutput
onehotCreates binary columns for each categoryExpands column count; original removed
ordinalMaps categories to integersSame column, Float64 dtype
targetReplaces categories with mean target valueSame column, Float64 dtype

Normalization & Outlier Removal

When normalize=True, StandardScaler is applied to numeric columns. When remove_outliers=True, extreme rows are dropped based on the outlier_threshold percentile.

Key API Reference

Class / MethodPurpose
PreprocessingPipeline()Instantiate the pipeline
pipeline.setup(df, target, ...)Fit and transform: detect types, impute, encode, scale, split

pipeline.inverse_transform(data)Reverse scaling for interpretability

Code Walkthrough

Step 1: Create Data with Missing Values

import polars as pl

df = pl.DataFrame({
    "age": [25.0, 30.0, None, 40.0, 45.0, 50.0, 55.0, 60.0, 65.0, 70.0],
    "income": [30000.0, 45000.0, 55000.0, None, 70000.0, 80000.0,
               90000.0, 100000.0, 110000.0, 120000.0],
    "department": ["eng", "eng", "sales", "sales", "hr", None,
                   "eng", "sales", "hr", "eng"],
    "churned": [0, 0, 1, 0, 1, 0, 1, 1, 0, 0],
})

Step 2: Setup with One-Hot Encoding

from kailash_ml.engines.preprocessing import PreprocessingPipeline, SetupResult

pipeline = PreprocessingPipeline()
result = pipeline.setup(
    df,
    target="churned",
    train_size=0.8,
    seed=42,
    normalize=True,
    categorical_encoding="onehot",
    imputation_strategy="mean",
)

assert result.task_type == "classification"
assert result.train_data.height == 8
assert result.test_data.height == 2
Sync by design: Unlike DataExplorer, setup() is synchronous — no await needed. All computation happens in-memory with polars.

Step 3: Verify One-Hot Encoding

train_cols = result.train_data.columns
assert "department" not in train_cols  # Original column removed
onehot_cols = [c for c in train_cols if c.startswith("department_")]
assert len(onehot_cols) >= 2

Step 4: Transform New Data at Inference Time

new_data = pl.DataFrame({
    "age": [33.0, None],
    "income": [52000.0, 68000.0],
    "department": ["eng", "hr"],
    "churned": [0, 1],
})

transformed = pipeline.transform(new_data)
assert "department" not in transformed.columns
Order matters: You must call setup() before transform() or get_config(). Calling them on an unfitted pipeline raises RuntimeError.

Step 5: Regression Task Detection

regression_df = pl.DataFrame({
    "sqft": [800.0, 1200.0, 1500.0, 1800.0, 2200.0, 2600.0, 3000.0, 3500.0],
    "bedrooms": [1.0, 2.0, 2.0, 3.0, 3.0, 4.0, 4.0, 5.0],
    "price": [200000.0, 350000.0, 420000.0, 510000.0,
              620000.0, 730000.0, 850000.0, 980000.0],
})

pipeline_reg = PreprocessingPipeline()
result_reg = pipeline_reg.setup(regression_df, target="price", normalize=True)
assert result_reg.task_type == "regression"

Common Mistakes

MistakeWhat HappensFix
Calling transform() before setup()RuntimeErrorAlways call setup() first
Setting train_size outside (0, 1)ValueErrorKeep between 0 and 1
Using an invalid encoding nameValueErrorUse onehot, ordinal, or target
Specifying a non-existent targetValueErrorVerify column exists in the DataFrame
Passing an empty DataFrameValueErrorEnsure data has at least one row

Key Takeaways

  • PreprocessingPipeline automates imputation, encoding, scaling, and splitting.
  • Task type is auto-detected from the target column.
  • Three imputation strategies and three encoding modes cover common needs.
  • Fitted transformers ensure identical processing at inference time via transform().
  • inverse_transform() reverses scaling for human-readable interpretation.
