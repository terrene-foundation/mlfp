Opening Case: The Feature That Killed a Clinical Trial

Foundations

The Story

A pharmaceutical company built a model to predict patient outcomes in a cancer drug trial. The model achieved 98.7% accuracy on historical data.

They used it to select patients for Phase III. The trial failed catastrophically — the drug showed no effect.

Cost: $2.1 billion and 4 years.

What Happened?

Feature leakage. One feature — "days since last blood test" — encoded information about whether the patient had already started responding to treatment. The model was not predicting outcomes. It was reading the answer sheet.

Feature Quality: What Makes a Good Feature?

Foundations

Good Features Are:

  • Informative — correlated with the target
  • Non-redundant — each adds unique information
  • Stable — consistent meaning over time
  • Available — accessible at prediction time
  • Interpretable — humans can explain them

Red Flags:

  • Leaky — encodes future information
  • Noisy — mostly random
  • Highly correlated with another feature — redundant
  • Missing 90%+ of values — unreliable
  • Changes meaning over time — concept drift
Time: ~2 min. "A feature that perfectly predicts the target is suspicious — it might be leaking. A feature that correlates 0.99 with another feature is redundant — keep one, drop the other." Beginners: "Think of features like witnesses in court. You want witnesses who saw different things, not witnesses who all saw the same thing."

Feature Contracts: Declaring What You Expect

Theory

A feature contract declares the expected type, range, and meaning of each feature. If reality violates the contract, the system raises an alert.

from kailash_ml import FeatureSchema, FeatureField

schema = FeatureSchema(
    name="hdb_resale_features_v1",
    features=[
        FeatureField("floor_area_sqm", dtype="float", min_val=20, max_val=300),
        FeatureField("num_rooms", dtype="int", allowed_values=[1,2,3,4,5]),
        FeatureField("town", dtype="category", allowed_values=["Tampines","Bishan",...]),
        FeatureField("remaining_lease_years", dtype="float", min_val=0, max_val=99),
    ]
)
schema.validate(df)  # raises if data violates contract
Why contracts? Without them, a column could silently change meaning (e.g., "floor_area" switches from sqm to sqft). The model trains on garbage and nobody notices until production fails.
Time: ~2 min. "Feature contracts are like data types in programming. They catch errors early instead of late. This is especially important in teams where multiple people create features." For beginners: "Think of it like a form that rejects invalid entries." For experts: "This is the typed, versioned, auditable feature definition that FeatureStore enforces."

Numeric Scaling: Why Scale at All?

Foundations

Without Scaling

  • Floor area: 20–300 sqm
  • Price: $100,000–$2,000,000
  • Bedrooms: 1–5

The model thinks price is "more important" because its numbers are bigger. That is wrong.

With Scaling

  • Floor area: -1.2 to 2.1
  • Price: -1.5 to 1.8
  • Bedrooms: -1.0 to 1.5

All features on the same scale. The model can compare them fairly.

Required for: linear regression, logistic regression, SVM, k-NN, neural networks, PCA.
Not required for: tree-based models (random forest, XGBoost) — they split on thresholds, not magnitudes.
Time: ~2 min. "Scaling does not change the information in your data — it changes the representation so that algorithms treat all features equally." Beginners: "Imagine measuring one thing in millimetres and another in kilometres — the model thinks the kilometre feature is more important just because the number is bigger."

Scaling Methods: Standardization (Z-Score)

Theory
$$z = \frac{x - \mu}{\sigma}$$
$x$ = original value, $\mu$ = mean, $\sigma$ = standard deviation

Result

  • Mean becomes 0
  • Standard deviation becomes 1
  • Values typically range -3 to +3

When to Use

  • Data is approximately normal
  • No extreme outliers
  • Default choice for most algorithms
Time: ~2 min. "Z-score standardization is the most common scaling method. It centres data at zero and scales by spread. After standardization, a value of +2 means 'two standard deviations above average'." Walk through an example: if mean price is $500k and std is $200k, a $700k flat has z = (700-500)/200 = 1.0.

Scaling Methods: Min-Max, Robust, MaxAbs

Theory
MethodFormulaRangeBest For
Min-Max $\frac{x - x_{\min}}{x_{\max} - x_{\min}}$ [0, 1] Bounded data, neural networks
Robust $\frac{x - Q_2}{Q_3 - Q_1}$ Unbounded Data with outliers
MaxAbs $\frac{x}{|x_{\max}|}$ [-1, 1] Sparse data, centred around 0
Robust scaling uses the median ($Q_2$) and interquartile range ($Q_3 - Q_1$) instead of mean and standard deviation. Outliers affect mean/std but NOT median/IQR — so robust scaling is resistant to outliers.
Time: ~2 min. "If you have outliers, use Robust. If you need [0,1] range, use Min-Max. If your data is mostly zeros with a few non-zero values (sparse), use MaxAbs." For experts: "Robust scaling is essentially the same as standardisation but with median and IQR replacing mean and std — giving breakdown point of 25%."

Power Transforms: Box-Cox and Yeo-Johnson

Theory

Box-Cox Transform

$$y^{(\lambda)} = \begin{cases} \frac{y^\lambda - 1}{\lambda} & \lambda \neq 0 \\ \ln(y) & \lambda = 0 \end{cases}$$

Requires: strictly positive values ($y > 0$).
$\lambda$ is found by maximum likelihood.

Yeo-Johnson Transform

$$y^{(\lambda)} = \begin{cases} \frac{(y+1)^\lambda - 1}{\lambda} & y \geq 0, \lambda \neq 0 \\ \ln(y+1) & y \geq 0, \lambda = 0 \\ \frac{-((-y+1)^{2-\lambda}-1)}{2-\lambda} & y < 0, \lambda \neq 2 \\ -\ln(-y+1) & y < 0, \lambda = 2 \end{cases}$$

Handles: positive AND negative values.

Plain English: Power transforms make skewed data more symmetric (closer to a bell curve). Many algorithms work better with bell-shaped data. Use Box-Cox for positive data, Yeo-Johnson for any data.
Time: ~3 min. "Many real-world features — like income, house prices, transaction amounts — are right-skewed. Power transforms pull in the long tail and make the data more symmetric." Beginners: "Do not memorise the formula. Remember: skewed data -> power transform -> more symmetric." Experts: "The optimal lambda is found via profile likelihood. In practice, Yeo-Johnson almost always works and handles zeros and negatives."

Log and Quantile Transforms

Theory

Log Transform

$$x' = \ln(x + 1)$$
  • Special case of Box-Cox ($\lambda = 0$)
  • Compresses large values, expands small ones
  • Perfect for: prices, counts, populations
  • $+1$ handles zeros (log1p)

Quantile Transform

$$x' = \Phi^{-1}(F_n(x))$$
  • Maps to uniform or Gaussian distribution
  • $F_n(x)$ = empirical CDF
  • $\Phi^{-1}$ = Gaussian inverse CDF
  • Handles ANY distribution shape
  • Destroys outlier information
Caution: Quantile transform is aggressive — it forces ANY distribution into a bell curve. Use sparingly. Information about outliers is lost permanently.
Time: ~2 min. "Log transform is the most common — if you have prices or counts, try log first. Quantile transform is the nuclear option — forces everything into a bell curve, but at the cost of losing outlier information." For experts: "Quantile transform is non-parametric and handles multimodal distributions, but beware of information loss and overfitting on small datasets."

Categorical Encoding: The Challenge

Foundations

Models require numbers. Categories are words. Encoding converts categories to numbers.

The trap: Assigning Tampines = 1, Bishan = 2, Woodlands = 3 implies Woodlands > Bishan > Tampines. That ordering is false and will mislead the model. This is called ordinal encoding for non-ordinal data — a common mistake.

We will cover 9 encoding methods. Each has specific use cases and failure modes.

Time: ~2 min. "This is one of the most important practical skills in ML. Wrong encoding is silent — the model trains fine but learns the wrong thing. We will go through every method so you know exactly when to use each one."

Target Encoding

Theory

Replace each category with the mean of the target variable for that category.

$$\text{encoded}(c) = \frac{\sum_{i: x_i = c} y_i}{|\{i: x_i = c\}|}$$
For category $c$: average target value of all rows where feature equals $c$
DANGER: Overfitting. Naive target encoding uses the target to create a feature — this leaks target information into the features. A rare category with 2 observations gets a near-perfect encoding.
Solutions: (1) K-fold target encoding — encode using out-of-fold target means. (2) Smoothing — blend category mean with global mean. (3) Add noise — Gaussian noise proportional to 1/count.
Time: ~3 min. "Target encoding is powerful for high-cardinality categoricals but it is the most dangerous encoding method. Without regularisation, you are literally using the answer to create the question." Walk through the smoothing formula next slide.

Target Encoding: Smoothing and Regularisation

Theory
$$\text{smoothed}(c) = \frac{n_c \cdot \bar{y}_c + m \cdot \bar{y}_{\text{global}}}{n_c + m}$$
$n_c$ = count for category $c$, $\bar{y}_c$ = category mean, $m$ = smoothing factor, $\bar{y}_{\text{global}}$ = global mean

When $n_c$ is large, the encoding approaches the category mean. When $n_c$ is small, it approaches the global mean. The parameter $m$ controls how much data you need before trusting the category mean.

James-Stein Shrinkage: A Bayesian formulation — the optimal shrinkage is $B = \frac{\sigma^2_{\text{within}}}{\sigma^2_{\text{within}} + \sigma^2_{\text{between}}}$. Categories with high within-group variance shrink more toward the global mean. This is the theoretically optimal approach but is rarely implemented in practice.
Time: ~2 min. "The smoothing parameter m is like a prior sample size. Setting m=30 means you need at least 30 observations in a category before you start trusting its own mean over the global mean." For experts: "James-Stein is the admissible estimator here — it dominates the MLE for 3+ categories (Stein's paradox)."

Leave-One-Out and CatBoost Encoding

Advanced

Leave-One-Out (LOO)

$$\text{LOO}_i = \frac{\sum_{j \neq i, x_j = c} y_j}{n_c - 1}$$
  • Excludes current row from the mean
  • Reduces leakage vs naive target encoding
  • Still leaks for small categories

CatBoost Encoding

$$\text{CB}_i = \frac{\sum_{j < i, x_j = c} y_j + a \cdot p}{|\{j < i: x_j = c\}| + a}$$
  • Only uses rows BEFORE current row (ordered)
  • $a$ = smoothing, $p$ = prior (global mean)
  • Most leakage-resistant target encoding
Time: ~2 min. "LOO helps but does not fully solve leakage. CatBoost encoding is the state of the art — it processes data in order and only uses previously seen rows, which prevents any target leakage." Expert tangent: "CatBoost uses random permutations of the data order to reduce variance from the ordering."

Weight of Evidence (WoE) Encoding

Theory
$$\text{WoE}(c) = \ln\left(\frac{P(X=c|Y=1)}{P(X=c|Y=0)}\right) = \ln\left(\frac{\text{Distribution of Events}_c}{\text{Distribution of Non-Events}_c}\right)$$
  • Originated in credit scoring (Basel regulations)
  • Only for binary classification
  • Positive WoE = higher proportion of events
  • Negative WoE = higher proportion of non-events
  • Handles categories and continuous (binned)
  • Monotonic relationship with log odds
  • Often paired with Information Value (IV)
  • IV > 0.3 = strong predictor, IV < 0.02 = useless
Time: ~2 min. "WoE comes from the banking industry where it has been a regulatory standard for decades. It directly measures how much each category separates events from non-events." For experts: "IV = sum of (Distribution of Events - Distribution of Non-Events) * WoE across all categories. It is essentially the KL divergence between the event and non-event distributions."

Hash Encoding

Advanced

For very high cardinality (millions of categories — user IDs, product SKUs, IP addresses).

$$\text{column}_j = \text{hash}(x) \bmod n_{\text{components}},\quad j = 1, \ldots, n_{\text{components}}$$
Advantages:
  • Fixed dimensionality regardless of cardinality
  • Handles unseen categories at inference
  • Very fast, low memory
Disadvantages:
  • Hash collisions — different categories, same bucket
  • Not interpretable
  • Cannot recover original categories
Time: ~2 min. "Hash encoding is the go-to for production systems with millions of categories. It is the backbone of many recommendation systems." Expert tangent: "The feature hashing trick (Weinberger et al., 2009) uses signed hashes to reduce collision bias."

Cyclical Encoding: Why sin/cos?

Theory
$$x_{\sin} = \sin\left(\frac{2\pi \cdot h}{H}\right), \quad x_{\cos} = \cos\left(\frac{2\pi \cdot h}{H}\right)$$
$h$ = hour (0-23), $H$ = period (24 for hours, 12 for months, 7 for days of week)
Without cyclical: hour 23 and hour 0 appear maximally different (23 vs 0), but they are actually adjacent.
With cyclical: sin/cos encoding preserves the circular nature. 11pm and midnight are close in both features.
Time: ~2 min. "If you encode hour as a simple number (0-23), the model thinks midnight (0) and 11pm (23) are far apart. But they are one hour apart! Sin/cos encoding wraps the number line into a circle." Draw a clock on the board.

Fourier Features for Seasonality

Advanced
$$f(t) = a_0 + \sum_{k=1}^{K}\left[a_k \cos\left(\frac{2\pi k t}{P}\right) + b_k \sin\left(\frac{2\pi k t}{P}\right)\right]$$
$P$ = period, $K$ = number of harmonics (complexity control), $a_k, b_k$ = amplitudes

Fourier features capture complex seasonality — weekly, monthly, and yearly patterns simultaneously. More harmonics ($K$) = more flexible but higher overfit risk.

Production tip: Start with K=3 Fourier pairs for each known period. Use cross-validation to select K. Too many harmonics overfit; too few underfit the seasonal pattern.
Time: ~2 min. "Fourier features are how Prophet and other forecasting tools model seasonality. They decompose complex periodic patterns into a sum of simple sine and cosine waves." Expert tangent: "For irregular time series, consider using wavelet features instead of Fourier."

Theory

Filter: Variance Threshold

Foundations
$$\text{Var}(X) = \frac{1}{n}\sum_{i=1}^{n}(x_i - \bar{x})^2$$

If a feature has zero or near-zero variance, it provides no information. A column that is 99.9% constant tells the model nothing.

Rule of thumb: Drop features with variance below a threshold (e.g., 0.01 after standardisation). This is the cheapest, fastest filter — do it first.
Time: ~1 min. "This is the simplest feature selection method. If a column barely changes, it cannot help distinguish between outcomes. Drop it." Quick slide.

Filter: Correlation Threshold

Theory
$$r_{xy} = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2 \sum(y_i - \bar{y})^2}}$$
Pearson correlation coefficient: ranges from -1 (perfect negative) to +1 (perfect positive)

Feature-Target Correlation

Keep features with |r| > threshold with the target. Low correlation = likely uninformative.

Feature-Feature Correlation

If two features correlate > 0.95, drop one. They carry the same information — keeping both wastes capacity and can cause instability.

Limitation: Pearson correlation only measures linear relationships. A feature with a U-shaped relationship to the target has correlation near zero but is highly informative. Use mutual information instead.
Time: ~2 min. "Correlation is fast and intuitive, but it only catches linear relationships. Always pair with mutual information for a more complete picture."

Filter: Mutual Information

Theory
$$I(X; Y) = \sum_{x}\sum_{y} p(x, y) \log\frac{p(x, y)}{p(x)p(y)}$$
Mutual information: measures ANY dependency (linear or non-linear) between feature X and target Y
  • $I(X;Y) = 0$ ⇒ X and Y are independent
  • $I(X;Y) > 0$ ⇒ X carries information about Y
  • Captures non-linear relationships
  • Works for continuous and categorical
Connection: $I(X;Y) = H(Y) - H(Y|X)$ where $H$ is entropy. Mutual information tells you how much knowing X reduces your uncertainty about Y.
Time: ~2 min. "Mutual information is strictly more powerful than correlation — it captures any relationship, not just linear ones. The cost is slightly more computation and the need for binning or KDE for continuous variables." For experts: "MI estimation for continuous variables uses k-nearest neighbours (Kraskov estimator). The bin-based estimator is biased downward."

Filter: Chi-Squared and ANOVA F-Test

Theory

Chi-Squared ($\chi^2$)

$$\chi^2 = \sum\frac{(O_i - E_i)^2}{E_i}$$
  • Tests independence of categorical feature vs categorical target
  • $O_i$ = observed count, $E_i$ = expected count
  • Higher $\chi^2$ = more dependent = better feature

ANOVA F-Test

$$F = \frac{\text{Between-group variance}}{\text{Within-group variance}}$$
  • Tests if numeric feature differs across target classes
  • Higher F = more separation = better feature
  • Assumes normality (robust to violations)
Quick guide: Categorical feature + categorical target → Chi-squared. Numeric feature + categorical target → ANOVA F-test. Both numeric → correlation or MI.
Time: ~2 min. "These are classical statistical tests repurposed for feature selection. They are fast and well-understood. The p-value tells you if the association is statistically significant." For experts: "Note that chi-squared requires non-negative features and assumes no ordering."

Wrapper: Forward, Backward, and RFE

Theory

RFE (Recursive Feature Elimination) Train model, remove least important, repeat $O(p \cdot \log p)$ Model has feature importance (trees, linear)
Wrapper methods are expensive. Each step trains a full model. With 500 features, forward selection trains ~125,000 models. Use filters first to reduce the candidate set, then wrappers for fine-tuning.

Embedded: L1 (Lasso) Regularisation

Theory
$$\hat{\beta}_{\text{Lasso}} = \arg\min_\beta \left\{\sum_{i=1}^{n}(y_i - x_i^T\beta)^2 + \lambda\sum_{j=1}^{p}|\beta_j|\right\}$$
L1 penalty ($\lambda|\beta_j|$) drives coefficients to exactly zero — automatic feature selection

Why L1 selects features

The diamond-shaped constraint region of L1 has corners on the axes. The optimal solution is more likely to hit a corner, setting some coefficients to exactly zero.

L2 (Ridge) does NOT select

The circular constraint region of L2 has no corners. Coefficients shrink toward zero but never reach it. All features remain.

Time: ~3 min. "Lasso is the only linear model that does feature selection and prediction simultaneously. The lambda parameter controls how aggressively features are eliminated." Draw the diamond vs circle geometry on the board. "Imagine a ball rolling downhill (loss function) hitting a diamond-shaped wall (constraint). It hits a corner — that corner means some coefficients are zero."

Embedded: Tree-Based Feature Importance

Theory

Gini Importance (MDI)

$$\text{Imp}(j) = \sum_{t \in T_j} \frac{n_t}{n}\Delta \text{Gini}(t)$$
  • Sum of impurity decrease across all splits using feature $j$
  • Fast — computed during training
  • Biased toward high-cardinality features

Permutation Importance

$$\text{PI}(j) = \text{Score} - \text{Score}(\text{shuffle } j)$$
  • Shuffle one feature, measure accuracy drop
  • Model-agnostic — works with any model
  • Unbiased but slower
Key insight: Gini importance and permutation importance can disagree. A feature may have high Gini importance because it has many unique values (more split opportunities) but low permutation importance because those splits do not actually help prediction. Always use permutation importance for final decisions.
Time: ~3 min. "Tree-based importance is the most commonly used feature selection method in practice because tree models (Random Forest, XGBoost) are the most commonly used models for tabular data." For experts: "Gini importance is biased toward features with more categories or more unique values — this is a well-documented problem (Strobl et al., 2007)."

Boruta Algorithm

Advanced

Boruta is a wrapper around random forest that creates shadow features — randomised copies of each real feature — and tests whether real features outperform their shadows.

1For each feature, create a "shadow" by shuffling its values randomly
2Train a random forest on real + shadow features
3Compare each real feature's importance against the best shadow feature
4Use a statistical test (binomial) to decide: confirmed, rejected, or tentative
5Repeat until all features are decided
Why Boruta works: The shadow features represent the null hypothesis — "this feature is no better than random." If a real feature consistently beats the best shadow, it is genuinely informative, not just noise.
Time: ~2 min. "Boruta answers a different question than most methods. Instead of 'rank features by importance,' it asks 'is this feature more important than random?' That is a much more useful question." Expert tangent: "The binomial test gives a p-value for each feature. The default uses Bonferroni correction for multiple testing."

Stability Selection

Advanced
$$\hat{\Pi}_j = \frac{1}{K}\sum_{k=1}^{K}\mathbb{1}[\text{feature } j \text{ selected in subsample } k]$$
Selection probability: how often feature $j$ is selected across $K$ random subsamples

Run Lasso (or any selector) on many random subsamples of your data. Features that are consistently selected are robust; features selected only sometimes are unstable.

Stable feature: Selected in 90/100 subsamples → genuinely important
Unstable feature: Selected in 15/100 subsamples → likely noise
Theorem (Meinshausen & Buhlmann, 2010): With stability selection, the expected number of false positives is bounded by $\frac{q^2}{(2\pi_{\text{thr}} - 1) \cdot p}$ where $q$ is the expected number selected and $\pi_{\text{thr}}$ is the threshold. This gives finite-sample false positive control.
Time: ~2 min. "Stability selection adds a layer of statistical rigour on top of any selection method. If a feature is only important on one particular subset of your data, you should not trust it." For experts: "The key insight is that subsampling + thresholding gives error control even when the base method (Lasso) has none."

SHAP-Based Feature Selection

Advanced

SHAP (SHapley Additive exPlanations) values measure each feature's contribution to each individual prediction. Aggregate SHAP values across the dataset for selection.

$$\text{SHAP Importance}(j) = \frac{1}{n}\sum_{i=1}^{n}|\phi_j^{(i)}|$$
Mean absolute SHAP value for feature $j$ across all $n$ observations
SHAP vs Permutation Importance: Permutation importance measures how much accuracy drops when you destroy a feature's information. SHAP measures how much each feature contributes to each prediction. SHAP is more fine-grained — it tells you which features matter FOR WHICH OBSERVATIONS, not just overall.

We will derive SHAP axioms fully in Module 3. For now: use mean absolute SHAP as a feature importance metric.

Time: ~2 min. "SHAP is the gold standard for feature importance in 2026. It is the only method that satisfies all four Shapley axioms — efficiency, symmetry, dummy, and linearity." Preview of M3 content.

Multicollinearity: VIF

Theory
$$\text{VIF}_j = \frac{1}{1 - R_j^2}$$
$R_j^2$ = R-squared from regressing feature $j$ on ALL other features

Interpretation

  • VIF = 1: no multicollinearity
  • VIF = 5: moderately correlated
  • VIF > 10: severely multicollinear — drop one
  • VIF = ∞: perfect collinearity

Why It Matters

  • Inflates standard errors of coefficients
  • Coefficients become unstable
  • Model appears to have no significant features
  • Critical for linear/logistic regression
Plain English: VIF measures how much one feature can be predicted by all others. If VIF is high, the feature is redundant — other features already contain its information.
Time: ~2 min. "VIF is essential for linear models. Tree models are immune to multicollinearity but still benefit from removing redundant features for speed and interpretability." Walk through: "If floor_area and num_rooms have VIF > 10, they share so much information that keeping both confuses the linear model."

The Curse of Dimensionality

Theory

As the number of features grows, the amount of data needed to cover the feature space grows exponentially.

Hughes Phenomenon

  • With fixed sample size $n$, adding features initially improves performance
  • Past a critical point, adding features degrades performance
  • The model overfits the noise in the extra dimensions

Geometric Intuition

  • In 1D: 10 data points cover a line well
  • In 2D: 10 points are sparse on a plane
  • In 100D: 10 points are isolated in a vast void
  • In high dimensions, all points are far from each other
Volume of a hypersphere: As dimensions increase, the volume concentrates in a thin shell near the surface. The ratio $V_{\text{sphere}}/V_{\text{cube}} \to 0$. This means distance-based methods (k-NN, SVM with RBF kernel) break down in high dimensions.
Time: ~2 min. "This is the fundamental reason feature selection matters. More features is not always better. At some point, you are just adding noise dimensions that drown out the signal." For beginners: "Imagine trying to find a needle in a room. Now imagine the room is the size of a football field. Now imagine it is the size of the universe. That is what happens in high dimensions."

"Data leakage is the single most dangerous mistake in applied ML."

It makes your model look brilliant in evaluation and worthless in production.

What Is Data Leakage?

Foundations
Definition: Data leakage occurs when information from outside the training dataset is used to create the model. This includes using the target variable (or proxies for it) as features, or using future data that would not be available at prediction time.

Two Main Types

  • Target leakage: A feature is derived from or correlated with the target in a way that would not be available at prediction time
  • Train-test contamination: Information from the test set leaks into training (e.g., fitting scaler on all data before splitting)

The Tell-Tale Sign

Suspiciously high accuracy.
If your model achieves 99%+ accuracy on a real-world problem, your first reaction should be suspicion, not celebration.
Time: ~2 min. "Leakage is invisible in your metrics. The model looks great on evaluation data. It only fails in production where the leaked information is not available." Beginners: "Imagine studying for a test with the answer key. You score 100%. Does that mean you learned the material? No."

Train-Test Contamination

Theory

Wrong (Leakage)

# Fit scaler on ALL data
scaler.fit(all_data)
all_data = scaler.transform(all_data)
# THEN split
train, test = split(all_data)
# Test set statistics leaked
# into training via the scaler

Correct (No Leakage)

# Split FIRST
train, test = split(all_data)
# Fit scaler on TRAIN only
scaler.fit(train)
train = scaler.transform(train)
test = scaler.transform(test)
# Test remains unseen
This applies to ALL preprocessing: scaling, encoding, imputation, feature selection, PCA. Everything must be fit on training data only, then applied to test data with the same parameters.
Time: ~2 min. "This is the most common form of leakage and the easiest to make. The rule is simple: SPLIT FIRST, then preprocess." Show both code blocks side by side.

Detecting Leakage

Theory
1Suspiciously high accuracy — real-world problems rarely exceed 95%. If yours does, investigate.
2Feature importance analysis — if one feature dominates, ask "why?" It may be a proxy for the target.
3Temporal validation — train on past, test on future. If accuracy drops dramatically, you have temporal leakage.
4Feature audit — for each feature, ask: "Would this be available at prediction time?"
5Adversarial validation — train a model to distinguish train from test. If it can (AUC > 0.8), the splits are contaminated.
Time: ~2 min. "Adversarial validation is a clever technique: you label train data as 0 and test data as 1, then train a classifier. If it can distinguish them, your train and test are drawn from different distributions — which means contamination."

Prevention: The Feature Pipeline

Theory
1Define prediction time — the exact moment the prediction is needed
2Split BEFORE preprocessing — temporal split for time-series, random for i.i.d.
3Fit on training data only — scalers, encoders, imputers, selectors
4Apply to test with fitted parameters — no recomputing statistics
5Use FeatureStore — point-in-time correctness is structural, not manual discipline
Kailash FeatureStore enforces point-in-time correctness by design. When you request features for a given timestamp, only data available at that timestamp is used. Leakage is structurally impossible.
Time: ~2 min. "The key word is STRUCTURAL. Manual discipline fails — someone forgets, someone is tired, someone does not know. FeatureStore makes it impossible to leak, the same way a compiler makes it impossible to assign a string to an integer in a typed language."

Advanced Imputation: KNN and MICE

Theory

KNN Imputation

  • Find K nearest neighbours (using non-missing features)
  • Impute with neighbours' average value
  • Captures local patterns
  • Slow for large datasets
  • Sensitive to scale — always standardise first

MICE (Multiple Imputation by Chained Equations)

  • Each feature with missing values gets its own imputation model
  • Models are conditional on all other features
  • Iterate: impute all → re-impute using updated values → repeat
  • Produces multiple datasets (captures uncertainty)
  • Gold standard for MAR data
Time: ~2 min. "MICE is the most principled imputation method. It creates multiple imputed datasets and combines them using Rubin's rules — this properly accounts for the uncertainty introduced by imputation." For experts: "MissForest uses random forests as the conditional model in MICE, which handles non-linear relationships."

Multiple Imputation: Rubin's Rules

Advanced
$$\bar{Q} = \frac{1}{M}\sum_{m=1}^{M} \hat{Q}_m$$
Final estimate = average across $M$ imputed datasets
$$T = \bar{U} + \left(1 + \frac{1}{M}\right)B$$
Total variance = within-imputation variance ($\bar{U}$) + between-imputation variance ($B$)

Multiple imputation creates $M$ complete datasets (typically $M = 5$-$20$). Analyse each separately, then combine using Rubin's rules. The between-imputation variance captures the additional uncertainty from the missing data.

Time: ~2 min. "The beauty of Rubin's rules is that they properly account for the fact that imputed values are uncertain. Single imputation pretends the imputed values are real. Multiple imputation says 'I am not sure what the missing value was — here are my best guesses, and here is how uncertain I am.'" For experts: "M=5 is sufficient for point estimates. M=20+ is needed for confidence interval coverage."

Causation vs Correlation

Foundations

Correlation

  • Ice cream sales and drownings both increase in summer
  • Does ice cream cause drowning? No.
  • Both are caused by a third variable: hot weather
  • Correlation cannot distinguish cause from confounding

Causation

  • Requires intervention: change X, observe Y
  • Hold everything else constant
  • The gold standard: Randomized Controlled Trial
  • Random assignment eliminates confounders
Business decisions require causation. "Should we spend $10M on this new feature?" requires knowing it CAUSES improvement, not just correlates with it.
Time: ~2 min. "This is perhaps the most important distinction in all of data science. Billions of dollars are wasted every year because teams confuse correlation with causation." Use the ice cream example — it is memorable.

The Randomized Controlled Trial (RCT)

Foundations
1Define hypothesis: "The new checkout flow increases conversion rate"
2Randomly assign: 50% of users see old flow (control), 50% see new flow (treatment)
3Measure outcome: Conversion rate in each group
4Compare: Is the difference statistically significant?
Why random assignment works: Because users are randomly assigned, the two groups are (on average) identical in every way EXCEPT the treatment. Any difference in outcomes must be caused by the treatment.
Time: ~2 min. "RCTs are the gold standard in medicine (drug trials), tech (A/B tests), and social science. The key insight is that randomisation makes the groups comparable on ALL variables — even ones you did not measure."

A/B Testing: The Tech Industry RCT

Foundations

How A/B Tests Work

  • User visits website/app
  • System randomly assigns to variant A (control) or B (treatment)
  • User experience differs ONLY in the tested element
  • System logs behaviour (clicks, purchases, time on page)
  • After enough data, compare groups statistically

What Makes a Good A/B Test

  • Single variable: change only ONE thing
  • Sufficient sample size: calculated in advance
  • Random assignment: no self-selection
  • Adequate duration: capture weekly cycles
  • Pre-registered metrics: define success before running
Time: ~2 min. "A/B testing is the most common form of experimentation in tech. Every major tech company runs thousands of A/B tests simultaneously. Google alone runs over 10,000 experiments per year."

Sample Size Calculation

Theory
$$n = \frac{(z_{\alpha/2} + z_\beta)^2 \cdot 2\sigma^2}{\delta^2}$$
$n$ = sample size per group, $z_{\alpha/2}$ = critical value for significance (1.96 for 5%), $z_\beta$ = critical value for power (0.84 for 80%), $\sigma^2$ = variance of metric, $\delta$ = minimum detectable effect (MDE)

Inputs You Need

  • Significance level ($\alpha$): usually 0.05 (5% false positive rate)
  • Power ($1-\beta$): usually 0.80 (80% chance of detecting real effect)
  • MDE ($\delta$): smallest effect worth detecting
  • Baseline variance ($\sigma^2$): from historical data

Key Insight

Halving the MDE quadruples the required sample size. If you want to detect smaller effects, you need exponentially more data.
Time: ~3 min. "Always calculate sample size BEFORE running the experiment. Running until you see significance is called p-hacking — and it inflates your false positive rate far above 5%." Walk through the formula with real numbers.

Minimum Detectable Effect (MDE)

Theory
$$\text{MDE} = (z_{\alpha/2} + z_\beta) \cdot \sqrt{\frac{2\sigma^2}{n}}$$
Given your sample size, what is the smallest effect you can reliably detect?

Business Translation

"With 10,000 users per group and 80% power, we can detect a 0.5 percentage point change in conversion rate. If the true effect is smaller, we will miss it."

Why MDE Matters

An underpowered test that shows "no significant difference" does NOT mean there is no effect. It means the effect (if any) is smaller than your MDE. This distinction is critical for decision-making.

Time: ~2 min. "'No significant effect' and 'no effect' are completely different statements. If your MDE is 5% but the true effect is 2%, a non-significant result is expected — not evidence of no effect."

Novelty, Primacy, and Network Effects

Advanced

Novelty Effect

Users engage more with something new simply because it IS new. Metrics inflate initially, then decay to true level. Mitigation: run experiments for 2+ weeks, analyse time-decayed effects.

Primacy Effect

Users prefer what they are used to. A change looks bad initially because users resist change, not because it IS bad. Mitigation: analyse new users separately (no prior exposure).

Network Effects (SUTVA Violation): If treated users interact with control users (social networks, marketplaces), treatment effects "leak" across groups. The Stable Unit Treatment Value Assumption (SUTVA) is violated. Solutions: cluster randomisation (randomise by city/network), switchback experiments (alternate treatment over time), ego-network randomisation.
Time: ~2 min. "Network effects are the biggest challenge in modern A/B testing. If you show a new feature to half your marketplace sellers but they compete with the control sellers, you cannot cleanly separate the effects." For experts: "Graph cluster randomisation (GCR) partitions the social graph into clusters and randomises at the cluster level."

H. Causal Inference

Foundations

"When you cannot run an experiment, causal inference lets you reason about cause and effect from observational data."

This section covers the two major frameworks (Rubin and Pearl) and the key methods used in industry.

Time: ~1 min. Section header. "A/B tests are the gold standard, but you cannot always randomise. You cannot randomly assign people to smoke or not smoke. You cannot randomly assign cities to get a new policy. Causal inference methods let you estimate causal effects from observational data."

Rubin's Potential Outcomes Framework

Theory
$$\tau_i = Y_i(1) - Y_i(0)$$
Individual treatment effect = outcome under treatment minus outcome under control
The Fundamental Problem: We can NEVER observe both $Y_i(1)$ and $Y_i(0)$ for the same individual

Key Estimands

  • ATE = $E[\tau_i]$ — Average Treatment Effect (everyone)
  • ATT = $E[\tau_i | D_i=1]$ — Effect on the Treated
  • ATU = $E[\tau_i | D_i=0]$ — Effect on the Untreated
  • CATE = $E[\tau_i | X_i = x]$ — Conditional (heterogeneous)
Plain English: For each person, there are two possible futures — one where they get the treatment, one where they do not. The treatment effect is the difference. But we can only see ONE future — the one that actually happened. Causal inference estimates the unseen future.
Time: ~3 min. "This is the foundational framework of modern causal inference. The fundamental problem is that we never see the counterfactual — what WOULD have happened if we had done something different." Beginners: "Imagine two parallel universes — in one you took the medicine, in the other you didn't. The treatment effect is the difference. But you only live in one universe."

Propensity Score Methods

Theory
$$e(X) = P(D = 1 | X) \quad \text{(Propensity Score)}$$
Probability of receiving treatment, given observed covariates $X$

Propensity Score Matching

  • Estimate $e(X)$ with logistic regression
  • Match treated units with control units that have similar $e(X)$
  • Compare outcomes within matched pairs
  • Intuition: "apples to apples" comparison

Inverse Propensity Weighting (IPW)

$$\hat{\tau}_{\text{IPW}} = \frac{1}{n}\sum_{i=1}^{n}\left[\frac{D_i Y_i}{e(X_i)} - \frac{(1-D_i)Y_i}{1-e(X_i)}\right]$$

Weight each observation by the inverse of its probability of receiving the treatment it actually received. Upweights rare treatment assignments.

Time: ~3 min. "Propensity score methods answer: what if treatment and control groups were comparable? By matching on the probability of treatment, we create a pseudo-randomised comparison." For experts: "IPW is unbiased if the propensity model is correct. Matching is biased but more robust to misspecification."

Doubly Robust Estimation

Advanced
$$\hat{\tau}_{\text{DR}} = \frac{1}{n}\sum_{i=1}^{n}\left[\hat{\mu}_1(X_i) - \hat{\mu}_0(X_i) + \frac{D_i(Y_i - \hat{\mu}_1(X_i))}{e(X_i)} - \frac{(1-D_i)(Y_i - \hat{\mu}_0(X_i))}{1-e(X_i)}\right]$$
$\hat{\mu}_d(X)$ = outcome model, $e(X)$ = propensity model
Doubly robust = consistent if EITHER model is correct. If the propensity model is wrong but the outcome model is right, the estimate is still valid. If the outcome model is wrong but the propensity model is right, still valid. Both wrong = biased. This is the recommended approach in practice.
Time: ~2 min. "Doubly robust estimation is the state of the art for observational causal inference. It gives you two chances to be right." For experts: "The DR estimator also achieves the semiparametric efficiency bound when both models are correct."

Pearl's Causal DAGs

Theory

Directed Acyclic Graphs (DAGs) represent causal relationships visually.

Z → T → Y

Z also → Y (confounder creates back-door path)

Backdoor Criterion

To identify the causal effect of T on Y, block all backdoor paths (paths from T to Y through common causes) by conditioning on the right variables.

d-Separation

Two variables are d-separated if all paths between them are blocked. Conditioning on a collider opens a path (do not condition on colliders!).

Time: ~3 min. "Pearl's framework gives a visual language for causal reasoning. Draw the DAG, identify backdoor paths, condition on the right variables." Beginners: "Think of the DAG as a map showing which things cause which other things. The arrows show the direction of causation." For experts: "The front-door criterion is useful when you cannot observe the confounder but can observe a mediator."

Difference-in-Differences (DiD)

Theory
$$\hat{\tau}_{\text{DiD}} = (\bar{Y}_{T,\text{post}} - \bar{Y}_{T,\text{pre}}) - (\bar{Y}_{C,\text{post}} - \bar{Y}_{C,\text{pre}})$$
Treatment effect = (change in treatment group) - (change in control group)

How It Works

  • Compare two groups (treatment vs control)
  • At two time points (before vs after intervention)
  • Subtract the time trend (captured by control)
  • What remains = causal effect of treatment

Key Assumption

Parallel Trends: Absent treatment, both groups would have followed the same trend. If trends diverge pre-treatment, DiD is invalid. Visualise pre-treatment trends to check.
Time: ~3 min. "DiD is the workhorse of policy evaluation. Did a new law reduce crime? Compare the treated state with untreated states, before and after the law." Draw the classic DiD diagram on the board — two parallel lines, one gets a treatment, measure the gap. "The parallel trends assumption is crucial. Always plot pre-treatment data to verify."

DiD: Regression Formulation

Theory
$$Y_{it} = \beta_0 + \beta_1 \text{Treat}_i + \beta_2 \text{Post}_t + \beta_3 (\text{Treat}_i \times \text{Post}_t) + \epsilon_{it}$$
$\beta_3$ = DiD estimator = causal effect of treatment
CoefficientMeaning
$\beta_0$Control group, pre-period baseline
$\beta_1$Baseline difference between treatment and control
$\beta_2$Time trend (common to both groups)
$\beta_3$Treatment effect — the causal parameter of interest
Placebo tests: Run DiD on pre-treatment periods (where there should be no effect). If $\beta_3 \neq 0$ in the pre-period, parallel trends is violated. Also test with fake treatment dates and alternative control groups.
Time: ~2 min. "The regression formulation is more flexible — you can add covariates, fixed effects, and cluster standard errors." For experts: "For staggered treatment adoption, use the Callaway-Sant'Anna estimator to avoid negative weighting bias."

Regression Discontinuity Design (RDD)

Theory
$$\tau_{\text{RDD}} = \lim_{x \downarrow c} E[Y|X=x] - \lim_{x \uparrow c} E[Y|X=x]$$
Treatment effect = jump in outcome at the threshold $c$

Sharp RDD

  • Treatment assigned if $X \geq c$ (deterministic)
  • Example: scholarship if GPA ≥ 3.5
  • Compare outcomes just above and just below cutoff
  • Units near cutoff are "as-if" randomly assigned

Fuzzy RDD

  • Cutoff changes probability of treatment
  • Example: recommended (not required) programme
  • Uses cutoff as instrumental variable
  • Estimates LATE (Local Average Treatment Effect)
Time: ~2 min. "RDD exploits an arbitrary threshold. Students with GPA 3.49 and 3.51 are essentially identical — but one gets a scholarship and the other doesn't. That threshold creates a natural experiment." For experts: "Bandwidth selection is crucial. Too narrow = high variance. Too wide = bias from non-comparable units."

Instrumental Variables

Theory
Z → T → Y Z → T → Y (but Z does NOT directly affect Y)
$$\hat{\tau}_{\text{2SLS}} = \frac{\text{Cov}(Y, Z)}{\text{Cov}(T, Z)} = \frac{\text{Reduced form}}{\text{First stage}}$$

Two Conditions

  • Relevance: Z affects T ($\text{Cov}(Z,T) \neq 0$)
  • Exclusion: Z affects Y ONLY through T

Classic Example

Effect of education on earnings. Instrument: proximity to college. Living near a college increases education (relevance) but does not directly affect earnings (exclusion — debatable).

Time: ~2 min. "Finding a valid instrument is the hardest part of IV. The exclusion restriction is untestable — it requires domain knowledge and argumentation." For experts: "Weak instruments (F-stat < 10) produce biased estimates. Use Anderson-Rubin confidence intervals for robust inference."

Synthetic Control Method

Theory

When you have one treated unit (a country, a city, a company), construct a "synthetic" control by optimally weighting multiple untreated units.

$$\hat{Y}_{1t}^{N} = \sum_{j=2}^{J+1} w_j^* Y_{jt}, \quad \text{s.t. } w_j \geq 0, \sum w_j = 1$$
Synthetic control = weighted average of donor units that best matches treated unit pre-treatment
  • Weights chosen to match pre-treatment outcomes
  • Treatment effect = actual minus synthetic post-treatment
  • Inference via permutation: apply to each donor unit
Abadie et al. (2010): Estimated the effect of California's tobacco control programme using a synthetic California constructed from other states. This method is now standard for policy evaluation with a single treated unit.
Time: ~2 min. "Synthetic control is brilliant when you cannot randomise and only have one treated unit. You build a counterfactual by combining untreated units." For experts: "Recent work combines DiD and synthetic control in a doubly robust framework (Arkhangelsky et al.)."

CUPED: Variance Reduction for Experiments

Theory
$$Y_{\text{adj}} = Y - \theta(X - \bar{X})$$
$Y$ = post-experiment metric, $X$ = pre-experiment metric (covariate), $\theta = \text{Cov}(Y,X)/\text{Var}(X)$
$$\text{Var}(Y_{\text{adj}}) = \text{Var}(Y)(1 - \rho^2)$$
$\rho$ = correlation between Y and X. If $\rho = 0.7$, variance reduced by $1 - 0.49 = 51\%$
Plain English: CUPED uses pre-experiment data (what users did BEFORE the experiment) to reduce noise in the experiment measurement. If a user was already a heavy spender before the test, CUPED adjusts for that, making the treatment effect easier to detect.
Time: ~3 min. "CUPED is one of the most impactful innovations in A/B testing. By using pre-experiment behaviour as a covariate, it can cut variance by 50% or more — which means you need half the sample size." Walk through the proof: $\text{Var}(Y - \theta X) = \text{Var}(Y) - 2\theta\text{Cov}(Y,X) + \theta^2\text{Var}(X)$. Setting derivative to zero: $\theta^* = \text{Cov}(Y,X)/\text{Var}(X)$.

CUPED: Derivation of Optimal Theta

Theory
$$\text{Var}(Y_{\text{adj}}) = \text{Var}(Y) - 2\theta\text{Cov}(Y,X) + \theta^2\text{Var}(X)$$

Take the derivative with respect to $\theta$ and set to zero:

$$\frac{d}{d\theta}\text{Var}(Y_{\text{adj}}) = -2\text{Cov}(Y,X) + 2\theta\text{Var}(X) = 0$$ $$\theta^* = \frac{\text{Cov}(Y,X)}{\text{Var}(X)}$$

Substituting back:

$$\text{Var}(Y_{\text{adj}}) = \text{Var}(Y)\left(1 - \frac{\text{Cov}(Y,X)^2}{\text{Var}(Y)\text{Var}(X)}\right) = \text{Var}(Y)(1 - \rho^2)$$
CUPAC extension: CUPAC (Controlled-experiment Using Predictions as Covariates) replaces $X$ with ML model predictions, achieving even greater variance reduction when the model is predictive.
Time: ~2 min. "This is one of the cleanest derivations in statistics. The optimal theta is exactly the OLS coefficient. The variance reduction is exactly one minus r-squared." For experts: "Recent work (2025) combines pre-experiment and in-experiment data for even greater variance reduction."

Double/Debiased Machine Learning

Advanced

Chernozhukov et al. (2018): Use ML to estimate nuisance parameters, then use residuals for causal estimation.

1Partial out X from Y: $\tilde{Y} = Y - \hat{m}(X)$ using ML model for $E[Y|X]$
2Partial out X from D: $\tilde{D} = D - \hat{e}(X)$ using ML model for $E[D|X]$
3Regress residuals: $\hat{\tau} = \frac{\sum \tilde{D}_i \tilde{Y}_i}{\sum \tilde{D}_i^2}$
Key innovation: Cross-fitting. Split data into K folds. For each fold, train ML models on the other K-1 folds and predict on the held-out fold. This prevents overfitting bias and ensures $\sqrt{n}$-consistency of $\hat{\tau}$ even though the ML nuisance estimates converge at slower rates.
Time: ~3 min. "Double ML allows you to use ANY ML model (random forests, neural nets, boosting) for the nuisance estimation while still getting valid confidence intervals for the causal parameter. This is the bridge between ML and causal inference." For experts: "The Neyman orthogonality condition ensures that the estimation error in the nuisance parameters does not contaminate the causal estimate."

Causal Forests for Heterogeneous Effects

Advanced

Wager & Athey (2018): Estimate CATE — how treatment effects vary across subgroups.

$$\hat{\tau}(x) = E[Y(1) - Y(0) | X = x]$$
Conditional Average Treatment Effect: the treatment effect for individuals with characteristics $x$

How It Works

  • Modified random forest: splits to maximise treatment effect heterogeneity
  • Honest estimation: separate sample for splitting and estimation
  • Pointwise confidence intervals via infinitesimal jackknife

Business Application

"Who benefits most from this treatment?" Instead of one average effect, get personalised treatment effects. Example: which customers respond to a discount? Target only those customers — save money and increase impact.

Time: ~2 min. "Causal forests are the state of the art for personalised treatment effect estimation. They answer not just 'does it work?' but 'for WHOM does it work?'" For experts: "The honesty condition ensures that the trees are not overfitting to the treatment assignment. Use the grf package in R or EconML in Python."

Bayesian A/B Testing

Advanced
$$P(\theta_B > \theta_A | \text{data}) = \int_0^1 \int_0^{\theta_B} p(\theta_A | \text{data}) \, d\theta_A \, p(\theta_B | \text{data}) \, d\theta_B$$
Posterior probability that treatment B is better than A

Bayesian Approach

  • Prior: Beta(1,1) for conversion rates
  • Posterior: Beta($\alpha + s$, $\beta + f$)
  • Decision: $P(B > A) > 0.95$
  • Natural stopping: compute probability at any time

Advantages over Frequentist

  • No peeking problem — posterior valid at any point
  • Direct probability statements ("95% chance B is better")
  • Expected loss: "how much do we lose by choosing wrong?"
  • Natural incorporation of prior knowledge
Time: ~2 min. "Bayesian A/B testing is gaining rapid adoption because it solves the peeking problem. You can check results at any time without inflating false positive rates." For experts: "The expected loss criterion — 'choose B if expected loss from choosing B is below threshold' — is often more actionable than the posterior probability."

Sequential Testing: Always-Valid P-Values

Advanced

Sequential testing provides valid inference at every point in time — you can check your experiment daily without inflating false positives.

The Peeking Problem

With fixed-horizon tests, checking daily inflates false positive rate from 5% to 30%+. Sequential tests construct confidence sequences that are valid at every sample size.

Methods

  • mSPRT: Mixture Sequential Probability Ratio Test (Johari et al.)
  • Alpha spending: O'Brien-Fleming, Pocock boundaries
  • Confidence sequences: Time-uniform confidence intervals
  • E-values: Measure of evidence (Grunwald et al.)
Industry adoption (2025-2026): Always-valid inference is now standard in major experimentation platforms. It is the recommended approach for any experiment where you want to monitor results before the planned end date.
Time: ~2 min. "Sequential testing is the modern solution to the peeking problem. It lets you have your cake and eat it too — continuous monitoring with valid inference." For experts: "mSPRT uses a mixture of likelihood ratios over the effect size parameter, producing an anytime-valid p-value."

When requesting features for a prediction at t=March:

  • April and May data is excluded — even though it exists now
  • This prevents temporal leakage automatically
Kailash FeatureStore timestamps every feature value. When you call store.get_features(entity_id, timestamp), only values available at that timestamp are returned.

Feature Versioning and Lineage

Theory

Versioning

  • v1: remaining_lease = 99 - (year - lease_commence)
  • v2: remaining_lease = lease_end_date - today
  • Model A uses v1, Model B uses v2
  • Both versions coexist — no breaking changes
  • Gradual migration: new models use v2, old models keep v1

Lineage

  • Track: raw data → transformation → feature
  • Upstream change? Know which features and models are affected
  • Feature breaks? Trace back to the raw data change
  • Audit: who created this feature, when, why?

Synthesis: If You Followed the Maths

Theory
THEORY — If you followed the maths:
  • Target encoding needs regularisation (smoothing, LOO, CatBoost) to prevent leakage.
  • Mutual information captures non-linear dependencies that correlation misses.
  • L1 (Lasso) drives coefficients to exactly zero — L2 (Ridge) does not. The geometry explains why.
  • CUPED reduces variance by $(1 - \rho^2)$ — the derivation is clean OLS.
  • DiD identifies causal effects under parallel trends; RDD under continuity at the cutoff.
  • Propensity score methods reweight observations to simulate randomisation.

Synthesis: For the Experts

Advanced
ADVANCED — For the experts:
  • Double ML uses cross-fitting to achieve $\sqrt{n}$-consistency despite slow ML nuisance estimation.
  • Causal forests estimate CATE with honest splitting — separate samples for tree structure and leaf estimation.
  • Always-valid p-values (mSPRT) solve the peeking problem with mixture sequential probability ratio tests.
  • Stability selection provides finite-sample false positive control over any base selector.
  • Doubly robust estimation is consistent if EITHER the propensity or outcome model is correct.
  • James-Stein shrinkage is the admissible target encoding estimator for 3+ categories (Stein's paradox).
