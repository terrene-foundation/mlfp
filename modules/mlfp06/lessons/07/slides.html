🟢 Foundations: the code IS the governance. If it is not in code, it does not exist.

PACT D/T/R addressing

from kailash_pact import GovernanceEngine, Address

engine = GovernanceEngine()
org = engine.compile_org({
    "domains": ["finance", "operations"],
    "teams": {"finance": ["risk", "compliance"]},
    "roles": {"risk": ["analyst", "manager"]}
})

addr = Address(domain="finance", team="risk", role="analyst")
allowed = engine.can_access(addr, resource="customer_data")
reason = engine.explain_access(addr, resource="customer_data")

Governance testing


def test_denied_access_stays_denied():
    addr = Address(domain="finance", team="risk", role="analyst")
    assert not engine.can_access(addr, resource="model_deployment")

def test_envelope_only_tightens():
    agent.tighten_envelope(max_cost=5.00)
    with pytest.raises(GovernanceViolation):
        agent.loosen_envelope(max_cost=10.00)
⚠ Key principle: test what does NOT happen. Governance tests verify denied access stays denied.

Common mistakes

  1. Fail-open — if governance check fails, DENY. Never default to allow.
  2. Testing only happy paths — test denied access, not just allowed access.
  3. No audit trail — compliance needs proof. Log every access decision.
