Lesson 3.8 Production Pipeline — DataFlow, Drift, and Deployment

A model in a notebook is a prototype. A model in production persists results to a database, monitors for drift, documents its limitations, and reports prediction uncertainty. This lesson completes the production story: DataFlow for database persistence, DriftMonitor for detecting distribution shift with PSI and KS tests, Model Cards for documentation, and conformal prediction for distribution-free prediction intervals.

DataFlow: Persisting ML Results

⚙ Kailash Engine

DataFlow provides zero-config database operations. Define a model with @db.model, and DataFlow generates CRUD operations automatically. For ML, this means persisting predictions, model metadata, and evaluation results to a database without writing SQL. 

from kailash_dataflow import DataFlow, field

db = DataFlow()

@db.model
class Prediction:
    id: int = field(primary_key=True)
    customer_id: str
    probability: float
    model_version: int
    created_at: str

# Create
await db.express.create(Prediction,
    customer_id="C001", probability=0.73,
    model_version=1, created_at="2024-01-15")

# Query
high_risk = await db.express.list(Prediction,
    filters={"probability__gt": 0.7})

DriftMonitor

A model trained on January data may fail on July data if the distribution has shifted. DriftMonitor detects this using two statistical tests:

PSI: Population Stability Index

$$\text{PSI} = \sum_{i=1}^{k} (p_i - q_i) \cdot \ln\!\left(\frac{p_i}{q_i}\right)$$
$p_i$ = proportion in bin $i$ (reference) $q_i$ = proportion in bin $i$ (current)
PSIInterpretationAction
< 0.1No significant shiftContinue serving
0.1 - 0.25Moderate shiftInvestigate
> 0.25Significant shiftRetrain

KS test: Kolmogorov-Smirnov

The KS test compares two CDFs and returns the maximum vertical distance between them. A small p-value indicates the distributions differ significantly. 

from kailash_ml import DriftMonitor, DriftSpec

monitor = DriftMonitor(
    spec=DriftSpec(
        features=["income", "age", "credit_score"],
        psi_threshold=0.15,
        ks_alpha=0.05,
        check_interval="daily",
    )
)

# Compare reference (training) to current (production)
drift_report = monitor.check(df_reference, df_current)

for alert in drift_report.alerts:
    print(f"  {alert.feature}: PSI={alert.psi:.3f}, "
          f"KS p={alert.ks_pvalue:.4f}")

A Model Card (Mitchell et al., 2019) documents a model's purpose, performance, limitations, and fairness findings. Every model promoted to production should have one.

Conformal Prediction

Traditional ML gives a point prediction: "this customer will default with probability 0.73". Conformal prediction adds a prediction interval with guaranteed coverage:

🔵 Theory

Conformal prediction is distribution-free: it makes no assumptions about the data distribution. Given a calibration set and a desired coverage level $\alpha$, it produces prediction sets that contain the true value with probability at least $1 - \alpha$. For regression, this means prediction intervals; for classification, this means sets of possible classes. 

# Conformal prediction interval for regression
residuals = np.abs(y_cal - model.predict(X_cal))
q = np.quantile(residuals, 1 - alpha)

# For a new point:
y_hat = model.predict(X_new)
interval = (y_hat - q, y_hat + q)
print(f"Prediction: {y_hat:.0f}, 90% interval: [{interval[0]:.0f}, {interval[1]:.0f}]")

Common Mistakes

⚠ Mistake 1: No drift monitoring

A model without drift monitoring degrades silently. By the time someone notices, the predictions have been wrong for weeks.

⚠ Mistake 2: PSI on the wrong features

Monitor the features the model actually uses, not all features in the dataset. Irrelevant features drifting do not affect predictions.

⚠ Mistake 3: Point predictions without uncertainty

"The predicted price is $450,000" is less useful than "The predicted price is $450,000 with a 90% interval of [$420,000, $480,000]". Conformal prediction provides this for free.
