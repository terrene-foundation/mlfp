A. Text Preprocessing

FOUNDATIONS

"Raw text is the messiest data type in machine learning."

Before any model can learn from text, we must transform unstructured character sequences into structured numerical representations.

Time: ~1 min. Section opener. "Every NLP pipeline starts here. Get this wrong, and nothing downstream works."

Text as Data: The Challenge

FOUNDATIONS

Structured Data (M1-M7)

  • Fixed number of columns
  • Each column has a type
  • Rows are independent
  • Numbers are numbers

Text Data (M8)

  • Variable length
  • Words are symbols, not numbers
  • Order matters enormously
  • "Not bad" ≠ "bad" + "not"
Core problem: ML models need fixed-size numerical inputs. Text is variable-length symbolic sequences. The entire field of NLP is about bridging this gap.
Time: ~2 min. "Think about what you did in M1: load a CSV, each row has the same columns. Text is nothing like that. A tweet has 20 words, a legal document has 20,000. How do you feed both to the same model?"

The NLP Pipeline

FOUNDATIONS
1Raw text → "The HDB flat in Ang Mo Kio costs $450,000!!!"
2Normalise → "the hdb flat in ang mo kio costs $450,000"
3Tokenise → ["the", "hdb", "flat", "in", "ang", "mo", "kio", "costs", "$450,000"]
4Remove stops → ["hdb", "flat", "ang", "mo", "kio", "costs", "$450,000"]
5Stem/lemmatise → ["hdb", "flat", "ang", "mo", "kio", "cost", "$450,000"]
6Vectorise → [0, 1, 0, 0, 1, 1, ...] (numbers!)
Time: ~2 min. "This is the full pipeline. We will spend the next 15 slides understanding each step. Notice the Singapore context — HDB, Ang Mo Kio. Real-world text is messy."

Tokenisation: Word-Level

FOUNDATIONS

The simplest approach: split on whitespace and punctuation.

text = "The cat sat on the mat."
tokens = text.lower().split()
# ["the", "cat", "sat", "on", "the", "mat."]
Problems with word tokenisation:
  • Vocabulary explodes (English: 170,000+ words)
  • Out-of-vocabulary (OOV) words: "unfriending", "GPT-4o"
  • Morphological variants: "run", "runs", "running", "ran"
  • Multi-word expressions: "New York", "ice cream"
Time: ~2 min. "Word tokenisation is where everyone starts. It is also where the first problems appear. What do you do with a word you have never seen before?"

Tokenisation: Character-Level

FOUNDATIONS

The other extreme: every character is a token.

The _ cat

Advantages

  • Tiny vocabulary (~100 chars)
  • No OOV problem ever
  • Handles any language

Disadvantages

  • Sequences become very long
  • Characters carry little meaning
  • Hard to learn word-level semantics
Time: ~2 min. "Character-level solves OOV but creates a new problem: your sequences are 5-10x longer, and each token is nearly meaningless on its own. We need a middle ground."

Subword Tokenisation: The Middle Ground

FOUNDATIONS

Modern NLP uses subword tokens: smaller than words, larger than characters.

unfriending

The word "unfriending" is split into meaningful subunits.

Why subwords?
  • Manageable vocabulary (30K-50K tokens)
  • No OOV: any word can be decomposed
  • Morphological structure preserved: "un" + "friend" + "ing"
  • All modern LLMs use subword tokenisation
Time: ~2 min. "This is the breakthrough insight: you do not need to choose between words and characters. Subwords give you the best of both. Every transformer you have heard of uses this."

BPE Algorithm: Step by Step

THEORY

Byte Pair Encoding (Sennrich et al., 2016) — the most common subword method.

1Start with character-level vocabulary: {a, b, c, d, ...}
2Count all adjacent character pairs in training corpus
3Merge the most frequent pair into a new token
4Repeat steps 2-3 until desired vocabulary size reached
Example corpus: "low lower lowest low"
Iteration 1: most frequent pair = (l, o) → merge to "lo"
Iteration 2: most frequent pair = (lo, w) → merge to "low"
Common substrings emerge as tokens.
Time: ~3 min. "BPE is beautifully simple: start with characters, keep merging the most common pairs. Frequent words become single tokens. Rare words stay decomposed. Walk through the example on the board."

Corpus: ["hug", "pug", "pun", "bun", "hugs"]

WordPiece and Unigram

THEORY

WordPiece (BERT)

  • Similar to BPE but uses likelihood
  • Merge pair that maximises training data likelihood
  • Score: $\frac{P(\text{ab})}{P(\text{a}) \cdot P(\text{b})}$
  • Uses "##" prefix for continuation
  • "playing" → ["play", "##ing"]

Unigram (T5, XLNet)

  • Starts with large vocabulary
  • Iteratively removes tokens
  • Keeps tokens that minimise loss
  • Probabilistic: multiple segmentations
  • Scores: $P(\mathbf{x}) = \prod P(x_i)$
SentencePiece: Language-agnostic framework that implements both BPE and Unigram. Treats input as raw bytes — no pre-tokenisation needed. Used by T5, LLaMA, and many multilingual models.
Time: ~3 min. "BPE merges bottom-up, Unigram prunes top-down. WordPiece uses likelihood instead of frequency. SentencePiece wraps both in a language-agnostic package. In practice, the differences are small — all produce good subword tokenisations."

Stop Words

FOUNDATIONS

Stop words are high-frequency words that carry little semantic content on their own.

the cat is on the mat

Common English stop words: the, is, at, which, on, a, an, and, or, but, in, ...

When NOT to remove stop words:
  • Sentiment analysis: "not good" ≠ "good" — "not" is a stop word!
  • Question answering: "what", "where", "who" are stop words
  • Machine translation: every word matters for fluency
  • Modern transformers: learn to ignore irrelevant words themselves
Time: ~2 min. "Stop word removal was standard practice for 20 years. Modern models with attention often do not need it — they learn which words to ignore. But for BoW and TF-IDF, it still matters."

Stemming

FOUNDATIONS

Stemming reduces words to their root form using rule-based suffix stripping.

Porter Stemmer (1980)

  • "studies" → "studi" (not a word!)

Time: ~2 min. "Stemming is fast and crude. It does not care about meaning — it applies rules to strip suffixes. Sometimes it produces nonsense. That is why we also have lemmatisation."

FOUNDATIONS

Stemming vs Lemmatisation: Stemming is fast, rule-based, and sometimes wrong. Lemmatisation is slower, uses a dictionary, and always produces real words. For bag-of-words models, stemming is often good enough. For anything semantic, use lemmatisation.

N-grams

FOUNDATIONS

N-grams capture local word order by grouping consecutive tokens.

Sentence: "the cat sat on the mat"

TypeN-grams
Unigrams (n=1)the, cat, sat, on, the, mat
Bigrams (n=2)the cat, cat sat, sat on, on the, the mat
Trigrams (n=3)the cat sat, cat sat on, sat on the, on the mat
Skip-grams: Allow gaps between words. "the __ sat" captures (the, sat) with skip=1. This is the key idea behind Word2Vec's skip-gram model (Section C).
Time: ~2 min. "N-grams add local context to bag-of-words models. 'New York' as a bigram is much more useful than 'New' and 'York' separately. But the vocabulary explodes: bigrams give V-squared possible tokens."

Text Normalisation

FOUNDATIONS

Cleaning raw text before tokenisation:

import re

text = "Check out https://t.co/abc!!! 😊 #NLP @user"

# Lowercase

# Remove URLs
text = re.sub(r'https?://\S+', '', text)

Time: ~2 min. "Normalisation is where you make your first modelling choices. Remove too much and you lose signal. Remove too little and you add noise. There is no universal right answer — it depends on your task."

Preprocessing: Common Mistakes

FOUNDATIONS
Mistake 1: Fit tokeniser on test data
Vocabulary must be built on training data only. If test words leak into vocabulary, you have data leakage.
Mistake 2: Over-aggressive cleaning
Removing all punctuation destroys "don't" → "dont" and "$100" → "100". Be deliberate.
Mistake 3: Ignoring encoding
UTF-8 vs Latin-1 mismatches corrupt text silently. Always specify encoding when loading text.
Time: ~2 min. "These three mistakes account for most text preprocessing bugs. The first one — tokeniser leakage — is especially insidious because your metrics look great until you deploy."

B. Bag of Words & TF-IDF

FOUNDATIONS

"Throw away word order and you still capture surprisingly much."

The simplest way to convert text to numbers: count the words.

Time: ~1 min. Section opener. "BoW is from the 1950s and it still works for many tasks. Understanding it deeply helps you appreciate why we needed something better."

Bag of Words: The Idea

FOUNDATIONS

Represent each document as a vector of word counts.

Doc 1: "the cat sat on the mat"
Doc 2: "the dog sat on the log"

thecatsatonmatdoglog
Doc 12111100
Doc 22011011
This is a term-document matrix. Each row is a document, each column is a word in the vocabulary. The values can be:
  • Binary: 1 if word appears, 0 otherwise
  • Count: number of occurrences (shown above)
  • Normalised: count / document length
Time: ~2 min. "This is surprisingly powerful. You lose word order entirely — 'dog bites man' and 'man bites dog' get the same vector. But for topic classification, it often works."

BoW: Limitations

FOUNDATIONS

What BoW Ignores

  • Word order: "dog bites man" = "man bites dog"
  • Semantics: "happy" and "joyful" are unrelated vectors
  • Context: "bank" (river) = "bank" (finance)

Practical Problems

  • High dimensionality: V can be 100K+
  • Extreme sparsity: most entries are 0
  • Common words dominate: "the" has highest counts

The last problem is exactly what TF-IDF solves.

Time: ~2 min. "BoW gives every word equal importance. But 'the' appearing 50 times tells you nothing. A rare domain-specific word appearing once tells you everything. TF-IDF captures this intuition."

TF-IDF: The Formula

THEORY

Term Frequency – Inverse Document Frequency

$$\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \text{IDF}(t)$$

Term Frequency

$$\text{TF}(t, d) = \frac{\text{count}(t, d)}{|d|}$$

How often does term $t$ appear in document $d$, normalised by document length?

Inverse Document Frequency

$$\text{IDF}(t) = \log \frac{N}{\text{df}(t)}$$

$N$ = total documents, $\text{df}(t)$ = documents containing $t$. Rare words get high IDF.

Time: ~3 min. "TF-IDF is elegant: words that appear often in one document but rarely across all documents get the highest scores. 'The' gets a near-zero IDF because it appears everywhere."

Corpus of 10,000 documents. Document $d$ has 200 words.

Sublinear TF

THEORY

Problem: raw TF grows linearly. A word appearing 20 times is not 20x more important than appearing once.

$$\text{TF}_{\text{sublinear}}(t, d) = 1 + \log(\text{count}(t, d)) \quad \text{if count} > 0$$

Sublinear TF compresses the scale: 100 occurrences are only ~5.6x more important than 1 occurrence.

Time: ~2 min. "This is a practical improvement. In long documents, some words repeat heavily due to topic, not importance. Logarithmic compression prevents these from dominating the vector."

BM25: The Search Engine Standard

THEORY

BM25 (Best Matching 25) improves TF-IDF with two key innovations:

$$\text{BM25}(q, d) = \sum_{t \in q} \text{IDF}(t) \cdot \frac{\text{TF}(t,d) \cdot (k_1 + 1)}{\text{TF}(t,d) + k_1 \cdot \left(1 - b + b \cdot \frac{|d|}{\text{avgdl}}\right)}$$

TF Saturation ($k_1$)

TF has diminishing returns. As $\text{TF} \to \infty$, the score approaches $(k_1 + 1)$, not infinity. Typical $k_1 = 1.2$.

Length Normalisation ($b$)

Long documents score higher just by having more words. $b$ penalises length relative to average. $b = 0.75$ is standard.

Time: ~3 min. "BM25 is what Google used for years. It is still the baseline for information retrieval. Two parameters, both well-understood. k1 controls TF saturation, b controls length normalisation. This formula powers most search engines."

C. Word Embeddings

THEORY

"You shall know a word by the company it keeps." — J.R. Firth, 1957

The idea that changed NLP: represent words as dense vectors where similar words have similar vectors.

Time: ~1 min. Section opener. "This quote is the entire theoretical foundation of word embeddings. Firth said it in 1957. It took until 2013 to make it computationally practical."

The Distributional Hypothesis

THEORY

Words that appear in similar contexts have similar meanings.

"I had a glass of ___ with dinner."

  • wine
  • water
  • juice
  • beer

"The ___ chased the mouse."

  • cat
  • dog
  • kitten
  • terrier
Key insight: We never tell the model that wine and water are beverages. It learns this because they appear in the same contexts. Meaning emerges from usage patterns.
Time: ~2 min. "If you replace every word in a sentence with a blank and ask what fits, you are defining that word's meaning through its context. That is the distributional hypothesis. It is the foundation of all embedding methods."

From Sparse to Dense

THEORY

One-Hot / BoW (Sparse)

dim = |V| (e.g., 100,000)

"cat"  = [0, 0, 1, 0, ..., 0]
"dog"  = [0, 0, 0, 1, ..., 0]
"kitten"= [0, 0, 0, 0, ..., 1]

cat · dog = 0 (orthogonal!)
No notion of similarity.

Embeddings (Dense)

dim = d (e.g., 300)

"cat"   = [0.2, -0.4, 0.7, ...]
"dog"   = [0.3, -0.3, 0.6, ...]
"kitten" = [0.2, -0.5, 0.8, ...]

cat · dog = 0.95 (similar!)
Similarity is meaningful.

Time: ~2 min. "In one-hot space, every word is equally different from every other word. Cat and dog are as different as cat and democracy. That is clearly wrong. Dense embeddings fix this by placing similar words near each other."

Word2Vec: Skip-gram

THEORY

Given a center word, predict the surrounding context words.

the cat sat on the

Center word: "sat" (window = 2) → Predict: "the", "cat", "on", "the"

$$P(o \mid c) = \frac{\exp(\mathbf{u}_o^\top \mathbf{v}_c)}{\sum_{w \in V} \exp(\mathbf{u}_w^\top \mathbf{v}_c)}$$
$\mathbf{v}_c$ = center word vector, $\mathbf{u}_o$ = context word vector, denominator = softmax over entire vocabulary
Time: ~3 min. "Skip-gram asks: given the word 'sat', what words are likely to appear nearby? It learns two vectors per word: one for when the word is the center, one for when it is the context. The softmax normalises over the entire vocabulary."

Skip-gram: The Objective

THEORY

Maximise the probability of observing the actual context words:

$$J(\theta) = \frac{1}{T} \sum_{t=1}^{T} \sum_{\substack{-m \leq j \leq m \\ j \neq 0}} \log P(w_{t+j} \mid w_t)$$
$T$ = corpus length, $m$ = window size, $w_t$ = word at position $t$
Problem: The softmax denominator sums over the entire vocabulary (e.g., 100K words). Computing this for every training example is prohibitively expensive.

Solution: Negative sampling — turn a V-class softmax into binary classification.

Time: ~2 min. "The objective is beautiful in theory: maximise the log probability of context words given center words. But that denominator requires summing over every word in the vocabulary for every single training step. Negative sampling is the breakthrough that made Word2Vec practical."

Negative Sampling

THEORY

Instead of a full softmax, sample $k$ negative (random) words and train binary classifiers.

$$J = \log \sigma(\mathbf{u}_o^\top \mathbf{v}_c) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n(w)} \left[ \log \sigma(-\mathbf{u}_{w_i}^\top \mathbf{v}_c) \right]$$
$\sigma$ = sigmoid, $P_n(w) \propto f(w)^{3/4}$ = noise distribution, $k$ = 5-20 negatives

Positive pair

("sat", "cat") → push vectors together

Negative pairs

("sat", "elephant") → push apart
("sat", "quantum") → push apart

Why $f(w)^{3/4}$? Raises the probability of sampling rare words. Without this, common words like "the" would be sampled too often as negatives.
Time: ~3 min. "Negative sampling is genius in its simplicity. Instead of asking 'which word is it?' (100K-way classification), ask 'is it this word or not?' (binary). Sample 5-20 random words as negatives. Training becomes 1000x faster."

Word2Vec: CBOW

THEORY

Continuous Bag of Words: the reverse of skip-gram. Predict the center word from the context.

Skip-gram

Input: center word
Output: context words

"sat" → {"the", "cat", "on", "the"}

Better for rare words. Slower to train.

CBOW

Input: context words
Output: center word

{"the", "cat", "on", "the"} → "sat"

Better for frequent words. Faster to train.

In practice: Skip-gram with negative sampling is the most widely used configuration because it handles rare words better.
Time: ~2 min. "CBOW averages the context vectors and predicts the center word. It is faster because each training example updates fewer weights. But skip-gram generally produces better embeddings, especially for rare words."

GloVe: Global Vectors

THEORY

GloVe (Pennington et al., 2014) learns embeddings from the co-occurrence matrix.

1Build co-occurrence matrix $X$ where $X_{ij}$ = how often word $i$ appears near word $j$
2Learn vectors such that $\mathbf{w}_i^\top \mathbf{w}_j + b_i + b_j \approx \log X_{ij}$
$$J = \sum_{i,j=1}^{V} f(X_{ij}) \left( \mathbf{w}_i^\top \mathbf{w}_j + b_i + b_j - \log X_{ij} \right)^2$$
$f(X_{ij})$ = weighting function that caps very frequent co-occurrences
Word2Vec vs GloVe: Word2Vec learns from local contexts (windows). GloVe learns from global co-occurrence statistics. In practice, they produce similar quality embeddings.
Time: ~3 min. "GloVe's insight is that co-occurrence ratios encode meaning. The ratio P(solid|ice)/P(solid|steam) is large, telling us ice is solid. GloVe captures this in the vector geometry."

FastText: Subword Embeddings

THEORY

FastText (Bojanowski et al., 2017) extends Word2Vec with character n-grams.

<wh whe her ere re> <where>

"where" = sum of character 3-gram vectors + whole word vector

Key Advantage

Handles out-of-vocabulary words! If "unhappily" is OOV, its character n-grams ("un", "hap", "app", "ppi", "pil", "ily") still produce a meaningful vector.

Works Well For

  • Morphologically rich languages
  • Misspellings ("teh" ≈ "the")
  • Domain jargon and neologisms
  • Small training corpora
Time: ~2 min. "FastText solves the OOV problem that plagues Word2Vec and GloVe. It is especially valuable for languages like Turkish or Finnish where a single root can have hundreds of inflected forms."

Bias in Word Embeddings

FOUNDATIONS

Word embeddings trained on real-world text inherit real-world biases.

Documented Biases

  • Gender: "doctor" closer to "man", "nurse" closer to "woman"
  • Racial: Names associated with different sentiments
  • Age: "elderly" associated with negative terms

Measurement

WEAT (Word Embedding Association Test):

  • Compare cosine similarity of target words (careers) to attribute words (male/female names)
  • Statistically significant differences indicate bias
Consequence: A hiring system using biased embeddings will rank "he is a doctor" higher than "she is a doctor" even if the embedding is never explicitly told about gender roles.
Time: ~2 min. "Embeddings mirror their training data. If the internet says 'doctor' near 'he' more often than 'she', the embedding learns that. This is not a theoretical concern — biased embeddings have been deployed in hiring, loan, and criminal justice systems."

ELMo: Contextual Embeddings

ADVANCED

ELMo (Peters et al., 2018): the first major contextual embedding model.

Static (Word2Vec/GloVe)

"bank" always has the same vector, whether it means:

  • "river bank" (geography)
  • "bank account" (finance)
  • "bank shot" (billiards)

Contextual (ELMo)

"bank" gets a different vector depending on the sentence:

  • Bidirectional LSTM reads full sentence
  • Combines representations from all layers
  • Each word's vector is context-dependent

D. Recurrent Neural Networks

THEORY

Vanilla RNN: The Equation

THEORY

At each time step $t$, the RNN reads input $\mathbf{x}_t$ and updates its hidden state:

$$\mathbf{h}_t = \tanh(\mathbf{W}_{hh} \mathbf{h}_{t-1} + \mathbf{W}_{xh} \mathbf{x}_t + \mathbf{b})$$
$\mathbf{h}_t$ = hidden state, $\mathbf{W}_{hh}$ = recurrent weights, $\mathbf{W}_{xh}$ = input weights

$\mathbf{h}_2$

$\mathbf{x}_3$

$\mathbf{h}_3$

The same weights $\mathbf{W}_{hh}$ and $\mathbf{W}_{xh}$ are shared across all time steps (weight sharing).

Time: ~3 min. "The hidden state is the memory. At each step, it combines new input with the previous memory. The tanh squashes values to [-1, 1]. Weight sharing means the network can handle sequences of any length."

Backpropagation Through Time (BPTT)

THEORY

To train an RNN, we unroll it across time steps and apply backpropagation.

1Forward pass: compute $\mathbf{h}_1, \mathbf{h}_2, \ldots, \mathbf{h}_T$ and loss $L$
2Backward pass: compute $\frac{\partial L}{\partial \mathbf{W}_{hh}}$ using chain rule through time
$$\frac{\partial L}{\partial \mathbf{W}_{hh}} = \sum_{t=1}^{T} \frac{\partial L}{\partial \mathbf{h}_t} \prod_{k=t+1}^{T} \frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}} \frac{\partial \mathbf{h}_t}{\partial \mathbf{W}_{hh}}$$
The product of derivatives $\prod \frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}}$ is the source of RNNs' biggest problem. When this product involves many multiplications of values < 1, the gradient vanishes.
Time: ~3 min. "BPTT is just backpropagation applied to the unrolled computation graph. The gradient flows backwards through every time step. That product of Jacobians is the crux of the vanishing gradient problem."

The Vanishing Gradient Problem

THEORY

For a vanilla RNN, each $\frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}}$ involves:

$$\frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}} = \text{diag}(\tanh'(\cdot)) \cdot \mathbf{W}_{hh}$$

Since $|\tanh'(x)| \leq 1$ and if $||\mathbf{W}_{hh}|| < 1$:

$$\prod_{k=t+1}^{T} \frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}} \to 0 \quad \text{as } (T-t) \to \infty$$
Consequence: The gradient from time step $T$ to time step $1$ goes to zero exponentially. The RNN cannot learn long-range dependencies. If "the cat, which sat on the mat, was fluffy", the RNN struggles to connect "cat" (subject) to "was" (verb) because they are too far apart.
Time: ~3 min. "This is the fundamental limitation of vanilla RNNs. The gradient signal dies as it flows backwards through time. In practice, vanilla RNNs can only learn dependencies spanning about 10-20 time steps. The LSTM was invented specifically to solve this."

LSTM: Long Short-Term Memory

THEORY

Hochreiter & Schmidhuber (1997) introduced the LSTM with a cell state $\mathbf{c}_t$ that provides a highway for gradient flow.

Key Idea

Three gates control information flow:

  • Forget gate: what to discard
  • Input gate: what to write
  • Output gate: what to reveal

Why It Works

The cell state $\mathbf{c}_t$ is updated additively, not multiplicatively. Gradients flow through addition without vanishing:

$\frac{\partial \mathbf{c}_t}{\partial \mathbf{c}_{t-1}} = \mathbf{f}_t$ (the forget gate)

Time: ~2 min. "The LSTM is one of the most important architectures in deep learning history. The key insight is the cell state: it acts like a conveyor belt that carries information unchanged through time, with gates controlling what gets added or removed."

LSTM: Forget Gate

THEORY

Forget Gate $\mathbf{f}_t$

$$\mathbf{f}_t = \sigma(\mathbf{W}_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f)$$

What it does: Decides what information to discard from the cell state. $\sigma$ outputs values in [0, 1].

  • $f_t = 0$ → completely forget this information
  • $f_t = 1$ → completely keep this information

Example: When the subject changes from "she" to "he", the forget gate discards the old gender information so the model can generate the correct pronoun.

Time: ~2 min. "The forget gate looks at the previous hidden state and current input, and outputs a value between 0 and 1 for each element of the cell state. Zero means forget, one means keep. This is element-wise multiplication."

LSTM: Input Gate & Candidate

THEORY

Input Gate $\mathbf{i}_t$ + Candidate $\tilde{\mathbf{c}}_t$

$$\mathbf{i}_t = \sigma(\mathbf{W}_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i)$$ $$\tilde{\mathbf{c}}_t = \tanh(\mathbf{W}_c [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c)$$

Two steps:

  1. $\mathbf{i}_t$ decides which values to update (0 = skip, 1 = update)
  2. $\tilde{\mathbf{c}}_t$ proposes new candidate values to add

Example: When the model reads "she is a doctor", the input gate writes "female" and "doctor" into the cell state.

Time: ~2 min. "The input gate is like a write filter. The candidate is what we want to write. The input gate decides how much of each candidate value actually gets written. This two-step process gives fine-grained control."

LSTM: Cell State Update

THEORY
$$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t$$
$\odot$ = element-wise (Hadamard) product
1Forget: $\mathbf{f}_t \odot \mathbf{c}_{t-1}$ — selectively erase old information
2Add: $\mathbf{i}_t \odot \tilde{\mathbf{c}}_t$ — selectively write new information
This is the magic. The cell state update is additive. Compare with the vanilla RNN where $\mathbf{h}_t = \tanh(\mathbf{W}\mathbf{h}_{t-1} + \ldots)$ is multiplicative. Addition preserves gradients — the gradient of $c_t$ w.r.t. $c_{t-1}$ is simply $f_t$, not a product of many small numbers.
Time: ~3 min. "This is the single most important equation in the LSTM. It is addition, not matrix multiplication. That is why gradients can flow across hundreds of time steps without vanishing. The forget gate controls how much old information to keep."

LSTM: Output Gate & Hidden State

THEORY

Output Gate $\mathbf{o}_t$ + Hidden State $\mathbf{h}_t$

$$\mathbf{o}_t = \sigma(\mathbf{W}_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o)$$ $$\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t)$$

What it does: The cell state $\mathbf{c}_t$ contains everything the LSTM remembers. The output gate decides what to reveal as the hidden state $\mathbf{h}_t$.

Example: The cell state remembers both "female" and "doctor". When predicting the next word after "She works as a", the output gate reveals the "doctor" information.

Time: ~2 min. "The cell state is the long-term memory. The hidden state is the working memory — what the LSTM chooses to expose at this particular time step. Not everything remembered needs to be relevant right now."

LSTM: Complete Equations

THEORY
$$\mathbf{f}_t = \sigma(\mathbf{W}_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f) \quad \text{(forget)}$$ $$\mathbf{i}_t = \sigma(\mathbf{W}_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i) \quad \text{(input)}$$ $$\tilde{\mathbf{c}}_t = \tanh(\mathbf{W}_c [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c) \quad \text{(candidate)}$$ $$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t \quad \text{(cell update)}$$ $$\mathbf{o}_t = \sigma(\mathbf{W}_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o) \quad \text{(output)}$$ $$\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t) \quad \text{(hidden state)}$$
Parameters: 4 weight matrices (W_f, W_i, W_c, W_o), each of size $[h+x, h]$, plus 4 bias vectors. Total: $\approx 4 \times (d_h + d_x) \times d_h$ parameters per layer.
Time: ~2 min. "All six equations on one slide for reference. Students should be able to trace the flow: input and previous hidden state go through four parallel transformations, which then interact to update the cell state and produce the output."

GRU: Gated Recurrent Unit

THEORY

The GRU (Cho et al., 2014) simplifies the LSTM with only two gates.

$$\mathbf{z}_t = \sigma(\mathbf{W}_z [\mathbf{h}_{t-1}, \mathbf{x}_t]) \quad \text{(update gate)}$$ $$\mathbf{r}_t = \sigma(\mathbf{W}_r [\mathbf{h}_{t-1}, \mathbf{x}_t]) \quad \text{(reset gate)}$$ $$\tilde{\mathbf{h}}_t = \tanh(\mathbf{W} [\mathbf{r}_t \odot \mathbf{h}_{t-1}, \mathbf{x}_t]) \quad \text{(candidate)}$$ $$\mathbf{h}_t = (1 - \mathbf{z}_t) \odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \tilde{\mathbf{h}}_t \quad \text{(interpolation)}$$

Bidirectional RNNs

THEORY

A forward-only RNN at position $t$ only knows words $1, \ldots, t$. But future context matters too!

$$\overrightarrow{\mathbf{h}}_t = \text{RNN}_{\text{fwd}}(\mathbf{x}_t, \overrightarrow{\mathbf{h}}_{t-1})$$ $$\overleftarrow{\mathbf{h}}_t = \text{RNN}_{\text{bwd}}(\mathbf{x}_t, \overleftarrow{\mathbf{h}}_{t+1})$$ $$\mathbf{h}_t = [\overrightarrow{\mathbf{h}}_t; \overleftarrow{\mathbf{h}}_t] \quad \text{(concatenation)}$$
Use case: "The bank by the river" vs "The bank approved the loan." A bidirectional RNN at "bank" sees both the preceding and following words, disambiguating the meaning. Essential for classification and NER, but cannot be used for generation (you cannot see the future while generating).
Time: ~2 min. "Two RNNs: one reads left-to-right, one reads right-to-left. Concatenate their hidden states. This doubles the representation size but captures both past and future context. BERT uses bidirectional attention for the same reason."

Teacher Forcing

THEORY

During seq2seq training, what input does the decoder get at each step?

Without Teacher Forcing

Decoder feeds its own previous prediction as input. If it predicts wrong early, all subsequent tokens are conditioned on the mistake.

Result: Error accumulation, slow convergence.

With Teacher Forcing

Decoder receives the ground truth previous token as input, regardless of its own prediction.

Result: Faster convergence, but train-test mismatch.

Exposure bias: During inference, the model must use its own predictions (no teacher). This mismatch between training and inference is called exposure bias. Scheduled sampling gradually shifts from teacher forcing to self-generated tokens during training.
Time: ~2 min. "Teacher forcing is a training trick: give the decoder the right answer at each step so it learns faster. But it creates a gap between training and inference. Scheduled sampling helps bridge this gap."

Gradient Clipping for RNNs

THEORY

While vanishing gradients kill learning, exploding gradients destabilise training:

$$\text{if } ||\mathbf{g}|| > \theta: \quad \mathbf{g} \leftarrow \frac{\theta}{||\mathbf{g}||} \mathbf{g}$$
$\mathbf{g}$ = gradient vector, $\theta$ = max norm threshold (typically 1.0-5.0)
Why gradients explode: If $||\mathbf{W}_{hh}|| > 1$, the product $\prod \frac{\partial \mathbf{h}_k}{\partial \mathbf{h}_{k-1}}$ grows exponentially instead of shrinking. Gradient clipping caps the maximum gradient norm, preventing parameter updates from being catastrophically large.

Vanishing gradients are structural (need LSTM/attention). Exploding gradients are a practical nuisance (clipping suffices).

Time: ~2 min. "Exploding gradients are the mirror problem of vanishing gradients. One drives gradients to zero, the other to infinity. Clipping is a simple fix: if the gradient is too large, rescale it. This is standard practice for all RNN training."

E. Attention Mechanisms

THEORY

"Instead of compressing everything into one vector, let the model look where it needs to."

Attention is the single most important idea in modern NLP.

Bahdanau Attention (2015)

THEORY

Instead of using just the final encoder state, the decoder can attend to all encoder states.

1Alignment score: How relevant is encoder state $j$ to decoder state $i$?
$$e_{ij} = \mathbf{v}^\top \tanh(\mathbf{W}_1 \mathbf{s}_{i-1} + \mathbf{W}_2 \mathbf{h}_j)$$
2Attention weights: Softmax over all encoder positions
$$\alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k=1}^{T} \exp(e_{ik})}$$
3Context vector: Weighted sum of encoder states
$$\mathbf{c}_i = \sum_{j=1}^{T} \alpha_{ij} \mathbf{h}_j$$
Time: ~3 min. "Bahdanau attention computes a separate context vector for each decoder step. The alignment score measures relevance. The softmax normalises. The weighted sum creates a focused representation. When translating, the model can focus on the relevant source words."

Self-Attention: Q, K, V

THEORY

What if a sequence attends to itself? This is self-attention — the foundation of transformers.

Each token produces three vectors from its embedding $\mathbf{x}$:

  • Query (Q): "What am I looking for?"
    $\mathbf{q} = \mathbf{x} \mathbf{W}^Q$
  • Key (K): "What do I contain?"
    $\mathbf{k} = \mathbf{x} \mathbf{W}^K$
  • Value (V): "What do I provide?"
    $\mathbf{v} = \mathbf{x} \mathbf{W}^V$
Analogy: Think of a library. You have a query (your question). Each book has a key (its title/index). The attention score is how well your query matches each key. The value is the book's content. You read a weighted combination of the most relevant books.
Time: ~3 min. "Q, K, V is the most important abstraction in modern deep learning. Each token asks a question (Q), advertises what it knows (K), and offers its information (V). The dot product of Q and K determines relevance."

Scaled Dot-Product Attention

THEORY
$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\mathbf{V}$$
1Compute dot products: $\mathbf{Q}\mathbf{K}^\top$ gives an $n \times n$ score matrix
2Scale by $\sqrt{d_k}$ to stabilise gradients
3Softmax: normalise each row to a probability distribution
4Multiply by $\mathbf{V}$: weighted sum of value vectors
Time: ~3 min. "This is the equation. QK-transpose gives the attention scores. Scale. Softmax. Multiply by V. The output is a weighted combination of value vectors, where the weights come from Q-K similarity. Beautiful in its simplicity."

Why Scale by $\sqrt{d_k}$?

THEORY

If $\mathbf{q}, \mathbf{k} \sim \mathcal{N}(0, 1)$ are independent with dimension $d_k$:

$$\mathbf{q} \cdot \mathbf{k} = \sum_{i=1}^{d_k} q_i k_i \sim \mathcal{N}(0, d_k)$$
The dot product has variance $d_k$, so standard deviation $\sqrt{d_k}$

Without scaling ($d_k = 512$)

Dot products range ~$[-45, 45]$
Softmax saturates → gradients ≈ 0
Model cannot learn

With scaling ($\div\sqrt{512} \approx 22.6$)

Scaled scores range ~$[-2, 2]$
Softmax stays in informative region
Healthy gradients

Time: ~2 min. "Without scaling, the dot products grow with dimension. Large values push softmax into saturation where all probability goes to one element and gradients vanish. Dividing by root-dk normalises the variance back to 1."

Cross-Attention vs Self-Attention

THEORY

Self-Attention

Q, K, V all come from the same sequence.

Used in: encoder, decoder (masked)

"How does each word in this sentence relate to every other word?"

Cross-Attention

Q from decoder, K and V from encoder output.

Used in: decoder (encoder-decoder attention)

"Which parts of the input are relevant to generating this output word?"

Cross-attention is Bahdanau attention generalised. In Bahdanau, the decoder attends to encoder states. In the transformer, cross-attention does the same thing, but using Q/K/V and multi-head projections.
Time: ~2 min. "Self-attention is a sequence looking at itself. Cross-attention is one sequence looking at another. The decoder uses both: masked self-attention for its own tokens, cross-attention to look at the encoder output."

Attention as Soft Dictionary Lookup

FOUNDATIONS

Hard Lookup (Dictionary)

Given a query key, find the exact match and return its value.

d = {"cat": [0.8, 0.1], "dog": [0.7, 0.2]}
result = d["cat"]  # exact match

Returns one value. Fails if key missing.

Soft Lookup (Attention)

Given a query, compute similarity to all keys, return weighted average of values.

# query similar to "cat" (0.9) and "dog" (0.1)
result = 0.9 * v_cat + 0.1 * v_dog

Returns blended value. Always works.

This is why it is called Q, K, V: The terminology comes directly from database/dictionary lookups. Attention is a differentiable, soft version of key-value retrieval.
Time: ~2 min. "If you struggle with Q, K, V, think of a dictionary. Your query matches against all keys with varying degrees of similarity. The output is a weighted blend of all values. That is all attention does."

Encoder Block: Detailed

THEORY
1Multi-Head Self-Attention: Each token attends to all other tokens
$$\mathbf{z} = \text{MultiHead}(\mathbf{x}, \mathbf{x}, \mathbf{x})$$
2Add & LayerNorm: Residual connection + normalisation
$$\mathbf{x}' = \text{LayerNorm}(\mathbf{x} + \mathbf{z})$$
3Feed-Forward Network: Two linear layers with ReLU
$$\text{FFN}(\mathbf{x}') = \max(0, \mathbf{x}'\mathbf{W}_1 + \mathbf{b}_1)\mathbf{W}_2 + \mathbf{b}_2$$
4Add & LayerNorm: Another residual connection
$$\text{output} = \text{LayerNorm}(\mathbf{x}' + \text{FFN}(\mathbf{x}'))$$
Time: ~3 min. "Four steps per encoder block. Self-attention mixes information across positions. The FFN processes each position independently. Residual connections allow gradient flow. LayerNorm stabilises training. Simple, but the combination is incredibly powerful."

Residual Connections

THEORY
$$\text{output} = \mathbf{x} + f(\mathbf{x})$$
$f$ = any sub-layer (attention or FFN)

Why residuals?

  • Gradient flows directly through addition
  • Deep networks (12-96 layers) can train
  • Each layer only needs to learn the residual change

Same idea as

  • ResNet (M7) for images
  • LSTM cell state for sequences
  • Highway networks
Pattern: Every successful deep architecture uses some form of skip/residual connection. Without them, gradients vanish after a few layers.
Time: ~2 min. "You saw this in M7 with ResNets. Same principle: let gradients bypass transformations through a shortcut. Without residuals, a 12-layer transformer would be untrainable."

Layer Normalisation

THEORY
$$\text{LayerNorm}(\mathbf{x}) = \gamma \odot \frac{\mathbf{x} - \mu}{\sigma + \epsilon} + \beta$$
$\mu, \sigma$ computed across the feature dimension for each token independently

Batch Norm (M7)

Normalise across the batch dimension. Depends on batch size. Problematic for variable-length sequences.

Layer Norm (Transformers)

Normalise across the feature dimension for each sample. Independent of batch size. Works naturally with sequences.

Time: ~2 min. "Layer norm normalises each token's feature vector independently. It does not depend on the batch, so it works the same during training and inference. This is why transformers use layer norm instead of batch norm."

Mask matrix $\mathbf{M}$:

Positional Encoding: Why We Need It

THEORY
Problem: Self-attention is permutation-invariant. Without position information, "dog bites man" and "man bites dog" produce identical attention patterns. We must inject position information.

Sinusoidal positional encoding (Vaswani et al., 2017):

$$\text{PE}(\text{pos}, 2i) = \sin\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right)$$ $$\text{PE}(\text{pos}, 2i+1) = \cos\left(\frac{\text{pos}}{10000^{2i/d_{\text{model}}}}\right)$$

Each dimension oscillates at a different frequency. Even dimensions use sin, odd dimensions use cos.

Time: ~3 min. "Without positional encoding, the transformer sees a bag of tokens — just like BoW but fancier. The sinusoidal encoding creates unique position signatures and has a beautiful property: relative positions can be represented as linear transformations."

Positional Encoding: Why Sin/Cos?

THEORY

Sinusoidal encoding allows the model to attend to relative positions:

Key property: For any fixed offset $k$, there exists a linear transformation $M$ such that:
$\text{PE}(\text{pos} + k) = M \cdot \text{PE}(\text{pos})$

This means relative position information is accessible via linear operations, which attention can learn.

Low dimensions

High frequency oscillation → fine-grained position (nearby words)

High dimensions

Low frequency oscillation → coarse position (distant words)

In practice, most modern models use learned positional embeddings instead, which perform similarly.

Time: ~2 min. "Think of binary counting: the least significant bit oscillates fastest, the most significant bit slowest. Sinusoidal encoding is analogous but continuous. Each dimension carries position information at a different scale."

RoPE: Rotary Position Embeddings

ADVANCED

RoPE (Su et al., 2021) encodes position by rotating Q and K vectors.

$$\mathbf{q}_m = R_{\theta, m} \mathbf{W}^Q \mathbf{x}_m, \quad \mathbf{k}_n = R_{\theta, n} \mathbf{W}^K \mathbf{x}_n$$
$R_{\theta, m}$ is a block-diagonal rotation matrix with angles $m\theta_i$
Why RoPE wins:
  • The dot product $\mathbf{q}_m^\top \mathbf{k}_n$ depends only on relative position $(m-n)$, not absolute
  • No separate position embedding needed — position is baked into Q and K
  • Extrapolates to longer sequences better than learned embeddings
  • Used by: LLaMA, Mistral, Gemma, and most modern LLMs
Time: ~2 min. "RoPE is elegant: instead of adding position vectors, rotate the query and key. The dot product between rotated vectors naturally depends on their distance. This is why modern LLMs can generalise to longer contexts than they were trained on."

ALiBi: Attention with Linear Biases

ADVANCED

ALiBi (Press et al., 2022) adds a linear position-dependent bias to attention scores.

$$\text{softmax}\left(\mathbf{q}_i^\top \mathbf{k}_j - m \cdot |i - j|\right)$$
$m$ is a head-specific slope (geometric sequence across heads)
How it works: Instead of adding position embeddings to the input, ALiBi subtracts a penalty proportional to distance directly from attention scores. Nearby tokens get higher scores, distant tokens get lower scores.
  • No position embeddings needed at all
  • Trains on short sequences, generalises to long ones
  • Different heads use different slopes → multi-scale attention
  • Used by: BLOOM, MPT
Time: ~2 min. "ALiBi is the simplest position method: just penalise distant tokens. Different heads use different penalty rates, so some heads look nearby and others look far away. It trains fast and generalises well."

Feed-Forward Network (FFN)

THEORY

Each transformer block contains a position-wise feed-forward network:

$$\text{FFN}(\mathbf{x}) = \text{GELU}(\mathbf{x}\mathbf{W}_1 + \mathbf{b}_1)\mathbf{W}_2 + \mathbf{b}_2$$
$\mathbf{W}_1 \in \mathbb{R}^{d \times 4d}$, $\mathbf{W}_2 \in \mathbb{R}^{4d \times d}$ (typical expansion factor = 4)
What the FFN does: Self-attention mixes information across positions. The FFN processes each position independently, applying the same transformation to every token. Recent research suggests FFNs act as key-value memories that store factual knowledge.

Modern models use SwiGLU instead of GELU: $\text{SwiGLU}(\mathbf{x}) = (\mathbf{x}\mathbf{W}_1 \odot \text{swish}(\mathbf{x}\mathbf{V}))\mathbf{W}_2$

Time: ~2 min. "The FFN is surprisingly important. It has 2/3 of the parameters in each block. Research has shown that factual knowledge (like 'Paris is the capital of France') is stored in FFN weights, while attention handles relational reasoning."

Training: Warmup + Cosine Decay

THEORY

Transformers use a specific learning rate schedule:

1Linear warmup: LR increases linearly from 0 to peak over first ~2K-10K steps
2Cosine decay: LR decreases following a cosine curve to near zero
$$\text{lr}(t) = \begin{cases} \text{lr}_{\max} \cdot \frac{t}{t_{\text{warmup}}} & t < t_{\text{warmup}} \\ \text{lr}_{\min} + \frac{1}{2}(\text{lr}_{\max} - \text{lr}_{\min})(1 + \cos(\pi \cdot \frac{t - t_{\text{warmup}}}{t_{\max} - t_{\text{warmup}}})) & t \geq t_{\text{warmup}} \end{cases}$$
Why warmup? Early in training, gradients are noisy. A high learning rate causes divergence. Warming up stabilises the Adam optimiser's variance estimates before ramping up the learning rate.
Time: ~2 min. "Every transformer uses this schedule. Without warmup, training often diverges in the first few hundred steps. The cosine decay smoothly reduces the learning rate, allowing fine-grained convergence in the later stages."

Flash Attention

ADVANCED

Standard attention materialises the $n \times n$ attention matrix — Flash Attention never does.

Standard Attention

  • Compute $\mathbf{S} = \mathbf{Q}\mathbf{K}^\top$ ($n \times n$ matrix)
  • Store in GPU HBM
  • Apply softmax
  • Multiply by $\mathbf{V}$
  • Memory: $O(n^2)$

Flash Attention

  • Tile Q, K, V into blocks
  • Compute attention per block in SRAM
  • Accumulate output with online softmax
  • Never materialise full $n \times n$ matrix
  • Memory: $O(n)$
IO-aware: Flash Attention is not mathematically different — it computes the exact same result. The innovation is how it schedules computation to minimise reads/writes between GPU SRAM (fast, small) and HBM (slow, large). 2-4x speedup, enabling longer contexts.
Time: ~3 min. "Flash Attention is a systems innovation, not an algorithmic one. It produces identical results but uses tiling and online softmax to avoid materialising the huge attention matrix. This is why modern models can handle 100K+ token contexts."

ADVANCED

In standard multi-head attention, each head has its own Q, K, V projections. GQA shares K and V across groups of Q heads.

KV Cache: Why Transformers Are Fast at Inference

THEORY

During autoregressive generation, each new token reuses all previous K and V computations.

1Generate token 1: compute Q1, K1, V1, attention, output
2Generate token 2: compute Q2, reuse K1, V1, add K2, V2
3Generate token $n$: compute Q$n$, reuse K$_{1:n-1}$, V$_{1:n-1}$, add K$n$, V$n$
Without KV cache: Each new token requires recomputing attention over all previous tokens — $O(n^2)$ total for $n$ tokens.
With KV cache: Each new token computes attention with one new Q against cached K,V — $O(n)$ per token, $O(n^2)$ total but with much better constants.

KV cache memory grows linearly with sequence length. For GPT-3 at 2K tokens: ~3 GB of KV cache.

Time: ~2 min. "KV caching is why LLMs can generate tokens quickly. Without it, generating each token would require reprocessing the entire context from scratch. The trade-off is memory: the cache grows with each token."

  • Flash Attention: exact same result, O(n) memory via tiling
  • GQA: shares K,V to reduce inference memory
  • The transformer wins because it scales

G. Pre-training Paradigms

THEORY

"Train once on everything, then fine-tune for anything."

Pre-training on massive unlabelled text creates foundation models that can be adapted to any downstream task.

BERT: Masked Language Model

THEORY

BERT (Devlin et al., 2019) masks 15% of input tokens and predicts them.

The [MASK] sat on the [MASK] → cat mat
The 15% masking strategy:
  • 80% of selected tokens: replace with [MASK]
  • 10%: replace with a random word
  • 10%: keep original word
Why not 100% [MASK]? The model never sees [MASK] during fine-tuning, creating a train-test mismatch. Random replacement and keeping the original reduce this mismatch.
Time: ~3 min. "BERT's MLM is like a fill-in-the-blank test. The model reads the entire sentence bidirectionally and predicts the masked words. The 80/10/10 split is a clever trick to avoid train-test mismatch."

GPT: Causal Language Model

THEORY

GPT (Radford et al., 2018) predicts the next token given all previous tokens.

$$P(x_1, x_2, \ldots, x_n) = \prod_{t=1}^{n} P(x_t \mid x_1, \ldots, x_{t-1})$$
Autoregressive factorisation: model the joint probability as a product of conditionals
The cat sat on the → mat
BERT vs GPT: BERT sees the full sentence (bidirectional, fill-in-the-blank). GPT only sees what came before (causal, predict-next-token). GPT can generate text naturally; BERT cannot.
Time: ~3 min. "GPT's training objective is deceptively simple: predict the next word. But this simple objective, applied at massive scale, produces models that can write, reason, code, and more. The autoregressive factorisation is the foundation of all modern LLMs."

Scaling Laws: Chinchilla

THEORY

Hoffmann et al. (2022) established that compute-optimal training requires balancing model size and data:

$$C \approx 6ND$$
$C$ = compute (FLOPs), $N$ = parameters, $D$ = tokens trained on
The Chinchilla finding: For compute-optimal training, $N$ and $D$ should scale equally. A 70B parameter model should see ~1.4T tokens. GPT-3 (175B, 300B tokens) was significantly undertrained by this rule.

Mixture of Experts (MoE)

ADVANCED

MoE uses sparse activation: only a subset of parameters are active for each token.

Dense Model

Every token activates every parameter. 70B model uses 70B params per token.

MoE Model

Each token is routed to top-k experts. 8x70B = 560B total, but only 2x70B = 140B active per token.

$$y = \sum_{i=1}^{E} g_i(\mathbf{x}) \cdot \text{Expert}_i(\mathbf{x})$$
$g(\mathbf{x})$ = router (softmax over expert scores), sparse: only top-k experts have $g_i > 0$
Used by: GPT-4 (rumoured 8 experts), Mixtral (8 experts, top-2 routing), Switch Transformer (top-1 routing). MoE gives the capacity of a huge model with the inference cost of a smaller one.
Time: ~2 min. "MoE is how you get the best of both worlds: the knowledge of a 1 trillion parameter model with the inference cost of a 140 billion parameter model. The router decides which experts are relevant for each token."

Decoding: Greedy and Beam Search

THEORY

Greedy Decoding

At each step, pick the most likely token:

$x_t = \arg\max P(x \mid x_{1:t-1})$

  • Fast: one forward pass per token
  • Suboptimal: can miss better sequences
  • "The cat sat on the mat." (boring but correct)

Beam Search

Keep top-$k$ hypotheses at each step:

  • Beam width $k = 4-8$ typical
  • Better: explores multiple paths
  • Slower: $k \times$ computation
  • Good for: translation, captioning
  • Bad for: creative generation (repetitive)
Time: ~2 min. "Greedy is fast but gets stuck in local optima. Beam search explores k paths in parallel and picks the best one. But for creative tasks, beam search produces generic, repetitive text because it always prefers high-probability sequences."

Decoding: Sampling Strategies

THEORY

Temperature

$$P(x_i) = \frac{\exp(z_i / T)}{\sum_j \exp(z_j / T)}$$
$T$EffectUse Case
$T \to 0$Argmax (greedy)Factual answers
$T = 1$Original distributionBalanced
$T > 1$Flatter, more randomCreative writing
Time: ~2 min. "Temperature controls randomness. Low temperature makes the model more deterministic and confident. High temperature makes it more creative and surprising. T=0.7 is a common default for conversational AI."

Top-k and Top-p (Nucleus) Sampling

THEORY

Top-k Sampling

Only consider the $k$ most likely tokens, then sample from those.

  • $k = 50$ typical
  • Problem: sometimes 50 tokens are all reasonable, sometimes only 3 are
  • Fixed $k$ does not adapt

Top-p (Nucleus) Sampling

Include tokens until their cumulative probability exceeds $p$.

  • $p = 0.9$ typical
  • Adaptive: fewer tokens when confident, more when uncertain
  • Generally preferred over top-k
In practice: Most systems combine temperature + top-p. For example, GPT-4 defaults: $T = 1.0$, top-p = 1.0 (all tokens). For controlled generation: $T = 0.7$, top-p = 0.9.
Time: ~2 min. "Top-p is more adaptive than top-k. When the model is confident, top-p considers very few tokens. When it is uncertain, it considers many. This produces more natural text than fixed top-k."

BLEU Score: Detailed

THEORY
$$\text{BLEU} = \text{BP} \cdot \exp\left(\sum_{n=1}^{N} w_n \log p_n\right)$$
$p_n$ = modified n-gram precision, $w_n = 1/N$ (uniform weights), BP = brevity penalty

Modified Precision

Count n-gram matches between candidate and reference, capped by reference count. Prevents gaming with repeated words.

Brevity Penalty

$\text{BP} = \min(1, e^{1-r/c})$
Penalises translations that are too short. Without BP, a one-word translation could get high precision.

BLEU limitations: Does not capture meaning ("the cat is on the mat" vs "the mat is under the cat" get different BLEU scores despite identical meaning). BERTScore handles this better.
Time: ~2 min. "BLEU is the standard metric for machine translation. It is simple, reproducible, and correlates reasonably with human judgments. But it only measures surface-level overlap. Two sentences with identical meaning can get very different BLEU scores."

Summarisation and Generation

FOUNDATIONS

Extractive Summarisation

Select the most important sentences from the original document.

  • Faithful: never generates text
  • Can be choppy and repetitive

Generate a new summary in the model's own words.

  • Risk of hallucination
