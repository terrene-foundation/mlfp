Chapter 5: Middleware

Overview

Middleware in Nexus follows the onion model: each middleware wraps the handler, processing requests on the way in and responses on the way out. Nexus provides built-in middleware for CORS, security headers, CSRF protection, and response caching, plus introspection via MiddlewareInfo.

Concepts

The Onion Model

Middleware executes in LIFO order (last added = outermost):

Request  -> Middleware C -> Middleware B -> Middleware A -> Handler
Response <- Middleware C <- Middleware B <- Middleware A <- Handler

Built-In Middleware

MiddlewarePurposeConfiguration
SecurityHeadersMiddlewareSecure-by-default response headersSecurityHeadersConfig
CSRFMiddlewareStateless Origin/Referer validationallowed_origins, exempt_paths
ResponseCacheMiddlewareTTL + LRU + ETag cachingCacheConfig

Code Walkthrough

Step 1: CORS via Constructor

from nexus import Nexus

app = Nexus(
    api_port=8000,
    cors_origins=["https://app.example.com", "https://admin.example.com"],
    cors_allow_methods=["GET", "POST", "PUT", "DELETE"],
    cors_allow_headers=["Authorization", "Content-Type"],
    cors_allow_credentials=False,
    cors_max_age=3600,
    enable_durability=False,
)

Step 2: Security Headers

from nexus.middleware.security_headers import (
    SecurityHeadersConfig,
    SecurityHeadersMiddleware,
)

app.add_middleware(
    SecurityHeadersMiddleware,
    config=SecurityHeadersConfig(
        frame_options="SAMEORIGIN",
        hsts_max_age=31536000,
        hsts_include_subdomains=True,
    ),
)

Defaults: frame_options="DENY", content_type_options="nosniff", hsts_max_age=31536000 (1 year).

Step 3: CSRF Protection

from nexus.middleware.csrf import CSRFMiddleware

app2 = Nexus(api_port=8001, enable_durability=False)
app2.add_middleware(
    CSRFMiddleware,
    allowed_origins=["https://app.example.com"],
    exempt_paths=["/webhooks/stripe", "/healthz"],
)

Step 4: Response Caching

from nexus.middleware.cache import CacheConfig, ResponseCacheMiddleware

cache_config = CacheConfig(
    default_ttl=300,
    max_entries=5000,
    no_cache_handlers={"create_user", "delete_user"},
)
app2.add_middleware(ResponseCacheMiddleware, config=cache_config)

Step 5: Method Chaining

app3 = Nexus(api_port=8002, enable_durability=False)

(
    app3.add_middleware(SecurityHeadersMiddleware, config=SecurityHeadersConfig())
    .add_middleware(CSRFMiddleware, allowed_origins=["https://example.com"])
    .add_middleware(ResponseCacheMiddleware, config=CacheConfig())
)
Ordering matters: Because of LIFO, add SecurityHeaders BEFORE CSRF so security headers appear even on CSRF-rejected requests. Response caching should be outermost to cache final responses.

Step 6: Introspection

from nexus import MiddlewareInfo

for mw_info in app2.middleware:
    assert isinstance(mw_info, MiddlewareInfo)
    print(f"{mw_info.name} added at {mw_info.added_at}")

names = [m.name for m in app2.middleware]
assert "CSRFMiddleware" in names
assert "ResponseCacheMiddleware" in names

Common Mistakes

MistakeProblemFix
Adding caching before security headersSecurity headers missing from cached responsesAdd security headers first (innermost)
Exempting too many CSRF pathsAttack surface widenedOnly exempt webhook paths
Forgetting LIFO orderingUnexpected execution orderLast added = outermost

Key Takeaways

  • CORS is best configured via the Nexus constructor.
  • add_middleware() follows the LIFO onion model.
  • Three built-in middleware: SecurityHeaders, CSRF, ResponseCache.
  • app.middleware provides runtime introspection via MiddlewareInfo.
  • add_middleware() returns self for fluent chaining.
