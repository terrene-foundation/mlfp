From scripts to workflows

Workflow as a node graph

WorkflowBuilder code


from kailash import WorkflowBuilder, PythonCodeNode
from kailash import ConditionalNode, Runtime

workflow = WorkflowBuilder("credit_pipeline")
load = PythonCodeNode("load", code=load_fn)
prep = PythonCodeNode("prep", code=prep_fn)
train = PythonCodeNode("train", code=train_fn)
evl = PythonCodeNode("eval", code=eval_fn)
gate = ConditionalNode("gate",
    condition=lambda ctx: ctx["auc"] > 0.85)
reg = PythonCodeNode("register", code=reg_fn)

workflow.connect(load, prep)
workflow.connect(prep, train)
workflow.connect(train, evl)
workflow.connect(evl, gate)
workflow.connect(gate, reg, branch="yes")

result = Runtime().execute(workflow.build())

Bayesian hyperparameter search

Bayesian optimisation: surrogate model + acquisition function. Finds good params in ~50 trials, not 3000.

HyperparameterSearch code


from kailash_ml import HyperparameterSearch
from kailash_ml import SearchSpace, SearchConfig

space = SearchSpace({
    "n_estimators": (100, 1000),
    "max_depth": (3, 10),
    "learning_rate": (0.01, 0.3, "log"),
})
config = SearchConfig(
    method="bayesian", n_trials=50,
    metric="auc", cv_folds=5,
)

search = HyperparameterSearch(model_class="xgboost")
best = search.run(X_train, y_train,
                  space=space, config=config)
print(f"Best AUC: {best.score:.4f}")

Bayesian search focuses where it matters

Grid vs Random vs Bayesian

MethodEvaluationsInformed?Best for
Grid$k^d$ (exponential)No1-2 hyperparameters
RandomFixed budgetNoHigh-dimensional, cheap evals
BayesianFixed budgetYes (surrogate)Expensive evals, complex spaces

ModelRegistry


from kailash_ml import ModelRegistry, MetricSpec
from kailash_ml import ModelSignature

registry = ModelRegistry()
registry.register(
    name="credit_scorer", model=best.model,
    metrics=MetricSpec(auc=best.score),
    signature=ModelSignature(
        input_schema=X_train.schema,
        output_schema={"probability": "Float64"},
    ),
)
registry.promote("credit_scorer", version=1,
                 stage="production")

Model lifecycle

Three mistakes to avoid

1. Grid search with 5+ hyperparameters
2. Promoting models without a signature
3. No conditional gate on promotion

What you can now do

  • Build ML workflows as node graphs with WorkflowBuilder
  • Use Bayesian optimisation for hyperparameter search
  • Register, version, and promote models through the lifecycle
