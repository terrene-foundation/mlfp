Why RAG beats fine-tuning for knowledge

  • Freshness: re-index daily (trivial) vs retrain daily (absurd).
  • Attribution: RAG cites source chunks. Fine-tuning blurs facts into weights.
  • Cost: indexing 1M docs = pennies. Fine-tuning on 1M docs = thousands.
🟢 Foundations: fine-tune for skills and style. RAG for facts and freshness.

The RAG pipeline — two paths meet at the vector store

Chunking strategies

StrategyHowWhen
Fixed sizeSplit every N tokensSimple, fast, baseline
SentenceSplit on sentence boundariesPreserves meaning
SemanticEmbed, cluster, split at topic shiftsBest quality, slower

Always overlap chunks (10-20%) to avoid splitting key passages at boundaries.

Three retrieval approaches

TypeMethodStrength
SparseBM25 (from 4.6)Exact keyword match
DenseSentence embeddings + cosineSemantic similarity
HybridCombine dense + sparse scoresBest of both
🔵 Theory: hybrid retrieval with re-ranking (cross-encoder) is the production standard.

RAGAS evaluation

MetricMeasures
FaithfulnessIs the answer grounded in retrieved context?
Answer relevanceDoes the answer address the question?
Context relevanceAre the retrieved chunks relevant?
Context recallDid we retrieve all relevant chunks?

HyDE — Hypothetical Document Embeddings

Generate a hypothetical answer, embed it, use that embedding for retrieval. Retrieves documents that look like answers, not just keyword matches.

RAG code


from sentence_transformers import SentenceTransformer
import numpy as np

encoder = SentenceTransformer("all-MiniLM-L6-v2")
chunk_embeddings = encoder.encode(chunks)

query_emb = encoder.encode([query])
sims = np.dot(chunk_embeddings, query_emb.T).flatten()
top_k = np.argsort(sims)[-5:][::-1]

context = "\n".join([chunks[i] for i in top_k])
answer = llm.generate(f"Context: {context}\nQ: {query}\nA:")

Common mistakes

  1. Chunks too large — LLM gets distracted by irrelevant context. 200–500 tokens.
  2. No overlap — key passages split at chunk boundaries.
  3. Dense-only retrieval — misses exact keyword matches. Use hybrid.
  4. No RAGAS evaluation — "it looks right" is not a metric.
