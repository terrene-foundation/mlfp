Lesson 3.6 Interpretability and Fairness

A model that scores 0.95 AUC means nothing if you cannot explain why it makes its predictions, and even less if it discriminates against protected groups. This lesson introduces SHAP (global and local explanations from Shapley values), LIME (model-agnostic local explanations), ALE (accumulated local effects that handle correlated features), and the fairness metrics every deployed model must report. We close with the impossibility theorem: you cannot simultaneously satisfy demographic parity, equalised odds, and calibration.

SHAP: Shapley Additive Explanations

Shapley axioms

Shapley values come from cooperative game theory. A "game" has players (features) and a payoff (prediction). The Shapley value of feature $i$ is the average marginal contribution of $i$ across all possible coalitions of features:

$$\phi_i = \sum_{S \subseteq F \setminus \{i\}} \frac{|S|!\,(|F| - |S| - 1)!}{|F|!} \left[ f(S \cup \{i\}) - f(S) \right]$$
$S$ = subset of features without $i$ $f(S)$ = model prediction using only features in $S$

Shapley values satisfy four axioms that make them the unique fair attribution method:

AxiomMeaning
EfficiencyContributions sum to the total prediction: $\sum_i \phi_i = f(x) - \mathbb{E}[f(x)]$
SymmetryFeatures with identical contributions get identical values
DummyA feature that never changes the prediction gets $\phi_i = 0$
LinearityFor a sum of two models, Shapley values add

TreeSHAP and KernelSHAP

Computing exact Shapley values requires evaluating $2^{|F|}$ coalitions. Two approximations make this practical:

  • TreeSHAP: exact Shapley values for tree-based models (XGBoost, LightGBM, random forests) in polynomial time. Use this whenever you have a tree model.
  • KernelSHAP: model-agnostic approximation using weighted linear regression on coalitions. Slower but works for any model.
import shap

# TreeSHAP for a gradient boosting model
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)

# Summary plot: global feature importance
shap.summary_plot(shap_values, X_test)

# Waterfall for one prediction
shap.plots.waterfall(shap_values[0])

LIME: Local Interpretable Model-agnostic Explanations

LIME explains individual predictions by fitting a simple, interpretable model (usually linear) in the local neighbourhood of the instance. It perturbs the input, gets predictions from the black-box model, and fits a weighted linear model where closer perturbations get higher weight. 

from lime.lime_tabular import LimeTabularExplainer

explainer = LimeTabularExplainer(
    X_train.to_numpy(),
    feature_names=X_train.columns,
    class_names=["low", "high"],
)

explanation = explainer.explain_instance(
    X_test[0].to_numpy(),
    model.predict_proba,
    num_features=5,
)
explanation.show_in_notebook()
🔵 Theory

SHAP gives exact attributions that sum to the prediction (efficiency axiom). LIME gives approximate local explanations that may not sum exactly. Use SHAP when you need consistency and additivity; use LIME when you need a quick, model-agnostic explanation.

ALE: Accumulated Local Effects

Partial Dependence Plots (PDP) show the average effect of a feature, but they assume features are independent. When features are correlated, PDPs produce unrealistic data points. ALE fixes this by accumulating local effects within small intervals:

🟣 Advanced

ALE partitions the feature range into intervals, computes the average change in prediction within each interval, and accumulates these changes. Unlike PDP, ALE never evaluates the model on unrealistic feature combinations (e.g., a flat that is both 20 sqm and has 5 bedrooms).

Fairness

Disparate impact

The four-fifths rule: the selection rate for the minority group should be at least 80% of the selection rate for the majority group:

$$\text{Disparate Impact} = \frac{P(\hat{Y}=1 \mid G=\text{minority})}{P(\hat{Y}=1 \mid G=\text{majority})} \;\geq\; 0.8$$
Below 0.8 = potential discrimination Common in hiring and lending

Equalised odds

A model satisfies equalised odds if the true positive rate and false positive rate are equal across groups:

$$P(\hat{Y}=1 \mid Y=1, G=a) = P(\hat{Y}=1 \mid Y=1, G=b)$$ $$P(\hat{Y}=1 \mid Y=0, G=a) = P(\hat{Y}=1 \mid Y=0, G=b)$$

The impossibility theorem

⚠ The impossibility theorem

Chouldechova (2017) and Kleinberg et al. (2016) proved that when base rates differ between groups, you cannot simultaneously satisfy demographic parity, equalised odds, and calibration. Any model that satisfies two of the three necessarily violates the third.

This is not a limitation of any particular model — it is a mathematical fact. Fairness is an engineering tradeoff: decide which criterion matters most for your deployment context, measure it, report it, and mitigate where possible.

Worked Example: Credit Scoring Fairness Audit

📝 Goal

Audit the credit scoring model from Lesson 3.5 for fairness across demographic groups.

Step 1 — Compute SHAP values. 
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)
shap.summary_plot(shap_values, X_test)
Step 2 — Measure disparate impact. 
pred = model.predict(X_test)
rate_minority = pred[group == "minority"].mean()
rate_majority = pred[group == "majority"].mean()
di = rate_minority / rate_majority
print(f"Disparate impact: {di:.3f} {'PASS' if di >= 0.8 else 'FAIL'}")
Step 3 — Measure equalised odds. 
for g in ["minority", "majority"]:
    mask = group == g
    tpr = recall_score(y_test[mask], pred[mask])
    fpr = (pred[mask] & ~y_test[mask]).sum() / (~y_test[mask]).sum()
    print(f"  {g}: TPR={tpr:.3f}, FPR={fpr:.3f}")
Step 4 — Report findings.

Document: which fairness criteria were measured, which passed, which failed, and what the tradeoffs are. The impossibility theorem means you cannot fix everything — be transparent about the choice.

Common Mistakes

⚠ Mistake 1: Using SHAP for causal claims

SHAP tells you which features the model uses. It does not tell you which features cause the outcome. "Income has high SHAP importance" does not mean income causes creditworthiness.

⚠ Mistake 2: Removing a protected feature and assuming fairness

Other features (postcode, school, employment history) are often proxies for the protected attribute. Removing gender from the model does not prevent gender discrimination.

⚠ Mistake 3: Optimising for one fairness metric blindly

The impossibility theorem means optimising for demographic parity may hurt calibration. Always measure and report multiple metrics.
