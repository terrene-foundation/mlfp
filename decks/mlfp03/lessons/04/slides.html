Bagging vs Boosting

The boosting recipe

\[ F_m(x) = F_{m-1}(x) + \eta \cdot f_m(x) \]
  • $F_0$ is a constant (mean or log-odds)
  • $f_m$ is the next tree, fit to the residuals/gradients of $F_{m-1}$
  • $\eta$ is the learning rate — bias-variance dial
  • Small $\eta$ with many trees $>$ large $\eta$ with few trees

XGBoost — second-order objective

\[ \mathcal{L}^{(t)} = \sum_i L(y_i, \hat{y}_i^{(t-1)} + f_t(x_i)) + \Omega(f_t) \]

where $\Omega(f_t) = \gamma T + \tfrac{1}{2}\lambda \sum_j w_j^2$.

Taylor-expand to second order around $\hat{y}_i^{(t-1)}$:

\[ L(y_i, \hat{y}_i^{(t-1)} + f_t(x_i)) \approx L_0 + g_i f_t + \tfrac{1}{2} h_i f_t^2 \]

Group by leaf, solve in closed form

Let $G_j = \sum_{i \in I_j} g_i$, $H_j = \sum_{i \in I_j} h_i$. Then:

\[ \tilde{\mathcal{L}}^{(t)} = \sum_{j=1}^T \bigl[G_j w_j + \tfrac{1}{2}(H_j + \lambda) w_j^2\bigr] + \gamma T \]

Quadratic in $w_j$. Minimum:

\[ w_j^* = -\frac{G_j}{H_j + \lambda} \qquad \tilde{\mathcal{L}}^{(t)}_{\min} = -\tfrac{1}{2} \sum_j \frac{G_j^2}{H_j + \lambda} + \gamma T \]

The split gain — the whole point

\[ \mathrm{Gain} = \tfrac{1}{2}\Bigl[\frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L + G_R)^2}{H_L + H_R + \lambda}\Bigr] - \gamma \]
  • Evaluate for every candidate split, keep the one with largest gain
  • $\gamma$ is the pruning cost — no split if gain $< \gamma$
  • $\lambda$ shrinks leaf weights (L2)
  • Loss-agnostic: any twice-differentiable loss plugs in via $g_i, h_i$

LightGBM — speed tricks

  • Histogram splits: bin features into ~256 buckets, evaluate splits at bucket boundaries — $O(n + 256)$ instead of $O(n \log n)$
  • GOSS: keep top $a\%$ of rows by $|g_i|$, randomly sample $b\%$ from the rest — ~30% of data per tree
  • Leaf-wise growth: always split the leaf with largest gain, regardless of depth. Unbalanced trees, faster convergence, num_leaves is the main dial

GOSS intuition

Small-gradient rows are already well-predicted. Large-gradient rows carry most of the signal. Subsample the small ones, keep all the large ones.

  1. Sort by $|g_i|$ descending
  2. Keep top $a\%$ (default $a=20$)
  3. Random sample $b\%$ from the rest (default $b=10$)
  4. Train on those $a + b \approx 30\%$ of rows
  5. Up-weight the sampled low-gradient rows by $(1 - a)/b$

~3× speedup, minimal accuracy loss.

CatBoost — ordered boosting

Problem: mean-target encoding leaks the target into the features.

Solution: impose an artificial row ordering and encode row $i$ using only rows $j < i$:

\[ \text{enc}(x_i^{\text{cat}}) = \frac{\sum_{j<i,\, x_j^{\text{cat}} = x_i^{\text{cat}}} y_j + a\bar{y}}{\sum_{j<i,\, x_j^{\text{cat}} = x_i^{\text{cat}}} 1 + a} \]

Reshuffle the ordering every boosting iteration. No leakage, no hand-engineering.

Train all three on credit scoring


models = {
  "XGBoost":  xgb.XGBClassifier(n_estimators=400, learning_rate=0.05,
              max_depth=6, reg_lambda=1, gamma=0.1, eval_metric="aucpr"),
  "LightGBM": lgb.LGBMClassifier(n_estimators=400, learning_rate=0.05,
              num_leaves=31, reg_lambda=1),
  "CatBoost": cb.CatBoostClassifier(iterations=400, learning_rate=0.05,
              depth=6, l2_leaf_reg=3, verbose=False),
}

for name, m in models.items():
    m.fit(X_tr, y_tr)
    p = m.predict_proba(X_te)[:, 1]
    print(name, "AUC-PR =", average_precision_score(y_te, p))

Early stopping — free tuning


xgb_model = xgb.XGBClassifier(
    n_estimators=1000,
    learning_rate=0.05,
    early_stopping_rounds=25,
    eval_metric="aucpr",
)
xgb_model.fit(X_tr, y_tr, eval_set=[(X_te, y_te)])
print("Stopped at round", xgb_model.best_iteration)

Halt when the validation metric hasn't improved for 25 rounds. Standard way to pick n_estimators without grid search.

Which booster?

BoosterPick whenTune first
XGBoostclean numeric, small-medium, reproducibilitylr, max_depth, reg_lambda
LightGBMmillions of rows, many features, speed mattersnum_leaves, lr, min_data_in_leaf
CatBoostmany high-cardinality categoricalsiterations, depth, l2_leaf_reg

Three traps

1. ROC-AUC on imbalanced data — use AUC-PR (average precision) instead. More in 3.5.
2. No early stopping — you under- or overfit. Always use eval_set.
3. Tuning lr and n_estimators independently — they interact. Fix small lr, let early stopping pick n_estimators.
