One chart, one question

Before you build a chart, ask yourself:

"What decision should the reader make after seeing this?"

If you cannot state the question in one sentence, the chart is not ready.

Gestalt principles: how eyes group shapes

Six chart types cover 95% of questions

QuestionChart
How is one number distributed?Histogram
How does one number change over time?Line chart
Which category ranks highest?Horizontal bar
Do two numbers relate?Scatter plot
How do many numbers correlate?Heatmap
How is a whole composed?Stacked bar

🔵 Cleveland–McGill perceptual ranking

Bars use #1. Pies use #4. 3D uses #5/#6. Always encode the key comparison using the most accurate task.

ModelVisualizer API


from kailash_ml import ModelVisualizer

viz = ModelVisualizer()

fig = viz.histogram(
    data=hdb,
    column="resale_price",
    bins=40,
    title="HDB Resale Price Distribution",
)
fig.write_html("price_histogram.html")

Same shape for every method: data, columns, title, get back a Plotly Figure. fig.show() in Jupyter, fig.write_html() to save.

Histogram — distribution


fig = viz.histogram(
    data=hdb,
    column="resale_price",
    bins=40,
    title="HDB Resale Price Distribution",
)

HDB prices are right-skewed: most in 350k–600k, long tail into million-dollar territory. Mean > median as a result.

Scatter — relationship between two numbers


# SAMPLE before plotting — 500k points is a grey blob
hdb_sample = hdb.sample(n=5_000, seed=42)

fig = viz.scatter(
    data=hdb_sample,
    x="floor_area_sqm",
    y="resale_price",
    title="HDB Resale Price vs Floor Area",
)

Clear positive slope, wide vertical spread — area explains maybe 40–50% of price variance. The rest is town, floor, lease, amenities.

Bar — ranking categories


district_prices = (
    hdb.group_by("town")
    .agg(pl.col("resale_price").median().alias("median_price"))
    .sort("median_price", descending=True)
)
# feature_importance = horizontal bar
fig = viz.metric_comparison({
    town: {"Median Price (S$)": price}
    for town, price in zip(
        district_prices["town"].to_list(),
        district_prices["median_price"].to_list(),
    )
})

Horizontal bars — long town names readable, lengths on a common axis.

Line — change over time


monthly = (
    hdb.group_by("transaction_date")
    .agg(pl.col("price_per_sqm").median().alias("median_psm"))
    .sort("transaction_date")
)

fig = viz.training_history({
    "median_price_sqm": monthly["median_psm"].to_list()
})

Line = continuity Gestalt. The eye reads a line as a trend automatically.

Heatmap — correlations at a glance


numeric = hdb.select("resale_price", "floor_area_sqm", "price_per_sqm")
corr = numeric.corr()
fig = viz.confusion_matrix(
    matrix=corr.to_numpy(),
    labels=numeric.columns,
    title="HDB Feature Correlations",
)

Colour encodes magnitude; diverging palette around zero. Use for many pairwise comparisons in a single small panel.

Stacked bar — composition


import plotly.express as px

composition = (
    hdb.group_by("town", "flat_type")
    .agg(pl.len().alias("count"))
)
fig = px.bar(
    composition.to_pandas(),
    x="town", y="count", color="flat_type",
    title="Flat Type Composition by Town",
)

Stacked bar = parts-of-whole comparison across categories.

Four charts to avoid

❌ 3D bars / pies. Perspective distorts quantities.
❌ Pies with > 5 slices. Angle comparison is hard; labels get crowded.
❌ Truncated y-axis on bars. Starts at 400, not 0 → misleading.
❌ Scatter with 500k points. Grey blob. Sample or hexbin.

Title = takeaway, not topic

❌ Topic: "HDB Resale Prices by District"
✅ Takeaway: "Central districts command a 60% premium over peripheral towns"

Your chart title is the first (often only) thing the reader reads. Put the conclusion there, not the topic.
