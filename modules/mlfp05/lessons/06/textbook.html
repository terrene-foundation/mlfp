Lesson 5.6 Graph Neural Networks

Not all data fits neatly into grids (images) or sequences (text). Social networks, molecular structures, knowledge graphs, and supply chains are naturally graphs — nodes connected by edges. Today you learn three architectures — GCN, GraphSAGE, and GAT — that extend neural networks to graph-structured data using message passing: each node updates its representation by aggregating information from its neighbours. You implement graph classification on molecular data using torch_geometric and visualise learned node embeddings to see what GNNs discover.

Why This Matters

Singapore's MRT network is a graph: stations are nodes, rail connections are edges, and passenger flow is a signal on that graph. DBS Bank's fraud detection system models transactions as a graph — accounts are nodes, transfers are edges, and suspicious patterns emerge as unusual subgraph structures. NUS chemistry researchers model molecules as graphs — atoms are nodes, bonds are edges, and predicting molecular properties (toxicity, solubility, drug efficacy) is a graph classification task.

Whenever your data has explicit relationships between entities, a graph representation captures structure that tabular data loses. Flattening a social network into a feature table discards the connection topology — who knows whom, who influences whom — which is often the most predictive signal.

Core Concepts

Graph representation

A graph $G = (V, E)$ consists of nodes $V$ (vertices) and edges $E$ (connections). Each node has a feature vector $\mathbf{h}_i \in \mathbb{R}^d$. The graph structure is encoded in the adjacency matrix $A \in \{0, 1\}^{|V| \times |V|}$ where $A_{ij} = 1$ if nodes $i$ and $j$ are connected.

Message passing — the GNN paradigm

All GNN architectures share one core idea: message passing. At each layer, every node collects information from its neighbours, aggregates it, and updates its own representation. One round of message passing = one GNN layer. After $L$ layers, each node's representation encodes information from its $L$-hop neighbourhood.

Math Foundations

GCN — Graph Convolutional Network

The GCN (Kipf and Welling, 2017) applies a spectral convolution approximation that simplifies to a neighbourhood aggregation with symmetric normalisation:

$$ H^{(l+1)} = \sigma\bigl(\tilde{D}^{-1/2}\,\tilde{A}\,\tilde{D}^{-1/2}\,H^{(l)}\,W^{(l)}\bigr) $$
$\tilde{A} = A + I_N$ — adjacency with self-loops $\tilde{D}_{ii} = \sum_j \tilde{A}_{ij}$ — degree matrix $W^{(l)} \in \mathbb{R}^{d_l \times d_{l+1}}$ — learnable weights $\sigma$ — activation (ReLU)

The term $\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}$ normalises the adjacency matrix so that high-degree nodes do not dominate. Adding the identity matrix $I_N$ to $A$ creates self-loops — each node also aggregates its own features. The result: each node's new representation is a weighted average of its neighbours' features (including itself), transformed by $W^{(l)}$.

Theory

The symmetric normalisation $\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}$ is key. Without it, nodes with many neighbours would have much larger feature values after aggregation (since they sum more neighbours). The normalisation divides by $\sqrt{\deg(i) \cdot \deg(j)}$, making the aggregation scale-invariant. This is analogous to batch normalisation in CNNs — it prevents features from growing or shrinking uncontrollably across layers.

GraphSAGE — inductive learning

GCN is transductive: it requires the full graph at training time and cannot generalise to unseen nodes. GraphSAGE (Hamilton et al., 2017) is inductive: it learns an aggregation function that works on any node, even new ones not seen during training. This is essential for dynamic graphs where new users or products appear daily.

GraphSAGE with mean aggregation:

$$ \mathbf{h}_v^{(l+1)} = \sigma\Bigl(W^{(l)} \cdot \text{CONCAT}\bigl(\mathbf{h}_v^{(l)},\;\text{MEAN}\bigl\{\mathbf{h}_u^{(l)} : u \in \mathcal{N}(v)\bigr\}\bigr)\Bigr) $$
$\mathcal{N}(v)$ — sampled neighbours of $v$ MEAN — mean aggregation (or LSTM, max pooling) CONCAT — concatenate self and aggregated neighbour

The critical difference: GraphSAGE samples a fixed-size neighbourhood (e.g., 10 neighbours per node per layer) rather than using all neighbours. This makes computation tractable on large graphs — Facebook's social graph has billions of edges, but each aggregation step only touches a fixed number.

GAT — attention on graphs

GCN weights all neighbours equally (after normalisation). GraphSAGE uses a fixed aggregation function. GAT (Velickovic et al., 2018) learns different attention weights for each neighbour pair — the same attention mechanism from Lesson 5.4, applied to graph neighbourhoods:

$$ e_{ij} = \text{LeakyReLU}\bigl(\mathbf{a}^T [W\mathbf{h}_i \| W\mathbf{h}_j]\bigr) $$
$W$ — shared weight matrix $\mathbf{a}$ — attention weight vector $\|$ — concatenation $e_{ij}$ — attention coefficient

Attention coefficients are normalised with softmax over the neighbourhood:

$$ \alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k \in \mathcal{N}(i)} \exp(e_{ik})} $$

The updated representation uses the attention-weighted sum:

$$ \mathbf{h}_i^{(l+1)} = \sigma\Bigl(\sum_{j \in \mathcal{N}(i)} \alpha_{ij}\, W\mathbf{h}_j^{(l)}\Bigr) $$

Multi-head attention (as in Transformers) is applied by running $K$ independent attention heads and concatenating or averaging the results.

Advanced

GAT attention is structurally identical to the self-attention in Transformers (Lesson 5.4). The difference: Transformers attend over all tokens (global attention), while GATs attend only over graph neighbours (local attention). This makes GAT computationally cheaper on sparse graphs — attention is $O(|E|)$ instead of $O(|V|^2)$. On dense graphs, the distinction disappears.

Foundations

The key distinction: node classification uses the node embeddings directly as input to a classifier. Graph classification needs to aggregate all node embeddings into a single graph-level representation — this is global_mean_pool (or global_max_pool). Forgetting the pooling step for graph classification is a common error that causes shape mismatches.

Worked Example: Molecular Graph Classification

Goal

Classify molecules in the MUTAG dataset (188 molecules, binary labels: mutagenic or not) using a 2-layer GCN. Then compare with a GAT and visualise the learned node embeddings.

Step 1. Load the dataset. 
import torch
import torch.nn.functional as F
from torch_geometric.datasets import TUDataset
from torch_geometric.loader import DataLoader

dataset = TUDataset(root='/tmp/MUTAG', name='MUTAG')
print(f"Graphs: {len(dataset)}")
print(f"Classes: {dataset.num_classes}")
print(f"Node features: {dataset.num_features}")
# → Graphs: 188, Classes: 2, Node features: 7

# Train/test split
train_dataset = dataset[:150]
test_dataset = dataset[150:]
train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
test_loader = DataLoader(test_dataset, batch_size=32)
Step 2. Build a 2-layer GCN for graph classification. 
from torch_geometric.nn import GCNConv, global_mean_pool

class GCN(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels):
        super().__init__()
        self.conv1 = GCNConv(in_channels, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, hidden_channels)
        self.classifier = torch.nn.Linear(hidden_channels, out_channels)

    def forward(self, data):
        x, edge_index, batch = data.x, data.edge_index, data.batch

        # Message passing layers
        x = F.relu(self.conv1(x, edge_index))
        x = F.dropout(x, p=0.3, training=self.training)
        x = F.relu(self.conv2(x, edge_index))

        # Graph-level pooling (critical for graph classification)
        x = global_mean_pool(x, batch)  # [num_graphs, hidden_channels]

        # Classification head
        return self.classifier(x)

model = GCN(dataset.num_features, 64, dataset.num_classes)

Note global_mean_pool(x, batch): this averages all node embeddings within each graph to produce a single graph-level vector. The batch tensor tells the pooling layer which nodes belong to which graph (torch_geometric batches multiple graphs into one large disconnected graph for efficiency).

Step 3. Train and evaluate. 
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

def train():
    model.train()
    total_loss = 0
    for data in train_loader:
        optimizer.zero_grad()
        out = model(data)
        loss = F.cross_entropy(out, data.y)
        loss.backward()
        optimizer.step()
        total_loss += loss.item() * data.num_graphs
    return total_loss / len(train_dataset)

def test(loader):
    model.eval()
    correct = 0
    for data in loader:
        out = model(data)
        pred = out.argmax(dim=1)
        correct += (pred == data.y).sum().item()
    return correct / len(loader.dataset)

for epoch in range(1, 101):
    loss = train()
    train_acc = test(train_loader)
    test_acc = test(test_loader)
    if epoch % 20 == 0:
        print(f"Epoch {epoch:3d} | Loss: {loss:.4f} | "
              f"Train: {train_acc:.3f} | Test: {test_acc:.3f}")

# Epoch  40 | Loss: 0.3912 | Train: 0.840 | Test: 0.789

Step 4. Compare GCN with GAT. 
from torch_geometric.nn import GATConv

class GAT(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels, heads=4):
        super().__init__()
        self.conv1 = GATConv(in_channels, hidden_channels, heads=heads)
        self.conv2 = GATConv(hidden_channels * heads, hidden_channels, heads=1)
        self.classifier = torch.nn.Linear(hidden_channels, out_channels)

    def forward(self, data):
        x, edge_index, batch = data.x, data.edge_index, data.batch
        x = F.relu(self.conv1(x, edge_index))
        x = F.dropout(x, p=0.3, training=self.training)
        x = F.relu(self.conv2(x, edge_index))
        x = global_mean_pool(x, batch)
        return self.classifier(x)

gat_model = GAT(dataset.num_features, 64, dataset.num_classes)
# Train similarly — GAT typically achieves 0.85-0.87 test accuracy on MUTAG

The GAT model uses heads=4 in the first layer — four independent attention mechanisms that each learn different neighbour importance patterns. The second layer uses heads=1 to produce the final node embedding. Note that the input to the second layer is hidden_channels * heads because multi-head attention concatenates head outputs.

Common Mistakes

Mistake 1: Ignoring graph structure

Flattening a graph into a feature table loses relationship information. If you have a social network and compute "number of friends" as a feature, you lose who those friends are and how they connect to each other. Use GNNs when edges carry signal — when the pattern of connections matters, not just the count.

Mistake 2: Too many GNN layers

GNNs over-smooth with depth. After many layers, every node's representation converges to the same value because it has aggregated information from the entire graph. For most tasks, 2-3 layers is optimal. If you need deeper models, use skip connections (residual GNN) or jumping knowledge networks that concatenate representations from all layers.

Mistake 3: Wrong pooling for the task

Node classification needs no pooling — you classify each node directly from its embedding. Graph classification needs global_mean_pool or global_max_pool to produce a graph-level vector. Mixing these up causes dimension mismatches or, worse, silently incorrect results.

Mistake 4: Forgetting self-loops

Without self-loops ($\tilde{A} = A + I$), each node aggregates only its neighbours' features and loses its own. After several layers, a node's representation contains no information about its original features — only neighbourhood statistics. Self-loops ensure the node retains its own identity. GCNConv in torch_geometric adds self-loops by default; custom implementations must add them explicitly.

Reflection

With GNNs, you have now covered all major deep learning data structures: grids (CNN, Lesson 5.2), sequences (RNN/Transformer, Lessons 5.3-5.4), generative models (VAE/GAN/Diffusion, Lessons 5.1/5.5), and graphs (GNN, today). Each architecture is optimised for a different data structure — the key skill is recognising which structure your data naturally has.

The message passing paradigm — aggregate neighbours, update self, repeat — is a beautiful unification. CNNs are GNNs on a fixed grid. Transformers are GNNs on a fully connected graph. Understanding message passing gives you a conceptual framework that connects all the architectures you have learned.

The next two lessons consolidate: transfer learning (5.7) leverages pre-trained models so you rarely train from scratch, and reinforcement learning (5.8) introduces learning from interaction rather than static data. Then Module 6 applies everything to LLMs and production systems.
