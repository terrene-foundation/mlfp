Lesson 4.2 EM Algorithm and Gaussian Mixture Models

K-means gave every customer a single cluster label. That is a lie. Most customers sit somewhere between "high-value" and "average" — they belong 70% to one segment and 30% to another. Today you will meet the Expectation-Maximisation (EM) algorithm, the twenty-line procedure that powers Gaussian mixture models, hidden Markov models, topic models, and half of classical machine learning. You will implement it yourself, then watch sklearn's GaussianMixture do exactly the same thing on real data.

Why This Matters

Hard clustering forces every point into exactly one group. For a point sitting directly on the boundary between two clusters, this is information loss: the algorithm knows the point is ambiguous but reports confident labels anyway. Soft clustering fixes this by producing responsibilities — for each point, a probability vector over clusters. A point can be 70% in cluster A, 25% in cluster B, and 5% in cluster C. That extra information unlocks downstream decisions that hard labels cannot support.

🟢 Foundations

Think of GMM as "K-means with error bars". The cluster centres exist; the membership is probabilistic. When your boss asks "how confident are we that this customer is in segment A?", K-means has no answer. GMM gives you a number.

Core Concepts

The generative story

A Gaussian mixture model is a generative model — it describes how the data was created rather than how to classify it. The story is:

  1. Pick a cluster index $k$ with probability $\pi_k$. The $\pi_k$ are the mixing coefficients and sum to 1.
  2. Draw a point from the multivariate normal distribution $\mathcal{N}(\boldsymbol{\mu}_k,\,\boldsymbol{\Sigma}_k)$.

If we wrote this as math, the overall density of a single point $\mathbf{x}$ is the sum over all the "ways it could have been generated":

$$ p(\mathbf{x}) \;=\; \sum_{k=1}^{K}\ \pi_k\ \mathcal{N}\!\bigl(\mathbf{x}\,\big|\,\boldsymbol{\mu}_k,\,\boldsymbol{\Sigma}_k\bigr) $$

Our job is to find the $K$ triples $(\pi_k, \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)$ that make the observed data most likely — i.e., maximise the log-likelihood $\log \prod_n p(\mathbf{x}_n)$. That is a hard optimisation because the log of a sum has no closed form. EM is the clever workaround.

The EM algorithm

EM introduces a latent variable $z_{nk}$ — a binary indicator that is 1 if point $n$ was generated by cluster $k$, 0 otherwise. We do not observe $z$; if we did, the problem would reduce to fitting $K$ independent Gaussians. The trick: alternate between (a) computing the expected value of $z_{nk}$ given the current parameters (the E-step), and (b) maximising the complete-data likelihood as if those expected values were observed counts (the M-step).

E-step: compute responsibilities

The E-step computes, for each point $n$ and each cluster $k$, the probability that cluster $k$ generated point $n$ given the current parameters:

$$ r_{nk} \;=\; \frac{\pi_k\ \mathcal{N}\!\bigl(\mathbf{x}_n\,\big|\,\boldsymbol{\mu}_k,\,\boldsymbol{\Sigma}_k\bigr)}{\sum_{j=1}^{K}\ \pi_j\ \mathcal{N}\!\bigl(\mathbf{x}_n\,\big|\,\boldsymbol{\mu}_j,\,\boldsymbol{\Sigma}_j\bigr)} $$

This is Bayes' rule: prior times likelihood, normalised. The denominator is $p(\mathbf{x}_n)$ — the marginal density of point $n$ under the current mixture. By construction $\sum_k r_{nk} = 1$ for every point. The $r_{nk}$ are the soft assignments: the fractional membership of point $n$ in cluster $k$.

M-step: update parameters as weighted means

Given the responsibilities, the M-step updates each parameter as if $r_{nk}$ were the number of times point $n$ was observed in cluster $k$:

$$ N_k \;=\; \sum_{n=1}^{N} r_{nk} $$ $$ \boldsymbol{\mu}_k^{\text{new}} \;=\; \frac{1}{N_k}\sum_{n=1}^{N}\ r_{nk}\ \mathbf{x}_n $$ $$ \boldsymbol{\Sigma}_k^{\text{new}} \;=\; \frac{1}{N_k}\sum_{n=1}^{N}\ r_{nk}\ (\mathbf{x}_n - \boldsymbol{\mu}_k^{\text{new}})(\mathbf{x}_n - \boldsymbol{\mu}_k^{\text{new}})^\top $$ $$ \pi_k^{\text{new}} \;=\; \frac{N_k}{N} $$

Each update is a weighted version of the maximum-likelihood estimate for a single Gaussian. $N_k$ is the "effective number of points" assigned to cluster $k$. The new mean is the weighted average of the data; the new covariance is the weighted scatter around the new mean; the new mixing coefficient is the fraction of effective mass.

🔵 Theory

The guarantee that each EM iteration does not decrease the log-likelihood comes from a Jensen's-inequality argument: EM is performing coordinate ascent on a lower bound of the log-likelihood (the ELBO), and each step tightens or improves the bound. This is why EM converges even though the objective is non-convex. It may converge to a local maximum — in practice, run EM a few times from different random initialisations and keep the best.

Twenty lines of EM, from scratch

import numpy as np
from scipy.stats import multivariate_normal as mvn

def em_gmm(X, K, max_iter=100, tol=1e-4):
    N, D = X.shape
    # Random init: pick K data points as means, identity covariances, uniform mixing
    rng = np.random.default_rng(42)
    mu = X[rng.choice(N, size=K, replace=False)]
    Sigma = np.array([np.eye(D) for _ in range(K)])
    pi = np.full(K, 1.0 / K)

    log_lik = -np.inf
    for iteration in range(max_iter):
        # E-step: responsibilities r_{nk}
        weighted = np.stack([pi[k] * mvn.pdf(X, mean=mu[k], cov=Sigma[k]) for k in range(K)], axis=1)
        total = weighted.sum(axis=1, keepdims=True) + 1e-12
        r = weighted / total                       # shape (N, K)

        # M-step: weighted mean, covariance, mixing
        Nk = r.sum(axis=0)                         # effective count per cluster
        pi = Nk / N
        mu = (r.T @ X) / Nk[:, None]
        for k in range(K):
            diff = X - mu[k]
            Sigma[k] = (r[:, k:k+1] * diff).T @ diff / Nk[k]

        # Log-likelihood and convergence check
        new_log_lik = np.log(total + 1e-12).sum()
        if abs(new_log_lik - log_lik) < tol:
            break
        log_lik = new_log_lik

    return pi, mu, Sigma, r, log_lik

Twenty substantive lines. Compare against sklearn.mixture.GaussianMixture — for well-conditioned data the implementations agree to within numerical tolerance, which is the right way to validate your own EM.

Mathematical Foundations

The complete-data log-likelihood

If we actually observed $z_{nk}$ (whether point $n$ came from cluster $k$), the log-likelihood would split into $K$ independent Gaussian fits:

$$ \log p(X, Z) \;=\; \sum_{n=1}^{N}\ \sum_{k=1}^{K}\ z_{nk}\bigl[\log \pi_k + \log \mathcal{N}(\mathbf{x}_n | \boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)\bigr] $$

EM replaces the unknown $z_{nk}$ with their expected values $r_{nk}$ and maximises this expression — exactly the M-step formulas above. The procedure iterates because changing the parameters changes the responsibilities, which changes the optimal parameters, and so on.

Choosing the number of components

GMM has a principled criterion for choosing $K$: the Bayesian Information Criterion (BIC):

$$ \text{BIC} \;=\; -2\log \hat{\mathcal{L}} + p\log N $$

where $\hat{\mathcal{L}}$ is the maximised log-likelihood, $p$ is the number of free parameters, and $N$ is the sample size. Lower BIC is better. BIC penalises model complexity, so it naturally picks a $K$ that balances fit against parsimony. In practice, fit GMMs for $K = 2, 3, \ldots, 10$ and take the $K$ with the smallest BIC.

Mixture of Experts

A Mixture of Experts (MoE) generalises the mixture model by replacing the fixed $\pi_k$ with a gating network $g_k(\mathbf{x})$ that depends on the input — a different mixture for every point. GPT-4 and other state-of-the-art large language models use MoE layers to scale to trillions of parameters efficiently: for each token, the gating network picks a few experts (typically 2 out of 8) to run, saving compute while keeping capacity high.

🟣 Advanced

The classical GMM is a special case of MoE with constant gates. EM still applies to certain MoE variants; most modern MoE is trained end-to-end with backpropagation and load-balancing losses. We will revisit this in M6 when we build Kaizen agents — the "routing" in multi-agent systems is structurally the same idea.

Worked Example: Three Gaussians in 2D

import numpy as np
import matplotlib.pyplot as plt
from sklearn.mixture import GaussianMixture

# Generate 3 ground-truth Gaussians
rng = np.random.default_rng(0)
means = np.array([[0, 0], [5, 5], [-3, 4]])
covs = [np.eye(2) * 0.8, np.eye(2) * 1.2, np.eye(2) * 0.6]
sizes = [200, 150, 120]
X = np.vstack([
    rng.multivariate_normal(m, c, size=s) for m, c, s in zip(means, covs, sizes)
])

# Fit our EM
pi, mu, Sigma, r, ll = em_gmm(X, K=3)
print(f"Log-likelihood: {ll:.2f}")
print(f"Mixing:     {pi.round(3)}")

# Cross-check with sklearn
gm = GaussianMixture(n_components=3, random_state=0).fit(X)
print(f"sklearn LL: {gm.score(X) * len(X):.2f}")
print(f"sklearn π: {gm.weights_.round(3)}")

Both implementations should report log-likelihoods within 1% of each other. The soft assignments r let you visualise responsibility heat maps — points near cluster centres get high responsibility for their cluster; points near boundaries get diffuse responsibilities. Look at the points with r.max(axis=1) < 0.6 — those are the "uncertain" customers, and they are exactly the ones you should flag for manual review in a real segmentation task.

Common Mistakes

  1. Initialising every covariance to zero. The Gaussian density is undefined when $\boldsymbol{\Sigma}$ is singular. Always initialise to the identity (or to the sample covariance) and add a small $\epsilon I$ regulariser each iteration.
  2. Running for a fixed number of iterations without checking convergence. EM can converge in 10 iterations on clean data and 500 on ambiguous data. Always check $|\Delta \log \mathcal{L}| < \text{tol}$ and stop early.
  3. Comparing GMM and K-means by accuracy against labels. Both are unsupervised — they have no way to match their cluster indices to your labels. Use ARI or NMI if you must compare to a ground truth.
  4. Using covariance_type="full" on high-dimensional data. Full covariance has $\mathcal{O}(D^2)$ parameters per cluster and overfits fast. Use "diag" or "tied" when $D > 20$.

Reflection

EM is one of the handful of algorithms that genuinely earns the description "beautiful". It takes a problem that looks hopeless — fit a model with hidden variables — and reduces it to two steps, each of which is trivial, with a mathematical guarantee that the loop converges. Once you see the pattern you will start recognising it everywhere: hidden Markov models, latent Dirichlet allocation, the Baum-Welch algorithm, probabilistic matrix factorisation. They are all EM.
