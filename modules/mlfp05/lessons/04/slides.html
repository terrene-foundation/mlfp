Why attention replaced recurrence

  • LSTMs are sequential — to compute h5 you need h1…h4. Cannot parallelise across time.
  • Self-attention lets every token look at every other token in parallel.
  • Same asymptotic work, fundamentally different parallelism.

2017: Vaswani et al., "Attention is all you need". NLP has never been the same.

Query, Key, Value — database metaphor

  • Each token emits a Query (what am I looking for?)
  • Each token emits a Key (what do I contain?)
  • Each token emits a Value (what do I return if retrieved?)

Compare queries against keys → similarity scores → softmax → weights → weighted sum of values.

Scaled dot-product attention

\[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V \]
  1. QKᵀ → L×L similarity matrix
  2. Divide by √d_k
  3. Row-wise softmax
  4. Multiply by V

Why divide by √d_k?

Dot product of two random d_k-dim vectors has variance d_k, so typical magnitude √d_k.

Feed that into softmax → saturates to one-hot → gradients vanish → training stalls.

🔵 Dividing by √d_k brings scores back to unit variance. One character in code, load-bearing for training.

From scratch with einsum


def scaled_dot_product_attention(q, k, v, mask=None):
    d_k = q.size(-1)
    scores = torch.einsum("bld,bmd->blm", q, k) / math.sqrt(d_k)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))
    attn = F.softmax(scores, dim=-1)
    out = torch.einsum("blm,bmd->bld", attn, v)
    return out, attn

Two einsum calls, one softmax. That's the whole operation.

Multi-head attention

One head = one viewpoint. Multi-head = h parallel attention operations on smaller subspaces.

\[ \text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) W^O \]

With d_model = 512 and h = 8 heads, each operates in d_k = 64. Same total parameters, eight different relationships.

Positional encoding

Self-attention is permutation invariant. But "dog bites man" ≠ "man bites dog".

\[ \text{PE}(\text{pos}, 2i) = \sin\!\left(\frac{\text{pos}}{10000^{2i/d}}\right) \] \[ \text{PE}(\text{pos}, 2i+1) = \cos\!\left(\frac{\text{pos}}{10000^{2i/d}}\right) \]

Unique per-position fingerprint, deterministic, extrapolates beyond training lengths.

PositionalEncoding in PyTorch


class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(torch.arange(0, d_model, 2).float()
                             * (-math.log(10000.0) / d_model))
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        self.register_buffer("pe", pe)

    def forward(self, x):
        return x + self.pe[:x.size(1)]

The block that repeats N times

Stack N identical blocks. Input and output shapes are identical: (seq_len, d_model). PyTorch has nn.TransformerEncoderLayer built in.

TinyTransformer in PyTorch


class TinyTransformer(nn.Module):
    def __init__(self, n_tokens, d_model=64, n_heads=4, n_layers=2):
        super().__init__()
        self.embed = nn.Embedding(n_tokens, d_model)
        self.pos   = PositionalEncoding(d_model, max_len=50)
        layer      = nn.TransformerEncoderLayer(
            d_model=d_model, nhead=n_heads,
            dim_feedforward=128, batch_first=True, norm_first=True)
        self.encoder = nn.TransformerEncoder(layer, num_layers=n_layers)
        self.head    = nn.Linear(d_model, 2)

    def forward(self, tokens):
        x = self.pos(self.embed(tokens))
        x = self.encoder(x)
        return self.head(x.mean(dim=1))

Layer norm, not batch norm

Layer norm normalises across features per token. Batch norm normalises across the batch per feature.

Sequence lengths vary → batch statistics become noisy or meaningless. Layer norm is robust to that.

Always layer norm in Transformers.

Four traps

  • Forgetting positional encoding — model becomes a set classifier, stuck at 50%
  • Missing √d_k — softmax saturates, gradients vanish
  • Wrong mask type — attn_mask vs key_padding_mask are different
  • O(L²) memory — doubling L quadruples RAM; use FlashAttention or Longformer for very long contexts
