LoRA: Low-Rank Adaptation

Theory

Core Idea

Instead of updating the full weight matrix $W \in \mathbb{R}^{d \times d}$, decompose the update into two small matrices:

$$W' = W_0 + \Delta W = W_0 + BA$$
$B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times d}$, where $r \ll d$
  • $W_0$ is frozen (pretrained weights)
  • Only $A$ and $B$ are trained
  • $r$ is the rank — typically 4, 8, 16, or 32
Time: ~3 min. "This is the key insight: we do not need to modify every single weight in a large model. We can express the weight update as a low-rank matrix, which requires far fewer parameters to store and train."

Why Low Rank Works: The SVD Connection

Theory

When we fine-tune a model, the weight change $\Delta W$ has a surprising property:

$$\Delta W = U \Sigma V^T$$
SVD of the weight update shows most singular values are near zero
Connection to M6 (PCA): Just as PCA showed that data variance concentrates in a few principal components, SVD of $\Delta W$ shows that fine-tuning updates concentrate in a few directions. LoRA exploits this by directly parameterising the low-rank structure.
Time: ~2 min. "Remember PCA from Module 6? You learned that most of the information in high-dimensional data lives in a small subspace. The same principle applies to weight updates during fine-tuning. This is why LoRA works so well."

LoRA: Parameter Savings

Theory

The Maths

Full fine-tuning:

$$d^2 \text{ parameters}$$

For $d = 4096$: 16,777,216 params

LoRA (rank $r$):

$$2 \times d \times r \text{ parameters}$$

For $d = 4096, r = 16$: 131,072 params

128x fewer parameters! From 16.8M to 131K per layer. A 7B model goes from 7 billion trainable params to ~17 million.
Time: ~2 min. "The savings are dramatic. You can fine-tune a 7-billion parameter model with the same GPU memory that would barely handle forward passes during full fine-tuning."

LoRA Training Process

Foundations
1
Freeze $W_0$ — pretrained weights are locked
2
Initialise $A \sim \mathcal{N}(0, \sigma^2)$, $B = 0$ (so $\Delta W = 0$ at start)
3
Forward pass: $h = (W_0 + BA)x = W_0 x + BAx$
4
Backward pass: gradients flow only through $A$ and $B$
5
Update: $A \leftarrow A - \eta \nabla_A \mathcal{L}$, $B \leftarrow B - \eta \nabla_B \mathcal{L}$
Time: ~2 min. "Key detail: B is initialised to zero so the model starts exactly where the pretrained model left off. No initial disruption. Training then gradually moves the weights in the learned low-rank direction."

LoRA: Merging for Inference

Foundations
$$W_{\text{final}} = W_0 + BA$$
Merge once, deploy with zero additional inference cost

During Training

  • Keep $W_0$, $A$, $B$ separate
  • Compute $W_0 x + BAx$
  • Swap adapters without reloading base

At Deployment

  • Merge: $W_{\text{final}} = W_0 + BA$
  • Discard $A$ and $B$
  • Same architecture, same speed
Time: ~2 min. "This is the elegance of LoRA. At inference time, there is ZERO overhead. You merge the adapter into the base weights and deploy a standard model. No extra computation, no extra memory."

LoRA: Which Modules to Target

Theory

Attention Modules

  • q_proj — query projection
  • k_proj — key projection
  • v_proj — value projection
  • o_proj — output projection

MLP Modules

  • gate_proj — gating layer
  • up_proj — expansion layer
  • down_proj — contraction layer
Research finding: q_proj and v_proj give the best parameter-efficiency trade-off. Adding MLP modules increases capacity but doubles parameter count. Target all modules only when r is very small (4-8).
Time: ~2 min. "The original LoRA paper found that targeting query and value projections gives the best results per parameter. This is because attention matrices control what information flows between tokens — the most impactful part of the transformer."

NF4: Why It Works

Theory

Standard INT4

  • Uniform spacing between levels
  • Wastes resolution on rare outlier values
  • Not optimal for any specific distribution

NormalFloat4 (NF4)

  • Levels spaced by quantiles of $\mathcal{N}(0,1)$
  • More levels near zero where density is highest
  • Provably optimal for normal distributions
Intuition: Most weights cluster near zero. NF4 puts more representation levels where the weights actually are, like how a histogram with more bins near the peak captures more detail.
Time: ~2 min. "Think of it as adaptive resolution. Standard INT4 gives equal precision to all values, but neural network weights are concentrated around zero. NF4 puts more precision where it matters most."

Double Quantization: Details

Theory

Standard quantization groups weights into blocks of 64, each with a FP32 scaling constant. That is 32/64 = 0.5 bits/param overhead.

$$\text{Double Quant: } c^{\text{FP32}} \xrightarrow{\text{quantize}} c^{\text{FP8}}$$
Quantize the scaling constants: 0.5 bits/param → 0.127 bits/param overhead
Savings: For a 65B model, double quantization saves ~3 GB of memory. Combined with NF4 and paged optimizers, this is what makes the 48GB target achievable.
Time: ~2 min. "Double quantization is recursive quantization. You quantize the weights, then you quantize the numbers used to quantize the weights. It sounds absurd but it saves significant memory at negligible quality cost."

DoRA: Weight-Decomposed Low-Rank

Advanced
$$W' = m \cdot \frac{W_0 + BA}{\|W_0 + BA\|_c}$$
$m$ = learnable magnitude vector, direction from LoRA update, $\|\cdot\|_c$ = column-wise norm
Why it helps: Full fine-tuning changes both magnitude and direction of weight vectors. Standard LoRA conflates these. DoRA separates them, allowing independent optimisation of "how much" vs "which direction."
Time: ~2 min. "DoRA consistently outperforms LoRA across benchmarks with minimal additional parameters. The magnitude vector m adds only d parameters per layer."

Prefix Tuning and Prompt Tuning

Advanced

Prefix Tuning

  • Prepend learnable vectors to keys/values at every layer
  • ~0.1% parameters
  • Strong for NLG (summarisation, translation)
  • Harder to combine multiple tasks

Prompt Tuning

  • Prepend soft tokens to input only
  • Even fewer parameters
  • Scales well with model size
  • Best for classification tasks
(IA)³: Infused Adapter by Inhibiting and Amplifying Inner Activations. Learns three vectors that rescale key, value, and FFN activations. Fewer than 10K parameters total.
Time: ~2 min. "Prefix tuning and prompt tuning predate LoRA. They work by adding learnable context to the model's input or attention layers. (IA)3 is the extreme end — just three small vectors that rescale existing activations."

Reward Model: Bradley-Terry

Theory

How do we model preferences?

Given prompt $x$ and two responses $y_1, y_2$, a human labels which is better. The reward model learns:

$$P(y_1 \succ y_2 \mid x) = \sigma\big(r(x, y_1) - r(x, y_2)\big)$$
$\sigma$ = sigmoid, $r(x, y)$ = scalar reward for response $y$ given prompt $x$
Bradley-Terry model: The probability of preferring $y_1$ depends only on the difference in rewards. This makes the reward invariant to constant shifts.
Time: ~3 min. "The Bradley-Terry model is from sports ranking — the same maths used to rank chess players by Elo. Applied to LLMs, it tells us the probability that a human would prefer one response over another, based on a learned reward function."

Reward Model Training

Theory

Training loss for the reward model:

$$\mathcal{L}_{\text{RM}} = -\mathbb{E}_{(x, y_w, y_l)}\left[\log \sigma\big(r(x, y_w) - r(x, y_l)\big)\right]$$
$y_w$ = preferred (winning) response, $y_l$ = rejected (losing) response
  • Collect thousands of $(x, y_w, y_l)$ preference pairs
  • Train reward model $r_\phi$ (often same architecture as LLM, minus generation head)
  • Output: scalar reward per (prompt, response) pair
Time: ~2 min. "The reward model is essentially a classifier trained on human preferences. For each prompt, it learns to assign higher reward scores to responses that humans prefer."

RLHF Objective: Reward + KL Constraint

Theory

The RL step maximises reward while staying close to the SFT model:

$$\max_\pi \; \mathbb{E}_{x \sim \mathcal{D}, \, y \sim \pi(\cdot|x)}\big[r(x, y)\big] - \beta \cdot \text{KL}\big(\pi \| \pi_{\text{ref}}\big)$$
$\pi$ = policy (LLM), $\pi_{\text{ref}}$ = reference model (SFT), $\beta$ = KL penalty weight
Why the KL term? Without it, the model would "hack" the reward by finding degenerate outputs that score high but are meaningless. The KL penalty keeps it close to the SFT model.
Time: ~3 min. "This objective has a beautiful tension: maximise reward (be helpful) while not straying too far from the reference (stay coherent). The beta parameter controls this trade-off."

DPO: Direct Preference Optimisation

Theory

The Big Idea

DPO shows that you can skip the reward model and RL entirely by deriving a closed-form loss from the RLHF objective.

Pre-train
→
SFT
→
DPO
Direct Loss

No reward model. No PPO. Just a supervised loss on preference pairs.

Time: ~2 min. "DPO is one of the most elegant results in modern ML. It reduces the four-stage RLHF pipeline to two stages by proving that the optimal policy has a closed-form relationship to the reward function."

DPO Derivation — Step 1: The RLHF Objective

Theory
Start with the RLHF objective: $$\max_\pi \; \mathbb{E}_{x, y \sim \pi}\big[r(x, y)\big] - \beta \cdot \text{KL}\big(\pi(\cdot|x) \| \pi_{\text{ref}}(\cdot|x)\big)$$

Expand the KL divergence:

$$\max_\pi \; \mathbb{E}_{y \sim \pi}\left[r(x, y) - \beta \log \frac{\pi(y|x)}{\pi_{\text{ref}}(y|x)}\right]$$
Time: ~2 min. "We start with exactly the objective we saw earlier. The KL divergence expands into a log ratio between the policy and reference. This is the starting point for the derivation."

DPO Derivation — Step 2: Optimal Policy

Theory
The optimal policy has a closed-form solution: $$\pi^*(y|x) = \frac{1}{Z(x)} \cdot \pi_{\text{ref}}(y|x) \cdot \exp\!\left(\frac{r(x,y)}{\beta}\right)$$

where $Z(x) = \sum_y \pi_{\text{ref}}(y|x) \cdot \exp(r(x,y)/\beta)$ is the partition function.

Intuition: The optimal policy is the reference model reweighted by the exponentiated reward. High-reward responses get more probability mass. $\beta$ controls how aggressively.
Time: ~3 min. "This is a standard result from KL-regularised RL. The optimal policy has this Boltzmann-like form — exactly what you see in physics with energy minimisation."

DPO Derivation — Step 3: Solve for Reward

Theory
Rearrange to express reward in terms of the policy: $$r(x, y) = \beta \log \frac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$$

Key insight: we now have the reward as a function of the policy ratio, plus a term that depends only on $x$.

Time: ~2 min. "This is the critical rearrangement. We have expressed the reward function in terms of the ratio between the optimal policy and the reference model. The partition function Z(x) depends only on the prompt, not on the specific response."

DPO Derivation — Step 4: Into Bradley-Terry

Theory
Substitute reward into the preference model: $$r(x, y_w) - r(x, y_l) = \beta \left(\log \frac{\pi^*(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log \frac{\pi^*(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)$$
The $Z(x)$ terms cancel! Since both rewards share the same prompt $x$, the partition function subtracts out completely.
Time: ~2 min. "This is the magic moment. When we take the difference of rewards for the winning and losing response, the intractable partition function Z(x) cancels completely. We never need to compute it."

DPO Derivation — Step 5: The DPO Loss

Theory
Final DPO loss function:
$$\mathcal{L}_{\text{DPO}}(\pi_\theta) = -\mathbb{E}_{(x, y_w, y_l)}\left[\log \sigma\!\left(\beta \left(\log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right)\right]$$
  • No reward model — the policy IS the reward
  • No RL loop — standard supervised training
  • Two forward passes per example (one per response)
Time: ~3 min. "This is the DPO loss. Increase the log probability of the preferred response relative to the reference, and decrease the log probability of the rejected response relative to the reference."

Default: $\beta = 0.1$ is a safe starting point for most applications.

DPO: Implicit Reward

Theory

Even though DPO has no explicit reward model, we can extract the implicit reward:

$$\hat{r}(x, y) = \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$$
The log-probability ratio IS the reward
Diagnostic tool: Track the implicit reward during training. If it diverges wildly, the model is overfitting to preference artifacts rather than learning genuine quality preferences.
Time: ~2 min. "This is a powerful insight for debugging. You can compute the implicit reward for any response and check whether it correlates with your notion of quality."

GRPO: Group Relative Policy Optimisation

Theory

The DeepSeek-R1 Method

Key Innovation

  • No reward model at all
  • Generate $G$ responses per prompt
  • Rank them by verifiable reward
  • Use relative ranking as the learning signal

Advantage Estimation

$$\hat{A}_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}$$

Group-normalised advantage — no separate value network needed

Time: ~3 min. "GRPO was used to train DeepSeek-R1, one of the strongest reasoning models. Instead of training a reward model, generate multiple responses and let their relative quality be the signal."

GRPO Loss Function

Theory
$$\mathcal{L}_{\text{GRPO}} = -\frac{1}{G}\sum_{i=1}^{G}\left[\min\!\left(\frac{\pi_\theta(y_i|x)}{\pi_{\text{old}}(y_i|x)}\hat{A}_i, \;\text{clip}\!\left(\frac{\pi_\theta}{\pi_{\text{old}}}, 1{-}\epsilon, 1{+}\epsilon\right)\hat{A}_i\right) - \beta \cdot \text{KL}_i\right]$$
PPO-style clipping with group-relative advantages and per-example KL penalty
Connection to PPO: GRPO uses the same clipped objective as PPO (Section C) but replaces the critic network with group statistics. This removes an entire model from the training pipeline.
Time: ~2 min. "GRPO is essentially PPO without a critic. Instead of learning a value function to estimate advantages, it uses the statistics of the group to normalise rewards."

GRPO: Verifiable Rewards

Foundations

Verifiable Tasks

  • Mathematics: check final answer
  • Code: run tests
  • Logic: verify proofs
  • Retrieval: check citations

Non-Verifiable Tasks

  • Creative writing: subjective quality
  • Summarisation: partial verifiability
  • Conversation: user satisfaction
  • Use DPO or RLHF instead
GRPO shines when: you can automatically verify correctness. No human annotation needed. No reward model needed. Just run the test and score.
Time: ~2 min. "The key insight is that GRPO needs verifiable rewards. For math, you can check the answer. For code, you can run tests. For tasks where quality is subjective, DPO or RLHF are better choices."

ORPO: Odds Ratio Preference Optimisation

Theory

SFT + Preference in One Step

$$\mathcal{L}_{\text{ORPO}} = \mathcal{L}_{\text{SFT}}(y_w) - \lambda \cdot \log \sigma\!\left(\log \frac{\text{odds}(y_w|x)}{\text{odds}(y_l|x)}\right)$$
$\text{odds}(y|x) = \frac{P(y|x)}{1-P(y|x)}$, combines SFT and alignment in a single loss
Why ORPO? DPO requires a separately-trained SFT model as reference. ORPO does both at once — reducing the pipeline to a single training stage.
Time: ~2 min. "ORPO merges SFT and preference alignment into one loss. The first term teaches the model to generate good responses. The second term teaches it to prefer good responses over bad ones."

SimPO: Simple Preference Optimisation

Theory

Average Log Probability as Reward

$$r_{\text{SimPO}}(x, y) = \frac{1}{|y|}\log \pi_\theta(y|x)$$
Length-normalised log probability — no reference model needed

Advantages

  • No reference model in memory
  • Length-normalised: no verbosity bias
  • Simpler than DPO

SimPO Loss

$$\mathcal{L} = -\log\sigma\!\left(\frac{\beta}{|y_w|}\log\pi(y_w|x) - \frac{\beta}{|y_l|}\log\pi(y_l|x) - \gamma\right)$$

$\gamma$ = target margin between preferred and rejected

Time: ~2 min. "SimPO removes the reference model entirely. The average log probability of the model's own output serves as the implicit reward. Length normalisation prevents the model from learning that longer responses are always better."

KTO: Kahneman-Tversky Optimisation

Theory

When You Only Have Good/Bad Labels

DPO requires paired preferences $(y_w, y_l)$ for the same prompt. KTO works with unpaired labels:

Good examples: $(x, y)$ where $y$ is desirable
Bad examples: $(x, y)$ where $y$ is undesirable
Prospect Theory connection: KTO uses Kahneman-Tversky's insight that losses loom larger than gains. The loss function is asymmetric — penalising bad outputs more than rewarding good ones.
Time: ~2 min. "KTO is practical when you cannot pair responses to the same prompt. If you only have thumbs-up/thumbs-down labels from users, KTO is your method."

Markov Decision Process (MDP)

Theory

The Mathematical Framework

An MDP is defined by the tuple $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$:

ComponentSymbolMeaning
States$\mathcal{S}$All possible situations the agent can be in
Actions$\mathcal{A}$All possible decisions the agent can make
Transitions$P(s'|s,a)$Probability of reaching $s'$ from $(s,a)$
Rewards$R(s,a)$Immediate signal after taking action $a$ in state $s$
Discount$\gamma \in [0,1)$How much future rewards matter vs immediate
Time: ~3 min. "The MDP is to reinforcement learning what the dataset is to supervised learning — it defines the problem. Every RL algorithm operates on an MDP."

The Markov Property

Theory
$$P(S_{t+1} | S_t, A_t) = P(S_{t+1} | S_t, A_t, S_{t-1}, A_{t-1}, \ldots)$$
The future depends only on the current state, not the history
Intuition: The state encodes everything the agent needs to decide. If you know the current position on a chess board, you do not need to know the sequence of moves that got you there. The Markov property is what makes RL tractable.
Time: ~2 min. "The Markov property is what makes the MDP work. The current state contains all relevant information for deciding the next action. If your state representation does not satisfy this property, your RL algorithm will struggle."

Policy, Value, and Q Functions

Theory
$$\pi(a|s): \text{probability of taking action } a \text{ in state } s$$

State Value $V^\pi(s)$

$$V^\pi(s) = \mathbb{E}_\pi\!\left[\sum_{t=0}^{\infty} \gamma^t R_{t+1} \;\middle|\; S_0 = s\right]$$

"How good is it to BE in state $s$?"

Action Value $Q^\pi(s,a)$

$$Q^\pi(s,a) = \mathbb{E}_\pi\!\left[\sum_{t=0}^{\infty} \gamma^t R_{t+1} \;\middle|\; S_0{=}s, A_0{=}a\right]$$

"How good is it to DO action $a$ in state $s$?"

Time: ~3 min. "The policy is the agent's strategy. The value function answers 'how good is this state?' and the Q function answers 'how good is this action in this state?'"

The Discount Factor $\gamma$

Theory
$\gamma$BehaviourUse Case
0Only care about immediate rewardGreedy, short-horizon tasks
0.9Moderate planning horizonMost practical RL tasks
0.99Long planning horizonGames, complex control
1.0No discounting (if episode ends)Episodic tasks only
Mathematical role: $\gamma < 1$ ensures the infinite sum $\sum \gamma^t R_t$ converges. It also encodes the agent's "patience" — how willing it is to sacrifice now for a bigger payoff later.
Time: ~2 min. "Gamma is the agent's time preference. A gamma of 0.99 means a reward 100 steps from now is worth about 37% of its face value today. A gamma of 0.9 means a reward 10 steps away is worth about 35%."

Bellman Expectation Equation

Theory

Step-by-Step Derivation

Start: $V^\pi(s) = \mathbb{E}_\pi[\sum_{t=0}^{\infty}\gamma^t R_{t+1} | S_0 = s]$
Split first step: $V^\pi(s) = \mathbb{E}_\pi[R_1 + \gamma \sum_{t=1}^{\infty}\gamma^{t-1} R_{t+1} | S_0 = s]$
Recognise recursion: The inner sum IS $V^\pi(S_1)$
$$V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)\big[R(s,a) + \gamma V^\pi(s')\big]$$
Value = expected immediate reward + discounted future value
Time: ~3 min. "The Bellman equation is the foundation of all RL algorithms. It says that the value of any state equals the immediate reward plus the discounted value of the next state."

Bellman Optimality Equation

Theory

The optimal value function satisfies:

$$V^*(s) = \max_a \sum_{s'} P(s'|s,a)\big[R(s,a) + \gamma V^*(s')\big]$$
The optimal policy always takes the best action
$$Q^*(s,a) = R(s,a) + \gamma \sum_{s'} P(s'|s,a) \max_{a'} Q^*(s',a')$$
Optimal Q = immediate reward + best future value
Key difference: Expectation equation uses $\sum_a \pi(a|s)$ (weighted by policy). Optimality equation uses $\max_a$ (always greedy).
Time: ~3 min. "The Bellman optimality equation replaces the expectation over actions with a maximum. If we can solve this equation, we have the optimal policy."

Policy Evaluation & Improvement

Theory

Policy Evaluation

Given $\pi$, compute $V^\pi$:

$$V_{k+1}(s) = \sum_a \pi(a|s)\sum_{s'}P[R + \gamma V_k(s')]$$

Iterate until convergence

Policy Improvement

Given $V^\pi$, improve $\pi$:

$$\pi'(s) = \arg\max_a \sum_{s'}P[R + \gamma V^\pi(s')]$$

Be greedy with respect to value

Policy Iteration: Alternate evaluation and improvement until $\pi$ stops changing. Guaranteed to converge to the optimal policy.
Time: ~3 min. "Policy iteration has two steps: evaluate how good the current policy is, then improve the policy by being greedy. This alternation provably converges to the optimal policy."

Value Iteration

Theory

Combine evaluation and improvement into a single update:

$$V_{k+1}(s) = \max_a \sum_{s'} P(s'|s,a)\big[R(s,a) + \gamma V_k(s')\big]$$
Direct application of the Bellman optimality equation as an update rule
Limitation: Policy evaluation, policy iteration, and value iteration all require knowledge of $P(s'|s,a)$. What if we do not know the transition model?
Time: ~2 min. "Value iteration is simpler than policy iteration. But both require the full transition model. In real problems, we rarely know the exact dynamics. This motivates model-free methods."

Monte Carlo Methods

Theory

Learning from Complete Episodes

1
Run episode to completion using policy $\pi$
2
For each state visited, compute return $G_t = \sum_{k=0}^{T-t-1}\gamma^k R_{t+k+1}$
3
Update: $V(s) \leftarrow V(s) + \alpha [G_t - V(s)]$

First-Visit MC

Average $G_t$ only for the first time $s$ is visited in each episode

Every-Visit MC

Average $G_t$ for every time $s$ is visited, including repeats

Time: ~3 min. "Monte Carlo methods learn from experience — no model needed. Run an episode, see what happened, update your estimates. The downside: you must wait until the episode ends."

TD(0): Temporal Difference Learning

Theory

The Core Update

$$V(s) \leftarrow V(s) + \alpha\big[\underbrace{r + \gamma V(s')}_{\text{TD target}} - V(s)\big]$$
$\delta = r + \gamma V(s') - V(s)$ is the TD error

Key Insight: Bootstrapping

  • MC waits for the full episode return $G_t$
  • TD uses the current estimate $V(s')$
  • Learn at every step
TD error $\delta$: the surprise — how much better (or worse) the actual experience was compared to expectation.
Time: ~3 min. "TD learning is one of the most important ideas in RL. Instead of waiting for the episode to end, TD updates immediately using the current estimate. This is called bootstrapping."

MC vs TD: The Bias-Variance Trade-off

Theory

VarianceHigh (full episode)Low (one step)
Needs episodes to end?YesNo

In practice: TD methods dominate because most real-world tasks are continuous and TD's lower variance leads to faster convergence.
Time: ~2 min. "Monte Carlo gives truth with high variance. TD gives biased estimates with low variance and instant feedback. TD usually wins."

SARSA: On-Policy TD Control

Theory
$$Q(s,a) \leftarrow Q(s,a) + \alpha\big[r + \gamma Q(s',a') - Q(s,a)\big]$$
$s,a,r,s',a'$ — State, Action, Reward, State', Action' (hence SARSA)
On-policy: SARSA evaluates and improves the SAME policy it uses to generate experience. The action $a'$ is chosen by the current policy (e.g., $\epsilon$-greedy). SARSA is "cautious" — it accounts for its own exploration noise.
Time: ~3 min. "SARSA learns Q-values by following its own policy. Because it uses the actual next action a-prime, it learns a policy that is safe under exploration."

Q-Learning: Off-Policy TD Control

Theory
$$Q(s,a) \leftarrow Q(s,a) + \alpha\big[r + \gamma \max_{a'} Q(s',a') - Q(s,a)\big]$$
Uses $\max$ instead of the actual next action — learns optimal policy regardless of behaviour

SARSA (on-policy)

Uses $Q(s', a')$ where $a'$ is actually taken

Learns the value of what it does

Q-Learning (off-policy)

Uses $\max_{a'} Q(s', a')$ regardless

Learns the value of the best it could do

Time: ~3 min. "Q-learning is the most famous RL algorithm. The key difference from SARSA is the max: Q-learning always imagines taking the best possible next action."

ε-Greedy Exploration

Foundations
$$a = \begin{cases} \arg\max_a Q(s,a) & \text{with probability } 1-\epsilon \\ \text{random action} & \text{with probability } \epsilon \end{cases}$$
Explore with probability $\epsilon$, exploit with probability $1-\epsilon$
Exploration-exploitation dilemma: Common schedule: start $\epsilon = 1.0$, decay to $0.01$ over training.
Time: ~2 min. "Epsilon-greedy is the simplest solution to exploration: with probability epsilon, take a random action. With probability 1-epsilon, take the best known action."

Deep Q-Networks (DQN)

Theory

Scaling Q-Learning with Neural Networks

Tabular Q-learning cannot handle large state spaces. DQN uses a neural network $Q_\theta(s,a)$.

Two Key Innovations

  • Experience Replay: store $(s,a,r,s')$, sample random mini-batches
  • Target Network: separate $Q_{\theta^-}$ updated slowly
$$\mathcal{L} = \mathbb{E}\!\left[\big(r + \gamma \max_{a'} Q_{\theta^-}(s',a') - Q_\theta(s,a)\big)^2\right]$$
Time: ~3 min. "DQN combined deep learning with RL. Experience replay breaks correlations in sequential data, and the target network prevents the moving-target problem."

Why Experience Replay?

Theory
Without Replay
  • Sequential data is correlated
  • Recent experience dominates
  • Catastrophic forgetting
With Replay
  • Random sampling breaks correlations
  • Past experiences revisited uniformly
  • Data-efficient (reuse experiences)
Time: ~2 min. "Experience replay turns RL into something closer to supervised learning by breaking the temporal correlation in the training data."

Policy Gradient: REINFORCE

Theory

Learning the Policy Directly

$$\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\!\left[\nabla_\theta \log \pi_\theta(a|s) \cdot G_t\right]$$
Policy gradient theorem: gradient of expected return w.r.t. policy parameters
Intuition: If action $a$ led to high return $G_t$, increase its log-probability. If it led to low return, decrease it.
Time: ~3 min. "REINFORCE is the foundational policy gradient algorithm. You can compute the gradient of expected reward even though the expectation is over trajectories."

REINFORCE: High Variance and Baselines

Theory
$$\nabla_\theta J(\theta) = \mathbb{E}\!\left[\nabla_\theta \log \pi_\theta(a|s) \cdot (G_t - b(s))\right]$$
$b(s)$ is a baseline (typically $V(s)$) — reduces variance without adding bias
$G_t - V(s) = A(s,a)$ — this is the advantage function. It tells us how much better action $a$ was compared to the average action in state $s$.
Time: ~2 min. "Without a baseline, every action in a good episode gets reinforced. The advantage function fixes this by asking: was this action better or worse than average?"

Actor-Critic Architecture

Theory

Actor $\pi_\theta(a|s)$

  • Outputs the policy
  • Updated by policy gradient
  • "Which action to take?"

Critic $V_\phi(s)$

  • Estimates state value
  • Updated by TD error
  • "How good is this state?"
$$A(s,a) = r + \gamma V_\phi(s') - V_\phi(s)$$
Advantage = TD error using the critic's value estimates
Time: ~3 min. "Actor-critic combines the best of both worlds: the actor learns a policy directly, and the critic provides a low-variance baseline."

A2C and A3C

Theory

A2C (Advantage Actor-Critic)

  • Synchronous: collect from all workers, then update
  • Simpler implementation
  • GPU-friendly batched updates

A3C (Asynchronous)

  • Multiple workers update asynchronously
  • Natural exploration through diverse workers
  • Historically important, less used now
In practice: A2C with PPO's clipped objective (next slide) dominates. A3C is historically important but largely superseded.
Time: ~2 min. "A2C and A3C extend actor-critic to parallel environments. A2C collects batches synchronously, A3C updates asynchronously. PPO built on these ideas and improved stability."

PPO: Proximal Policy Optimisation

Theory

The Most Important RL Algorithm

PPO prevents the policy from changing too much in a single update:

$$r_t(\theta) = \frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\text{old}}}(a_t|s_t)}$$
Probability ratio: how much has the policy changed?
$$L^{\text{CLIP}} = \mathbb{E}_t\!\left[\min\!\Big(r_t(\theta) \hat{A}_t, \; \text{clip}(r_t(\theta), 1{-}\epsilon, 1{+}\epsilon)\hat{A}_t\Big)\right]$$
$\epsilon$ typically $0.1$-$0.2$. Clips updates that change the policy too aggressively.
Time: ~3 min. "PPO is used everywhere — ChatGPT, game-playing agents, robotics. The key idea: if the policy changes too much, clip the objective."

PPO: Why Clipping Works

Theory

When $\hat{A}_t > 0$ (good action)

  • We want to increase $\pi_\theta(a|s)$
  • $r_t$ increases as policy increases
  • Clip at $1+\epsilon$: prevent over-committing

When $\hat{A}_t < 0$ (bad action)

  • We want to decrease $\pi_\theta(a|s)$
  • $r_t$ decreases as policy decreases
  • Clip at $1-\epsilon$: prevent over-avoiding
The min operation: takes the more pessimistic (conservative) estimate, ensuring the policy update is never dangerously large.
Time: ~2 min. "The genius of PPO is the combination of the ratio and the clip. The min ensures we take the conservative bound in both directions."

GAE: Generalised Advantage Estimation

Theory
$$\hat{A}_t^{\text{GAE}} = \sum_{l=0}^{\infty}(\gamma\lambda)^l \delta_{t+l}$$
$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$, $\lambda$ controls bias-variance
$\lambda$Equivalent ToBiasVariance
0TD(0) advantageHighLow
1MC advantageLowHigh
0.95-0.99Practical sweet spotLowMedium
Time: ~2 min. "GAE interpolates between TD and Monte Carlo advantage estimation using lambda. In practice, lambda around 0.95 gives the best balance."

PPO Training Loop

Foundations
1
Collect: Run policy $\pi_{\theta_{\text{old}}}$ in environment, gather trajectories
2
Compute advantages: Use GAE with collected rewards and critic values
3
Optimise: Multiple epochs of SGD on the clipped objective
4
Update: $\theta_{\text{old}} \leftarrow \theta$
5
Repeat from step 1
Key detail: Step 3 does $K$ epochs on the same batch. The clipping ensures multiple passes do not move the policy too far.
Time: ~2 min. "PPO is on-policy but squeezes maximum value from each batch by doing multiple optimisation epochs."

SAC: Soft Actor-Critic

Theory

Maximum Entropy RL

$$J(\pi) = \sum_t \mathbb{E}\!\left[r(s_t, a_t) + \alpha \mathcal{H}(\pi(\cdot|s_t))\right]$$
Maximise reward AND entropy. $\alpha$ = temperature parameter.

Why Entropy?

  • Encourages exploration
  • Prevents premature convergence
  • Robust to perturbations

Architecture

  • Off-policy (uses replay buffer)
  • Twin Q-networks
  • Automatic temperature tuning
  • Continuous action spaces
Time: ~3 min. "SAC adds entropy to the reward, encouraging the policy to be as random as possible while still achieving high reward. SAC is the go-to for continuous control."

Decision Transformer

Advanced

RL as Sequence Modelling

$$\tau = (\hat{R}_1, s_1, a_1, \hat{R}_2, s_2, a_2, \ldots, \hat{R}_T, s_T, a_T)$$
$\hat{R}_t$ = return-to-go (desired cumulative reward from time $t$)
Key insight: Condition on the desired return. At test time, specify a high return-to-go and the transformer generates actions to achieve it. No Bellman equation, no TD error, no policy gradient.
Time: ~2 min. "Decision Transformer reimagines RL through the lens of sequence modelling. Instead of learning 'what is the best action?', it learns 'what actions lead to this reward?'."

Linear Merge & SLERP

Theory

Linear (Weighted Average)

$$W_{\text{merged}} = \alpha W_A + (1-\alpha) W_B$$

Simple, fast, works well for similar models

SLERP (Spherical Linear)

$$W = \frac{\sin((1{-}t)\Omega)}{\sin\Omega}W_A + \frac{\sin(t\Omega)}{\sin\Omega}W_B$$

Preserves weight norms, better for diverse models

Time: ~2 min. "Linear merge is just averaging. SLERP interpolates on the unit sphere, preserving magnitude. SLERP tends to produce better results when models have diverged significantly."

TIES: Trim, Elect, Merge

Theory
1
Trim: Set small-magnitude changes to zero (top-k by magnitude)
2
Elect sign: For conflicting parameters, majority vote on sign
3
Merge: Average the remaining agreeing parameters
Why TIES? Naive averaging cancels out important task-specific changes when models disagree on direction. TIES resolves conflicts before merging.
Time: ~2 min. "TIES addresses interference in model merging. When two adapters disagree on direction, naive averaging cancels both contributions. TIES resolves this by voting on direction first."

DARE: Drop And REscale

Theory
$$\tilde{\delta}_t = \mathbf{m} \odot \delta_t \cdot \frac{1}{1-p}$$
$\delta_t$ = task vector, $\mathbf{m}$ = random binary mask with drop rate $p$
Intuition: Most fine-tuned weight changes are redundant. Randomly dropping ~90% and rescaling the rest preserves performance while reducing interference during merging. Similar to dropout but for weight deltas.
Time: ~2 min. "DARE sparsifies the fine-tuning delta, making it possible to merge many more adapters without interference."

Model Soups: Checkpoint Averaging

Theory
