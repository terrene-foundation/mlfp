ETL: the universal data pattern

→ "The first new tool: null handling."
Slide 4~3 minNull handling

Null handling: three tools

Walk the table. Then show the code. "is_null() tells you where the holes are. fill_null() fills them. drop_nulls() removes rows with holes. The order matters: inspect first, then decide whether to fill or drop."

Emphasise the rule: "Never drop nulls without checking how many there are. If 30% of rows have nulls, you are throwing away a third of your data."

For beginners: "Think of nulls as blank cells in a spreadsheet. You can fill them with a reasonable guess, or you can delete the whole row. Filling is usually better."
→ "Which filling strategy?"
Slide 5~3 minImputation strategies

Which imputation strategy?

Walk the three boxes. Forward-fill for time-series data (taxi fares over time). Median for cross-sectional data (robust to outliers). Group-aware for the best accuracy (median fare per zone).

The bottom box: dropping is the last resort.

→ "Where does data come from? Not always a CSV."
Slide 6~3 minREST APIs

Extracting data from APIs

"Many datasets are live services. Singapore's OneMap API gives you geolocation for any address. The pattern: send a GET request with query parameters, get JSON back."

Run the code live if you have network access. Show the JSON response. Point out: "The API returns a dictionary with a 'results' key containing a list of dictionaries. Each dictionary is one search result."

If beginners are confused by APIs: "Think of it as typing a URL in your browser and getting data back instead of a webpage. The URL is the API endpoint. The parameters are the search terms."
→ "How to turn JSON into a DataFrame."

Slide 8~3 minPreprocessingPipeline

PreprocessingPipeline: automated cleaning

"Just like DataExplorer automates profiling, PreprocessingPipeline automates cleaning. Create it, configure it, call fit_transform(). One method does everything: detects types, imputes nulls, encodes categoricals, scales numerics."

Point out the shape change: "5 columns in, 22 columns out. That is because one-hot encoding expands the pickup_zone column into multiple binary columns."

→ "A critical distinction."
Slide 9~3 minfit vs transform

fit_transform vs transform

This is one of the most important concepts in the entire course. Walk the diagram slowly. "fit_transform on training data: the pipeline learns the medians, the category sets, the scaling parameters. Then on test data, you call transform only — applying the same parameters. Never fit on test data."

"Why? Because if the pipeline learns from test data, the test results are no longer a fair evaluation. This is called data leakage and we will come back to it in Module 3."

For experts: "This is the same reason you never standardise before splitting. The test set must be truly unseen."

"This is the simplest form of software engineering. In Module 3 you will use WorkflowBuilder to do this with automatic error handling and parallel execution."
