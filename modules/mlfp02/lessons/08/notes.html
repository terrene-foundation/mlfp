Speaker Notes — Lesson 2.8 Capstone — Statistical Analysis Project

Total time: ~60 minutes  ·  16 slides  ·  ~3.5 min per slide average

This is the capstone. The new content (FeatureEngineer, FeatureStore) is light — most of the lesson is applying skills from 2.1-2.7 to a real project. Let learners choose their project option early (Slide 6) and then walk the wine quality example as the guided demo. The exercise time is the heart of the lesson — give at least 25 minutes for independent work.

Slide 1~2 minTitle

Capstone — Statistical Analysis Project

"This is it — everything from Module 2 in one project. Choose a dataset, run the full pipeline, and present your findings."

→ "The pipeline."
Slide 2~2 minPipeline

The statistical analysis pipeline

Walk the six stages. Point out which lesson each stage comes from.

→ "Two new engines."
Slides 3-5~8 minFeatureEngineer + FeatureStore

FeatureEngineer and FeatureStore

Slide 3: Walk the FeatureEngineer code. "Domain knowledge decides which features to create. The engine handles the implementation. Temporal features for time-series, interactions for non-linear relationships."

Slide 4: Walk the FeatureStore code. Emphasise point-in-time correctness. "When you ask for features as of January, you get January's data — not March's. This prevents data leakage."

Slide 5: Walk the lifecycle diagram. "Create features, store them, retrieve them at the right point in time, train. This is the pattern every ML team uses."

For beginners: "Think of FeatureStore as a library catalogue. You check out the book (features) that existed on the date you need — not tomorrow's edition."
→ "Choose your project."
Slide 6~3 minProject options

Three project options

Let learners choose. Option A (wine) is the most straightforward — good for beginners. Option B (economic indicators) is the most technically interesting — good for experts. Option C (experiment design) is the most creative — good for learners who enjoyed 2.4 and 2.7.

→ "The guided example uses wine quality."
Slides 7-12~15 minGuided example

Wine quality pipeline (6 slides)

Run the wine quality example live, step by step. For each slide:

  • Slide 7 (Load): Show shape and alerts.
  • Slide 8 (EDA): Compute correlations with quality. Identify top predictors: alcohol, volatile acidity, sulphates.
  • Slide 9 (Engineer): Create the alcohol-acidity interaction. Store features.
  • Slide 10 (Model): Fit regression. Report $R^2$.
  • Slide 11 (Test): Show significant coefficients with effect sizes. Emphasise: "p-values without effect sizes are useless."
  • Slide 12 (Report): Walk the report structure. A non-technical reader should understand every sentence.
For experts: "After the guided example, try adding polynomial terms, interaction selection via LASSO (preview of M3), or a bootstrap confidence interval on the key coefficient."
→ "Traps."
Slide 13~2 minMistakes

Three mistakes

Walk each briefly. Mistake 3 (p-values without effect sizes) is the most important for the report.

→ "Module 2 complete."
Slide 14~3 minReflection

Module 2 complete

Walk the lesson list. Bridge to Module 3: "You can now reason about uncertainty, test hypotheses, and build regression models. Module 3 takes this foundation and adds the ML engineering layer: regularisation, the complete model zoo, gradient boosting, SHAP, fairness, and production deployment. Same pipeline; much more powerful models."

→ End of Module 2. Celebrate — this was hard.
