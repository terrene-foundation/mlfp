Lesson 5.4 Transformers and Self-Attention

"Attention is all you need" — the 2017 paper by Vaswani et al. replaced recurrence with pure attention and kicked off the modern era of deep learning. In this lesson you derive scaled dot-product attention from scratch with torch.einsum, understand why we divide by $\sqrt{d_k}$, implement multi-head attention, add sinusoidal positional encoding, and assemble a working Transformer encoder for sequence classification.

The Big Shift

An LSTM processes a sequence one step at a time: to compute $h_5$ you must first compute $h_1, h_2, h_3, h_4$. The computation is sequential — you cannot parallelise it across time. For long sequences on modern GPUs this is a tremendous waste of compute.

The Transformer asks: what if every token could look at every other token directly, in parallel, using an attention mechanism? No hidden state passed forward; no sequential recurrence; all positions computed at once. The answer is the self-attention layer, and it dominates modern NLP (BERT, GPT, T5), speech (Whisper), vision (ViT, DeiT), protein structure (AlphaFold), and more.

Scaled Dot-Product Attention — From Scratch

Each token in the sequence is first projected into three vectors — a query $q$, a key $k$, and a value $v$ — by three learnable linear maps.

🟢 The metaphor

Think of a database lookup. Each token emits a question (query), each token emits a key describing what it contains, and each token emits a value to return. A token's query is compared against every key (via dot product) to produce a similarity score. The scores are softmaxed into weights; the output is a weighted sum of the values.

Stack all the queries, keys, and values into matrices $Q, K, V \in \mathbb{R}^{L \times d_k}$ (one row per token, $L$ the sequence length). The attention output for all positions is computed in one shot:

$$\text{Attention}(Q, K, V) \;=\; \text{softmax}\!\left(\frac{Q K^{\top}}{\sqrt{d_k}}\right) V$$
$Q K^\top$ — raw similarity scores ($L \times L$) $\sqrt{d_k}$ — variance rescaling softmax — row-wise

Read the formula right to left:

  1. $Q K^\top$ is an $L \times L$ matrix. Entry $(i, j)$ is the dot product of query $i$ and key $j$ — a raw similarity score.
  2. Divide every entry by $\sqrt{d_k}$. We explain why next.
  3. Softmax each row, so row $i$ becomes a probability distribution over the sequence: "how much should token $i$ pay attention to each other token?"
  4. Multiply by $V$: token $i$'s output is the weighted sum of value vectors according to its attention distribution.

Why divide by $\sqrt{d_k}$?

If $q$ and $k$ are random vectors of dimension $d_k$ with unit-variance entries, the dot product $q^\top k$ is a sum of $d_k$ products of unit-variance numbers — so its variance is $d_k$ and its typical magnitude is $\sqrt{d_k}$. Feed $\sqrt{d_k}$-magnitude scores into a softmax and the softmax saturates: one entry gets essentially all the weight, the rest get nothing, and the gradient with respect to all the other entries is zero. Training stalls.

Dividing by $\sqrt{d_k}$ brings the scores back to unit variance, keeps the softmax in its useful region, and lets gradients flow. It is a single character in code but absolutely load-bearing for training stability. Transformers without the scaling factor do not work.

🔵 Theory — The variance calculation

Let $q, k \in \mathbb{R}^{d_k}$ with independent zero-mean unit-variance entries. Then $q^\top k = \sum_{i=1}^{d_k} q_i k_i$. Each term has variance $\text{Var}(q_i)\text{Var}(k_i) = 1$. Independent sum gives variance $d_k$, so standard deviation $\sqrt{d_k}$. Dividing the dot product by $\sqrt{d_k}$ normalises it back to unit variance.

Attention from scratch with einsum

import math
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(q, k, v, mask=None):
    """q, k, v: (B, L, d_k) — queries, keys, values per sample/position"""
    d_k = q.size(-1)
    # (B, L, L) — one similarity matrix per sample in the batch
    scores = torch.einsum("bld,bmd->blm", q, k) / math.sqrt(d_k)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float("-inf"))
    attn = F.softmax(scores, dim=-1)        # row-wise over key positions
    out = torch.einsum("blm,bmd->bld", attn, v)
    return out, attn

Two einsum calls and a softmax — that is the entire attention computation. The mask argument is how we handle padding (ignore positions beyond the true sequence length) and causal generation (each position only attends to earlier positions).

Multi-Head Attention

A single attention head gives every token one "viewpoint" on the rest of the sequence. Different linguistic phenomena (syntactic agreement, coreference, topic relevance) live on different viewpoints. Multi-head attention runs $h$ separate attention computations in parallel on smaller subspaces and concatenates the results:

$$\text{MultiHead}(Q, K, V) \;=\; \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W^O$$
$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$ $W^O$ — output projection

If the full model dimension is $d_{\text{model}} = 512$ and we have $h = 8$ heads, each head operates in a $d_k = 64$-dimensional subspace. Total parameter count is roughly the same as a single-head $d_k = 512$ attention, but the model can attend to eight different relationships simultaneously.

Positional Encoding

Self-attention is permutation invariant — shuffle the sequence and you get the same attention scores (with rows and columns permuted). But in language, order matters: "dog bites man" is not "man bites dog". The Transformer injects position information by adding a position vector to each token embedding.

The original paper uses sinusoidal positional encodings with different frequencies per dimension:

$$\text{PE}(\text{pos}, 2i) = \sin\!\left(\frac{\text{pos}}{10000^{2i/d}}\right), \quad \text{PE}(\text{pos}, 2i+1) = \cos\!\left(\frac{\text{pos}}{10000^{2i/d}}\right)$$
pos — position index in sequence $i$ — feature dimension $d$ — model dimension

The particular choice gives every position a unique fingerprint that can be computed deterministically (no parameters), extrapolated beyond training sequence lengths, and lets the model learn relative position via trigonometric identities. Later Transformers (BERT, GPT) use learned positional embeddings instead — one vector per position, trained from scratch. 

class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
        )
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        self.register_buffer("pe", pe)

    def forward(self, x):
        return x + self.pe[:x.size(1)]   # broadcast-add (L, d) to (B, L, d)

The Encoder Block and Full Encoder

One Transformer encoder block has six components:

  1. Multi-head self-attention
  2. Residual connection (add input to attention output)
  3. Layer normalisation
  4. Feed-forward network (two linear layers with ReLU)
  5. Residual connection (add to FFN output)
  6. Layer normalisation again

The full encoder stacks $N$ identical blocks. PyTorch provides nn.TransformerEncoderLayer and nn.TransformerEncoder which implement this exactly, with the option of norm_first=True to apply layer norm before attention (the "pre-norm" variant, which trains more stably). 

class TinyTransformer(nn.Module):
    def __init__(self, n_tokens=10, d_model=64, n_heads=4, n_layers=2,
                 max_len=50, n_classes=2):
        super().__init__()
        self.embed = nn.Embedding(n_tokens, d_model)
        self.pos   = PositionalEncoding(d_model, max_len)
        enc_layer  = nn.TransformerEncoderLayer(
            d_model=d_model, nhead=n_heads, dim_feedforward=128,
            batch_first=True, norm_first=True,
        )
        self.encoder = nn.TransformerEncoder(enc_layer, num_layers=n_layers)
        self.head = nn.Linear(d_model, n_classes)

    def forward(self, tokens):
        x = self.embed(tokens)               # (B, L, d)
        x = self.pos(x)                      # add position info
        x = self.encoder(x)                  # (B, L, d)
        pooled = x.mean(dim=1)               # mean-pool over sequence
        return self.head(pooled)
🔵 Theory — Layer norm, not batch norm

Transformers use layer norm (normalise across the feature dimension per token) rather than batch norm (normalise across the batch dimension per feature). Why? Because sequence lengths vary across batches, and batch statistics become noisy or meaningless when the effective batch size at each position varies. Layer norm has no dependency on batch shape, so it is robust.

Worked Example: A Tiny Transformer Sequence Classifier

📝 Goal

Build and train a small Transformer encoder on a synthetic majority-bit task: given a sequence of bits, output 1 if the majority are 1, else 0. Simple enough to train in 10 seconds, non-trivial enough to require attention (the model must count).

Step 1 — data. 
import torch, math
import torch.nn as nn, torch.nn.functional as F
from torch.utils.data import DataLoader, TensorDataset

torch.manual_seed(42)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

def make_dataset(n=3000, L=20):
    X = torch.randint(0, 2, (n, L))
    y = (X.sum(dim=1) > (L // 2)).long()
    return X, y

X, y = make_dataset()
X, y = X.to(device), y.to(device)
loader = DataLoader(TensorDataset(X, y), batch_size=64, shuffle=True)
Step 2 — assemble the model.

Use the TinyTransformer class defined earlier. With n_tokens=2 the embedding is a 2-entry lookup.

model = TinyTransformer(n_tokens=2, d_model=32, n_heads=4,
                        n_layers=2, max_len=20, n_classes=2).to(device)
opt = torch.optim.Adam(model.parameters(), lr=1e-3)
Step 3 — train. 
for epoch in range(20):
    for xb, yb in loader:
        logits = model(xb)
        loss = F.cross_entropy(logits, yb)
        opt.zero_grad()
        loss.backward()
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        opt.step()

Same gradient clipping as for RNNs — Transformers explode in exactly the same way if you forget. Same optimiser, same loss, same structure.

Step 4 — inspect attention weights. 
model.eval()
with torch.no_grad():
    test = X[:4]
    attn_maps = []
    def hook(module, inp, out):
        # out is (attn_output, attn_weights)
        attn_maps.append(out[1].cpu())
    handle = model.encoder.layers[0].self_attn.register_forward_hook(hook)
    _ = model(test)
    handle.remove()
print(attn_maps[0].shape)    # (B, L, L)

PyTorch's built-in attention returns the attention matrix if you hook it correctly. Plotting these matrices lets you see which positions are attending to which — invaluable for debugging.

Common Mistakes

⚠ Mistake 1: Forgetting positional encoding

Without PE, a Transformer is a set classifier — it cannot distinguish "dog bites man" from "man bites dog". Symptom: the model gets stuck at ~50% on any task that depends on order. Fix: add sinusoidal or learned positional encoding to the embedding before the first encoder layer.

⚠ Mistake 2: Missing / sqrt(d_k)

Without the scaling, the softmax saturates almost immediately and training stalls. Symptom: loss drops for one epoch and then flatlines at a high value. Double-check the scaling before anything else.

⚠ Mistake 3: Wrong mask shape or type

nn.MultiheadAttention expects attn_mask (per-position, broadcast) and key_padding_mask (per-sample, padding indicators) as separate arguments. Mixing them up is the #1 source of "my Transformer ignores the last half of the sentence" bugs. For causal masking, use an upper-triangular boolean matrix with True where positions should be masked.

⚠ Mistake 4: Quadratic memory surprise

Self-attention is $\mathcal{O}(L^2)$ in memory. Double the sequence length and memory quadruples. Token sequences of 4096 are painful on a consumer GPU. This is why you see "efficient attention" variants (Longformer, Performer, FlashAttention) in the literature.

Reflection — The Four Paradigms Consolidated

You have now seen all four core deep-learning paradigms:

  • Autoencoders (5.1) — representation learning via compress-and-reconstruct.
  • CNNs (5.2) — spatial feature extraction via weight sharing.
  • RNNs / LSTMs (5.3) — temporal feature extraction via gated memory.
  • Transformers (5.4) — direct attention between all positions, in parallel.

The remaining four lessons combine these paradigms: GANs and diffusion (5.5) pair a decoder/generator with an adversarial or noise-based training signal; GNNs (5.6) generalise convolutions and attention to graph-structured data; transfer learning (5.7) reuses pre-trained backbones from any of the four; and reinforcement learning (5.8) changes the game entirely by learning from interaction.

You derived scaled dot-product attention from scratch, understood why $\sqrt{d_k}$ matters, implemented multi-head attention, added positional encoding, and trained a working Transformer encoder. That is the single most important architecture in modern AI and you now know how it works end to end.
