The confusion matrix

Each metric reads different cells

The formulas

\[ \text{Precision} = \frac{TP}{TP+FP}\qquad \text{Recall} = \frac{TP}{TP+FN} \]
\[ F_1 = 2\cdot\frac{\text{Prec}\cdot\text{Rec}}{\text{Prec}+\text{Rec}} \]

On 12% credit data, predicting "no default" for everyone scores 88% accuracy. F1 = 0. Accuracy lies.

ROC vs PR — why PR is honest on imbalanced data

Why ROC fails on imbalanced data

  • $\text{FPR} = FP / (FP + TN)$ — when negatives dominate, FP is tiny relative to TN
  • Moving $FP$ from 10 to 1000 barely changes FPR
  • ROC stays beautiful while precision is miserable
  • PR has no blind spot: ignores $TN$ entirely. Every false positive counts.
Rule of thumb: positive class < 20%? report PR-AUC. Otherwise, either works.

Four imbalance techniques

  1. SMOTE — synthesise minority samples via $k$-NN interpolation. Fragile in high-dim or noisy minority.
  2. Cost-sensitive — class_weight in the loss. No fake data.
  3. Focal loss — $(1 - p_t)^\gamma$ factor down-weights easy examples. Focus on the hard.
  4. Threshold tuning — derive from business cost matrix, not default 0.5.

Focal loss

\[ \text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t) \]
  • $p_t$ = predicted probability of the true class
  • $\gamma = 0$ → ordinary weighted cross-entropy
  • $\gamma = 2$: easy example at $p_t = 0.9$ gets factor $0.01$ — 100× less influence
  • Hard example at $p_t = 0.3$ gets factor $0.49$ — barely changed
  • Gradient is dominated by hard cases

Threshold from the cost matrix

Given costs $(c_{FP}, c_{FN})$, predict positive iff $p > c_{FP}/(c_{FP}+c_{FN})$.

\[ t^* = \frac{c_{FP}}{c_{FP} + c_{FN}} \]

100:1 credit costs: $t^* = 100/10100 \approx 0.0099$.

Not 0.5! Any doubt about a loan is enough to flag when missing a default is 100× more expensive.

Calibration — reliability diagram

Two calibration methods

  • Platt scaling: fit $\sigma(Ax + B)$ on model scores. One parameter plus bias. Cheap. Works when miscalibration is sigmoid-shaped.
  • Isotonic regression: non-decreasing piecewise-constant function. Flexible but needs more data.

from sklearn.calibration import CalibratedClassifierCV

cal = CalibratedClassifierCV(base_model, method="isotonic", cv=5)
cal.fit(X_tr, y_tr)

Brier score — proper scoring rule

\[ \text{BS} = \frac{1}{N}\sum_i (p_i - y_i)^2 \]
  • Lower is better
  • Proper scoring rule: minimised by reporting the true probability
  • Cheating (push everything to 0/1) scores worse than honest uncertainty
  • Report before and after calibration to show improvement

Full pipeline — credit scoring


# 1. Baseline
base = lgb.LGBMClassifier(n_estimators=400).fit(X_tr, y_tr)

# 2. Cost-sensitive
weighted = lgb.LGBMClassifier(n_estimators=400,
    class_weight={0: 1, 1: 100}).fit(X_tr, y_tr)

# 3. Threshold from cost matrix
t_opt = 100 / (100 + 10_000)  # 0.0099
pred = (weighted.predict_proba(X_te)[:, 1] >= t_opt).astype(int)

# 4. Post-hoc calibration
cal = CalibratedClassifierCV(weighted, method="isotonic", cv=3).fit(X_tr, y_tr)
p_cal = cal.predict_proba(X_te)[:, 1]
print("Brier:", brier_score_loss(y_te, p_cal))

Three traps

1. Reporting accuracy on imbalanced data
2. SMOTE-ing the test set (SMOTE is training-only)
3. Leaving the decision threshold at 0.5
