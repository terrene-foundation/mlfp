Lesson 5.3 RNNs and Sequence Models

Convolutions handle grids; recurrent networks handle sequences. In this lesson you meet the vanilla RNN, then the Long Short-Term Memory (LSTM) cell and its four gates, then the GRU. You will derive all six LSTM gate equations, see why the cell state is a "gradient highway", and train an LSTM on a synthetic financial time series that looks a lot like an SGX stock price.

Why Sequences Are Different

A sequence is an ordered list: words in a sentence, prices in a time series, trip points in a taxi route, heart rates in an ECG. Two properties distinguish sequence data from images:

  • Variable length. One sentence has 5 words, another has 50. A CNN with a fixed input size cannot handle this gracefully.
  • Temporal dependency. The meaning of a word depends on the words before it ("bank" near "river" vs "bank" near "deposit"). The price tomorrow depends on prices today and last week.

The idea of a recurrent network is to maintain a hidden state $h_t$ that summarises everything seen so far, and update it step by step as each new input arrives:

$$h_t \;=\; \phi(W_{xh} \, x_t + W_{hh} \, h_{t-1} + b_h)$$
$x_t$ — input at step $t$ $h_{t-1}$ — previous hidden state $\phi$ — non-linearity (tanh)

The same weight matrices $W_{xh}$ and $W_{hh}$ are reused at every time step — just like a conv filter reuses weights across positions. That weight sharing is what lets a recurrent network handle any length of sequence with a fixed number of parameters.

Vanishing and exploding gradients

The elegant picture has a cruel flaw. When you backpropagate through a long sequence, the gradient of the loss with respect to $h_1$ involves a product of $T$ Jacobians — one per time step. If each is smaller than 1, the product shrinks exponentially (vanishing gradient) and $h_1$ never gets updated. If each is larger than 1, it blows up (exploding gradient) and training NaNs out.

Vanilla RNNs are almost unusable on sequences longer than ~20 steps for this reason. The fix came in 1997 from Hochreiter and Schmidhuber: the LSTM cell.

The LSTM Cell

An LSTM adds a second state on top of the hidden state $h_t$: a cell state $C_t$. The cell state flows forward with only additive updates, protected by three gates that choose what to remember, what to add, and what to reveal. The additive path is a gradient highway — it does not get multiplied by a Jacobian at every step, so gradients do not vanish.

All six gate equations

Let $[h_{t-1}, x_t]$ denote the concatenation of the previous hidden state and the current input. The LSTM update rule is:

$$\begin{aligned} f_t &= \sigma\!\bigl(W_f \, [h_{t-1}, x_t] + b_f\bigr) & \text{(forget gate)}\\[2pt] i_t &= \sigma\!\bigl(W_i \, [h_{t-1}, x_t] + b_i\bigr) & \text{(input gate)}\\[2pt] \tilde{C}_t &= \tanh\!\bigl(W_C \, [h_{t-1}, x_t] + b_C\bigr) & \text{(candidate cell)}\\[2pt] C_t &= f_t \odot C_{t-1} + i_t \odot \tilde{C}_t & \text{(cell update)}\\[2pt] o_t &= \sigma\!\bigl(W_o \, [h_{t-1}, x_t] + b_o\bigr) & \text{(output gate)}\\[2pt] h_t &= o_t \odot \tanh(C_t) & \text{(hidden state)} \end{aligned}$$

Read row by row:

  • Forget gate $f_t$. A sigmoid in $[0, 1]$ per cell dimension. Multiplies $C_{t-1}$ elementwise — zero means "forget this component", one means "keep it entirely".
  • Input gate $i_t$. Another sigmoid: how much of the new candidate to let into the cell state.
  • Candidate $\tilde{C}_t$. A tanh in $[-1, 1]$: the new content to potentially add.
  • Cell update. Combine: forget some of the past, accept some of the new. The update is additive — this is the gradient highway.
  • Output gate $o_t$. How much of the cell state to expose as the hidden state.
  • Hidden state $h_t$. Output-gated tanh of the cell state.
🔵 Theory — Why the cell state dodges vanishing gradients

In a vanilla RNN, $\partial h_t / \partial h_{t-1}$ is a product involving $W_{hh}$ and a tanh derivative. Repeated over $T$ steps it becomes a product of matrices with small singular values, which decays exponentially. In an LSTM, the cell-state update is $C_t = f_t \odot C_{t-1} + (\text{stuff})$. If the forget gate $f_t \approx 1$, then $\partial C_t / \partial C_{t-1} \approx I$ — essentially no decay. The gradient flows backward through the cell state nearly unchanged over hundreds of time steps.

The GRU

The GRU (Gated Recurrent Unit, 2014) simplifies the LSTM by merging the forget and input gates into a single update gate and dropping the separate cell state:

$$\begin{aligned} z_t &= \sigma(W_z \, [h_{t-1}, x_t]) & \text{(update gate)}\\ r_t &= \sigma(W_r \, [h_{t-1}, x_t]) & \text{(reset gate)}\\ \tilde{h}_t &= \tanh(W \, [r_t \odot h_{t-1}, x_t])\\ h_t &= (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t \end{aligned}$$

Fewer parameters, faster to train, usually competitive with LSTM. Rule of thumb: try a GRU first; if it underfits, upgrade to LSTM; if you need really long contexts (1000+ tokens), use a Transformer (Lesson 5.4).

Worked Example: LSTM on a synthetic price series

📝 Goal

Build an LSTM that predicts the next value of a noisy sine-plus-trend "price" series — a proxy for a stock on the Singapore Exchange. Compare vanilla RNN vs LSTM vs GRU.

Step 1 — synthetic series. 
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.utils.data import DataLoader, TensorDataset
import numpy as np

torch.manual_seed(42)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

def make_series(n=2000, seq_len=50):
    t = np.arange(n + seq_len).astype(np.float32)
    trend = 0.002 * t
    season = 0.5 * np.sin(2 * np.pi * t / 25) + 0.3 * np.sin(2 * np.pi * t / 8)
    noise = 0.1 * np.random.randn(len(t)).astype(np.float32)
    series = trend + season + noise
    X = np.stack([series[i:i+seq_len] for i in range(n)])         # (n, L)
    y = series[seq_len:seq_len+n]                                 # (n,)
    return X[..., None], y                                        # (n, L, 1)

X_np, y_np = make_series()
X = torch.from_numpy(X_np).to(device)
y = torch.from_numpy(y_np).to(device)
loader = DataLoader(TensorDataset(X, y), batch_size=64, shuffle=True)
Step 2 — an LSTM regressor. 
class LSTMRegressor(nn.Module):
    def __init__(self, input_size=1, hidden_size=32, num_layers=2):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=0.1,
        )
        self.head = nn.Linear(hidden_size, 1)

    def forward(self, x):
        out, (h_n, c_n) = self.lstm(x)      # out: (B, L, H)
        last = out[:, -1, :]                # take final time step
        return self.head(last).squeeze(-1)  # (B,)

nn.LSTM implements all six gate equations above for you. batch_first=True makes the expected input shape $(B, L, F)$ instead of the older $(L, B, F)$ convention. We take the output at the last time step as our regression head input.

Step 3 — train with gradient clipping. 
model = LSTMRegressor().to(device)
opt = torch.optim.Adam(model.parameters(), lr=1e-3)
history = []

for epoch in range(30):
    running = 0.0
    for xb, yb in loader:
        pred = model(xb)
        loss = F.mse_loss(pred, yb)
        opt.zero_grad()
        loss.backward()
        # ★ gradient clipping — essential for recurrent networks
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        opt.step()
        running += loss.item() * xb.size(0)
    history.append(running / len(loader.dataset))
print(f"Final LSTM MSE: {history[-1]:.5f}")

The star line is clip_grad_norm_. It rescales the gradient vector so its L2 norm never exceeds 1.0. Without it, occasional exploding-gradient spikes ruin training for long sequences. For LSTMs and Transformers, clipping is standard practice — always on.

Step 4 — GRU and vanilla RNN baselines. 
class GRURegressor(LSTMRegressor):
    def __init__(self, **kw):
        super().__init__(**kw)
        self.lstm = nn.GRU(**{**dict(input_size=1, hidden_size=32,
                                     num_layers=2, batch_first=True, dropout=0.1)})

class RNNRegressor(LSTMRegressor):
    def __init__(self, **kw):
        super().__init__(**kw)
        self.lstm = nn.RNN(input_size=1, hidden_size=32, num_layers=2,
                           batch_first=True, nonlinearity='tanh')

Subclassing with the same head; only the recurrent core differs. You should observe: vanilla RNN struggles to get below a modest plateau; GRU and LSTM both converge to noticeably lower MSE. On longer sequences (seq_len > 100), the vanilla RNN gap becomes dramatic.

Step 5 — temporal attention (optional).

Instead of taking the last time step, compute a weighted sum of all time steps where the weights are learned: 

class AttentionLSTM(nn.Module):
    def __init__(self, input_size=1, hidden_size=32):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, batch_first=True)
        self.attn = nn.Linear(hidden_size, 1)
        self.head = nn.Linear(hidden_size, 1)

    def forward(self, x):
        out, _ = self.lstm(x)                            # (B, L, H)
        scores = self.attn(out).squeeze(-1)              # (B, L)
        weights = F.softmax(scores, dim=-1).unsqueeze(-1)
        context = (weights * out).sum(dim=1)             # (B, H)
        return self.head(context).squeeze(-1)

This is a foreshadowing of Lesson 5.4 — the idea that "attend to a weighted combination of all time steps" is exactly what the Transformer generalises and takes to its logical conclusion.

🟣 Advanced — Teacher forcing vs free-running

When training a sequence-to-sequence model, you can either feed the ground-truth previous token as input (teacher forcing) or feed the model's own previous prediction (free-running). Teacher forcing is faster to train but creates exposure bias: at inference time the model has to condition on its own outputs, which it never saw during training. Scheduled sampling mixes the two to mitigate this. Transformers largely sidestep the issue with parallel attention training.

Common Mistakes

⚠ Mistake 1: Forgetting gradient clipping

RNNs without gradient clipping explode on ~1% of batches and NaN your loss. Always add torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0) between loss.backward() and opt.step().

⚠ Mistake 2: Wrong batch_first convention

PyTorch's recurrent layers default to (seq_len, batch, features) shape. Modern practice is batch_first=True which gives (batch, seq_len, features). Mix them up and every dimension is off by one axis — the bug is silent because shapes still "work", they just mean different things.

⚠ Mistake 3: Using LSTM state across unrelated sequences

By default nn.LSTM initialises the hidden state to zeros at the start of each forward pass — correct. If you manually pass the previous batch's final state as the next batch's initial state (across unrelated sequences), you contaminate each sequence with another's hidden memory. Only do stateful RNN training deliberately, for a single long stream cut into chunks.

⚠ Mistake 4: Sequence length too long for available memory

Backprop-through-time stores activations for every step. A seq_len of 1000 with a hidden size of 512 and batch 32 can eat many GBs. Truncated BPTT (reset gradients every $k$ steps) is the fix when memory bites.
