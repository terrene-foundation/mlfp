Chapter 3: MCP Transport

Overview

The MCP (Model Context Protocol) transport enables AI agents to interact with Nexus handlers as MCP tools. When registered, each handler becomes a tool that agents like Claude can invoke. This chapter covers MCPTransport configuration, tool naming conventions, the background thread architecture, and integration with Nexus.

Prerequisites

  • Completed Chapters 1–2 (Nexus basics, HTTP transport)
  • Understanding of the Transport ABC
  • Conceptual familiarity with MCP (Model Context Protocol)

Concepts

Tool Naming Convention

When a handler named "greet" is registered, MCPTransport creates an MCP tool named "{namespace}_{handler_name}". With the default namespace "nexus", the tool becomes "nexus_greet". This prevents collisions when multiple Nexus instances share the same MCP platform.

Background Thread Architecture

MCPTransport runs the FastMCP server in a daemon thread with its own asyncio event loop, because the main thread is blocked by uvicorn (HTTP). The daemon flag ensures the thread dies with the process.

Note: MCP is opt-in. Unlike HTTP (always present), MCP must be explicitly added via app.add_transport(MCPTransport(...)).

Code Walkthrough

Step 1: Create an MCPTransport

from nexus.transports import MCPTransport, Transport

mcp = MCPTransport(
    port=4001,
    namespace="myapp",
    server_name="my-nexus-service",
)

assert mcp.name == "mcp"
assert mcp.port == 4001
assert mcp._namespace == "myapp"
assert mcp.is_running is False

Step 2: Default Configuration

mcp_default = MCPTransport()

assert mcp_default.port == 3001
assert mcp_default._namespace == "nexus"
assert mcp_default._server_name == "kailash-nexus"

Step 3: Add to Nexus

from nexus import Nexus

app = Nexus(api_port=8000, enable_durability=False)
custom_mcp = MCPTransport(port=5001, namespace="tutorial")
app.add_transport(custom_mcp)

assert custom_mcp in app._transports

Step 4: Health Check

health = mcp.health_check()
assert health["transport"] == "mcp"
assert health["running"] is False
assert health["port"] == 4001
assert health["server"] is False

  3. Multi-Transport Nexus: Add two MCPTransport instances with different ports to one Nexus app.

Key Takeaways

  • MCPTransport registers handlers as MCP tools for AI agents.
  • Tool names follow "{namespace}_{handler_name}" to avoid collisions.
  • Runs in a background daemon thread with its own event loop.
  • Added to Nexus via app.add_transport() — opt-in.
  • Hot-registration supports adding tools at runtime.
