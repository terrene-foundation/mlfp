Dimensionality Reduction

PCA · SVD · Kernel PCA · t-SNE · UMAP · Intrinsic dimension

⏱ 90 minutes   ·   Outcome: compress 50 features into 6

From "which group" to "which axis"

  • Clustering partitioned the points. Dimensionality reduction rotates the features.
  • Goal: fewer features, most variance kept.
  • Uses: visualisation, compression, feature extraction.
🟢 Foundations: if two features carry the same information, you have one feature's worth of signal. Reduction finds the non-redundant directions.

PCA rotates the axes to the data

Deriving PCA — one Lagrangian

Maximise projected variance subject to unit norm:

\[ \max_{\mathbf{w}}\ \mathbf{w}^\top C\,\mathbf{w}\quad \text{s.t. } \lVert \mathbf{w}\rVert = 1 \]

Taking derivatives of the Lagrangian:

\[ C\mathbf{w} \;=\; \lambda\,\mathbf{w} \]
🔵 Theory: The solution is the top eigenvector of the covariance matrix. The variance captured = the corresponding eigenvalue.

SVD = PCA's computational engine

\[ X \;=\; U\,\Sigma\,V^\top \]

V = principal directions. Σ²/(N−1) = variances. Every serious PCA uses SVD.

Scree plot: how many components?

Rule of thumb: take the smallest $k$ where cumulative variance ≥ 90%.

Loadings make PCA interpretable


for i in range(3):
    top = np.argsort(-np.abs(pca.components_[i]))[:3]
    print(f"PC{i+1} (var={pca.explained_variance_ratio_[i]:.3f}):")
    for j in top:
        print(f"  {feature_names[j]}: {pca.components_[i][j]:+.3f}")

Example: PC1 = +0.4 spend + 0.4 visits + 0.3 items → "overall activity".

Kernel PCA — for curved structure

  • Standard PCA can only rotate.
  • Kernel PCA applies PCA in an implicit feature space via $k(\mathbf{x}, \mathbf{y}) = \phi(\mathbf{x})^\top \phi(\mathbf{y})$.
  • Common kernels: RBF, polynomial.
  • Cost: $O(N^2)$ memory — only for ~few thousand points.

t-SNE: good for plots, bad for features

\[ \text{KL}(P \,\|\, Q) \;=\; \sum_{i \neq j}\ p_{ij}\ \log \frac{p_{ij}}{q_{ij}} \]
  • Minimises KL between pair-distributions in high-dim and low-dim.
  • Preserves local structure, distorts global.
  • Non-deterministic, no inverse transform.
  • Hyperparameter: perplexity (~5–50).
⚠ Never use t-SNE for feature extraction. Use it only to see.

UMAP: newer, faster, better defaults

  • Preserves more global structure than t-SNE.
  • ~10× faster. Scales to millions of points.
  • Has an (approximate) inverse transform — can embed new points.
  • Deterministic (given a seed).

Default choice for any visualisation task. Fall back to t-SNE only when specifically needed.

Same data, different preservation guarantees

Eckart-Young: PCA is optimal

Best rank-$k$ approximation in Frobenius norm:

\[ \min_{\text{rank}(\hat X) \leq k}\ \lVert X - \hat X \rVert_F^2 \;=\; \sum_{i=k+1}^{r}\sigma_i^2 \]

achieved by $\hat X = U_k \Sigma_k V_k^\top$ — the truncated SVD.

🔵 Theory: PCA is provably the best linear low-rank approximation. No linear method beats it.

Intrinsic dimension

How many components do you truly need?

  • Scree elbow — visual estimate.
  • Variance threshold — smallest $k$ with $\sum \sigma_i^2 / \sum \sigma^2 > 0.9$.
  • Participation ratio: $(\sum \lambda_i)^2 / \sum \lambda_i^2$.

Worked example: compress customer features


pca = PCA().fit(StandardScaler().fit_transform(X))
cum = np.cumsum(pca.explained_variance_ratio_)
k90 = int(np.searchsorted(cum, 0.90)) + 1
# 15 features → 6 components for 90% variance

After PCA, downstream models train faster, overfit less, often score slightly better.

Four rookie mistakes

  1. Forgetting to centre/standardise the data.
  2. Using t-SNE as a feature extractor.
  3. Picking $k$ that is too small (over-aggressive compression).
  4. Reporting components without loadings — black-box PCA.
