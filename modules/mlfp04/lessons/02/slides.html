EM Algorithm and Gaussian Mixture Models

Soft clustering · E-step · M-step · GMM · Mixture of Experts

⏱ 75 minutes   ·   Outcome: implement EM in 20 lines

Hard labels lie

  • K-means says: "you are cluster 2."
  • GMM says: "you are 70% cluster 2, 25% cluster 1, 5% cluster 3."
  • For boundary points, the soft answer is the honest answer.
🟢 Foundations: GMM = "K-means with error bars".

Hard vs soft in one picture

The generative story

  1. Pick cluster $k$ with probability $\pi_k$.
  2. Draw from $\mathcal{N}(\boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)$.
\[ p(\mathbf{x}) \;=\; \sum_{k=1}^{K}\ \pi_k\ \mathcal{N}\!\bigl(\mathbf{x}\,\big|\,\boldsymbol{\mu}_k,\,\boldsymbol{\Sigma}_k\bigr) \]

Goal: find $(\pi, \mu, \Sigma)$ that maximises this for the data.

Why maximising the likelihood is hard

\[ \log \prod_n p(\mathbf{x}_n) \;=\; \sum_n \log \sum_k \pi_k\ \mathcal{N}(\mathbf{x}_n|\mu_k,\Sigma_k) \]

The "log of a sum" has no closed form. We need a trick.

🔵 Theory: The trick is to introduce a hidden variable $z_{nk}$ for "which cluster generated point $n$", then alternate between estimating it and updating parameters.

EM: ellipses find the clusters

E-step: responsibilities via Bayes' rule

\[ r_{nk} \;=\; \frac{\pi_k\ \mathcal{N}(\mathbf{x}_n | \mu_k, \Sigma_k)}{\sum_{j=1}^{K}\pi_j\ \mathcal{N}(\mathbf{x}_n | \mu_j, \Sigma_j)} \]
  • Numerator: prior × likelihood.
  • Denominator: marginal $p(\mathbf{x}_n)$.
  • $\sum_k r_{nk} = 1$ by construction.

M-step: weighted means

\[ N_k = \sum_n r_{nk},\quad \pi_k^{\text{new}} = \frac{N_k}{N} \] \[ \mu_k^{\text{new}} = \frac{1}{N_k}\sum_n r_{nk}\,\mathbf{x}_n \] \[ \Sigma_k^{\text{new}} = \frac{1}{N_k}\sum_n r_{nk}(\mathbf{x}_n - \mu_k)(\mathbf{x}_n - \mu_k)^\top \]

Each update is a weighted MLE. Weights are the responsibilities.

Convergence guarantee

Each EM iteration satisfies

\[ \log \mathcal{L}(\theta^{t+1}) \;\geq\; \log \mathcal{L}(\theta^{t}) \]

with equality only at a stationary point.

🔵 Theory: EM is coordinate ascent on the ELBO (evidence lower bound). Jensen's inequality does the heavy lifting.
⚠ Caveat: EM can get stuck in local maxima. Restart from several random inits and keep the best.

Twenty lines of EM (1/2)


def em_gmm(X, K, max_iter=100, tol=1e-4):
    N, D = X.shape
    rng = np.random.default_rng(42)
    mu = X[rng.choice(N, size=K, replace=False)]
    Sigma = np.array([np.eye(D) for _ in range(K)])
    pi = np.full(K, 1.0 / K)
    log_lik = -np.inf
    for _ in range(max_iter):
        # E-step
        weighted = np.stack(
            [pi[k] * mvn.pdf(X, mu[k], Sigma[k]) for k in range(K)],
            axis=1,
        )
        total = weighted.sum(1, keepdims=True) + 1e-12
        r = weighted / total

Twenty lines of EM (2/2)


        # M-step
        Nk = r.sum(0)
        pi = Nk / N
        mu = (r.T @ X) / Nk[:, None]
        for k in range(K):
            diff = X - mu[k]
            Sigma[k] = (r[:, k:k+1] * diff).T @ diff / Nk[k]

        new_ll = np.log(total + 1e-12).sum()
        if abs(new_ll - log_lik) < tol:
            break
        log_lik = new_ll
    return pi, mu, Sigma, r, log_lik

Picking $K$ with BIC

\[ \text{BIC} \;=\; -2\log \hat{\mathcal{L}} + p\log N \]
  • $\hat{\mathcal{L}}$ = maximised likelihood.
  • $p$ = number of free parameters (penalises complexity).
  • $N$ = sample size.
  • Lower is better. Try $K = 2..10$.

Mixture of Experts — where GMM meets GPT-4

  • Replace fixed $\pi_k$ with a gating network $g_k(\mathbf{x})$.
  • Each "expert" is a sub-network; the gate routes inputs.
  • GPT-4, Mixtral, DeepSeek — all use MoE layers for scale.
🟣 Advanced: Classical GMM is MoE with constant gates. We revisit this in M6 when we route between Kaizen agents.

Worked example: 3 Gaussians in 2D


means = np.array([[0, 0], [5, 5], [-3, 4]])
X = np.vstack([rng.multivariate_normal(m, np.eye(2), 200) for m in means])

pi, mu, Sigma, r, ll = em_gmm(X, K=3)
print(f"LL: {ll:.2f}, π: {pi.round(3)}")

# Cross-check with sklearn
gm = GaussianMixture(n_components=3, random_state=0).fit(X)
print(f"sklearn LL: {gm.score(X) * len(X):.2f}")

Both implementations should agree to 3 decimal places.

The uncertain points are where the money is


# Points with ambiguous membership
uncertain = X[r.max(axis=1) < 0.6]
print(f"{len(uncertain)} customers need manual review")

These are the customers worth flagging for human review in a real segmentation pipeline. Hard clustering hides them.

Four rookie mistakes

  1. Zero-initialised covariance → singular matrix.
  2. Fixed iteration count, no $\Delta \mathcal{L}$ check.
  3. Comparing GMM to K-means by "accuracy vs labels" (no ground truth!).
  4. covariance_type="full" with $D > 20$ — overfits.
