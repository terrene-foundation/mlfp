Functions: reusable logic


def format_sgd(amount: float) -> str:
    """Format a number as Singapore dollars."""
    return f"S${amount:,.0f}"

print(format_sgd(485_000))     # → S$485,000
print(format_sgd(1_234_567))   # → S$1,234,567

Four ingredients: def, name, parameters (with type hints), indented body, return.

group_by + agg: the pattern

group_by in code


district_stats = (
    hdb.group_by("town")
    .agg(
        pl.len().alias("transaction_count"),
        pl.col("resale_price").mean().alias("mean_price"),
        pl.col("resale_price").median().alias("median_price"),
        pl.col("resale_price").std().alias("std_price"),
    )
    .sort("median_price", descending=True)
)
print(district_stats.head(5))

One call → one row per town. Each .alias(...) names the new column.

Multi-key grouping


town_flat_stats = (
    hdb.group_by("town", "flat_type")
    .agg(
        pl.len().alias("count"),
        pl.col("resale_price").median().alias("median_price"),
    )
    .sort("town", "flat_type")
)
# → one row per unique (town, flat_type) combination

Every unique combination of the group keys becomes one row.

Measures of centre: mean vs median

\[ \bar{x} = \frac{1}{n}\sum_{i=1}^{n} x_i \qquad \text{median} = \text{middle of sorted data} \]
🟢 Rule: Mean is pulled by outliers; median is not. For skewed data like property prices, report median for "typical" and mean for "total market value".

Measure of spread: IQR

\[ \text{IQR} = Q_3 - Q_1 \]
  • $Q_1$ = 25th percentile (value below which 25% of data lies)
  • $Q_3$ = 75th percentile (value below which 75% of data lies)
  • IQR captures the spread of the middle 50% — ignores outliers

pl.col("resale_price").quantile(0.25).alias("q25"),
pl.col("resale_price").quantile(0.75).alias("q75"),
# then: (q75 - q25).alias("iqr")

🔵 Theory: Coefficient of variation

\[ \text{CV} = \frac{s}{\bar{x}} \times 100\% \]

Unitless — lets you compare spread across groups with different means.

  • CV ≈ 25%: moderate spread
  • CV ≈ 45%: wide spread relative to centre
  • High CV districts have mixed housing stock (budget next to premium)

Derived aggregates with with_columns


district_stats = district_stats.with_columns(
    (pl.col("q75") - pl.col("q25")).alias("iqr_price"),
    (pl.col("std_price") / pl.col("mean_price") * 100)
        .alias("cv_price_pct"),
)

After agg, the result is just a DataFrame. Every Lesson 1.2 tool — with_columns, filter, sort — still works.

Iterating with for + functions


def district_line(row: dict) -> str:
    return (
        f"  {row['town']:<20} "
        f"{format_sgd(row['median_price']):>12}  "
        f"{row['transaction_count']:>8,}"
    )

print("=" * 50)
print("  TOP 10 HDB TOWNS BY MEDIAN PRICE")
print("=" * 50)
for row in district_stats.head(10).iter_rows(named=True):
    print(district_line(row))

Three mistakes to avoid

1. Hardcoding inside functions. No parameters = no reuse. Always parameterise.
2. Forgetting .alias(...). Aggregations get auto-named garbage without it. Always alias.
3. Python loop instead of group_by. 10–100× slower. If you're looping to filter, rewrite as group_by.
