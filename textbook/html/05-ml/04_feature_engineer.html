Chapter 4: FeatureEngineer

Overview

FeatureEngineer automates the two-step feature engineering workflow: generate candidate features, then select the best subset. The generate() method produces interaction terms, polynomial features, and binned columns. The select() method ranks all features using importance, correlation, or mutual information, then picks the top-k.

Concepts

The Two-Step Pattern

Feature engineering follows a deliberate process:

  1. Generate — Cast a wide net. Create many candidate features without worrying about usefulness.
  2. Select — Narrow down. Rank all features by predictive value and keep only the best.

Generation Strategies

StrategyWhat It ProducesNaming

Selection Methods

Code Walkthrough

Step 1: Prepare Data and Schema

import polars as pl
from kailash_ml import FeatureField, FeatureSchema
from kailash_ml.engines.feature_engineer import FeatureEngineer

schema = FeatureSchema(
    name="churn_features",
    features=[
        FeatureField(name="age", dtype="int64"),
        FeatureField(name="income", dtype="float64"),
        FeatureField(name="tenure_months", dtype="int64"),
        FeatureField(name="num_products", dtype="int64"),
    ],
    entity_id_column="entity_id",
)

Step 2: Generate Candidates

engineer = FeatureEngineer(max_features=10)
candidates = engineer.generate(
    df, schema,
    strategies=["interactions", "polynomial", "binning"],
)

# With 4 numeric features:
# 6 interactions (C(4,2)), 4 polynomials, 4 binned = 14 candidates
Naming conventions: Interaction columns use _x_ (e.g., age_x_income), polynomial columns use _squared, and binned columns use _binned. This makes it easy to trace generated features back to their source.

Step 3: Select Best Features

selected = engineer.select(
    candidates.data, candidates,
    target="churned",
    method="importance",
    top_k=8,
)

# Rankings are sorted by score descending
for rank in selected.rankings:
    print(f"  #{rank.rank} {rank.column_name}: {rank.score:.4f} ({rank.source})")

Step 4: Try Different Selection Methods

# Correlation-based
selected_corr = engineer.select(
    candidates.data, candidates,
    target="churned", method="correlation", top_k=5,
)

# Mutual information
selected_mi = engineer.select(
    candidates.data, candidates,
    target="churned", method="mutual_info", top_k=5,
)

Key Takeaways

  • FeatureEngineer follows a two-step pattern: generate() then select().
  • Three generation strategies create diverse candidates from existing numeric columns.
  • Three selection methods rank features by different predictive value measures.
  • Rankings distinguish between original and generated features.
  • max_features provides a global cap when top_k is not specified.
