Anomaly Detection and Ensembles

Z-score · IQR · Isolation Forest · LOF · EnsembleEngine

⏱ 75 minutes   ·   Outcome: detect fraud without labels

The opposite question

  • Clustering: "Which group does this point belong to?"
  • Anomaly detection: "Which points don't belong to any group?"
  • Use cases: fraud, defects, sensor faults, intrusions, churn precursors.
🟢 Foundations: anomalies are rare. If "anomalies" are 20% of your data, you have a classification problem, not an anomaly detection one.

Three types of outliers

Different methods catch different types.

Z-score: univariate, assumes normality

\[ z_i \;=\; \frac{x_i - \bar x}{s}\qquad |z_i| > 3\ \Rightarrow\ \text{flag} \]
  • Simple, fast, one-line.
  • Breaks on right-skewed data (revenue, session length).
  • Outliers inflate $\bar x$ and $s$, hiding each other.

IQR: robust to skew

\[ \text{flag if } x < Q_1 - 1.5\,\text{IQR} \quad\text{or}\quad x > Q_3 + 1.5\,\text{IQR} \]
  • Quartiles are not affected by a few extreme values.
  • This is what box-plot whiskers use.
  • Default univariate detector for business data.
🔵 Theory: winsorisation = cap at IQR bounds instead of dropping. Preserves sample size for downstream stats.

Univariate methods miss combinations

  • Small transaction: normal.
  • Midnight transaction: normal.
  • Small + midnight + new device + unusual location: suspicious.

Need a multivariate method. Enter Isolation Forest.

Isolation Forest — few splits isolate anomalies

Isolation Forest score

\[ s(\mathbf{x}, n) \;=\; 2^{-E(h(\mathbf{x}))/c(n)} \]
  • $E(h(\mathbf{x}))$ = average path length across trees.
  • $c(n)$ = expected path in random binary tree.
  • $s \approx 1$ → strong anomaly. $s \approx 0.5$ → normal.
  • Threshold: typically $s > 0.6$.

LOF — local density matters

\[ \text{LOF}_k(\mathbf{x}) \;=\; \frac{1}{|N_k|}\sum_{\mathbf{o}\in N_k}\ \frac{\text{lrd}_k(\mathbf{o})}{\text{lrd}_k(\mathbf{x})} \]
  • Compares point density to neighbour density.
  • LOF ≈ 1: same density → normal.
  • LOF > 1.5: anomalous relative to local context.
  • Right choice when clusters have different densities.

Score blending — more robust than any single method


def normalise(x):
    return (x - x.min()) / (x.max() - x.min() + 1e-9)

z_s = normalise(np.abs(z_scores))
if_s = normalise(-iforest.decision_function(X))
lof_s = normalise(-lof.negative_outlier_factor_)

blended = (z_s + if_s + lof_s) / 3
anomalies = blended > 0.7

Caught-by-all-three → almost certainly anomaly. Caught-by-one → review.

EnsembleEngine does this with one API


from kailash_ml import EnsembleEngine

ensemble = EnsembleEngine()
blended = ensemble.blend(
    scores={"z": z_s, "iforest": if_s, "lof": lof_s},
    normalise=True,
    agg="mean",
)

Worked example: financial fraud


features = ["amount", "hour", "recipient_count_24h", "location_km"]
X = StandardScaler().fit_transform(...)

iforest = IsolationForest(contamination=0.01, random_state=42).fit(X)
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.01)
lof.fit_predict(X)

# ... blend scores, flag top 1% ...
print(f"Flagged {anomalies.sum()} of {len(X)} transactions")

Contamination matters

  • Isolation Forest / LOF need contamination — approx. anomaly fraction.
  • Set too high → false positives explode.
  • Set too low → real anomalies missed.
  • Rule of thumb: start with the real base rate, then tune by inspecting top-$k$.
⚠ Warning: there is no precision/recall without labels. You must inspect manually.

Four rookie mistakes

  1. Contamination set to 10% when the real rate is 0.1%.
  2. Z-score on right-skewed data without log-transform.
  3. Unscaled features in multivariate methods.
  4. Treating raw anomaly scores as probabilities across methods.

Production: DriftMonitor uses this

The same algorithms — trained on yesterday's distribution — flag today's data as "drifted" when too many points look anomalous compared to the reference. 


from kailash_ml import DriftMonitor
monitor = DriftMonitor(reference=X_train, method="iforest")
score = monitor.check(X_today)
if score.drift_detected:
    alert("Distribution shift — consider retraining")
