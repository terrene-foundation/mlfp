Chapter 9: Probes

Overview

Kubernetes probes determine whether a container is alive, ready to serve traffic, and finished starting up. Nexus provides a ProbeManager that models the Kubernetes probe lifecycle with thread-safe, atomic state transitions.

Concepts

Probe State Machine

STARTING  -->  READY  -->  DRAINING
    |            |            |
    +-----+------+-----+------+
          |            |
          v            v
        FAILED  <------+

Three Probe Types

ProbeEndpointPasses WhenPurpose
Liveness/healthzNot FAILEDShould container be restarted?
Readiness/readyzREADY + callbacks passShould traffic be sent?
Startup/startupPast STARTINGHas initial boot completed?

Code Walkthrough

Step 1: Initial State

from nexus.probes import ProbeManager, ProbeResponse, ProbeState

probes = ProbeManager()
assert probes.state == ProbeState.STARTING
assert probes.is_alive is True
assert probes.is_ready is False

Step 2: Transition to READY

probes.mark_ready()

assert probes.check_liveness().http_status == 200
assert probes.check_readiness().http_status == 200
assert probes.check_startup().http_status == 200

Step 3: Custom Readiness Checks

def check_database_connection() -> bool:
    return True

def check_model_loaded() -> bool:
    return True

probes.add_readiness_check(check_database_connection)
probes.add_readiness_check(check_model_loaded)
assert probes.check_readiness().http_status == 200

Step 4: Failing Callback

probes2 = ProbeManager()
probes2.mark_ready()

def always_fails() -> bool:
    return False

probes2.add_readiness_check(always_fails)

readiness = probes2.check_readiness()
assert readiness.http_status == 503
assert "always_fails" in readiness.details["failed_checks"]

Step 5: DRAINING

probes.mark_draining()

assert probes.is_alive is True    # Don't restart
assert probes.is_ready is False   # Stop sending traffic
assert probes.check_liveness().http_status == 200
assert probes.check_readiness().http_status == 503

Step 6: FAILED and Recovery

probes3 = ProbeManager()
probes3.mark_ready()
probes3.mark_failed(reason="Out of memory")

assert probes3.is_alive is False
assert probes3.check_liveness().http_status == 503

# Recovery via reset
probes3.reset()
assert probes3.state == ProbeState.STARTING
probes3.mark_ready()
assert probes3.state == ProbeState.READY

Step 7: Invalid Transitions

probes4 = ProbeManager()

# Can't skip STARTING -> DRAINING
assert probes4.mark_draining() is False

# FAILED is terminal (without reset)
probes4.mark_failed(reason="crash")
assert probes4.mark_ready() is False
Important: Invalid state transitions return False without changing state. Always check the return value to confirm the transition succeeded.

Common Mistakes

MistakeProblemFix
Skipping STARTING → READYmark_draining() returns FalseMust go through READY first
Expecting recovery from FAILEDmark_ready() returns FalseUse reset() first
Readiness callback that raisesUnexpected behaviorCallbacks must return bool
Forgetting mark_ready()Readiness always 503Transition after init

Key Takeaways

  • ProbeState: STARTING → READY → DRAINING (FAILED is terminal).
  • ProbeManager is thread-safe with atomic state transitions.
  • Liveness: 200 unless FAILED. Readiness: 200 only in READY + callbacks pass.
  • Custom readiness checks via add_readiness_check(callback).
  • Invalid transitions return False without changing state.
  • reset() is the only recovery from FAILED.
