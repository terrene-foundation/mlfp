Chapter 7: Event Bus

Overview

The EventBus is Nexus's internal event system for event-driven architectures. It provides thread-safe publishing, async subscriptions with optional filtering, bounded history with LRU eviction, and round-trip serialization.

Concepts

Event Types

Event TypeValueWhen Published
HANDLER_REGISTERED"handler.registered"New handler added
HANDLER_CALLED"handler.called"Handler begins execution
HANDLER_COMPLETED"handler.completed"Handler finishes successfully
HANDLER_ERROR"handler.error"Handler throws an error
HEALTH_CHECK"health.check"Health check performed
CUSTOM"custom"Application-defined

Janus Queue Bridge

The EventBus uses janus.Queue to bridge synchronous publishers (any thread) and asynchronous consumers (SSE endpoint, subscribers). publish() writes to the sync side; the dispatch loop reads from the async side.

Code Walkthrough

Step 1: Create and Serialize Events

from nexus.events import EventBus, NexusEvent, NexusEventType

event = NexusEvent(
    event_type=NexusEventType.HANDLER_CALLED,
    handler_name="greet",
    request_id="req-001",
    data={"input_name": "Alice"},
)

event_dict = event.to_dict()
assert event_dict["event_type"] == "handler.called"

restored = NexusEvent.from_dict(event_dict)
assert restored.event_type == event.event_type

Step 2: Publish and Dispatch

import asyncio

bus = EventBus(capacity=128)

bus.publish(NexusEvent(
    event_type=NexusEventType.HANDLER_CALLED,
    handler_name="add",
    data={"a": 1, "b": 2},
))

bus.publish(NexusEvent(
    event_type=NexusEventType.HANDLER_COMPLETED,
    handler_name="add",
    data={"result": 3},
))

async def run_bus_briefly():
    await bus.start()
    await asyncio.sleep(0.1)
    await bus.stop()

asyncio.run(run_bus_briefly())

Step 3: Query History

all_history = bus.get_history()
assert len(all_history) >= 2

called_only = bus.get_history(event_type="handler.called")
assert all(e["type"] == "handler.called" for e in called_only)

recent_two = bus.get_history(limit=2)
assert len(recent_two) == 2

Step 4: Subscribe with Filters

q_all = bus.subscribe()
assert bus.subscriber_count == 1

q_errors = bus.subscribe_filtered(
    lambda evt: evt.event_type == NexusEventType.HANDLER_ERROR
)
assert bus.subscriber_count == 2

Step 5: Bounded History

small_bus = EventBus(capacity=3)

for i in range(5):
    small_bus._history.append(
        NexusEvent(event_type=NexusEventType.CUSTOM, data={"index": i})
    )

history = small_bus.get_history()
assert len(history) == 3
assert history[0]["data"]["index"] == 2  # Oldest surviving
assert history[2]["data"]["index"] == 4  # Most recent
Note: Nexus creates an EventBus internally (capacity=256). The HandlerRegistry publishes HANDLER_REGISTERED events automatically when handlers are added.

Common Mistakes

MistakeProblemFix
Querying history before dispatch loopHistory is emptyCall bus.start() first
Capacity too smallEvents evicted quicklyUse capacity ≥ 128
Forgetting await on start/stopCoroutine never runsBoth are async methods

Key Takeaways

  • NexusEventType is a typed enum with dotted string values matching the Rust SDK.
  • NexusEvent provides to_dict()/from_dict() for round-trip serialization.
  • publish() is sync and thread-safe; works before the async loop starts.
  • subscribe() and subscribe_filtered() return async queues.
  • get_history() reads from the bounded deque with optional filters.
