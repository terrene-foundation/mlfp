Chapter 8: HyperparameterSearch

Overview

HyperparameterSearch wraps TrainingPipeline to systematically explore hyperparameter combinations. Define a SearchSpace, choose a strategy (grid, random, Bayesian), and call search(). The engine runs multiple trials and identifies the best configuration.

Concepts

ParamDistribution Types

TypeBehaviorUse Case
uniformContinuous float [low, high]General hyperparameters
log_uniformLog-scale sampling [low, high]Learning rates
int_uniformInteger [low, high]n_estimators, max_depth
categoricalDraws from choices listSolvers, activations

Search Strategies

StrategyBehavior
gridExhaustive enumeration (ignores n_trials)
randomSamples n_trials random combinations
bayesianPast results guide future sampling
Grid vs random: Grid search is exhaustive but expensive. Research shows random search often finds equally good results with fewer trials, especially when only a few parameters matter.

Key API Reference

Class / MethodPurpose
HyperparameterSearch(pipeline)Create with a TrainingPipeline

Code Walkthrough

Step 1: Define SearchSpace

from kailash_ml.engines.hyperparameter_search import (
    SearchSpace, ParamDistribution, SearchConfig,
)

space = SearchSpace(params=[
    ParamDistribution(name="n_estimators", type="int_uniform", low=5, high=30),
    ParamDistribution(name="max_depth", type="categorical", choices=[3, 5, 7]),
])

Step 2: Preview Samples

# Exhaustive grid
grid = space.sample_grid()

# Random samples
random_samples = space.sample_random(n=10)
for s in random_samples:
    assert 5 <= s["n_estimators"] <= 30
    assert s["max_depth"] in [3, 5, 7]

Step 3: Run Random Search

config = SearchConfig(
    strategy="random", n_trials=3,
    metric_to_optimize="accuracy",
    direction="maximize",
)

result = await search.search(
    data=df, schema=schema,
    base_model_spec=base_model_spec,
    search_space=space, config=config,
    eval_spec=eval_spec,
    experiment_name="random_search",
)

assert len(result.all_trials) == 3
assert "accuracy" in result.best_metrics

Step 4: Grid Search

grid_config = SearchConfig(
    strategy="grid", n_trials=10,  # Ignored for grid
    metric_to_optimize="accuracy", direction="maximize",
)

grid_result = await search.search(

Common Mistakes

MistakeWhat HappensFix
Grid with continuous paramsEnormous gridUse categorical or switch to random
Invalid strategy nameValueErrorUse grid, random, or bayesian
Mixing up maximize/minimizeWorst trial selected as bestMatch direction to metric semantics

Key Takeaways

  • Four distribution types cover numeric and categorical parameters.
  • Grid search is exhaustive; random search is often more efficient.
  • sample_grid() and sample_random() let you preview before committing.
  • Direction (maximize/minimize) determines best trial selection.
  • Grid search ignores n_trials — it always evaluates the full grid.
