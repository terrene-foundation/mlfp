The Story That Starts Everything

FOUNDATIONS

Singapore HDB Resale Flash Crash — Q4 2023

  • October 2023: HDB resale prices had risen 10 consecutive quarters
  • Then a sudden anomaly — median prices in three mature estates dropped 8-12% in a single month
  • Property portals and dashboards showed nothing unusual
  • The anomaly was only caught by a data analyst who ran exploratory analysis on raw transaction data
The Dashboard Said Everything Was Fine.
Aggregated dashboards masked the signal. Monthly medians smoothed out the crash. Only row-level EDA on the 15 million transaction dataset revealed the pattern.

What Went Wrong?

FOUNDATIONS

Dashboard Blindness

  • Aggregation hides outliers — a mean over 50,000 transactions masks 200 anomalous ones
  • Pre-built charts answer yesterday's questions — they cannot ask new ones
  • No distribution view — the dashboard showed a single number, not a histogram

What EDA Would Have Caught

  • A bimodal distribution in the price column for those estates
  • Temporal clustering — the low-price transactions all occurred in a 2-week window
  • Correlation with flat type — only 3-room and 4-room flats were affected
Lesson: EDA catches what dashboards miss. This is why you learn to explore data yourself.

This Affected YOUR Property Value

FOUNDATIONS

  • Banks used dashboard-reported medians for mortgage valuations
  • Some homeowners were denied refinancing because their property appeared to have lost value
  • The root cause? A batch of subsidised transfers entered the public dataset without proper classification
By the end of today, you will be able to load this exact dataset, filter for those estates, plot the distribution, and identify the anomaly yourself.

Functions: Reusable Code

FOUNDATIONS

Why Functions?

You will compute price-per-sqm for many towns. Instead of copy-pasting, write it once as a function.

def price_stats(df: pl.DataFrame, town: str) -> dict:
    """Compute price statistics for a given town."""
    filtered = df.filter(pl.col("town") == town)
    return {
        "town": town,
        "count": len(filtered),
        "median_price": filtered["resale_price"].median(),
        "mean_psm": (
            filtered["resale_price"] / filtered["floor_area_sqm"]
        ).mean(),
    }

# Use it
stats = price_stats(df, "QUEENSTOWN")
print(stats)

Anatomy of a Function

def function_name(param1, param2):
    """Docstring — what does it do?"""
    # Body — the instructions
    result = param1 + param2
    return result  # Send back the answer
  • def — "I am defining a function"
  • param1, param2 — inputs (arguments)
  • return — the output
  • The body is indented (4 spaces)
A function is a recipe. You write it once, then call it with different ingredients.

From Histograms to Probability

THEORY

The Histogram Is a Rough Draft

A histogram bins continuous data into discrete buckets. As you increase bins and increase data, the histogram converges to a smooth curve.

That smooth curve is the probability density function (PDF).

\[ P(a \leq X \leq b) = \int_a^b f(x)\, dx \]
The probability that X falls between a and b is the area under the curve f(x) between those points.
Plain English: The taller the curve at a point, the more likely values are near there. The total area under the entire curve equals 1 (100%).

The Normal Distribution

THEORY

The Bell Curve

The most important distribution in statistics. Many natural phenomena cluster around a central value with symmetric tails.

\[ f(x) = \frac{1}{\sqrt{2\pi} \cdot \color{#D97706}{\sigma}} \exp\left(-\frac{(x - \color{#DC2626}{\mu})^2}{2\color{#D97706}{\sigma}^2}\right) \]
μ = mean (centre), σ = standard deviation (spread)

What the Parameters Control

  • μ shifts the curve left or right — where is the centre?
  • σ stretches or compresses — how spread out?
  • 68% of data falls within ±1σ of μ
  • 95% falls within ±2σ
  • 99.7% falls within ±3σ
HDB context: If resale prices were normally distributed with μ = $500K and σ = $80K, then 95% of prices would fall between $340K and $660K.

Beyond Normal: Poisson and Exponential

THEORY

Poisson: Counting Events

"How many HDB transactions happen per day?"

\[ P(X = k) = \frac{\color{#D97706}{\lambda}^k e^{-\color{#D97706}{\lambda}}}{k!} \]
λ = average rate (e.g., 45 transactions per day).
k = the specific count you are asking about.
  • Counts of rare events in fixed intervals
  • Only one parameter: λ (rate)
  • Mean = Variance = λ

Exponential: Time Between Events

"How long until the next transaction?"

\[ f(x) = \color{#D97706}{\lambda} e^{-\color{#D97706}{\lambda} x}, \quad x \geq 0 \]
λ = rate. Mean waiting time = 1/λ.
  • The Poisson and Exponential are two sides of the same coin
  • Memoryless: the time you have already waited tells you nothing about how long you will still wait

The Exponential Family

THEORY

One Formula to Rule Them All

Normal, Poisson, Exponential, Bernoulli, Gamma, Beta — all are members of the exponential family.

\[ f(x \mid \color{#D97706}{\theta}) = h(x) \cdot \exp\!\Big(\color{#D97706}{\eta(\theta)}^\top \color{#0D9488}{T(x)} - A(\color{#D97706}{\theta})\Big) \]
η(θ) = natural parameter, T(x) = sufficient statistic, A(θ) = log-partition function, h(x) = base measure

Why Does This Matter?

  • T(x) is a sufficient statistic — it captures all information in the data about θ
  • MLE has a closed-form solution for exponential family distributions
  • Generalised Linear Models (GLMs) use the exponential family directly
Advanced: The log-partition function A(θ) generates all moments: A'(θ) = E[T(X)] and A''(θ) = Var[T(X)]. It is the cumulant generating function.

Sufficient Statistics: A Deeper Look

THEORY

What Does "Sufficient" Mean?

A statistic T(x) is sufficient for θ if, once you know T(x), the data tells you nothing more about θ.

\[ P(X \mid \color{#0D9488}{T(X)} = t) \text{ does not depend on } \color{#D97706}{\theta} \]
Given the sufficient statistic, the conditional distribution of the data is parameter-free.

Example: Normal Distribution

For \(X_1, \ldots, X_n \sim \mathcal{N}(\mu, \sigma^2)\) with known \(\sigma^2\):

  • The sufficient statistic for μ is \(\bar{X} = \frac{1}{n}\sum_{i=1}^n X_i\)
  • You do not need the individual data points — just the sample mean
  • This is why df.describe() is so powerful: the summary statistics often are the sufficient statistics
Fisher-Neyman factorisation theorem: T(x) is sufficient for θ iff the likelihood can be written as g(T(x), θ) · h(x).

Maximum Likelihood Estimation (MLE)

THEORY

The Big Question: Given the Data, What Are the Best Parameters?

Plain English: You have a bunch of HDB prices. You believe they follow a Normal distribution. MLE asks: "Which μ and σ make this exact data most likely to have occurred?"

The likelihood function is the probability of observing your data, viewed as a function of the parameters:

\[ \mathcal{L}(\color{#D97706}{\theta} \mid x_1, \ldots, x_n) = \prod_{i=1}^n f(x_i \mid \color{#D97706}{\theta}) \]
Multiply the probability of each observation. θ that maximises this is the MLE.

Why the Log?

Products of thousands of tiny probabilities cause numerical underflow. Taking the log turns products into sums:

\[ \ell(\color{#D97706}{\theta}) = \sum_{i=1}^n \log f(x_i \mid \color{#D97706}{\theta}) \]
The log-likelihood. Maximising ℓ(θ) is equivalent to maximising L(θ).

MLE for the Normal Distribution — Step by Step

THEORY

Deriving the Estimators

Step 1: Write the log-likelihood for \(X_i \sim \mathcal{N}(\mu, \sigma^2)\):

\[ \ell(\color{#DC2626}{\mu}, \color{#D97706}{\sigma^2}) = -\frac{n}{2}\log(2\pi) - \frac{n}{2}\log(\color{#D97706}{\sigma^2}) - \frac{1}{2\color{#D97706}{\sigma^2}} \sum_{i=1}^n (x_i - \color{#DC2626}{\mu})^2 \]
Each term has a job: constant, spread penalty, fit penalty.

Step 2: Differentiate with respect to μ and set to zero:

\[ \frac{\partial \ell}{\partial \color{#DC2626}{\mu}} = \frac{1}{\color{#D97706}{\sigma^2}} \sum_{i=1}^n (x_i - \color{#DC2626}{\mu}) = 0 \implies \hat{\color{#DC2626}{\mu}}_{\text{MLE}} = \bar{x} = \frac{1}{n}\sum_{i=1}^n x_i \]
The MLE of the mean is the sample mean. Intuitive and exact.

MLE for the Normal (continued)

THEORY

Step 3: Differentiate with respect to σ² and set to zero:

\[ \frac{\partial \ell}{\partial \color{#D97706}{\sigma^2}} = -\frac{n}{2\color{#D97706}{\sigma^2}} + \frac{1}{2(\color{#D97706}{\sigma^2})^2} \sum_{i=1}^n (x_i - \color{#DC2626}{\mu})^2 = 0 \] \[ \implies \hat{\color{#D97706}{\sigma^2}}_{\text{MLE}} = \frac{1}{n}\sum_{i=1}^n (x_i - \bar{x})^2 \]
Note: MLE uses n, not n−1. The MLE is biased — it slightly underestimates the true variance.
Production gotcha: Polars .var() uses n−1 (Bessel's correction) by default. The MLE and the sample variance differ by a factor of n/(n−1). For large n, the difference is negligible. For small samples, it matters. 
# In Polars — default uses n-1 (unbiased)
sample_var = df["resale_price"].var()

# MLE variance (uses n)
n = len(df)
mle_var = df["resale_price"].var() * (n - 1) / n

Fisher Information

THEORY

How Much Does the Data Tell You?

Plain English: Fisher Information measures how "sharp" the likelihood peak is. A sharp peak = the data strongly points to one parameter value. A flat peak = the data is ambiguous.
\[ \mathcal{I}(\color{#D97706}{\theta}) = -E\!\left[\frac{\partial^2 \ell}{\partial \color{#D97706}{\theta}^2}\right] = E\!\left[\left(\frac{\partial \ell}{\partial \color{#D97706}{\theta}}\right)^2\right] \]
The expected curvature of the log-likelihood at θ.

For the Normal (known σ²)

\[ \mathcal{I}(\mu) = \frac{n}{\sigma^2} \]
More data (larger n) or less noise (smaller σ²) = more information.

Cramér-Rao Bound

\[ \text{Var}(\hat{\theta}) \geq \frac{1}{\mathcal{I}(\theta)} \]
No unbiased estimator can have lower variance than 1/I(θ). The MLE achieves this bound asymptotically.

MLE Properties

THEORY

Why MLE Is the Default

PropertyMeaning
Consistent\(\hat{\theta}_n \xrightarrow{p} \theta\) as \(n \to \infty\)
Asymptotically Normal\(\sqrt{n}(\hat{\theta} - \theta) \xrightarrow{d} \mathcal{N}(0, \mathcal{I}(\theta)^{-1})\)
Asymptotically EfficientAchieves the Cramér-Rao bound as \(n \to \infty\)
InvariantMLE of \(g(\theta)\) is \(g(\hat{\theta})\)
Advanced — When MLE Fails:
  • Small samples: MLE can be biased and high-variance
  • Misspecified model: MLE converges to the "closest" distribution in KL divergence — which may not be useful
  • Multimodal likelihood: gradient methods find local optima
  • Infinite parameters: non-parametric MLE (kernel density) needs bandwidth selection

Information Geometry

ADVANCED

Distributions Live on a Curved Surface

The family of Normal distributions \(\{\mathcal{N}(\mu, \sigma^2)\}\) is a 2D surface (a manifold) parameterised by (μ, σ²).

The Fisher Information Matrix defines a Riemannian metric on this manifold:

\[ g_{ij}(\theta) = \mathcal{I}_{ij}(\theta) = E\!\left[\frac{\partial \ell}{\partial \theta_i} \frac{\partial \ell}{\partial \theta_j}\right] \]
The Fisher metric measures "distance" between nearby distributions.

Why This Matters

  • The "natural" distance between distributions is not Euclidean — moving from σ=1 to σ=2 is a bigger change than from σ=100 to σ=101
  • Natural gradient descent (used in modern RL and deep learning) follows the Fisher metric, not the Euclidean gradient
  • KL divergence is the local quadratic approximation of this Riemannian distance
Connection to Module 6: PPO and TRPO (reinforcement learning) use KL divergence constraints — they are implicitly doing information geometry on the policy manifold.

Bayesian Thinking: Prior to Posterior

THEORY
Plain English: You have a belief about HDB prices before looking at data (the prior). You observe data. Your updated belief is the posterior. Bayesian inference is the principled way to update beliefs with evidence.
\[ \underbrace{P(\color{#D97706}{\theta} \mid \color{#0D9488}{X})}_{\text{posterior}} = \frac{\overbrace{P(\color{#0D9488}{X} \mid \color{#D97706}{\theta})}^{\text{likelihood}} \cdot \overbrace{P(\color{#D97706}{\theta})}^{\text{prior}}}{\underbrace{P(\color{#0D9488}{X})}_{\text{evidence}}} \]
θ = parameter, X = observed data. Evidence is a normalising constant.

The Three Ingredients

  • Prior P(θ) — what you believed before seeing data. "HDB prices are probably between $300K and $800K"
  • Likelihood P(X|θ) — how probable is this data given a particular parameter value?
  • Posterior P(θ|X) — your updated belief after seeing the data
The posterior is always a compromise between prior and data. With lots of data, the posterior is dominated by the likelihood. With little data, the prior matters more.

Conjugate Priors: When the Math Works Out

THEORY

Normal-Normal Conjugacy

Prior: \(\mu \sim \mathcal{N}(\mu_0, \sigma_0^2)\), Likelihood: \(X_i \sim \mathcal{N}(\mu, \sigma^2)\) (known σ²)

\[ \mu \mid X \sim \mathcal{N}\!\left(\frac{\frac{\mu_0}{\sigma_0^2} + \frac{n\bar{x}}{\sigma^2}}{\frac{1}{\sigma_0^2} + \frac{n}{\sigma^2}}, \; \frac{1}{\frac{1}{\sigma_0^2} + \frac{n}{\sigma^2}}\right) \]
The posterior mean is a precision-weighted average of the prior mean and sample mean.

Beta-Binomial Conjugacy

"What fraction of HDB sales in Queenstown are 4-room flats?"

Prior: \(p \sim \text{Beta}(\alpha, \beta)\), Likelihood: \(k \sim \text{Binomial}(n, p)\)

\[ p \mid k \sim \text{Beta}(\alpha + k, \; \beta + n - k) \]
Just add successes to α and failures to β. Beautifully simple.
Why "conjugate"? Prior and posterior have the same distribution family. The data only updates the parameters. This makes computation exact — no approximations needed.

MAP Estimation

THEORY

Maximum A Posteriori = MLE + Prior

Instead of the full posterior, sometimes you just want the single most likely parameter value given the prior.

\[ \hat{\theta}_{\text{MAP}} = \arg\max_\theta \; \underbrace{\log P(X \mid \theta)}_{\text{log-likelihood}} + \underbrace{\log P(\theta)}_{\text{log-prior}} \]
MAP = MLE with a regularisation term. The prior penalises extreme parameter values.

Connection to Regularisation

  • Gaussian prior on θ → L2 regularisation (Ridge)
  • Laplace prior on θ → L1 regularisation (Lasso)
  • Uniform prior → MAP = MLE (no regularisation)
Module 3 preview: When you learn Ridge and Lasso regression, remember — they are Bayesian MAP estimates with different priors. The "regularisation strength" λ controls how much you trust the prior vs the data.

Bayesian Predictive Distribution

THEORY

Predicting the Next Observation

MLE gives you a point estimate. Bayesian inference gives you a distribution over predictions:

\[ P(\tilde{x} \mid X) = \int P(\tilde{x} \mid \theta) \, P(\theta \mid X) \, d\theta \]
Average over all possible parameter values, weighted by how likely each is (the posterior).

Why This Matters

  • MLE says: "The next HDB price will be $450K"
  • Bayesian says: "The next HDB price will be $450K ± $35K (with 95% probability)"
  • The predictive distribution captures parameter uncertainty — how unsure you are about μ and σ
Advanced — Bayesian Model Comparison: BIC approximates the log-evidence. WAIC and LOO-CV are better in practice. BIC overpenalises complex models; LOO-CV adapts to model misspecification.

Hypothesis Testing: Making Decisions from Data

THEORY
Plain English: "Are Queenstown HDB prices really higher than Yishun, or is the difference just random noise?" Hypothesis testing gives you a framework to answer this rigorously.

The Setup

  • H0 (null): There is no difference. Queenstown = Yishun.
  • H1 (alternative): There is a difference.
  • Collect data. Compute a test statistic. Ask: "How surprising is this result if H0 were true?"

Two Types of Errors

H0 TrueH0 False
Reject H0Type I (α)Correct!
Fail to rejectCorrect!Type II (β)
  • α = probability of false alarm (usually 0.05)
  • β = probability of missing a real effect
  • Power = 1 − β = probability of detecting a real effect

The Neyman-Pearson Framework

THEORY

The Likelihood Ratio Test

\[ \Lambda = \frac{L(\theta_0 \mid X)}{L(\hat{\theta} \mid X)} = \frac{\text{likelihood under } H_0}{\text{likelihood under best fit}} \]
Small Λ = data fits H0 poorly relative to the best possible fit → reject H0.

Neyman-Pearson Lemma: The likelihood ratio test is the most powerful test at any given significance level α.

The p-value

\[ p = P(\text{test statistic} \geq \text{observed} \mid H_0 \text{ true}) \]
The probability of seeing a result this extreme or more, assuming no effect exists.
What the p-value is NOT:
  • NOT the probability that H0 is true
  • NOT the probability of a false positive
  • NOT a measure of effect size
p < 0.05 does not mean "the effect is real." It means "the data is surprising under H0."

Power Analysis

THEORY

How Much Data Do You Need?

Before running an experiment, you must determine the sample size needed to detect a meaningful effect.

\[ n = \left(\frac{z_{1-\alpha/2} + z_{1-\beta}}{\delta / \sigma}\right)^2 \]
α = significance level, β = Type II error rate, δ = minimum detectable effect, σ = standard deviation

Singapore Example

"Is the average HDB price in Queenstown at least $20K higher than Yishun?"

# Rough power calculation
import math
alpha, power = 0.05, 0.80
z_alpha = 1.96   # two-sided
z_beta  = 0.84   # 80% power
delta   = 20_000  # $20K difference
sigma   = 80_000  # price std dev

n = ((z_alpha + z_beta) / (delta / sigma)) ** 2
print(f"Need {math.ceil(n)} transactions per group")
# Need ~126 transactions per group
FDR correction: Testing many towns at once? Bonferroni divides α by the number of tests. Benjamini-Hochberg controls the false discovery rate — less conservative, more powerful.

Permutation Tests vs Parametric Tests

ADVANCED

When You Cannot Assume a Distribution

Parametric tests (t-test, z-test) assume data follows a specific distribution. What if it does not?

Permutation test idea: If there is no real difference between Queenstown and Yishun, then shuffling the town labels should not change the result. Do this 10,000 times and see how extreme your actual result is. 
import numpy as np

observed_diff = qt_prices.mean() - ys_prices.mean()
combined = np.concatenate([qt_prices, ys_prices])
n_qt = len(qt_prices)

# Permutation test
perm_diffs = []
for _ in range(10_000):
    np.random.shuffle(combined)
    perm_diff = combined[:n_qt].mean() - combined[n_qt:].mean()
    perm_diffs.append(perm_diff)

p_value = np.mean(np.abs(perm_diffs) >= np.abs(observed_diff))
print(f"Permutation p-value: {p_value:.4f}")
Permutation tests are exact — no distributional assumptions. But they only test the null of exchangeability, not a specific alternative.

Bootstrap: Estimating Uncertainty Without Formulas

THEORY
Plain English: You have one sample. You want to know how precise your estimate is. The bootstrap idea: treat your sample as if it were the population, then resample from it thousands of times to see how much your estimate varies.

Efron's Insight (1979)

You cannot get more data from the population. But you can simulate having more data by resampling with replacement from your existing sample.

# Bootstrap the median HDB price
prices = df["resale_price"].to_numpy()
n_boot = 10_000
boot_medians = []

for _ in range(n_boot):
    resample = np.random.choice(prices, size=len(prices),
                                replace=True)
    boot_medians.append(np.median(resample))

# 95% confidence interval
ci_low  = np.percentile(boot_medians, 2.5)
ci_high = np.percentile(boot_medians, 97.5)
print(f"95% CI: ${ci_low:,.0f} to ${ci_high:,.0f}")

BCa Bootstrap Intervals

THEORY

Better Than Percentile Intervals

The simple percentile method assumes the bootstrap distribution is symmetric and centred. BCa (bias-corrected and accelerated) fixes both issues:

\[ \alpha_1 = \Phi\!\left(\hat{z}_0 + \frac{\hat{z}_0 + z_{\alpha/2}}{1 - \hat{a}(\hat{z}_0 + z_{\alpha/2})}\right) \]
&zcirc;0 = bias correction (how far the bootstrap median is from the estimate). â = acceleration (how fast the variance changes with the parameter).

When Bootstrap Fails

The bootstrap is not magic. It fails when:
  • Extreme quantiles — bootstrapping the maximum of a distribution is unreliable
  • Heavy tails — if the variance does not exist (e.g., Cauchy), the bootstrap CLT does not apply
  • Dependent data — time series requires block bootstrap or stationary bootstrap
Advanced — Wild Bootstrap: For heteroscedastic data (variance changes across groups), the standard bootstrap is inconsistent. The wild bootstrap resamples residuals with random sign flips, preserving the heteroscedastic structure.

Variance: Measuring Spread

THEORY
Plain English: The mean tells you where the centre is. Variance tells you how spread out the data is around that centre.
\[ \text{Var}(X) = \color{#D97706}{\sigma^2} = E\!\left[(X - \color{#DC2626}{\mu})^2\right] = E[X^2] - (E[X])^2 \]
Average of squared deviations from the mean. σ (the square root) is the standard deviation.

In Polars

# Sample variance (n-1, Bessel-corrected)
var = df["resale_price"].var()
std = df["resale_price"].std()

# Per-town variance
town_spread = (
    df.group_by("town")
    .agg(
        pl.col("resale_price").std().alias("price_std"),
        pl.col("resale_price").mean().alias("price_mean"),
    )
    .with_columns(
        (pl.col("price_std") / pl.col("price_mean"))
        .alias("cv")  # Coefficient of Variation
    )
)
CV (Coefficient of Variation) = σ/μ. Use it to compare spread across groups with different means.

Skewness and Kurtosis

THEORY

Skewness: Asymmetry

\[ \gamma_1 = E\!\left[\left(\frac{X - \mu}{\sigma}\right)^3\right] \]
Positive: right tail is longer (e.g., HDB prices).
Negative: left tail is longer.
Zero: symmetric.
  • HDB resale prices are right-skewed: most are $300K–$600K, but a few reach $1.5M+
  • The mean is pulled toward the tail; the median stays put

Kurtosis: Tail Heaviness

\[ \gamma_2 = E\!\left[\left(\frac{X - \mu}{\sigma}\right)^4\right] - 3 \]
Excess kurtosis (−3 adjustment makes Normal = 0).
Positive: heavier tails than Normal (more extreme values).
Negative: lighter tails.
Finance caution: Financial returns often have high kurtosis. The Normal distribution dramatically underestimates the probability of crashes. This is why Black-Scholes fails.

Correlation: Beyond Pearson

THEORY

Pearson Correlation

\[ r = \frac{\sum_{i=1}^n (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum_{i=1}^n (x_i - \bar{x})^2 \cdot \sum_{i=1}^n (y_i - \bar{y})^2}} \]
Measures linear relationship. Ranges from −1 to +1.
Pearson misses non-linear relationships entirely. A perfect parabola has r = 0.

Alternatives

MethodMeasuresUse When
SpearmanMonotonic relationshipOrdinal data, non-linear monotonic
KendallConcordanceSmall samples, many ties

The Central Limit Theorem

THEORY
Plain English: Take any distribution. Sample from it repeatedly. The average of your samples will be approximately Normal — regardless of the original distribution's shape. The more samples, the better the approximation.
\[ \bar{X}_n = \frac{1}{n}\sum_{i=1}^n X_i \xrightarrow{d} \mathcal{N}\!\left(\mu, \frac{\sigma^2}{n}\right) \]
As n grows, the sample mean converges to a Normal distribution with variance shrinking by 1/n.

Why This Matters

  • HDB prices are not Normal (right-skewed)
  • But the average of 100 randomly selected HDB prices is approximately Normal
  • This is why confidence intervals and hypothesis tests work — they operate on averages, not individual observations
How fast? Berry-Esséen theorem: the convergence rate is \(O(1/\sqrt{n})\). For heavy-tailed distributions, convergence is slow — the bootstrap may be more reliable than CLT-based intervals.

Covariance and the Covariance Matrix

THEORY

Covariance: Unstandardised Correlation

\[ \text{Cov}(X, Y) = E[(X - \mu_X)(Y - \mu_Y)] \]
Positive: X and Y increase together.
Negative: one increases while the other decreases.
Zero: no linear relationship.

Correlation is standardised covariance: \(r = \frac{\text{Cov}(X,Y)}{\sigma_X \sigma_Y}\)

The Covariance Matrix

\[ \Sigma = \begin{pmatrix} \text{Var}(X_1) & \text{Cov}(X_1, X_2) & \cdots \\ \text{Cov}(X_2, X_1) & \text{Var}(X_2) & \cdots \\ \vdots & \vdots & \ddots \end{pmatrix} \]
Symmetric, positive semi-definite. Diagonal = variances. Off-diagonal = covariances.
Module 4 preview: PCA decomposes the covariance matrix into eigenvectors (principal components) and eigenvalues (variance explained). The covariance matrix is the foundation of dimensionality reduction.

Law of Large Numbers

THEORY
Plain English: The more data you collect, the closer your sample average gets to the true population average. With enough HDB transactions, the sample mean converges to the true mean.
\[ \bar{X}_n \xrightarrow{a.s.} \mu \quad \text{as } n \to \infty \]
Strong Law: the sample mean converges almost surely to the population mean.

Weak vs Strong

VersionConvergenceRequirement
Weak LLNIn probabilityFinite variance
Strong LLNAlmost surelyIID, finite mean
The gambler's fallacy violates this. LLN says the average converges — it does NOT say that short-run deviations will be "corrected." Each coin flip is independent.

Bayesian Model Comparison

ADVANCED

BIC vs WAIC vs LOO-CV

CriterionFormulaStrength
BIC \(-2\ell(\hat{\theta}) + k \log n\) Simple, consistent
WAIC \(-2 \sum \log \overline{p}_i + 2 p_{\text{waic}}\) Fully Bayesian
LOO-CV \(\sum \log p(y_i \mid y_{-i})\) Robust to misspecification

When to Use Each

  • BIC: quick screening, well-specified models, large n
  • WAIC: Bayesian models, posterior is available, want a single number
  • LOO-CV (Pareto-smoothed): gold standard for Bayesian model comparison; robust when WAIC fails
BIC approximates the log marginal likelihood. It over-penalises complexity for small n. WAIC and LOO-CV estimate out-of-sample predictive accuracy — a more directly useful quantity.

False Discovery Rate

ADVANCED

The Multiple Testing Problem

Testing 26 towns at α = 0.05: you expect 1.3 false positives even if no effect exists anywhere.

\[ \text{FDR} = E\!\left[\frac{\text{false discoveries}}{\text{total discoveries}}\right] \]
Control the proportion of false positives among your significant results.

Benjamini-Hochberg Procedure

  1. Sort all m p-values: \(p_{(1)} \leq p_{(2)} \leq \cdots \leq p_{(m)}\)
  2. Find the largest k where \(p_{(k)} \leq \frac{k}{m} \cdot q\)
  3. Reject all hypotheses with \(p_{(i)} \leq p_{(k)}\)
# BH correction in NumPy
import numpy as np
p_values = np.array([0.001, 0.013, 0.029, 0.043, 0.15])
m = len(p_values)
q = 0.05  # FDR level

sorted_idx = np.argsort(p_values)
thresholds = q * np.arange(1, m + 1) / m
# Compare sorted p-values to thresholds

Wild Bootstrap for Heteroscedastic Data

ADVANCED

When Standard Bootstrap Fails

If variance differs across groups (heteroscedasticity), the standard bootstrap underestimates uncertainty for high-variance groups and overestimates for low-variance groups.

The Wild Bootstrap

  1. Fit the model: \(y_i = X_i \hat{\beta} + \hat{e}_i\)
  2. Generate \(y_i^* = X_i \hat{\beta} + \hat{e}_i \cdot v_i\)
  3. Where \(v_i\) is a random variable with \(E[v] = 0\), \(E[v^2] = 1\)
  4. Common choice: Rademacher (\(v_i = \pm 1\) with equal probability)

Singapore Application

HDB price variance differs enormously by town:

  • Bukit Timah: σ = $180K (huge spread)
  • Woodlands: σ = $45K (tight cluster)

Bootstrapping the price difference between these towns with a standard bootstrap gives incorrect confidence intervals. The wild bootstrap preserves each town's variance structure.

Pairs bootstrap (resample observation pairs) is a simpler alternative that is heteroscedasticity-consistent but less efficient than wild bootstrap.

Production Gotcha: Simpson's Paradox

THEORY

A Trend That Reverses When You Split by Group

HDB example: Overall, HDB prices appear to be falling this quarter. But when you split by town, every single town shows prices rising. How? The mix shifted — more transactions in cheaper towns this quarter. 
# Overall trend: looks like decline
overall = df.group_by("quarter").agg(
    pl.col("resale_price").mean()
)

# Per-town trend: every town is rising!
per_town = df.group_by("quarter", "town").agg(
    pl.col("resale_price").mean()
)

# The paradox: composition shift
mix = df.group_by("quarter", "town").agg(
    pl.col("resale_price").count().alias("n")
)
# More cheap-town transactions this quarter
Always stratify. An aggregate trend can be the opposite of every subgroup trend.

Production Gotcha: Survivorship Bias

THEORY

You Only See the Data That Survived

Plain English: Looking only at successful companies teaches you nothing about what causes success — because you never see the companies that did the same things and failed.
  • HDB example: Analysing only completed transactions ignores withdrawn listings — which may have been withdrawn because the price was too high
  • Your dataset is a survivor sample — it only includes flats that actually sold

How to Detect It

  • Ask: "What data is missing from this dataset?"
  • Ask: "What selection process created this dataset?"
  • Compare your sample to known population statistics
Formally: Survivorship bias is a form of selection bias where the sample is conditioned on the outcome variable. This violates the assumption that data was randomly drawn from the population.

If You Followed the Maths

  • Fisher Information measures how much data tells you about parameters
  • Conjugate priors give closed-form Bayesian updates
  • p-values are NOT the probability that H0 is true
  • Bootstrap resamples with replacement to estimate uncertainty

Synthesis: Expert Takeaways

For the Experts

  • The Fisher metric defines a Riemannian geometry on the distribution manifold — KL divergence is its local approximation
  • BIC approximates log-evidence; LOO-CV is more robust to misspecification
  • Permutation tests require only exchangeability, not distributional assumptions
  • Wild bootstrap preserves heteroscedastic structure where standard bootstrap fails
  • BCa intervals correct for bias and acceleration in the bootstrap distribution
