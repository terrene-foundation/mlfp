Chapter 4: LLM Providers

Overview

Kaizen supports multiple LLM providers through environment-variable-based configuration. This chapter covers provider detection from model name prefixes, API key pairing, cost estimation, and the security pattern for managing secrets.

Code Walkthrough

Step 1: Model from Environment

import os
from dotenv import load_dotenv
load_dotenv()

model = os.environ.get("DEFAULT_LLM_MODEL", "claude-sonnet-4-20250514")

Step 2: Provider Detection

def detect_provider(model_name: str) -> str:
    if model_name.startswith("claude-"):
        return "anthropic"
    elif model_name.startswith(("gpt-", "o1", "o3", "o4")):
        return "openai"
    elif model_name.startswith("gemini-"):
        return "google"
    return "unknown"

assert detect_provider("claude-sonnet-4-20250514") == "anthropic"
assert detect_provider("gpt-4o") == "openai"
assert detect_provider("gemini-pro") == "google"
Security: API keys must never appear in source code. Always use os.environ with load_dotenv(), and ensure .env is in .gitignore.

Key Takeaways

  • Provider detected from model name prefix automatically.
  • API keys always from environment variables, never hardcoded.
  • .env files must be in .gitignore.
  • Cost estimation uses per-1M-token rates for budget enforcement.
  • The SDK reads keys at runtime, not at import time.
