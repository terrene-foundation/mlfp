Bootstrapping and Hypothesis Testing

Resampling · p-values · Power · Multiple testing · Permutation tests

⏱ 45 minutes   ·   Prereq: 2.2 (sampling distributions, CLT)

The Bootstrap: sampling without extra samples

Problem: you have one sample, $n$ observations. You want the sampling distribution of some statistic. You cannot go get more data.

Solution: treat your sample as if it were the population. Resample with replacement. Compute the statistic. Repeat thousands of times.

It feels illegal. It is mathematically justified. And it works for statistics (median, correlation, regression coefficients) where no clean formula exists.

The resampling loop

Percentile CI (the simple version)

\[ \text{CI}_{95\%} = \left[\hat\theta^*_{(0.025)},\; \hat\theta^*_{(0.975)}\right] \] 

rng = np.random.default_rng(42)
B = 5000
reps = np.empty(B)
for b in range(B):
    resample = rng.choice(data, len(data), replace=True)
    reps[b] = np.median(resample)

lo, hi = np.percentile(reps, [2.5, 97.5])

Read the 2.5th and 97.5th percentiles straight off the empirical distribution.

BCa: when percentile is biased

The percentile method is biased when the sampling distribution is skewed. BCa corrects two things:

  • Bias $\hat z_0$ — is the median of the bootstrap distribution equal to $\hat\theta$?
  • Acceleration $\hat a$ — does skewness change with $\theta$? (jackknife)
\[ \alpha_i = \Phi\!\left(\hat z_0 + \frac{\hat z_0 + z_{\alpha_i'}}{1 - \hat a(\hat z_0 + z_{\alpha_i'})}\right) \]

BCa in scipy


from scipy.stats import bootstrap
res = bootstrap((data,), np.median, method="BCa",
                confidence_level=0.95, n_resamples=5000)
print(res.confidence_interval)

One line replaces the manual loop. Use "BCa" for skewed statistics (median, trimmed mean, ratio). Use "percentile" when the bootstrap distribution is approximately symmetric.

Hypothesis testing in one picture

\[ p \;=\; P\!\left(T \ge T_{\text{obs}} \;\middle|\; H_0\right) \]

The $t$-statistic and its distribution

\[ T = \frac{\bar x - \mu_0}{s / \sqrt n} \sim t_{n-1} \quad \text{under } H_0 \]

Large $T$ → observed mean is far from the null hypothesis → small p-value → reject $H_0$.

⚠ p-value is not $P(H_0 \mid \text{data})$. To get that you need a prior. The p-value is probability of extreme data given $H_0$.

Type I / Type II / Power

Power and MDE

Two-sample $z$-test with effect $\delta$, variance $\sigma^2$, $n$ per arm:

\[ \text{power} = \Phi\!\left(\frac{\delta\sqrt{n/2}}{\sigma} - z_{1-\alpha/2}\right) \]

Solve for the smallest detectable $\delta$:

\[ \delta_{\text{MDE}} = (z_{1-\alpha/2} + z_{1-\beta}) \cdot \sigma\sqrt{2/n} \]

More data → smaller MDE. More variance → larger MDE. Stricter $\alpha$ or higher power target → larger MDE.

Power = probability of detecting a real effect

Multiple testing: Bonferroni

Run 20 tests at $\alpha = 0.05$ on random data → expect 1 false positive.

\[ \alpha_{\text{Bonf}} = \frac{\alpha}{m} \]

For 20 tests → $\alpha = 0.0025$ per test. Controls family-wise error rate.

Very conservative. Good for high-stakes decisions; wasteful for exploration.

Multiple testing: BH-FDR

Sort p-values $p_{(1)} \le p_{(2)} \le \cdots \le p_{(m)}$.

Find the largest $k$ where $p_{(k)} \le \frac{k}{m}\alpha$. Reject all with rank $\le k$.

Controls false discovery rate — expected proportion of false positives among the rejects.


from statsmodels.stats.multitest import multipletests
bh_ok, bh_p, _, _ = multipletests(pvals, alpha=0.05, method="fdr_bh")

Permutation test

  1. Compute observed test statistic $T_{\text{obs}}$
  2. Pool the two groups
  3. Randomly re-split $B$ times; each shuffle gives $T^{*(b)}$
  4. p-value = fraction of $T^{*(b)}$ at least as extreme as $T_{\text{obs}}$

obs = treat.mean() - ctrl.mean()
pool = np.concatenate([ctrl, treat])
n_c = len(ctrl)
B = 5000
null = np.empty(B)
for b in range(B):
    rng.shuffle(pool)
    null[b] = pool[n_c:].mean() - pool[:n_c].mean()
p = np.mean(np.abs(null) >= abs(obs))

Worked: three-metric A/B test


pvals = np.array([p_conv, p_aov, p_rpv])
bonf_ok, bonf_p, _, _ = multipletests(pvals, 0.05, "bonferroni")
bh_ok,   bh_p,   _, _ = multipletests(pvals, 0.05, "fdr_bh")

print("Raw p:     ", pvals)
print("Bonferroni:", bonf_ok, bonf_p)
print("BH-FDR:    ", bh_ok,   bh_p)

BH-FDR typically rejects more than Bonferroni — more power at the cost of a weaker error guarantee.
