Chapter 2: Built-in Node Types

Overview

Kailash ships with a registry of built-in node types covering common processing tasks. This chapter teaches you how to discover available nodes through the NodeRegistry, inspect their metadata and parameters, and use them in workflows.

Prerequisites

  • Chapter 1: WorkflowBuilder Basics
  • Kailash SDK installed

Concepts

NodeRegistry

The NodeRegistry is a global dictionary that maps string names to Node classes. When you call add_node("PythonCodeNode", ...), the builder looks up "PythonCodeNode" in the registry. Built-in nodes are registered automatically at import time via the @register_node decorator.

Import path: from kailash.nodes.base import NodeRegistry

NodeMetadata

Every node class carries metadata: a human-readable name, description, version, and tags for categorization. This metadata powers tooling such as visual editors and documentation generators.

NodeParameter

NodeParameter describes a single input or output — its name, type, whether it is required, a description, and an optional default value. The collection of parameters defines the node's data contract.

Code Walkthrough

from kailash.nodes.base import Node, NodeMetadata, NodeParameter, NodeRegistry

# 1. Discover available nodes
registered_names = NodeRegistry.list()
print(f"Registered node types: {len(registered_names)}")

# PythonCodeNode is always available
python_code_cls = NodeRegistry.get("PythonCodeNode")
assert python_code_cls is not None
assert issubclass(python_code_cls, Node)

# 2. NodeMetadata
metadata = NodeMetadata(
    name="ExampleNode", description="A test node", version="1.0.0"
)
assert metadata.name == "ExampleNode"
assert isinstance(metadata.tags, set)  # Defaults to empty set

# 3. NodeParameter
param = NodeParameter(
    name="input_data", type="str",
    description="The input string to process", required=True,
)
optional_param = NodeParameter(
    name="separator", type="str",
    description="Optional separator", required=False, default=",",
)

# 4. Using PythonCodeNode in a workflow
from kailash import WorkflowBuilder, LocalRuntime

builder = WorkflowBuilder()
builder.add_node("PythonCodeNode", "make_list",
    {"code": "output = [1, 2, 3, 4, 5]", "output_type": "list"})
builder.add_node("PythonCodeNode", "sum_it",
    {"code": "output = sum(numbers)", "inputs": {"numbers": "list"},
     "output_type": "int"})
builder.connect("make_list", "sum_it", mapping={"output": "numbers"})

workflow = builder.build(name="node-types-demo")
runtime = LocalRuntime()
results, run_id = runtime.execute(workflow)
runtime.close()

print(f"Sum result: {results.get('sum_it', {})}")

Exercises

  1. Call NodeRegistry.list() and count available node types. Import additional submodules and check if the count increases.
  2. Create a NodeParameter with required=False and no default. What value does the node receive?
  3. Build a workflow that generates a dictionary and extracts a specific key from it.

Key Takeaways

  • NodeRegistry is the global catalog of all available node types
  • Built-in nodes register automatically via @register_node at import time
  • NodeMetadata provides human-readable node descriptions for introspection
  • NodeParameter defines the input/output contract: name, type, required, default
  • Results from runtime.execute() are returned as a (dict, run_id) tuple
