Logistic Regression and Classification

Sigmoid · Log-odds · Odds ratios · ANOVA · Tukey HSD

⏱ 60 minutes · Prereq: 2.2, 2.5

Binary outcomes — why not linear?

  • Linear predictions can exceed 1 or drop below 0 — no probability interpretation
  • Constant variance is wrong for binary data (variance = $p(1-p)$)
  • MLE under Bernoulli is cleaner and more efficient

We need a model that bounds the output in $(0, 1)$ and uses the Bernoulli likelihood.

The sigmoid

\[ \sigma(z) = \frac{1}{1 + e^{-z}} \]

From log-odds to sigmoid

Assume log-odds is linear in predictors:

\[ \log\!\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 x_1 + \cdots = z \]

Solve for $p$:

\[ \frac{p}{1-p} = e^z \;\Rightarrow\; p = \frac{e^z}{1 + e^z} = \frac{1}{1 + e^{-z}} = \sigma(z) \]

Log-odds live in $(-\infty, \infty)$; sigmoid maps them back to $(0, 1)$.

Bernoulli log-likelihood

\[ P(y_i \mid x_i, \beta) = p_i^{y_i}(1 - p_i)^{1 - y_i} \]

Log over $n$ observations:

\[ \ell(\beta) = \sum_{i=1}^{n}\big[y_i \log p_i + (1 - y_i)\log(1 - p_i)\big] \]
No closed form. Gradient is non-linear in $\beta$. Optimise with BFGS or IRLS.

Odds ratios — the multiplicative story

\[ \frac{\text{odds}(x_j + 1)}{\text{odds}(x_j)} = e^{\beta_j} \]

Odds ≠ probability

⚠ Odds ratio of 2 does not mean "twice as likely". For a baseline probability of 10%, doubling the odds gives about 18%.

Always convert back to probabilities when reporting to non-specialists. Coefficients live in log-odds space; audiences live in probability space.

Evaluation: confusion matrix

Accuracy = (TP+TN)/total. Precision = TP/(TP+FP). Recall = TP/(TP+FN). AUC is threshold-independent.

Worked: ICU mortality (MLE by hand)


def sigmoid(z):
    return np.where(z >= 0, 1/(1+np.exp(-z)), np.exp(z)/(1+np.exp(z)))

def neg_log_lik(beta, X, y):
    p = sigmoid(X @ beta)
    eps = 1e-12
    return -np.sum(y*np.log(p+eps) + (1-y)*np.log(1-p+eps))

result = minimize(neg_log_lik, np.zeros(X.shape[1]),
                  args=(X, y), method="BFGS")
for name, b in zip(names, result.x):
    print(f"{name}: β={b:+.4f}  OR={np.exp(b):.2f}")

Worked: evaluate the classifier


from sklearn.metrics import confusion_matrix, roc_auc_score

p_hat = sigmoid(X @ result.x)
y_hat = (p_hat >= 0.5).astype(int)

cm  = confusion_matrix(y, y_hat)
acc = (y == y_hat).mean()
auc = roc_auc_score(y, p_hat)

Always report accuracy with the confusion matrix — accuracy alone lies on imbalanced data.

ANOVA: generalising the $t$-test

Test whether 3+ groups have the same mean. Decompose variance:

\[ \text{SS}_{\text{total}} = \text{SS}_{\text{between}} + \text{SS}_{\text{within}} \] \[ F = \frac{\text{MS}_{\text{between}}}{\text{MS}_{\text{within}}} = \frac{\text{SS}_{\text{between}}/(k-1)}{\text{SS}_{\text{within}}/(n-k)} \]

Under $H_0$ (all group means equal), $F \sim F_{k-1, n-k}$. With 2 groups, ANOVA's $F$ is the square of the $t$-statistic.

Post-hoc: Tukey's HSD

ANOVA says "some group differs". Tukey's HSD says which.

\[ \text{HSD} = q_{\alpha, k, n-k} \cdot \sqrt{\frac{\text{MS}_{\text{within}}}{n_{\text{group}}}} \]

Two means differ significantly if their difference exceeds HSD. Family-wise error control is built in.


from statsmodels.stats.multicomp import pairwise_tukeyhsd
result = pairwise_tukeyhsd(prices, labels, alpha=0.05)
print(result)

Worked: HDB prices across flat types


groups = {k: df.filter(pl.col("flat_type") == k)["resale_price"].to_numpy()

labels = np.concatenate([[k]*len(v) for k, v in groups.items()])

Three rookie mistakes

1. Blind 0.5 threshold. It's arbitrary. Optimise for business cost.
2. Accuracy on imbalanced data. 95% sounds great — until base rate is 95%.
3. Multiple pairwise $t$-tests. Use ANOVA + Tukey.
