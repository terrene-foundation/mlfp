Speaker Notes — Lesson 4.7 Recommender Systems — THE PIVOT

Total time: ~90 minutes  ·  17 slides  ·  ~5 min per slide

This is the most important lesson in Module 4. The recommender system content is the vehicle; the actual learning objective is the pivot insight: optimisation drives feature discovery. Every minute spent on content-based and collaborative filtering is setup for the matrix factorisation reveal on Slide 6. Do not rush the pivot slides (Slides 6 and 13). If you are running long, cut Slide 11 (hybrid) to a single sentence and expand the pivot discussion.

Slide 1~3 minTitle

Recommender Systems — THE PIVOT

Set the stakes immediately: "This is the most important lesson in Module 4. Not because recommender systems are hard — they are actually simpler than clustering — but because they contain an insight that changes how you think about feature engineering. By the end of this lesson you will understand that optimisation can create features. That insight is the bridge to deep learning."

→ "Let me show you where this fits."
Slide 2~4 minSpectrum diagram

The Feature Engineering Spectrum

Walk the diagram left to right. "In M3, humans designed features. In M4.1 through 4.6, algorithms discovered features without any error signal — clustering, PCA, NMF, topic models. Today, in 4.7, we cross the line: the algorithm discovers features by minimising a loss function. The features are called embeddings. In 4.8, neural networks generalise this by adding non-linearity."

Make them see the yellow box. "This is the PIVOT. Everything before it is classical unsupervised. Everything after it is deep learning in disguise."

→ "Let's start with the simplest approach."
Slide 3~4 minContent-based

Content-based filtering

Quick and intuitive. "You liked Item A. Item B has similar features. I recommend Item B. That is content-based filtering. It works for new items (they have features from day one) but it only recommends more of the same — no serendipity."

Singapore context: "FairPrice Online's 'similar products' section is content-based — it matches on product category, brand, and description."

→ "What if we ignore item features entirely?"
Slide 4~5 minCF types

Collaborative filtering

Walk both boxes. User-based: "Find people who rate things similarly to you and recommend what they liked that you have not seen." Item-based: "Find items that get rated similarly to things you liked." Tell them Amazon's original recommendation engine was item-based CF — published in 2003.

For beginners: "Think of user-based CF as asking your friends for recommendations. Item-based CF is asking the store: 'what else do customers buy with this item?'"
→ "Both need the interaction matrix."
Slide 5~4 minInteraction matrix

The user-item matrix

Walk the diagram. "Rows are users, columns are items, cells are ratings. Most cells are empty — that is the sparsity problem. The goal is to fill in the question marks — predict unobserved ratings — so we can recommend high-predicted-rating items."

Tease the factorisation: "One way to fill the gaps: factorise this matrix into two smaller matrices whose product approximates the original. The rows of those smaller matrices are embeddings."

→ "The factorisation objective."
Slide 6~6 minTHE PIVOT — objective

Matrix factorisation objective

This is the pivot slide. Slow down. Walk the formula: "We want to find $U$ and $V$ such that $U V^T$ reconstructs the observed ratings. The first term is reconstruction error. The second term is regularisation — prevents overfitting on sparse data."

Then deliver the insight: "The embeddings $\mathbf{u}_u$ and $\mathbf{v}_i$ did not exist before we started optimising. They are features that the optimisation process created. This is the first time in this course that a loss function has created features. Write that down."

For experts: mention the connection to NMF (4.6). "NMF factorised the document-word matrix. Here we factorise the user-item matrix. Same math. The constraint is different: NMF requires non-negativity; CF MF uses L2 regularisation."
→ "How do we solve this?"
Slide 7~4 minALS algorithm

ALS — Alternating Least Squares

Walk the diagram. "The trick: if you know $V$, finding $U$ is just a bunch of independent least-squares problems — one per user. If you know $U$, finding $V$ is the same. So alternate: fix one, solve the other. Each step has a closed-form solution. Repeat 10–20 times and you converge."

→ "How does this connect to PCA?"
Slide 8~4 minPCA connection

Connection to PCA

Walk the table row by row. "Same decomposition. Different data. Same output: embeddings. The only difference is PCA reconstructs the full matrix while CF only reconstructs observed entries. This is why the regularisation term is needed — without it, you could make unobserved predictions arbitrarily large."

→ "Let's see the code."
Slide 9~5 minALS code

ALS from scratch

Walk the code. Point out: "The outer loop alternates between solving for U and solving for V. Each inner loop is a for-loop over users (or items). The solve call is np.linalg.solve — that is the closed-form least-squares solution."

If beginners are lost in the code: "Focus on the structure: two nested loops, alternating. The math inside is a standard least-squares regression — the same thing you learned in Module 2."
→ "What kind of feedback drives this?"
Slide 10~4 minImplicit vs explicit

Implicit vs explicit feedback

Walk the table. Emphasise the abundance gap: "For every user who leaves a star rating, a hundred users click, view, and scroll without rating. Implicit data is where the signal is."

→ "Combining both worlds."
Slide 11~3 minHybrid

Hybrid recommenders

Quick slide. "Combine content features (metadata, descriptions) with collaborative signals (interaction data). Most production systems at Shopee, Grab, and Netflix are hybrids."

→ "What do the learned embeddings look like?"
Slide 12~4 minEmbedding vis

Visualising learned embeddings

Point at the diagram. "These clusters emerged without any labels. The factorisation discovered that electronics items are rated similarly by certain users, groceries by others, fashion by others. The clusters ARE the evidence that the embeddings capture meaningful structure."

→ "Now the big statement."
Slide 13~6 minTHE PIVOT

THE PIVOT

This is the climax. Slow down and be deliberate. "Matrix factorisation learns embeddings by minimising reconstruction error. This is the first time in this course that optimisation has driven feature discovery. Before today, features were hand-crafted or discovered without error feedback. Today, a loss function created features that did not exist before."

Bridge to 4.8: "In the next lesson, neural networks generalise this. Hidden layer activations ARE embeddings. Learned by minimising a loss function. The difference: neural networks add non-linearity through activation functions, so they can learn non-linear combinations. Matrix factorisation produces linear embeddings; neural networks produce non-linear ones. But the core insight is identical."

If the pivot concept is not landing: "Think of it this way. In M3, a human decided 'price per square metre is a useful feature'. Today, the algorithm decided 'this 20-dimensional vector is a useful representation of a user'. Nobody designed that vector. Optimisation created it."
→ "The factorisation family."
Slide 14~3 minFactorisation family

The factorisation family

Walk the table. "PCA, NMF, CF, neural networks — all factorise a matrix into lower-rank components. The progression: PCA is linear and full-matrix; NMF adds non-negativity; CF optimises on observed entries only; neural networks add non-linearity. Each step is more powerful and more general."

→ "Mistakes to avoid."
Slide 15~3 minMistakes

Four common mistakes

Walk all four. #4 is the meta-mistake: "If you write in your exercise report 'I built a recommender system' and say nothing about embeddings and optimisation-driven features, you have missed the point of this lesson."
