Why 99.9% Was Useless

Foundations

The Numbers

  • 10 million transactions per day
  • Only ~100 are truly suspicious (~0.001%)
  • A model that says "everything is fine" is 99.99% accurate
  • The 0.1% false positive rate = 10,000 false alerts daily

The Impact

  • Compliance officers overwhelmed by false alerts
  • Real suspicious transactions buried in noise
  • Regulatory fines: CHF 2 billion (FINMA 2024)
  • Accuracy was the wrong metric
Time: ~2 min. Walk through the math slowly. When 0.001% of data is positive, accuracy is meaningless. This motivates our entire evaluation section and class imbalance section.

What Is Supervised Learning?

Foundations

Given inputs (features X) and known outputs (target y), learn a function that maps X to y.

Analogy: A tutor shows a student 1,000 solved math problems (X = problem, y = answer). The student learns the pattern. Then we give the student NEW problems and see if they can solve them.
\hat{y} = f(X) + \varepsilon
f is the true relationship; our model learns an approximation f̂; ε is irreducible noise
Time: ~2 min. This is review from M3. Reinforce: "supervised" means we have labelled examples. The goal is generalisation — performing well on data the model has never seen.

Training, Validation, and Test Splits

Overfitting vs Underfitting

Foundations
Key insight: More complex is not always better. The goal is to capture the signal (the real pattern) without memorising the noise (random variation).
Time: ~2 min. Walk through each panel slowly. Underfitting = straight line through a curve. Overfitting = wiggly line through every point. Goldilocks = smooth curve that follows the trend. If beginners look confused: "Think of it like writing. Too short (underfit) misses the point. Too long (overfit) includes every random thought. Just right captures the key ideas."

The Learning Curve

Foundations

Plot training and test error as you increase model complexity (or data size).

Reading a Learning Curve

  • Training error low, test error high → overfitting
  • Both errors high → underfitting
  • Both errors converge low → good fit
  • Gap between curves → generalisation gap

What To Do

  • Overfitting → more data, simpler model, regularisation
  • Underfitting → more complex model, more features
  • Good fit → ship it (after calibration!)
Time: ~2 min. Learning curves are the diagnostic tool students will use most. Show how to read the gap between train and test.

No Free Lunch Theorem

Theory

There is no single model that is best for all problems.

Wolpert & Macready (1997): Averaged over ALL possible data distributions, every learning algorithm performs equally well. Any advantage on one class of problems is paid for by disadvantage on another.

What this means in practice:

  • You MUST try multiple models on YOUR data
  • "Always use XGBoost" is wrong (even though it wins many competitions)
  • The best model depends on: data size, feature types, noise level, interpretability requirements
Kailash bridge: AutoMLEngine compares multiple models systematically, so you do not have to guess.
Time: ~2 min. No Free Lunch is why we need this module. We must know all the models to pick the right one. For experts: mention that NFL has interesting caveats — in practice, real-world data has structure that some algorithms exploit better than others.

Bias and Variance — The Darts Analogy

Foundations

Bias = how far the centre of your throws is from the bullseye. Variance = how spread out your throws are.

Time: ~2 min. Review from M3 but critical context for model comparison. Every model in today's zoo has a different bias-variance profile.

Bias-Variance Decomposition

Theory

Target: y = f(x) + \varepsilon where \varepsilon \sim \mathcal{N}(0, \sigma^2). Our model \hat{f}(x) is trained on random sample D.

Step 1: Expected prediction error
\text{EPE}(x) = \mathbb{E}_{D,\varepsilon}\!\left[(y - \hat{f}(x))^2\right]
Step 2: Substitute y = f(x) + ε and add/subtract E[f̂(x)]
= \mathbb{E}\!\left[\bigl(\underbrace{f(x) - \mathbb{E}[\hat{f}]}_{\text{bias}} + \underbrace{\mathbb{E}[\hat{f}] - \hat{f}}_{\text{variance}} + \underbrace{\varepsilon}_{\text{noise}}\bigr)^2\right]
Step 3: Cross terms vanish — result
\text{EPE}(x) = \underbrace{(f(x) - \mathbb{E}[\hat{f}])^2}_{\color{#DC2626}{\textbf{Bias}^2}} + \underbrace{\mathbb{E}[(\hat{f} - \mathbb{E}[\hat{f}])^2]}_{\color{#2563EB}{\textbf{Variance}}} + \underbrace{\sigma^2}_{\color{#6B7280}{\textbf{Noise}}}
Time: ~3 min. Full derivation. Cross terms vanish because noise is zero-mean and independent, and variance term is zero-mean by definition. For beginners: "We split the total error into three parts: how wrong we are on average, how much our predictions wobble, and unavoidable noise."

The Three Terms

Theory
TermWhat It MeasuresYou Control It By
Bias²Systematic error — model's average is off-targetMore complex model
VarianceSensitivity to training set — predictions fluctuateSimpler model, more data, ensembles
σ²Irreducible randomness in the dataNothing — this is the floor
The trade-off: Reducing bias (more complex model) typically increases variance. Reducing variance (simpler model) typically increases bias. The art of ML is finding the sweet spot.
Time: ~2 min. This is the most important equation in supervised ML. Every model choice is a bias-variance trade-off.

Double Descent — When the Trade-off Breaks

Advanced

Classical wisdom: more parameters = more overfitting. But modern over-parameterised models generalise well.

Belkin et al. (2019): Test error first decreases, then increases at the interpolation threshold (classical overfitting), then decreases again in the over-parameterised regime — the "double descent" curve.
  • Under-parameterised: classical bias-variance applies
  • Interpolation threshold: just enough params to memorise = worst generalisation
  • Over-parameterised: many solutions fit perfectly; gradient descent finds the smoothest one (implicit regularisation)

This does NOT mean "just make everything bigger." The classical picture is incomplete for modern architectures.

Time: ~3 min. For experts only. Mention epoch-wise double descent (Nakkiran et al. 2021). For beginners: "The takeaway is that the simple story has exceptions for very large models."

Simple Linear Regression

Foundations

The simplest model: draw a straight line through data.

Example: How does floor area affect HDB resale price?
Price = β₀ + β₁ × floor_area + noise
β₀ is the base price (intercept). β₁ is the price per square metre (slope).

OLS: Find the line that minimises the sum of squared distances.

\min_{\beta_0, \beta_1} \sum_{i=1}^{n} \left(y_i - (\beta_0 + \beta_1 x_i)\right)^2
Sum of squared residuals — penalises large errors quadratically
Time: ~2 min. Use the HDB example throughout. Ask: "Why squared? Why not absolute value?" Squared is differentiable everywhere.

OLS Solution — Normal Equations

Theory

In matrix form: y = X\beta + \varepsilon. Minimise \|y - X\beta\|^2.

Step 1: Expand the loss
L(\beta) = (y - X\beta)^T(y - X\beta) = y^Ty - 2\beta^TX^Ty + \beta^TX^TX\beta
Step 2: Differentiate and set to zero
\frac{\partial L}{\partial \beta} = -2X^Ty + 2X^TX\beta = 0
Step 3: Solve for β
\boxed{\hat{\beta} = (X^TX)^{-1}X^Ty}
Time: ~3 min. Walk through each step. Closed-form solution — no iterative optimisation. For beginners: "We found the exact best line using calculus." For experts: "Requires X'X invertible — when it is not, Ridge saves us."

Geometric Interpretation & R²

Theory

\hat{y} = X(X^TX)^{-1}X^Ty = Hy — projection onto the column space of X.

  • The hat matrix H projects y onto the space that X can describe
  • Residuals e = y - \hat{y} are orthogonal to the column space
  • R² = 1 - \text{SS}_\text{res}/\text{SS}_\text{tot} — proportion of variance explained
  • Adjusted R² penalises adding useless features: \bar{R}^2 = 1 - \frac{(1-R^2)(n-1)}{n-p-1}
Time: ~2 min. The geometric view: OLS is just projection. If students have linear algebra: "The best prediction lives in the world X can describe." Cook's distance, leverage (h_ii), and VIF > 10 for multicollinearity are the key diagnostic measures.

OLS Assumptions — LINE

Theory
LetterAssumptionWhat Breaks If Violated
LLinearity: E[y|X] is linear in XBiased estimates, wrong predictions
IIndependence: errors are independentStandard errors wrong, p-values unreliable
NNormality: errors ~ N(0, σ²)Inference invalid; predictions still OK
EEqual variance (homoscedasticity)OLS still unbiased but inefficient
In practice: Linearity and independence matter most. Normality matters least (CLT rescues us for large n). Always check residual plots.
Time: ~2 min. Diagnostics: residuals vs fitted (patterns), Q-Q plot (normality), Cook's distance > 1 (influential points), VIF > 10 (multicollinearity).

Ridge Regression (L2)

Theory

Problem: When features are correlated, (X^TX) is nearly singular and estimates explode.

\hat{\beta}_\text{ridge} = (X^TX + \lambda I)^{-1}X^Ty
λI adds a positive value to the diagonal — always invertible. Bayesian interpretation: Gaussian prior on β.
  • Shrinks all coefficients toward zero, but never exactly to zero
  • Effect on eigenvalues: shrinks poorly determined directions most — d_j^2/(d_j^2 + \lambda)
  • Adds bias but dramatically reduces variance — often reduces total error
Time: ~3 min. The lambda-I trick stabilises an unstable matrix. For experts: Ridge shrinks coefficients along the eigenvectors of X'X. Effective degrees of freedom: df(lambda) = sum d_j^2/(d_j^2 + lambda).

Lasso (L1) & ElasticNet

Theory

Lasso (L1)

\min_\beta \|y - X\beta\|^2 + \lambda \sum_{j}|\beta_j|
  • No closed form (|β| not differentiable at 0)
  • Solved by coordinate descent
  • Produces sparse solutions (automatic feature selection)
  • Diamond constraint meets loss contours at corners
  • Bayesian: Laplace prior (sharp peak at 0)

ElasticNet

\lambda[\alpha\|\beta\|_1 + (1\!-\!\alpha)\|\beta\|_2^2]
  • α = 1: pure Lasso; α = 0: pure Ridge
  • Groups correlated features (Lasso picks one arbitrarily)
  • Practical default: α = 0.5
  • Feature selection + stability
Time: ~3 min. The geometry is the key: L1 constraint is a diamond with corners on axes. Loss contours (ellipses) touch the corners where parameters = 0. L2 circle has no corners, so no exact zeros. ElasticNet combines both for correlated features. For experts: LARS algorithm computes the entire Lasso path efficiently.

Polynomial Regression

Foundations

Linear in parameters, not in features. Add polynomial features for non-linearity.

y = \beta_0 + \beta_1 x + \beta_2 x^2 + \beta_3 x^3 + \varepsilon
Still a linear model! Linear in the parameters β.
Danger: High-degree polynomials overfit catastrophically. Degree 2 or 3 is usually enough. Beyond that, use trees or splines.
Time: ~2 min. The key insight: "linear model" means linear in parameters, not in x. A degree-10 polynomial is still a linear model. For beginners: "We give the model more building blocks to draw curves instead of just straight lines."

Logistic Regression — From Log-Odds to Sigmoid

Theory
Model log-odds as linear
\log\!\left(\frac{p}{1-p}\right) = X\beta
Solve for p — get the sigmoid
p = \frac{1}{1 + e^{-X\beta}} = \sigma(X\beta)
MLE → Cross-entropy loss
\mathcal{L} = -\sum_{i=1}^{n}\!\left[y_i \log \hat{p}_i + (1\!-\!y_i)\log(1\!-\!\hat{p}_i)\right]
Beautiful fact: The gradient is \nabla_\beta \mathcal{L} = X^T(\hat{y} - y) — same form as OLS! Decision boundary: where X\beta = 0 (a hyperplane).
Time: ~3 min. Derive the sigmoid from log-odds — it is not arbitrary. The cross-entropy loss comes from maximum likelihood. Multinomial extension: softmax with K weight vectors. Ordinal: cumulative link model.

Multinomial & Generalised Linear Models

Theory

Multinomial logistic: P(Y\!=\!k) = e^{X\beta_k} / \sum_j e^{X\beta_j} (softmax)

GLMs unify linear and logistic regression via the exponential family:

DistributionLink FunctionUse Case
NormalIdentityContinuous (OLS)
BernoulliLogitBinary (logistic)
PoissonLogCount data (claims, events)
Neg. BinomialLogOverdispersed counts
GammaInversePositive continuous (amounts)
Time: ~2 min. GLMs are essential for insurance/finance. Poisson for counts, Gamma for positive amounts. Singapore context: MAS risk models commonly use GLMs.

Bayesian Linear Regression

Advanced

Full posterior distribution over β instead of a single point estimate:

p(\beta|y,X) \propto p(y|X,\beta) \cdot p(\beta)

Conjugate Gaussian

  • Prior: \beta \sim \mathcal{N}(\mu_0, \Sigma_0)
  • Posterior: \beta|y \sim \mathcal{N}(\mu_n, \Sigma_n)
  • \Sigma_n = (\Sigma_0^{-1} + \sigma^{-2}X^TX)^{-1}
  • With \mu_0\!=\!0, \Sigma_0\!=\!\tau^2 I → reduces to Ridge

Why Bayesian?

  • Uncertainty: credible intervals on every prediction
  • Prior knowledge: encode domain expertise
  • Small data: priors regularise naturally
  • Predictive: p(y^*|y) includes parameter uncertainty
Time: ~3 min. Bayesian linear regression is the principled version of Ridge. Ridge = MAP estimate with Gaussian prior. Full Bayesian gives you the entire distribution. For experts: predictive distribution naturally includes both parameter uncertainty and noise.

Splitting Criteria — Gini & Entropy

Theory

Gini Impurity

\text{Gini}(t) = 1 - \sum_{k=1}^K p_k^2

Probability of misclassification. Gini = 0 means pure node.

Entropy / Information Gain

H(t) = -\sum_{k=1}^K p_k \log_2 p_k

IG = H(parent) − weighted H(children). Gain Ratio (C4.5) normalises by split info.

Regression trees: split by MSE/MAE reduction. Leaf prediction = mean of samples in leaf.

Gini vs Entropy: Almost always select the same splits. Gini is slightly faster (no logarithm).
Time: ~2 min. Example: node with 70% approve, 30% deny: Gini = 1 - (0.49 + 0.09) = 0.42. For beginners: "Both measure how mixed the classes are."

Pruning — Controlling Tree Complexity

Theory

Pre-Pruning

  • max_depth, min_samples_leaf
  • min_samples_split
  • min_impurity_decrease

Post-Pruning (Cost-Complexity)

  • R_\alpha(T) = R(T) + \alpha|T|
  • Grow full tree, then prune branches
  • Select α by cross-validation
Unpruned trees: Low bias but VERY high variance. This is why single trees are weak learners — ensembles fix this.
Time: ~2 min. Advantages of trees: interpretable, handles mixed types, no scaling. Disadvantages: high variance, greedy, axis-aligned splits.

Random Forest — Wisdom of the Crowd

Foundations

Train hundreds of trees and let them vote. Two sources of randomness:

  1. Bootstrap sampling: each tree gets a random sample with replacement
  2. Feature randomness: at each split, only consider a random subset of features
\text{Var}\!\left(\frac{1}{B}\sum T_b\right) = \rho\sigma^2 + \frac{1-\rho}{B}\sigma^2
As B → ∞, variance → ρσ². Feature randomness reduces ρ. Defaults: max_features = √p (class), p/3 (reg).
Time: ~3 min. The variance formula is the key: averaging helps but correlation rho sets the floor. Feature randomness decorrelates trees. OOB error gives free validation (~36.8% of data unused per tree).

Gradient Boosting — The Algorithm

Theory

Key insight: gradient descent in function space. Each tree fits the negative gradients (pseudo-residuals).

The boosting loop (m = 1 to M)
  1. Pseudo-residuals: r_{im} = -\partial L(y_i, F_{m-1}(x_i))/\partial F_{m-1}
  2. Fit tree hm to pseudo-residuals
  3. Find optimal leaf values: \gamma_{jm} = \arg\min_\gamma \sum_{x_i \in R_j} L(y_i, F_{m-1}(x_i) + \gamma)
  4. Update: F_m = F_{m-1} + \eta \cdot h_m

Shrinkage: Learning rate η < 1 with more trees generalises better. Stochastic GB: subsample rows and columns.

Time: ~3 min. For MSE loss, pseudo-residuals = actual residuals. For other losses, they are the gradients. For beginners: "Each tree learns the mistakes of the previous ensemble." Loss functions: squared, absolute, Huber, logistic, exponential.

XGBoost — Second-Order Optimisation

Theory

2nd-order Taylor expansion with regularised tree structure:

\mathcal{L}^{(t)} \approx \sum_{i}\!\left[g_i f_t(x_i) + \tfrac{1}{2}h_i f_t(x_i)^2\right] + \gamma T + \tfrac{1}{2}\lambda\sum_{j=1}^T w_j^2
gi = first derivative, hi = second derivative, T = leaves, wj = leaf weights

Optimal Leaf Weight

w_j^* = -\frac{G_j}{H_j + \lambda}

Split Gain

\frac{1}{2}\!\left[\frac{G_L^2}{H_L\!+\!\lambda} + \frac{G_R^2}{H_R\!+\!\lambda} - \frac{(G_L\!+\!G_R)^2}{H_L\!+\!H_R\!+\!\lambda}\right] - \gamma

γ = minimum gain to split. Sparsity-aware splits handle missing values. Weighted quantile sketch for approximate split finding.

Time: ~3 min. Newton's method (2nd order) converges faster than gradient descent (1st order). XGBoost 2.0: vector-leaf trees for multi-target. XGBoost 3.x: SYCL device support. Key hyperparameters: n_estimators, learning_rate, max_depth, min_child_weight, gamma, lambda, alpha, subsample, colsample_bytree.

LightGBM — Speed Through Cleverness

Theory

Three Key Ideas

  • Histogram-based splitting: 256 bins → O(bins × features) not O(n × features)
  • GOSS: keep all high-gradient (hard) samples, sample low-gradient (easy) ones
  • EFB: bundle mutually exclusive features to reduce dimensionality

Leaf-Wise Growth

  • XGBoost: level-wise (all nodes at same depth)
  • LightGBM: leaf-wise (split leaf with highest gain)
  • Faster convergence, lower error with fewer leaves
  • Risk of overfitting on small data — control with num_leaves

Native categorical features: optimal subset splits using gradient statistics (Fisher's theorem for binary). 5-10x faster than XGBoost on large data.

Time: ~3 min. Histogram binning is the biggest speedup. GOSS is elegant: most learning happens from hard examples. For beginners: "LightGBM does the same thing as XGBoost but with tricks to make it much faster."

CatBoost — Robustness by Design

Theory
  • Ordered Target Statistics: encode categoricals using target mean with permutations to prevent leakage — for sample i, use only samples BEFORE i in a random permutation
  • Ordered Boosting: random permutations for unbiased gradient estimates
  • Symmetric (oblivious) trees: same split condition at each level — faster inference, natural regularisation
  • Best out-of-box: needs less tuning than XGBoost or LightGBM
Time: ~2 min. CatBoost addresses a subtle bias in standard gradient boosting. Its ordered boosting is theoretically principled. For beginners: "CatBoost works well straight out of the box."

Mahalanobis\sqrt{(x-y)^T\Sigma^{-1}(x-y)}Correlated features

Feature scaling is critical! Distance-based models require it. Curse of dimensionality: KNN fails above ~20-50 features — all points become equidistant. Use PCA first.
Time: ~2 min. KNN is a lazy learner — no training phase. KD-tree and Ball-tree accelerate search. Choosing k: cross-validation, sqrt(n) heuristic. For beginners: "Find the most similar examples."

Support Vector Machines

Theory

Find the hyperplane that separates classes with the widest margin.

\min_{w,b,\xi} \frac{1}{2}\|w\|^2 + C\sum\xi_i \quad \text{s.t.} \;\; y_i(w\!\cdot\!x_i + b) \geq 1 - \xi_i
C controls tolerance: high C = narrow margin, low training error; low C = wide margin, more tolerance.

Kernel trick: compute inner products in high-dimensional space WITHOUT explicit mapping.

KernelFormulaEffective Dim
Linearx_i \cdot x_jp
Polynomial(x_i\!\cdot\!x_j + c)^d\binom{p+d}{d}
RBF\exp(-\gamma\|x_i - x_j\|^2)Infinite

Guide: features > samples → linear. Default → RBF. Limitation: O(n²)-O(n³) training — impractical for n > 100K.

Time: ~3 min. The kernel trick is beautiful: RBF maps to infinite dimensions with one exponential. SVR uses epsilon-insensitive tube. SVMs were dominant 2000-2012; gradient boosting has largely replaced them for tabular data.

Naive Bayes

Theory
P(y|x_1,...,x_p) \propto P(y)\prod_{j=1}^{p}P(x_j|y)
"Naive" = features independent given class. Wrong but works (Zhang, 2004).
VariantP(xj|y)Best For
Gaussian\mathcal{N}(\mu_{jy}, \sigma^2_{jy})Continuous features
MultinomialCount-basedText (bag of words)
BernoulliBinaryBinary features
ComplementComplement of classImbalanced text

Laplace smoothing (α): prevents zero probabilities. When to use: baselines, text, small data, real-time (extremely fast).

Time: ~2 min. NB only needs the ranking of class probabilities to be correct, not exact values. That is why the independence assumption matters less than you think. NB is a linear classifier in log-probability space.

Gaussian Processes

Advanced

A distribution over ALL possible functions. Predict both mean AND variance.

f \sim \mathcal{GP}(m(x), k(x, x'))
m(x) = mean function, k(x, x') = kernel (RBF, Matern, periodic). Built-in uncertainty quantification.
  • Non-parametric: complexity grows with data
  • Bayesian Optimisation: GP surrogate + acquisition function (EI, UCB) — efficient hyperparameter tuning
  • Connection: infinite-width neural networks are GPs (Neal, 1996)
  • Limitation: O(n³) — practical for n < 10,000
Kailash bridge: HyperparameterSearch(config=SearchConfig(method="bayesian")) uses Bayesian Optimisation under the hood.
Time: ~3 min. GPs are the gold standard for uncertainty quantification on small data. For beginners: "GPs tell you not just 'what is my prediction' but 'how confident am I.'" Common kernels: RBF (smooth), Matern v=3/2 (realistic for physical data), periodic (repeating patterns).

TabPFN — Foundation Models for Tabular Data

Advanced
TabPFN: A transformer pre-trained on millions of synthetic datasets. Given your training data as context, it produces predictions in a single forward pass — no training needed.
  • TabPFN v2.5 (Nov 2025): up to 50K samples, 2K features
  • 100% win rate vs default XGBoost on small-medium classification (≤10K samples)
  • 87% win rate on larger datasets up to 100K samples
  • Distillation: convert to compact MLP or tree for fast inference
  • Paradigm shift: treats tabular ML as in-context learning

Source: Hollmann et al. (2024), Nature; PriorLabs TabPFN v2.5, 2025.

Time: ~2 min. TabPFN is cutting-edge. It implicitly performs Bayesian model selection during pre-training. Best for small data where gradient boosting struggles.

Confusion Matrix & Precision, Recall, F1

Foundations
Pred +Pred −
Actual +TPFN
Actual −FPTN
\text{Precision} = \frac{TP}{TP+FP}
\text{Recall} = \frac{TP}{TP+FN}
F_1 = \frac{2PR}{P+R} \quad F_\beta = \frac{(1+\beta^2)PR}{\beta^2 P + R}

Precision: "Of those I flagged, how many are real?" Recall: "Of all positives, how many did I find?" F1: harmonic mean (penalises imbalance). β=2 weights recall; β=0.5 weights precision.

Time: ~3 min. Use Credit Suisse: P = 1% means 99% false alarms. Which error is worse depends on the domain: medical (FN worse), spam (FP worse), fraud (FN worse).

ROC vs PR Curves

Theory

ROC Curve

  • TPR vs FPR as threshold varies
  • AUC = P(positive scores > negative)
  • Misleading with severe imbalance
  • FPR denominator (FP+TN) dominated by TN

PR Curve

  • Precision vs Recall as threshold varies
  • AP = area under PR curve
  • Honest with imbalanced data
  • Precision directly measures false alarm rate

More metrics: Log loss (penalises confident wrong predictions), Cohen's κ (chance-adjusted), MCC = \frac{TP\!\cdot\!TN - FP\!\cdot\!FN}{\sqrt{(TP\!+\!FP)(TP\!+\!FN)(TN\!+\!FP)(TN\!+\!FN)}} — most informative single metric for imbalanced data (−1 to +1).

Multi-class: macro (unweighted avg), micro (aggregate TP/FP/FN), weighted (by class support).

Time: ~3 min. MCC is underused. Chicco & Jurman (2020) showed MCC is more informative than F1 and accuracy for imbalanced data. Use ROC for balanced data, PR for imbalanced.

Cross-Validation Strategies

Theory
StrategyHowWhen
K-FoldK folds, rotate test foldDefault (K=5 or K=10)
Stratified K-FoldPreserves class proportionsClassification with imbalance
Leave-One-OutK = n, test on each sampleVery small data (<50)

Nested CV: 5×5 nested CV is the gold standard for rigorous model comparison. Non-nested CV overestimates performance by 2-5% (Varma & Simon, 2006).

Model selection: AIC = 2k − 2ln(L), BIC = k·ln(n) − 2ln(L). For non-parametric models, CV error is the practical standard. Statistical comparison: corrected resampled t-test, Friedman + Nemenyi post-hoc.

Time: ~3 min. Stratified K-Fold should be default for classification. Time Series Split is mandatory for temporal data. K=5 is usually sufficient; K=10 for critical decisions.

Section F: Class Imbalance

Foundations

"In the real world, the interesting things are rare."

  • Fraud: 0.1% | Disease: 0.01% | Defects: 0.5% | Money laundering: 0.001%

A model that always predicts "normal" gets 99.9%+ accuracy. This is the Credit Suisse problem.

Time: ~1 min. Every interesting ML problem involves imbalance.

SMOTE & Resampling

Theory
x_\text{new} = x_i + \lambda(x_\text{nn} - x_i), \quad \lambda \sim U(0,1)
SMOTE: interpolate between minority sample and its nearest neighbour

SMOTE Failure Modes

  • Noisy features → unrealistic samples
  • High dimensionality → meaningless neighbours
  • Overlapping classes → samples in wrong territory
  • Categorical features → interpolation meaningless

Better Alternatives

  • ADASYN: more samples near decision boundary
  • Tomek links: clean boundary (remove close opposite pairs)
  • ENN: remove misclassified majority samples
  • Combined: SMOTE + Tomek or SMOTE + ENN

Undersampling: random (discards info but fast), Tomek (cleans boundary), ENN (removes noise).

Time: ~3 min. SMOTE is not a magic bullet. Always compare with simpler approaches. For beginners: "Instead of copying examples, SMOTE creates new synthetic examples by blending."

Algorithmic Approaches

Theory

Class Weights

w_k = \frac{n}{K \cdot n_k}
  • Rare classes get higher weight
  • class_weight="balanced"
  • No risk of overfitting to synthetic samples
  • Try this FIRST

Focal Loss

\text{FL}(p_t) = -\alpha_t(1-p_t)^\gamma\log(p_t)
  • Downweights easy examples (γ = 2 recommended)
  • At pt = 0.9: 100x lower loss than CE
  • Focuses learning on hard examples
  • Lin et al. (2017), originally for object detection
Time: ~3 min. Class weights are the simplest fix. Focal loss is elegant: the modulating factor (1-p_t)^gamma automatically downweights easy examples. For experts: focal loss addresses the same problem as hard negative mining but in a differentiable way.

Threshold Moving & Ensemble Approaches

Foundations

Threshold moving: change decision threshold from 0.5 to match business capacity.

  • Lower threshold (0.1) → higher recall, more false alarms
  • Higher threshold (0.9) → higher precision, misses more positives
  • Choose based on business constraint (e.g. "max 50 alerts/day")
Ensemble MethodHow
BalancedRandomForestBalanced bootstrap per tree — uses all majority data across forest
EasyEnsembleMultiple classifiers on different undersampled subsets
RUSBoostAdaBoost with random undersampling each round
Decision guide: Mild (10:1) → class weights. Moderate (100:1) → weights + threshold. Severe (1000:1+) → resampling + focal loss + ensembles. Always: stratified CV, PR curve, MCC.
Time: ~3 min. Threshold moving is free — no retraining. The flowchart is the practical summary. Start simple (class weights), add complexity only if needed.

Section G: Model Calibration

Foundations

When your model says "70% chance of default," does it actually default 70% of the time?

Why it matters: If a model says "90% chance of cancer" but the real probability is 30%, you perform unnecessary surgery. If an insurance model says "5% risk" but reality is 20%, you underprice the policy.

Most models are NOT well-calibrated out of the box. Logistic regression is often OK. Random Forests push probabilities toward 0.5. Gradient boosting is often over-confident.

Time: ~2 min. Connect to Zillow: uncalibrated predictions without uncertainty led to overpaying for houses.

Reliability Diagram & Calibration Methods

Theory

Reliability diagram: bin predictions, plot predicted vs observed frequency. Perfect = diagonal.

MethodHowWhen
Platt scalingP_\text{cal} = \sigma(Af(x) + B)Small calibration set (<1K), sigmoid-shaped miscalibration
Isotonic regressionMonotonic step function fit to reliability diagramLarge calibration set (>1K), non-sigmoid patterns
Temperature scalingP_k = e^{z_k/T}/\sum e^{z_j/T}Multi-class, preserves accuracy (only changes confidence)
\text{ECE} = \sum_{b=1}^{B}\frac{n_b}{n}|\text{acc}(b) - \text{conf}(b)|
Expected Calibration Error: average gap between accuracy and confidence across bins
Time: ~3 min. Platt scaling fits a logistic on model outputs. Isotonic regression is non-parametric. Temperature scaling is the go-to for deep learning (Guo et al. 2017). Always evaluate calibration on a held-out set.

Proper Scoring Rules

Advanced

A scoring rule is proper if it is minimised when predicted probabilities equal true probabilities.

Proper (honest)

  • Brier: \frac{1}{n}\sum(y_i - \hat{p}_i)^2
  • Log loss: cross-entropy
  • Both minimised at \hat{p} = P(Y\!=\!1|X)

Not Proper (gameable)

  • Accuracy: incentivises over-confidence
  • Report extreme probabilities (0 or 1) and still score well
Proof Brier is proper: \partial \mathbb{E}[(Y\!-\!\hat{p})^2]/\partial\hat{p} = 2(\hat{p}-p) = 0 at \hat{p} = p. QED.
Time: ~2 min. Proper scoring rules reward honesty. Brier decomposes into calibration + refinement. Log loss is more sensitive to confident wrong predictions.

Conformal Prediction — Guaranteed Coverage

Advanced

Instead of point predictions, produce prediction sets with finite-sample coverage guarantee:

P(y \in C(x)) \geq 1 - \alpha
No distribution assumptions needed! Works with ANY model.
  1. Train model, compute nonconformity scores on calibration set: s_i = 1 - \hat{p}(y_i|x_i)
  2. Find threshold \hat{q} = (1−α)-quantile of scores
  3. Prediction set: C(x) = \{y : s(x,y) \leq \hat{q}\}

Properties: distribution-free, finite-sample, adaptive (smaller sets where model is confident), model-agnostic. Recent: Tail-Aware CP (AAAI 2025), Adaptive CI for distribution shift (Gibbs & Candes, 2021).

Time: ~3 min. Conformal prediction is one of the most exciting recent developments. For beginners: "Instead of 'this is category A,' the model says 'it is one of {A, B}' and guarantees the answer is in that set 95% of the time."
