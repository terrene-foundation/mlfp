Why convolutions?

A 224×224 colour image has 150,528 pixels. A fully connected layer to 1000 units needs 150M parameters and learns nothing about spatial structure.

Images have two properties we need to exploit:

  • Local structure — adjacent pixels are correlated
  • Translation equivariance — a cat is a cat regardless of position

A convolution uses the same small filter at every position.

The convolution op

Output size: \( (W - F + 2P)/S + 1 \)

Conv arithmetic — the one formula

\[ W_{\text{out}} = \left\lfloor \frac{W - F + 2P}{S} \right\rfloor + 1 \]
  • Same (stride 1, P = (F−1)/2): 28×28 → 28×28
  • Halving (stride 2, F=3, P=1): 28×28 → 14×14

The ResNet idea

Before ResNet: adding more layers hurt accuracy after ~20 layers. Not overfitting — gradient vanishing.

ResNet learns the residual and adds the input back:

\[ H(x) = F(x) + x \]

The skip gives gradients a clear path back. In the worst case, F can learn zero and the block becomes the identity — so adding layers can never hurt.

ResNet basic block

BasicBlock in PyTorch


class BasicBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn1   = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn2   = nn.BatchNorm2d(channels)

    def forward(self, x):
        identity = x
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = out + identity          # ← the magic line
        return F.relu(out)

Squeeze-and-Excitation

Drop-in block (2017) that rescales channels based on global context.

  1. Squeeze: global average pool each channel → one number
  2. Excite: tiny MLP → sigmoid → weight in (0, 1)
  3. Scale: multiply each channel by its weight
\( w = \sigma(W_2 \, \text{ReLU}(W_1 \, \text{GAP}(x))) \)

Adds ~3% parameters, buys ~1–2% accuracy on ImageNet.

SE block code


class SEBlock(nn.Module):
    def __init__(self, channels, reduction=8):
        super().__init__()
        hidden = max(channels // reduction, 4)
        self.fc1 = nn.Linear(channels, hidden)
        self.fc2 = nn.Linear(hidden, channels)

    def forward(self, x):
        b, c, _, _ = x.shape
        s = x.mean(dim=[2, 3])                # squeeze (GAP)
        s = F.relu(self.fc1(s))
        s = torch.sigmoid(self.fc2(s))        # excite
        return x * s.view(b, c, 1, 1)         # scale

Stacking them


class TinyResNetSE(nn.Module):
    def __init__(self, n_classes=3):
        super().__init__()
        self.stem   = nn.Sequential(nn.Conv2d(1, 16, 3, padding=1),
                                    nn.BatchNorm2d(16), nn.ReLU())
        self.block1 = BasicBlock(16)
        self.se1    = SEBlock(16)
        self.down   = nn.Conv2d(16, 32, 3, stride=2, padding=1)
        self.block2 = BasicBlock(32)
        self.se2    = SEBlock(32)
        self.head   = nn.Sequential(nn.AdaptiveAvgPool2d(1),
                                    nn.Flatten(),
                                    nn.Linear(32, n_classes))

Training loop — familiar territory


model = TinyResNetSE().to(device)
opt = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(15):
    for xb, yb in loader:
        logits = model(xb)
        loss = F.cross_entropy(logits, yb)
        opt.zero_grad(); loss.backward(); opt.step()

Identical structure to M4.8 and to Lesson 5.1. The only change: cross-entropy instead of MSE.

Vision Transformers in one slide

2020+: ViT matches ResNet on ImageNet without a single convolution.

  • Split image into 16×16 patches → flatten each to a vector
  • Add a learned position embedding per patch
  • Feed the sequence to a standard Transformer encoder (Lesson 5.4)

For very large datasets: ViT ≥ CNN. For small datasets: CNN still wins (stronger inductive bias).

Four traps

  • Forgetting model.eval() — BN uses batch stats in train mode, running avg in eval.
  • Spatial mismatch in the skip — when downsampling, use a 1×1 conv on the shortcut.
  • LR too high for deep nets — start 1e-3 Adam; halve on NaN.
  • .view(b, -1) vs flatten — use torch.flatten or nn.Flatten.
