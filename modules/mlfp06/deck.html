How LLMs Learn: Pre-training

Foundations

Next Token Prediction (GPT-style)

  • Given tokens t_1, t_2, \ldots, t_n, predict t_{n+1}
  • Trained on trillions of tokens from the internet
  • The model learns grammar, facts, reasoning — all from prediction
  • Autoregressive: each token depends on all previous tokens

Masked Language Modelling (BERT-style)

  • Mask 15% of tokens, predict the masked ones
  • Bidirectional: sees both left and right context
  • Better for classification; worse for generation
\mathcal{L} = -\sum_{i=1}^{N} \log P(t_i \mid t_1, \ldots, t_{i-1}; \theta)
Cross-entropy loss: maximise the probability of the next correct token
Key insight: Pre-training does not teach the model to follow instructions. It teaches it to complete text. Alignment (RLHF/DPO) is what makes it helpful.

LoRA: Low-Rank Adaptation

Theory

Core Idea

  • Pre-trained weights W_0 are frozen
  • Weight update decomposed into two small matrices
  • Instead of updating all d \times d parameters, learn d \times r and r \times d
  • Rank r \ll d — typically 4, 8, or 16
M4 Connection: LoRA IS SVD applied to weight updates. The low-rank matrices B and A approximate the full update \Delta W just like truncated SVD approximates a matrix.
W = W_0 + \Delta W = W_0 + BA
W_0 \in \mathbb{R}^{d \times d} (frozen)  |  B \in \mathbb{R}^{d \times r}  |  A \in \mathbb{R}^{r \times d}
h = W_0 x + BAx
Forward pass: original output + low-rank adaptation

Parameter savings: For d = 4096, r = 8: full update = 16.7M params. LoRA = 65K params. 256x reduction.

MLFP Module 6 — Terrene Open Academy
Time: ~4 min. This is the key formula of the lesson. Walk through the shapes carefully: W0 is dxd and frozen. B is dxr, A is rxd. The product BA is dxd but has only 2*d*r parameters. For d=4096 and r=8, that is 65,536 vs 16,777,216. If beginners confused: "Think of it as summarising a 1000-page book into a 4-page summary. You lose some detail but capture the essence."

LoRA: From-Scratch Implementation

Theory 
import torch
import torch.nn as nn

class LoRALayer(nn.Module):
    """Low-Rank Adaptation layer."""
    def __init__(self, in_features: int, out_features: int, rank: int = 8):
        super().__init__()
        self.rank = rank
        # Frozen original weights
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.linear.weight.requires_grad = False

        # Low-rank matrices
        self.A = nn.Parameter(torch.randn(rank, in_features) * 0.01)
        self.B = nn.Parameter(torch.zeros(out_features, rank))
        # A is initialised with small random values; B with zeros
        # So BA = 0 at init: adaptation starts from the original model

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        original = self.linear(x)           # W_0 @ x
        adaptation = (x @ self.A.T) @ self.B.T  # x @ A^T @ B^T = (BA)x
        return original + adaptation
Why B=0? At initialisation, BA = 0, so the model starts as the original pre-trained model. Training gradually learns the adaptation. This is why LoRA is safe: you cannot make the model worse at the start.

Time: ~4 min. Walk through every line. Emphasise: the original linear layer is frozen. Only A and B are trainable. The forward pass computes both the original and the adaptation, then adds them. Adapter Layers: Bottleneck Modules

Theory

Architecture

  • Insert small bottleneck modules between transformer layers
  • Down-project → Non-linearity → Up-project
  • Residual connection: if adapter output is zero, original behaviour preserved
  • Only adapter weights are trained; transformer weights frozen
Input h
→
Down W_d
→
ReLU
→
Up W_u
→
h + f(h)
class AdapterLayer(nn.Module):
    def __init__(self, hidden_size: int,
                 bottleneck: int = 64):
        super().__init__()
        self.down = nn.Linear(
            hidden_size, bottleneck)
        self.activation = nn.ReLU()
        self.up = nn.Linear(
            bottleneck, hidden_size)
        # Init up-project near zero
        nn.init.zeros_(self.up.weight)
        nn.init.zeros_(self.up.bias)

    def forward(self, x: torch.Tensor):
        residual = x
        x = self.down(x)
        x = self.activation(x)
        x = self.up(x)
        return residual + x  # Skip conn

Model Merging: Combining Fine-tuned Models

Advanced

TIES

Trim small values, elect sign by majority, merge. Resolves sign conflicts between adapters.

DARE

Drop and rescale: randomly drop weight deltas, rescale survivors. Reduces interference.

SLERP

Spherical linear interpolation between weight vectors. Smooth blending of two models.

Task Arithmetic

Add or subtract task vectors: \tau = W_{\text{ft}} - W_0. Combine: W = W_0 + \lambda(\tau_1 + \tau_2).

Application: Train separate LoRA adapters for sentiment analysis and summarisation. Merge them with TIES to get a model that does both — without retraining.

DPO: Direct Preference Optimization

Theory

From RLHF to DPO

  • RLHF objective: maximise reward while staying close to reference policy
  • Rafailov et al. (2023): the optimal policy has a closed-form solution
  • Rearranging: we can express the reward in terms of the policy directly
  • Result: train directly on preference pairs — no reward model needed
Bradley-Terry model: The probability that response y_w is preferred over y_l follows a logistic function of their reward difference.
\mathcal{L}_{\text{DPO}} = -\mathbb{E}\left[\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]
y_w = preferred response  |  y_l = rejected response  |  \beta = deviation penalty

Intuition: Increase probability of y_w relative to reference. Decrease probability of y_l relative to reference. \beta controls how far the policy can deviate.

MLFP Module 6 — Terrene Open Academy
Time: ~5 min. This is the key formula. Walk through it term by term. The log ratio pi/pi_ref measures how much the policy has changed from the reference for each response. The sigma (sigmoid) converts the difference into a probability. Beta is the temperature: higher beta means stricter adherence to the reference. If beginners confused: "DPO says: given two answers, make the good one more likely and the bad one less likely, but do not change too much from the original model."

DPO: Training Loop

Theory 
import torch
import torch.nn.functional as F

def dpo_loss(
    policy_chosen_logps: torch.Tensor,    # log pi(y_w|x)
    policy_rejected_logps: torch.Tensor,  # log pi(y_l|x)
    ref_chosen_logps: torch.Tensor,       # log pi_ref(y_w|x)
    ref_rejected_logps: torch.Tensor,     # log pi_ref(y_l|x)
    beta: float = 0.1,
) -> torch.Tensor:
    """Direct Preference Optimization loss."""
    chosen_rewards = beta * (policy_chosen_logps - ref_chosen_logps)
    rejected_rewards = beta * (policy_rejected_logps - ref_rejected_logps)
    loss = -F.logsigmoid(chosen_rewards - rejected_rewards).mean()
    return loss

# Training loop (simplified)
for batch in preference_dataloader:
    chosen_logps = model.log_prob(batch["chosen"])
    rejected_logps = model.log_prob(batch["rejected"])
    ref_chosen_logps = ref_model.log_prob(batch["chosen"])
    ref_rejected_logps = ref_model.log_prob(batch["rejected"])

    loss = dpo_loss(chosen_logps, rejected_logps,
                    ref_chosen_logps, ref_rejected_logps)
    loss.backward()
    optimizer.step()

GRPO: Group Relative Policy Optimization

Advanced

Key Innovation (DeepSeek-R1, 2025)

  • Sample G completions for each prompt
  • Score each completion with a verifier (not a learned reward model)
  • Compute advantage relative to group mean
  • Update policy with clipped objective (like PPO, but group-normalised)
\hat{A}_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}
Advantage of completion i relative to group statistics

Why RAG?

  • LLMs have a knowledge cutoff — no real-time information
  • LLMs hallucinate — confidently wrong answers

Chunks overlap by 10-20% to preserve context at boundaries. Without overlap, a fact split across two chunks is lost to both.

def rrf(dense_ranks, sparse_ranks,
        k=60):
    scores = {}
    for doc, r in dense_ranks:
        scores[doc] = 1/(k+r)
    for doc, r in sparse_ranks:
        scores[doc] = scores.get(
            doc,0) + 1/(k+r)

Theory

The ReAct Loop

Thought: I need to find the average HDB price in 2024.

Agent Memory

Theory

Short-term Memory

Current conversation context. Managed by the LLM's context window. Lost when the session ends. Used for: multi-turn reasoning within one task.

Long-term Memory

Persistent knowledge across sessions. Stored in a vector database. Retrieved by semantic similarity. Used for: learning from past interactions.

Structured knowledge about specific entities (people, projects, datasets). Key-value store with entity extraction. Used for: maintaining facts about recurring subjects.

Operating Envelopes

Theory

What Are Operating Envelopes?

  • Define boundaries for what agents can do
  • Task envelopes: restrict agent to specific task types
  • Role envelopes: restrict based on role in the organisation
  • Monotonic tightening: envelopes can only get stricter, never looser
Fail-closed: If the governance check fails (error, timeout), access is denied. Never fail-open. A broken guard is not a free pass. 
from kailash_pact import OperatingEnvelope

envelope = OperatingEnvelope(
    allowed_tasks=[
        "data_analysis",
        "model_training",
    ],
    forbidden_tasks=[
        "data_deletion",
        "model_deployment",
    ],
    max_cost_per_task=5.00,
    enforcement="block",  # warn | block | audit
)

# Monotonic tightening: child inherits
# parent restrictions + can add more
child_envelope = envelope.tighten(
    max_cost_per_task=1.00  # Stricter
)

This prevents privilege escalation attacks. If beginners confused: "An operating envelope is like a job description with hard limits. The agent can do anything within the envelope, but nothing outside it."

Budget Cascading

Theory

How It Works

  • Parent agent has a total budget: $10.00
  • Parent allocates portions to child agents
  • Child agents cannot exceed their allocation
  • When a child's budget runs out, it stops and reports partial results
  • Parent can reallocate remaining budget dynamically
from kailash_pact import (
    GovernanceEngine, BudgetAllocator)

allocator = BudgetAllocator(
    total_budget=10.00
)

allocator.allocate("data_scientist", 3.00)
allocator.allocate("feature_eng", 2.00)
allocator.allocate("model_trainer", 4.00)
allocator.allocate("report_writer", 1.00)

# Each agent receives its allocation
# via its operating envelope
# If data_scientist finishes early:
remaining = allocator.reclaim(
    "data_scientist")
allocator.reallocate(
    "model_trainer", remaining)

Key Formula Recap: Attention Mechanism

Theory
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V
From M5: the foundation of every LLM. Q = queries, K = keys, V = values, d_k = key dimension

Where It Appears in M6

  • 6.1: LLMs use multi-head attention at every layer
  • 6.2: LoRA targets attention weight matrices (q_proj, v_proj)
  • 6.4: Dense retrieval uses attention-based embeddings
  • 6.8: Flash Attention optimises this computation in production
The \sqrt{d_k} scaling prevents dot products from growing too large, which would push softmax into saturated regions (near-zero gradients). This is why training is stable at scale.
MLFP Module 6 — Terrene Open Academy
Time: ~2 min. Recap the attention formula from M5. It is the foundation of everything in M6. LoRA modifies the attention weight matrices. Dense retrieval uses attention-based embeddings. Flash Attention optimises the computation. If beginners confused: "This formula is the engine inside every LLM. Everything we built in M6 runs on this."

M6 Formula Summary

Theory
W = W_0 + BA \quad \text{(LoRA)}
\mathcal{L}_{\text{DPO}} = -\mathbb{E}\!\left[\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]
\hat{A}_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})} \quad \text{(GRPO advantage)}
\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V
Four formulas, four concepts: LoRA decomposes weight updates, DPO aligns with preferences, GRPO normalises rewards within a group, Attention is the computation substrate for all of them.
