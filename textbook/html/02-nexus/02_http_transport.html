Chapter 2: HTTP Transport

Overview

The HTTP transport is the primary channel through which Nexus serves workflows as REST API endpoints. Under the hood, Nexus wraps FastAPI via the Kailash Core SDK's enterprise gateway. This chapter covers the Transport abstract base class, HTTPTransport configuration, middleware queuing, endpoint registration, and how Nexus creates the transport internally.

Prerequisites

  • Completed Chapter 1: Hello Nexus
  • Familiarity with HTTP concepts (ports, CORS, middleware)
  • Understanding of abstract base classes in Python

Concepts

The Transport ABC

Every Nexus transport implements the Transport abstract base class, which defines a uniform contract: name, start(registry), stop(), is_running, and an optional on_handler_registered() hook for hot-reloading.

Middleware Queuing

HTTPTransport supports adding middleware and endpoints before the underlying FastAPI gateway exists. These are queued internally and applied when the gateway initializes during start().

Code Walkthrough

Step 1: Create an HTTPTransport

from nexus.transports import HTTPTransport, Transport

http = HTTPTransport(
    port=9090,
    cors_origins=["http://localhost:3000"],
    cors_allow_methods=["GET", "POST"],
    cors_allow_headers=["Authorization", "Content-Type"],
    cors_allow_credentials=False,
    cors_max_age=600,
    enable_auth=False,
    enable_monitoring=False,
    enable_durability=False,
    rate_limit=200,
)

assert http.name == "http"
assert http.port == 9090
assert http.is_running is False
assert http.app is None

Step 2: Queue Middleware

from nexus.middleware.security_headers import (
    SecurityHeadersConfig,
    SecurityHeadersMiddleware,
)

http.add_middleware(SecurityHeadersMiddleware, config=SecurityHeadersConfig())
assert len(http._middleware_queue) == 1

Middleware is queued internally and applied when the gateway is created during start().

Step 3: Register Custom Endpoints

async def custom_health(request):
    return {"status": "ok"}

http.register_endpoint("/custom-health", ["GET"], custom_health)
assert len(http._endpoint_queue) == 1

Step 4: HTTPTransport Inside Nexus

from nexus import Nexus

app = Nexus(
    api_port=8080,
    cors_origins=["https://app.example.com"],
    enable_durability=False,
)

assert isinstance(app._http_transport, HTTPTransport)
assert app._http_transport.port == 8080
assert app.fastapi_app is not None

Step 5: Add Middleware via Nexus

from nexus.middleware.cache import CacheConfig, ResponseCacheMiddleware

app.add_middleware(
    ResponseCacheMiddleware,
    config=CacheConfig(default_ttl=120, max_entries=500),
)

last_mw = app.middleware[-1]
assert last_mw.name == "ResponseCacheMiddleware"
Note: Nexus.add_middleware() delegates to the HTTP transport and records MiddlewareInfo objects for runtime introspection.

Exercises

  1. Transport Properties: Create an HTTPTransport with custom settings, verify all properties, and examine the health_check() output.
  2. Middleware Stack: Queue three middleware classes and verify the order. Explain why order matters in the onion model.
  3. Nexus Integration: Compare adding middleware via the transport directly vs. via Nexus.add_middleware().

Key Takeaways

  • HTTPTransport implements the Transport ABC with name "http".
  • Middleware, routers, and endpoints are queued before the gateway is created.
  • Nexus creates HTTPTransport automatically from constructor parameters.
  • app.fastapi_app provides access to the underlying FastAPI application.
  • health_check() returns transport status for monitoring.
