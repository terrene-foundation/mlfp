Graph Neural Networks

GCN · GraphSAGE · GAT · Message passing · torch_geometric

⏱ 75 minutes  ·  Outcome: build GCN and GAT for node and graph classification

Why graphs?

  • Social networks, molecular structures, supply chains, MRT map.
  • Relationships between entities carry information tables lose.
  • Graph data: nodes (entities), edges (relationships), features on both.
🟢 Foundations: if your data has explicit relationships between entities, a graph captures structure that a table cannot.

Message passing

Each node updates by aggregating neighbour features. 1 GNN layer = 1 round of message passing.

GCN update rule

\[ H^{(l+1)} = \sigma\bigl(\tilde{D}^{-1/2}\,\tilde{A}\,\tilde{D}^{-1/2}\,H^{(l)}\,W^{(l)}\bigr) \]
  • $\tilde{A} = A + I$ (add self-loops).
  • Degree normalisation ensures equal weight regardless of node degree.
  • $W^{(l)}$ = learnable weight matrix shared across all nodes.

GraphSAGE — inductive

  • Samples fixed-size neighbourhood (not all neighbours).
  • Aggregation: mean, LSTM, or max pooling.
  • Generalises to unseen nodes — essential for dynamic graphs.
🟢 Foundations: GCN requires all nodes at training. GraphSAGE can embed new nodes without retraining.

GAT — attention on neighbours

\[ e_{ij} = \text{LeakyReLU}\bigl(\mathbf{a}^T [W\mathbf{h}_i \| W\mathbf{h}_j]\bigr) \]
  • Learns different weights per neighbour pair.
  • Normalised with softmax → attention coefficients $\alpha_{ij}$.
  • Multi-head attention (same as Transformers, Lesson 5.4).
🔵 Theory: GCN = fixed weights. GraphSAGE = fixed aggregation. GAT = learned per-pair weights. Increasing flexibility.

GCN in PyG


from torch_geometric.nn import GCNConv, global_mean_pool

class GCN(torch.nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = GCNConv(num_features, 64)
        self.conv2 = GCNConv(64, 64)
        self.fc = torch.nn.Linear(64, num_classes)

    def forward(self, data):
        x, edge_index, batch = data.x, data.edge_index, data.batch
        x = torch.relu(self.conv1(x, edge_index))
        x = torch.relu(self.conv2(x, edge_index))
        x = global_mean_pool(x, batch)  # graph-level
        return self.fc(x)

Common mistakes

  1. Ignoring graph structure — flattening to table loses edges.
  2. Too many layers — over-smoothing. 2–3 layers is typical.
  3. Wrong pooling — node tasks need no pooling; graph tasks need global pooling.
