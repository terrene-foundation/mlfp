Speaker Notes — Lesson 4.8 DL Foundations — Neural Networks and the Training Toolkit

Total time: ~120 minutes  ·  18 slides  ·  ~6–7 min per slide

This is the densest lesson in the entire curriculum. It bridges USML to DL and equips students with the training toolkit for all of M5. Pace is critical. The conceptual payload is the bridge from 4.7 (Slides 1–2), the forward/backprop mechanics (Slides 3–7), and the hidden-layers-as-feature-engineering insight (Slide 14). The training toolkit (Slides 8–13) is reference material — walk it briskly and tell them to use the textbook for details. If running long, compress the toolkit slides and protect the bridge and insight slides.

Slide 1~3 minTitle

DL Foundations

Set context: "This is the final lesson of Module 4 and the most important one. Today we build a neural network from scratch and understand how it works from the ground up. Every architecture in Module 5 — CNNs for images, RNNs for sequences, Transformers for language — is a specialised version of what we build today. The training toolkit you learn here is equipment for the rest of the course."

→ "First, the bridge from 4.7."
Slide 2~5 minBridge

The bridge from 4.7

Walk the side-by-side diagram. "On the left: matrix factorisation from 4.7. It learns linear embeddings by minimising reconstruction error. On the right: a neural network. It learns non-linear embeddings by minimising any differentiable loss. The arrow in the middle is labelled 'add non-linearity' — that is the only difference. Everything else is the same: optimisation drives feature discovery."

If the bridge is unclear: "In 4.7, the optimiser created a 20-dimensional vector for each user. Today, the optimiser creates a vector at each hidden layer. Same process, more expressive because of activation functions."
→ "Let's see the architecture."
Slide 3~5 minArchitecture

Network architecture

Walk the diagram. "Input layer: your features — floor area, storey, town code. Hidden layers: learned representations. Output: predicted price. Each connection carries a weight. Each node applies a linear transform (multiply by weights, add bias) then an activation function (introduces non-linearity). The key insight: the hidden activations are embeddings — just like the user/item vectors in 4.7, but non-linear."

→ "The forward pass computes the prediction."
Slide 4~5 minForward pass

Forward pass

Walk the formula, then the code. "Layer 1: multiply input by weights W1, add bias b1, apply ReLU. Layer 2: same with W2, b2. Layer 3: same but no activation — linear output for regression. Three lines of code. That is the entire forward pass."

→ "Now we need a loss to minimise."
Slide 5~4 minLoss function

Loss function

"For regression: MSE — same as M2. For classification: cross-entropy — measures how far the predicted probability distribution is from the true labels. Both are differentiable, which is what backprop needs."

→ "How do we compute gradients?"
Slide 6~7 minBackprop

Backpropagation — chain rule

This is where many students get lost. Go slowly. "The chain rule says: to find how the loss changes with respect to a weight in layer 1, multiply together all the partial derivatives from the output back to that weight. It is just repeated multiplication — the chain rule from calculus applied layer by layer."

If the math is intimidating: "You do not need to memorise this formula. The point is: there is a systematic way to compute how each weight affects the loss. That is backprop. The code is three lines per layer."
For experts: mention that automatic differentiation (PyTorch, JAX) computes this for you. Understanding backprop matters for debugging gradient issues, not for daily use.
→ "Gradients tell us the direction. Descent takes the step."
Slide 7~4 minGradient descent

Gradient descent

Walk the diagram. "The loss surface is a landscape. The gradient points uphill. We step downhill by subtracting the gradient times the learning rate. Repeat until we reach a valley." Singapore analogy: "Imagine hiking on Bukit Timah in fog. You cannot see the bottom but you can feel which direction is downhill. Take a step. Feel again. Repeat."

→ "Now the toolkit: activations."
Slide 8~5 minActivations

Activation functions

Walk the six boxes. "ReLU is the default for hidden layers — simple, fast, works. Sigmoid for binary output (probability). Softmax for multi-class (probabilities summing to 1). GELU for Transformers. Tanh for RNNs. Leaky ReLU when you are worried about dead neurons." Tell them: "You do not need to memorise all six. Use ReLU unless there is a reason not to."

→ "Regularisation: dropout."
Slide 9~4 minDropout

Dropout

Walk the diagram. "During training, we randomly delete neurons — set them to zero. The network has to learn with missing pieces, which forces it to spread information across multiple neurons. At inference, all neurons are active."

Study group analogy: "Imagine a study group where one person is randomly absent each session. Everyone has to learn the material, because you cannot rely on one person to carry the group."
→ "Normalisation: batch norm."
Slide 10~4 minBatch norm

Batch normalisation

Walk the formula. "Normalise each layer's inputs to zero mean, unit variance. Then apply learnable scale and shift — gamma and beta — so the network can undo the normalisation if it helps. The effect: training is more stable, you can use higher learning rates, and there is a mild regularisation effect."

→ "Optimisers."
Slide 11~4 minOptimisers

Optimisers

Walk the table. "Adam is the default. It combines momentum (remembers past gradients) with per-parameter learning rates (adapts to each weight's scale). AdamW adds proper weight decay — use it for transformers. SGD only if you need precise control."

→ "LR schedules."
Slide 12~4 minLR schedule

LR schedule + early stopping

Walk the diagram. "Start low (warmup), ramp up, then decay following a cosine curve. The dashed red line is early stopping: when validation loss stops improving, stop training and restore the best weights. This prevents the model from memorising the training set."

→ "Now the deep insight."
Slide 13~3 minNN code

From-scratch code

Show the code. "Three lines for forward pass. In the exercise you will implement the full backprop. The textbook has the complete code — use it as reference."

→ "The deep insight."
Slide 14~6 minKey insight

Hidden layers = automated feature engineering

This is the conceptual climax. Slow down. "The hidden layers perform unsupervised feature discovery — they learn representations of the input that are useful for the task. The output layer performs supervised prediction on those learned features. This is representation learning: the network simultaneously discovers and uses features."

Connect to the full spectrum: "M3: human designs features. M4.1-6: algorithms discover features without error. M4.7: optimisation creates linear features. M4.8: optimisation creates non-linear features. M5: specialised architectures create domain-specific features (spatial, temporal, graph, generative)."

→ "Common mistakes."

Slide 16~3 minMistakes

Common mistakes

Walk all four. #1 (zero init) is the most common beginner trap. "If all weights are zero, every neuron computes the same thing. The gradients are the same. The updates are the same. Symmetry is never broken. The network effectively has one neuron per layer."

→ "Cross-references."
Slide 17~3 minCross-refs

Cross-references

Walk the three links. The forward link to M5 is the most important: "Every architecture in M5 is a specialised neural network. CNNs add spatial connectivity. RNNs add temporal connectivity. Transformers add attention. GANs add adversarial training. But the forward pass, backprop, and training toolkit are identical."
